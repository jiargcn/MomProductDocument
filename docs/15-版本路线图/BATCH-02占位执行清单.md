# BATCH-02 占位执行清单

> 状态：已启动。
> 范围：System + Infrastructure。
> 原则：只建立公共能力、权限模型、系统入口和基础服务的文档结构；不写未经取证的具体权限码、接口、配置字段、日志表、消息渠道或报表模板结论。

## 1. 执行边界

| 项目 | 本批处理方式 |
| --- | --- |
| System | 说明租户、认证、组织、用户、角色、权限、运维、消息、开发平台、系统参数等用户可见管理能力。 |
| Infrastructure | 说明标签、条码、打印、报表、看板、查询、导入导出、文件、消息、日志、任务、接口可靠性等跨模块公共能力。 |
| 业务模块引用 | WMS、MES、QMS、EAM、ANDON、SCP 等只在各自业务页说明“如何使用公共能力”，公共规则在本批维护。 |
| 权限取证 | 引用 [RBAC 与动作权限取证模板](RBAC与动作权限取证模板.md)，不直接推断具体角色能做什么。 |
| 图示与示例 | 保留流程图、架构图、权限矩阵、消息流、日志追踪、打印链路等占位，后续按真实实现补证据。 |

## 3. 每页首轮占位最低输出

| 章节 | 首轮最低要求 |
| --- | --- |
| 页面定位 | 明确这是 System 用户入口、Infra 公共能力，还是底层基础服务。 |
| 责任边界 | 说明平台规则与业务页使用规则的分工。 |
| 取证任务 | 保留菜单、前端、后端、DDL、测试环境的证据路径占位。 |
| 权限与审计 | 引用 RBAC 模板，说明是否涉及菜单、按钮、接口、数据范围或高风险动作。 |
| 业务引用 | 标记 WMS/MES/QMS/EAM/ANDON/SCP 等业务页需要如何引用该能力。 |
| 图示与示例 | 至少保留一个流程图、架构图、权限矩阵、消息流或操作示例占位。 |
| 差距记录 | 对归属冲突、实现缺失、证据冲突写入产品差距总账或问题记录清单。 |

## 4. 当前风险

| 编号 | 风险 | 处理方式 |
| --- | --- | --- |
| B2-RISK-001 | 标签、条码、打印当前入口位于 DBC，但目标归属更偏 Infra。 | 文档按“当前入口 + 目标归属”双写，不改菜单结构。 |
| B2-RISK-002 | 只看前端按钮权限可能误判真实授权。 | 必须结合后端注解、角色配置、数据范围和状态前置条件取证。 |
| B2-RISK-003 | 报表、看板、查询、导入导出可能由不同组件或模块承载。 | 先按公共能力建结构，具体实现来源后续逐项核验。 |
| B2-RISK-004 | 日志、审计、监控、任务调度可能横跨 System、Infra、Framework 和第三方组件。 | 先描述能力边界，不猜测表结构和组件职责。 |
| B2-RISK-005 | 开发平台能力范围不明，可能与业务配置、报表平台或代码生成混杂。 | 保留“待核验”并记录到问题清单，不阻塞其它页面。 |
