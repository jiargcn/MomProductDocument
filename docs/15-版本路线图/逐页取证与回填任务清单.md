# 逐页取证与回填任务清单

> 状态：进行中（P0 物料页已完成第二轮取证）。
> 前置：BATCH-01 至 BATCH-04 首轮占位已完成。
> 目的：从“结构占位”转入“证据取证 + 业务规则回填”，避免文档长期停留在待核验状态。

## 1. 阶段目标

本阶段不再扩大占位范围，重点把已建页面逐步变成可培训、可查证、可用于实施和测试的业务字典。

| 目标 | 完成标准 |
| --- | --- |
| 字段真实性 | 字段技术名、中文名、类型、长度、枚举、来源与 DDL / DO / DTO / VO / 前端配置一致。 |
| 新增约束 | 新增必填、唯一性、默认值、选择器数据源和筛选条件有证据。 |
| 编辑限制 | 可编辑字段、不可编辑字段、状态限制、引用限制和删除限制有证据。 |
| 导入规则 | 导入模板字段、系统填充字段、校验规则和错误回执明确。 |
| 列表与详情 | 列表字段顺序、筛选条件、详情分组、快速跳转和关联 Tab 明确。 |
| 状态动作 | 按按钮、接口、状态前置、数据影响、权限和审计形成矩阵。 |
| 终端操作 | PDA、线边端、工位端入口、扫码规则、状态前置和数据影响明确。 |
| 接口挂接 | 第三方系统、触发业务、同步方向、幂等、重试、失败补偿和日志明确。 |
| 图示与示例 | 复杂业务具备流程图、状态图、截图、示例数据或示例流转。 |

## 2. 推荐执行顺序

| 顺序 | 范围 | 原因 |
| --- | --- | --- |
| 1 | P0 样板复核：物料、采购收货、预计入、库存事务、库存余额 | 先确保样板页经得起复制和培训使用。 |
| 2 | BATCH-01：DBC 主数据 + WMS 核心库存链路 | 与库存、单据、终端和接口关系最强，优先形成可用业务字典。 |
| 3 | BATCH-02：System + Infrastructure | 权限、日志、导入导出、消息、报表、打印等会被其它模块引用。 |
| 4 | BATCH-03：MES + QMS + EAM + ANDON | 现场执行、质量、设备、异常状态机复杂，需在公共能力基础上补。 |
| 5 | BATCH-04：SCP + PS + API | 外部协同、排程和接口挂接需要在核心业务模型稳定后补。 |

## 3. 每页取证动作

| 动作 | 证据来源 | 输出位置 |
| --- | --- | --- |
| 菜单与路由确认 | `reference/menu.csv`、前端路由 | 页面“当前实现状态”或“功能入口”。 |
| 字段校正 | DDL、DO、DTO、VO、Excel VO、前端表格/表单配置 | 字段表、新增约束、编辑限制、导入规则。 |
| 选择器确认 | 前端选择器组件、接口、后端查询条件 | 新增约束中的“选择器数据源与筛选条件”。 |
| 动作按钮确认 | 前端按钮、权限标识、后端接口 | 状态动作矩阵、权限矩阵。 |
| 状态机确认 | 字典、后端状态常量、服务流转、测试环境 | 状态图、动作前置条件。 |
| 业务影响确认 | 服务层、Mapper、事务、库存/消息/日志写入 | 业务规则、库存挂接、接口挂接。 |
| 终端入口确认 | PDA、线边端、工位端源码和测试环境 | 终端操作章节。 |
| 示例补充 | 测试环境数据、脱敏样例、截图 | 图示、截图与示例章节。 |

## 4. 批次内优先级

| 优先级 | 页面类型 | 说明 |
| --- | --- | --- |
| P0 | 高频入口、培训必讲、库存/生产/质量/设备闭环关键页 | 必须先形成完整样板。 |
| P1 | 高频主数据、配置页、业务单据页、终端执行页 | 影响日常实施、测试和用户培训。 |
| P2 | 查询、报表、统计、低频维护页 | 在核心规则稳定后补齐。 |
| P3 | 历史页面、低频配置、暂不明确入口 | 保留待核验，不阻塞主线。 |

## 5. 问题处理

| 问题类型 | 处理方式 |
| --- | --- |
| 字段缺证据 | 页面标记“待核验”，登记到字段技术名真实性校正任务。 |
| 规则冲突 | 页面保留证据来源，登记到产品差距总账或问题记录清单。 |
| 实现不完整 | 不写成缺陷结论，记录当前实现、影响和后续确认人。 |
| 需要大范围改名/移动 | 暂停确认，不在取证阶段顺手处理。 |
| 需要源码修改或生产数据 | 暂停确认，不属于当前文档仓库工作范围。 |
