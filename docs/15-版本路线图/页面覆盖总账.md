### 6.1 文件名中文化迁移补充规则

执行 `INV-008` 时，同步处理“一个目录下只有一个 Markdown 文件”的冗余层级。原则如下：

| 场景 | 处理方式 | 判断标准 |
| --- | --- | --- |
| 目录只包含一个 `index.md`，且该目录只是叶子功能页 | 可以上移一层，并将文件命名为对应模块或功能名。 | 上移后语义更清晰，且不会破坏未来分组。 |
| 目录只包含一个普通 `.md` 文件，且无 assets、图片、附件、子目录 | 可以上移一层，并按导航名称重命名。 | 同步更新 `mkdocs.yml`、`.linkmap.json` 和所有内部链接。 |
| 目录虽然只有一个 `.md` 文件，但未来明确会继续拆子页 | 保留目录，将文件命名为 `index.md`。 | 适用于模块首页、业务分组首页、终端操作首页。 |
| 目录包含 assets、截图、图示或页面专属附件 | 默认保留目录。 | 避免资源路径混乱；如确需上移，必须同步迁移资源并校验引用。 |
| 目录层级对应菜单分组或导航分组 | 默认保留目录。 | 即使暂时只有一个页面，也承担信息架构作用。 |

执行顺序：

1. 先生成迁移映射表：旧路径、建议新路径、处理类型、是否保留目录、原因。
2. 再批量更新 `mkdocs.yml`、`.linkmap.json`、正文链接和图片链接。
3. 运行链接脚本和 `mkdocs build`。
4. 对无法确定是否未来扩展的目录，先保留目录，不做上移。
