# WMS 纵向试点任务清单

| 项目 | 内容 |
| --- | --- |
| 任务状态 | 已启动 |
| 目标 | 用一条完整的 WMS 链路验证文档结构、事实取证方法和差距记录方式。 |
| 分析基线 | 测试环境对应的 `dev` 分支，2026-07-15。 |
| 试点链路 | 物料 → 采购收货 → 预计入库 → 库存事务 → 库存余额 → 权限。 |
| 终端范围 | PDA/线边端作为所属业务的“终端操作”补充，不单列模块。 |

## 1. 试点要回答的问题

完成后，读者应能从文档中直接得到以下答案：

1. 为什么同一业务要区分申请、任务、执行记录；它们如何衔接。
2. 采购收货如何影响预计入库、库存事务和库存余额。
3. 物料主数据在新增、编辑、导入时的规则是什么。
4. 用户可在列表和详情页看到什么、可跳转到哪里、可做什么操作。
5. 谁可查看、创建、执行和审核；PDA 操作与 Web 操作有哪些差异。
6. 哪些内容已由当前基线证实，哪些是待补齐的产品能力。

## 2. 工作项与交付物

| 编号 | 工作项 | 主要证据 | 交付物 | 完成标准 |
| --- | --- | --- | --- |
| TRIAL-01 | 提炼“申请—任务—记录”公共模型 | WMS 菜单、后端代码、WMS DDL、历史设计参考 | 业务模型公共说明 | 说明每类对象的目的、主键关联、状态流转和适用边界。 |
| TRIAL-02 | 梳理采购收货 Web 业务 | `ui` 前端、`win-module-wms` 后端、WMS DDL | 采购收货页面事实卡与差距项 | 明确页面、接口、单据字段、操作、状态和异常规则。 |
| TRIAL-03 | 梳理库存挂接关系 | 预计入库、库存事务、库存余额相关菜单、代码与表结构 | 库存挂接关系说明 | 能从收货单追溯到预计入库、事务和余额；不能证实的关联明确标记。 |
| TRIAL-04 | 补齐物料主数据规则 | `itembasic` 页面、DBC 后端、DBC DDL | 物料主数据维护卡 | 覆盖新增约束、编辑限制、导入模板字段与校验。 |
| TRIAL-05 | 核对权限边界 | 用户、角色、菜单页面及系统后端 | 业务动作权限矩阵 | 标出查看、创建、执行、审核等动作与权限证据。 |
| TRIAL-06 | 增加 WMS 终端操作补充 | `pda`、线边端前端，WMS 后端和菜单 | WMS“终端操作”说明 | 只描述扫码、简化录入、设备能力等差异，并链接到对应 Web 业务页。 |
| TRIAL-07 | 汇总冲突与产品差距 | 上述全部证据与历史设计参考 | 差距总账条目 | 每项标明现状、证据、影响、建议方向和待确认人。 |

## 3. 证据优先级与判定规则

| 优先级 | 来源 | 用途 |
| --- | --- | --- |
| 1 | 测试环境实际行为（后续提供） | 判定当前可用能力。 |
| 2 | `dev` 分支前后端源码 | 判定页面、接口、校验、状态和权限实现。 |
| 3 | 数据库 DDL | 判定字段、类型、长度、约束和关联线索。 |
| 4 | 菜单清单 | 判定页面范围、路由与功能入口。 |
| 5 | 历史产品规划书 | 仅用于提取稳定设计理念和发现待验证项，不能单独作为当前事实。 |

当证据不一致时，不强行合并结论：在差距总账登记冲突，并暂停涉及该结论的规范性描述，等待业务确认。

## 5. 输出位置

试点不直接把未证实内容写成业务事实。已验证内容分别进入 `02-业务模型`、`04-DBC-主数据管理`、`05-WMS-库房管理`、`12-系统管理` 的对应页面；问题和缺口进入 `15-版本路线图` 的差距总账。

> 当前阶段先形成取证结果与页面结构；涉及业务口径、状态含义或权限策略的关键冲突，会暂停并向产品负责人确认。
