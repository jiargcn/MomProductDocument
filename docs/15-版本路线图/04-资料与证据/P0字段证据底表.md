# P0 字段证据底表

> 状态：P0 第一版证据底表。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。
> 适用范围：物料基本信息、采购收货、预计入、库存事务、库存余额。
> 使用方式：本页是后续回写业务页面前的字段依据，不替代正式业务说明页。

## 1. 本页结论

P0 链路的关键对象已能用当前 DDL 和后端 DO 初步确认：

| 业务对象 | 数据库表 | 后端 DO | 结论 |
| --- | --- | --- | --- |
| 物料基本信息 | `basic_itembasic` | `ItembasicDO` | 字段名应以 `basic_itembasic` 与 `ItembasicDO` 为准；`isRecycled` 语义为“回收件”。 |
| 采购收货申请 | `request_purchasereceipt_main` / `request_purchasereceipt_detail` | `PurchasereceiptRequestMainDO` / `PurchasereceiptRequestDetailDO` | 主表保存申请单号、状态、供应商、来源、订单等；明细保存物料、数量、批次、包装、库位等。 |
| 采购收货任务 | `job_purchasereceipt_main` / `job_purchasereceipt_detail` | `PurchasereceiptJobMainDO` / `PurchasereceiptJobDetailDO` | 任务主表通过 `request_number` 关联申请；任务明细继承库存定位字段。 |
| 采购收货记录 | `record_purchasereceipt_main` / `record_purchasereceipt_detail` | `PurchasereceiptRecordMainDO` / `PurchasereceiptRecordDetailDO` | 记录主表通过 `request_number`、`job_number` 追溯申请和任务；记录明细是形成库存事务的关键来源。 |
| 预计入库存 | `transaction_expectin` | `ExpectinDO` | 预计入按任务号、业务类型、物料、库存状态、库位、数量和包装/批次维度保存。 |
| 库存事务 | `transaction_transaction` | `TransactionDO` | 库存事务记录事务号、事务类型、库存动作、业务类型、记录号和库存粒度字段。 |
| 库存余额 | `transaction_balance` | `BalanceDO` | 库存余额表名不是 `balance`，而是 `transaction_balance`；库存粒度字段需单独说明。 |

## 6. 库存余额业务粒度

当前用户已确认库存余额业务粒度如下：

| 字段 | 数据库列 | 类型/长度 | 说明 |

| 箱包装号 | `package_number` | `varchar(64)` | 粒度字段。 |
| 物料代码 | `item_code` | `varchar(64)` | 粒度字段。 |
| 库存状态 | `inventory_status` | `varchar(64)` | 粒度字段。 |
| 库位代码 | `location_code` | `varchar(64)` | 粒度字段。 |

补充说明：

1. 当前代码 `BalanceLookupKey` 使用 `itemCode`、`packageNumber`、`locationCode`、`batchId`、`inventoryStatus` 作为组合查询键，未包含 `palletNumber`。
2. 用户确认的业务粒度包含 `pallet_number`。因此这里存在“代码查询键”和“业务唯一粒度”不完全一致的风险，需进入产品差距或技术差距跟踪。
3. 当前已确认存在唯一业务索引缺失问题。本阶段先标记，不直接要求修改数据库结构。
