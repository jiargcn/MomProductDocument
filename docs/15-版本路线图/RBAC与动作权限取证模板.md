# RBAC 与动作权限取证模板

> 状态：批量占位前通用取证模板。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。
> 目的：统一说明每个页面如何取证“谁能看、谁能点、什么状态能执行、执行后影响什么数据”，避免把前端按钮、菜单配置或后端接口单独误写成完整权限结论。

## 1. 适用范围

本模板适用于以下页面：

- 主数据、配置、业务单据、任务、记录、查询页。
- 有新增、编辑、删除、提交、审核、执行、撤销、导入、导出等动作的页面。
- PDA、线边端、EAM PDA 等终端操作入口。
- System / Infrastructure 中的用户、角色、菜单、权限、日志、导入导出、报表、打印、看板等公共能力页。

## 2. 基本原则

| 原则 | 说明 |
| --- | --- |
| 菜单权限、按钮权限、接口权限分开写 | 菜单可见不代表按钮可点；按钮存在不代表后端接口允许；接口存在不代表当前角色能执行。 |
| 前端显示条件和后端状态校验分开写 | 前端 `hide`、`v-if`、`hasPermi` 只说明页面入口；最终能否执行要看后端状态校验、权限注解和服务逻辑。 |
| 权限码不猜测 | 没有源码或菜单证据时写“待核验”，不按页面名称自行拼权限码。 |
| 角色结论必须有证据 | 没有角色菜单分配、租户套餐或测试账号验证时，不写“仓管员可执行”“质检员可审核”等结论。 |
| 数据范围单独记录 | 查询权限和数据可见范围不是一回事；需要区分租户、部门、用户组、仓库、工厂、供应商等范围。 |
| 发现不一致先登记 | 前端有按钮但后端无接口、后端有接口但前端隐藏、权限码命名错位等，先进入差距总账，不阻断其它占位。 |

## 3. 证据优先级

| 优先级 | 证据 | 用途 |
| --- | --- | --- |
| 1 | 测试环境账号实测 | 判定某角色当前能否看见、点击、执行和查看结果。 |
| 2 | 后端权限注解/拦截器/服务校验 | 判定接口实际权限、状态前置条件和数据范围。 |
| 3 | 前端按钮权限与显示条件 | 判定页面入口、按钮名称、按钮显示条件。 |
| 4 | 菜单清单与路由 | 判定页面入口、菜单权限码、页面归属。 |
| 5 | 系统角色、菜单、租户套餐、数据权限表 | 判定角色授权和数据范围设计。 |
| 6 | 历史设计文档 | 仅作为设计意图参考，不作为当前实现事实。 |

## 4. 每页权限取证最小表

批量占位时，每个页面至少保留以下表格。没有证据时填“待核验”，不要删除表格。

| 项目 | 当前结论 | 证据路径 | 待确认 |
| --- | --- | --- | --- |
| 菜单入口 | 待核验 | `reference/menu.csv`、前端路由 | 是 |
| 页面路由/组件 | 待核验 | 前端 `views` 路径 | 是 |
| 菜单权限码 | 待核验 | 菜单清单、系统菜单表、前端路由 | 是 |
| 顶部按钮权限 | 待核验 | `hasPermi`、`v-hasPermi`、按钮配置 | 是 |
| 行按钮权限 | 待核验 | `hasPermi`、`v-hasPermi`、按钮配置 | 是 |
| 后端接口权限 | 待核验 | Controller 注解、网关/拦截器、服务继承 | 是 |
| 状态前置条件 | 待核验 | 前端显示条件、后端服务校验、状态枚举/字典 | 是 |
| 数据范围 | 待核验 | 数据权限注解、角色数据范围、租户/部门/仓库/用户组字段 | 是 |
| 审计日志 | 待核验 | 操作日志注解、审计字段、日志页面 | 是 |

## 5. 动作权限矩阵模板

业务页必须用动作维度描述权限，不只罗列权限码。

| 动作 | 前端入口 | 前端权限码/显示条件 | 后端接口/权限 | 后端状态前置条件 | 数据范围 | 执行后影响 | 当前结论 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 查看列表 | 待核验 | 待核验 | 待核验 | 通常无状态限制，待核验 | 待核验 | 返回可见数据集合 | 待核验 |
| 查看详情 | 待核验 | 待核验 | 待核验 | 待核验 | 待核验 | 返回单据详情、明细、关联页签 | 待核验 |
| 新增 | 待核验 | 待核验 | 待核验 | 待核验 | 待核验 | 新增主数据/申请/任务/配置 | 待核验 |
| 编辑 | 待核验 | 待核验 | 待核验 | 待核验 | 待核验 | 修改字段，可能影响下游引用 | 待核验 |
| 删除 | 待核验 | 待核验 | 待核验 | 待核验 | 待核验 | 删除或逻辑删除对象 | 待核验 |
| 导入 | 待核验 | 待核验 | 待核验 | 待核验 | 待核验 | 批量写入，生成错误文件或导入日志 | 待核验 |
| 导出 | 待核验 | 待核验 | 待核验 | 待核验 | 待核验 | 生成导出文件或导出任务 | 待核验 |
| 提交/审批/驳回 | 待核验 | 待核验 | 待核验 | 必须逐状态核验 | 待核验 | 改变状态，可能生成任务或回写来源单据 | 待核验 |
| 执行/完成 | 待核验 | 待核验 | 待核验 | 必须逐状态核验 | 待核验 | 生成记录、库存事务、接口消息等 | 待核验 |
| 撤销/关闭/冲销 | 待核验 | 待核验 | 待核验 | 必须逐状态核验 | 待核验 | 删除或反向影响预计库存、库存事务、上游数量 | 待核验 |

## 6. 前端权限取证方法

| 取证点 | 需要记录什么 | 常见证据 |
| --- | --- | --- |
| 顶部按钮 | 新增、导入、导出、刷新、筛选、列设置等入口。 | `defaultAddBtn({hasPermi:'...'})`、`v-hasPermi`。 |
| 行按钮 | 编辑、删除、启用、禁用、提交、审批、执行、详情、打印等入口。 | `mainListEditBtn`、`mainListDeleteBtn`、自定义按钮配置。 |
| 按钮显示条件 | 按状态、字段、是否已生成标签、是否已有包装等隐藏或禁用。 | `hide`、`disabled`、`v-if`、`isShowMainButton`。 |
| 详情入口 | 列表字段点击、Tab 页、弹窗详情、终端详情。 | `openDetail`、`Detail` 组件、Tab 配置。 |
| 终端入口 | PDA/线边端菜单、扫码入口、提交入口、拒收入口。 | 终端路由、页面组件、按钮事件。 |

前端取证结论只能写成：

- “页面存在该入口”。
- “当前按钮绑定了某权限码”。
- “页面显示条件疑似与某状态或字段有关”。

不能仅凭前端写成：

- “该角色一定可以执行”。
- “后端一定限制了该状态”。
- “该动作已经完整可用”。

## 7. 后端权限与状态取证方法

| 取证点 | 需要记录什么 | 常见证据 |
| --- | --- | --- |
| Controller | 接口路径、方法、权限注解、操作日志注解。 | Controller 源码、继承的 Base Controller。 |
| Service | 状态校验、业务校验、下游影响。 | Service 实现、Base Service、状态枚举/常量。 |
| 数据权限 | 是否按租户、部门、用户组、仓库、工厂、供应商过滤。 | 数据权限注解、Mapper 条件、上下文工具类。 |
| 通用能力继承 | 子类中看不到接口时，检查是否来自通用父类。 | BaseRequest、BaseJob、BaseRecord 等继承链。 |
| 异步任务/日志 | 导入导出、审批、打印、接口回写是否异步处理。 | 导入服务、导出服务、Job、日志表。 |

若出现“前端按钮被注释但后端接口存在”“后端接口存在但前端没有入口”“子类 Controller 注释但父类提供接口”等情况，应写入差距总账或待确认清单。

## 8. 数据范围取证模板

| 范围类型 | 当前证据 | 可能影响 | 文档写法 |
| --- | --- | --- | --- |
| 租户 | 租户字段、租户上下文、租户套餐。 | 不同租户数据隔离、菜单授权差异。 | 未实测前写“租户范围待核验”。 |
| 部门/组织 | 角色数据范围、部门字段。 | 用户只能看到本部门、本部门及下级或全部数据。 | 需结合角色和数据权限表确认。 |
| 用户/创建人 | 创建者、负责人、承接人、处理人。 | 个人任务、我的申请、待办列表过滤。 | 需验证“本人数据权限”是否生效。 |
| 仓库/库区/库位 | 仓库权限、库位组、用户组。 | 仓管员只维护授权仓库或库位。 | 未证实时不写具体仓库范围。 |
| 工厂/车间/产线 | 工厂建模、人员组织、终端工位。 | MES、QMS、ANDON 的现场数据可见范围。 | 待 MES/QMS/ANDON 批次取证。 |
| 供应商/客户 | SCP 供应商账号、客户协同账号。 | 外部用户只能查看自己的订单或发货数据。 | 待 SCP 批次取证。 |

## 9. 页面文档中的推荐写法

在业务页面中引用本模板时，建议使用以下短句：

> 权限结论按“菜单入口、前端按钮权限、后端接口权限、状态前置条件、数据范围”五层取证。当前页面若只完成前端按钮取证，不代表对应角色已经能在测试环境执行该动作。

业务页可保留以下占位段落：

```markdown
### 权限、审计与日志

| 项目 | 当前结论 | 证据 | 待确认 |
| --- | --- | --- | --- |
| 菜单入口 | 待核验 | 待补充 | 是 |
| 按钮权限 | 待核验 | 待补充 | 是 |
| 后端权限 | 待核验 | 待补充 | 是 |
| 状态前置条件 | 待核验 | 待补充 | 是 |
| 数据范围 | 待核验 | 待补充 | 是 |
| 操作日志 | 待核验 | 待补充 | 是 |

> 说明：本页权限按 [RBAC 与动作权限取证模板](../../15-版本路线图/RBAC与动作权限取证模板.md) 逐项补齐。
```

## 10. 产品差距记录规则

发现以下问题时，进入[产品差距总账](产品差距总账.md)：

| 问题类型 | 记录方式 |
| --- | --- |
| 前端有按钮但后端无接口或接口不可用 | 记录按钮、权限码、接口路径和影响。 |
| 后端有接口但前端无入口 | 记录接口能力是否是预留、废弃、终端专用或权限隐藏。 |
| 前端状态显示条件与后端状态校验不一致 | 记录两个证据来源，不直接判定谁正确。 |
| 权限码命名与模块不一致 | 记录当前权限码、页面归属和可能的授权影响。 |
| 角色菜单配置与产品职责不一致 | 记录测试账号、角色、菜单差异和业务影响。 |
| 数据权限范围无法解释查询结果 | 记录账号、条件、结果和待确认范围规则。 |

## 11. 批量占位时的使用边界

| 可以批量复制 | 不允许批量复制 |
| --- | --- |
| 本模板的章节结构和表头。 | 某个页面的具体权限码。 |
| “前端/后端/状态/数据范围分层取证”的方法。 | 某个业务的状态前置条件。 |
| “待核验 / 待确认”的占位写法。 | 某个角色能执行某动作的结论。 |
| 差距登记规则。 | 某个按钮可见就等于业务可执行的结论。 |
