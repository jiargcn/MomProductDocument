# 批量占位前检查清单

> 状态：P0 样板闭环后的批量执行闸门。  
> 基线：测试环境 / `dev` 分支 / 2026-07-15。  
> 目的：避免把未验证的页面结构、字段规则、状态规则和图示占位大范围复制到全站。

## 1. 本页结论

当前可以进入“有限批量占位准备”，但还不建议直接全站铺开。

可复制的是页面结构、取证方法、占位章节和差距记录方式；不可直接复制的是采购收货的具体状态、动作、字段、导入模板和 PDA 执行规则。后者必须逐业务取证。

## 2. 批量占位闸门

| 闸门 | 检查项 | 当前结论 | 是否允许批量复制 |
| --- | --- | --- | --- |
| GATE-01 | 标准页面结构是否已验证 | 已通过 P0 采购收货样板验证基本结构。 | 是 |
| GATE-02 | 公共业务模型是否已抽取 | 申请、任务、记录模型已抽取首版。 | 是，但需保留例外标记 |
| GATE-03 | 字段技术名校正规则是否明确 | 已形成字段技术名真实性校正任务和 P0 证据底表。 | 是 |
| GATE-04 | 图示、截图、示例占位规则是否明确 | 已形成图示截图任务清单，并在采购收货页验证样板。 | 是 |
| GATE-05 | 详情分组和快速跳转规则是否明确 | 已在采购收货页和标准模板中形成样板。 | 是 |
| GATE-06 | 终端操作归属规则是否明确 | 已确认 PDA/线边端作为所属业务页补充，不单列孤立模块。 | 是 |
| GATE-07 | 权限边界是否已验证 | RBAC 和按钮权限尚未完整取证。 | 否，先占位不写结论 |
| GATE-08 | 状态动作是否可跨业务复用 | 通用状态机可引用，但业务动作差异仍大。 | 条件允许，只复制表结构 |
| GATE-09 | 导入规则是否可跨业务复用 | 导入模板差异大，且字段需逐类核验。 | 否，只复制章节占位 |
| GATE-10 | 测试环境截图是否已具备 | 尚未采集。 | 否，只保留截图占位 |

## 3. 可以批量复制的内容

| 内容 | 复制方式 | 使用范围 |
| --- | --- | --- |
| 页面最小占位结构 | 直接复制标准模板章节。 | 所有实体页、配置页、业务页。 |
| 当前实现状态表 | 复制表结构，逐页填证据路径。 | 所有页面。 |
| 字段证据表 | 复制字段表结构，不复制具体字段。 | 主数据、单据、任务、记录、查询页。 |
| 新增约束、编辑限制、导入规则章节 | 复制章节和表头，内容逐页取证。 | 主数据、配置、可导入业务单据。 |
| 列表显示、详情分组、快速跳转章节 | 复制章节和规划方法。 | 所有数据查询页。 |
| 动作状态表 | 复制表结构，状态和动作逐业务取证。 | 申请、任务、记录、审批、执行类页面。 |
| 终端操作入口章节 | 复制章节结构，入口逐业务取证。 | WMS、MES、EAM 等有 PDA/线边端业务。 |
| 图示、截图、示例占位表 | 复制占位表结构。 | 复杂业务页、库存相关页、接口相关页。 |
| 产品差距表 | 复制记录方式。 | 所有发现冲突或缺失的页面。 |

## 4. 不允许直接复制的内容

| 内容 | 原因 | 正确处理 |
| --- | --- | --- |
| 采购收货具体字段 | 字段属于采购收货业务，不代表其它业务。 | 逐页从 DDL、DO、DTO、前端配置取证。 |
| 采购收货动作状态 | 其它业务状态和异常动作可能不同。 | 只复用“动作状态表结构”。 |
| PDA 采购收货执行规则 | PDA 扫码、库位、数量限制因业务不同而变化。 | 逐终端入口取证。 |
| 导入模板字段 | 导入模板差异大，且存在系统字段误入模板风险。 | 逐模板核验。 |
| 库存挂接字段组合 | 不同业务可能影响预计入、预计出、事务、余额的不同对象。 | 逐业务说明库存影响。 |
| 接口回写规则 | SAP、SCP、ERP、设备接口触发时机不同。 | 接口专项取证。 |
| 权限结论 | 权限涉及菜单、按钮、后端注解、角色数据范围。 | RBAC 专项取证后再写。 |

## 6. 每页批量占位前必须填写的最小信息

| 信息 | 最低要求 | 如果缺失 |
| --- | --- | --- |
| 页面名称 | 使用菜单中文名或导航名称。 | 暂不建叶子页。 |
| 所属模块 | DBC、WMS、MES、QMS、EAM、ANDON、SCP、PS、System、Infra 等。 | 暂不建叶子页。 |
| 页面类型 | 主数据、配置、申请、任务、记录、查询、报表、终端操作、公共能力。 | 标为“待分类”，不套具体业务模板。 |
| 证据路径 | 至少有菜单、前端页面、后端接口、DDL 之一。 | 只建立模块占位，不写实体规则。 |
| 是否涉及导入 | 有、无、待核验。 | 默认保留导入规则占位。 |
| 是否涉及终端 | 有、无、待核验。 | 默认保留终端入口占位。 |
| 是否涉及库存 | 预计入、预计出、事务、余额、无、待核验。 | 默认不写库存影响结论。 |
| 是否涉及接口 | ERP/SAP/SCP/设备/报表/无/待核验。 | 默认保留接口挂接占位。 |

## 7. 单页占位输出标准

每个页面至少形成以下结果：

1. 页面定位清楚：说明它解决什么问题、由谁维护、被谁引用。
2. 证据路径清楚：不能只有业务猜测。
3. 字段名不猜测：没有证据时写“待核验”。
4. 列表和详情有占位：即使未确认，也要说明后续要补什么。
5. 动作和状态有占位：按钮、前置条件、结果影响分开写。
6. 图示和示例有占位：复杂业务不能只有文字和字段表。
7. 差距可追踪：冲突、缺失、产品不足进入差距总账。
