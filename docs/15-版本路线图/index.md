# 版本路线图

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：需要了解「当前能训什么、已知限制、版本如何演进」的测试、实施与运维读者。

## 业务目的与适用范围

本目录面向产品读者，说明版本能力范围、已知限制、变更影响和未来规划。读完本页，应能：知道正式路线图尚在建设中；当前以各模块文档中的已证实能力与限制为准；不要把内部建设清单当成培训事实。

## 如何使用本目录

| 你的目的 | 建议怎么做 |
| --- | --- |
| 查某模块当前可培训能力 | 进入对应业务模块概述与叶页（以各页「已证实 / 待确认」为准） |
| 查已知限制与缺口 | 模块页限制节；跨模块问题见内部总账（不进培训主线） |
| 等正式版本清单 / 变更记录 | 见下方「计划收录内容」；未发布前不以历史规划替代实现事实 |
| 做联调 / 接口发现 | [API 参考](../14-API参考/index.md) |

## 当前状态

当前产品文档仍处于“基于实现证据逐页补齐”的建设阶段。正式版本路线图将在各模块的能力边界、已验证限制和版本变更信息稳定后补充；在此之前，不以历史规划或内部任务清单替代当前实现事实。

!!! example "读完自检"
    **给定：** 需要对外说明「本版本 WMS/MES 能训哪些能力」。
    **期望做法：** 打开对应模块概述与分组页，按其已证实范围讲解；本目录尚未提供可发布的版本能力总表前，不要用内部任务清单冒充版本说明。

## 计划收录内容

| 页面 | 面向读者的用途 | 当前状态 |
| --- | --- | --- |
| 当前版本能力清单 | 说明各模块已交付、可培训的能力。 | 待逐模块确认后建立。 |

### 建议验证点（读者向）

- 能从本页转到至少一个业务模块，并找到该模块自己的能力/限制表述。
- 不把内部建设资料当作现场培训的版本承诺。

!!! note "内部建设资料"

    文档建设任务、取证底稿、问题登记和阶段执行档案属于内部工作资料，已移至仓库根目录 `project-docs/`，不作为产品文档站点导航内容。它们用于保证正式页面可追溯，但不能直接作为培训或业务事实依据。
