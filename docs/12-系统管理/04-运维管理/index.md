# 运维管理

> 状态：BATCH-02 首轮占位页。当前页面用于承接 System 模块中的运行维护入口；具体菜单、字段、接口和权限需后续从 System / Infra 源码和测试环境取证。

## 1. 页面定位

运维管理面向系统管理员和运维人员，重点说明系统运行过程中的管理入口、诊断入口和高风险维护动作。它与基础设施页面的关系如下：

| 范围 | 文档归属 |
| --- | --- |
| 运维人员在系统中能操作的菜单、按钮、配置和任务 | 系统管理 / 运维管理 |
| 日志、监控、告警、任务调度、接口可靠性等平台能力 | 基础设施 |
| 具体业务单据的异常处理、重试、撤销和补偿 | 对应业务模块页面 |

## 2. 后续取证范围

| 主题 | 需要确认的内容 |
| --- | --- |
| 运维菜单 | 当前有哪些运维入口、面向哪些角色、是否存在隐藏入口。 |
| 后台任务 | 与定时任务、补偿任务、重试任务的关系。 |
| 操作审计 | 哪些运维动作必须记录操作者、原因、前后值和影响范围。 |
| 风险动作 | 清理、重算、重试、关闭、补偿、手工同步等动作是否需要二次确认。 |
| 权限边界 | 运维权限是否与普通业务权限、数据权限、租户权限分离。 |
