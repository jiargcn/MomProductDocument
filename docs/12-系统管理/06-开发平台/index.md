# 开发平台

> 状态：BATCH-02 首轮占位页。当前页面用于承接 System / Infra 中面向配置、扩展、低代码或开发支撑的能力；具体实现范围待源码和菜单取证。

## 1. 页面定位

开发平台不是业务模块，它用于支撑页面、接口、报表、表单、字典、流程或代码生成等可配置能力。文档编写时需要避免把开发平台能力与业务规则混写：

| 内容 | 写作位置 |
| --- | --- |
| 平台提供什么配置能力 | 系统管理 / 开发平台 |
| 配置能力如何被监控、审计、发布和回滚 | 基础设施 |
| 某个业务页面如何使用配置结果 | 对应业务模块页面 |

## 2. 后续取证范围

| 主题 | 需要确认的内容 |
| --- | --- |
| 菜单与能力 | 当前开发平台包含哪些菜单：代码生成、表单、报表、流程、接口、字典等。 |
| 数据来源 | 配置数据来自数据库、文件、前端配置还是第三方组件。 |
| 发布机制 | 配置变更是否需要发布、生效范围、缓存刷新和回滚。 |
| 权限与审计 | 谁可以修改配置，变更是否留痕，是否影响生产业务。 |
| 与业务页关系 | 哪些业务页依赖开发平台配置，如何在业务文档中引用。 |
