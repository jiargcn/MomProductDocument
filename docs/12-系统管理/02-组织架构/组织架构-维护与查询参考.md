# 组织架构 - 维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 配套主文档：[组织架构](index.md)。本页只保留部门与用户归属的维护入口、配置项和查询联查。

## 维护入口

| 入口 | 作用 | 说明 |
| --- | --- | --- |
| 部门管理 | 维护部门树、负责人、状态、部门编号等。 | System 主入口。 |
| 用户管理 | 为用户指定所属部门。 | 数据范围计算依赖此项。 |
| 岗位管理 | 维护岗位及库位/物料条件。 | 不在本页展开，见岗位页。 |

## 配置项说明

| 配置项 | 业务含义 | 配置时注意 |
| --- | --- | --- |
| 上级部门 | 构成组织树。 | 调整上级会影响“本部门及下级”范围。 |
| 部门名称 / 编号 | 识别与检索。 | 编号规则按实施约定统一。 |
| 负责人 | 部门联络责任人。 | 不等于自动审批人。 |
| 状态 | 启用/停用。 | 停用前处理用户与下级。 |
| 用户所属部门 | 人员挂载点。 | 必填类场景应在验收清单中检查。 |

## 建议配置顺序

1. 先搭好部门树骨架（由粗到细）。
2. 再维护负责人与状态。
3. 再创建/导入用户并挂部门。
4. 再配置角色与数据范围、岗位条件。
5. 用非超管账号验证组织相关可见性。

## 查询与联查

| 场景 | 建议看什么 | 联查建议 |
| --- | --- | --- |
| 部门树缺失节点 | 是否未创建或已停用。 | 部门管理。 |
| 用户看不见本部门数据 | 用户部门、角色数据范围。 | [数据权限与决策权限-维护与查询参考](../03-用户与权限/数据权限与决策权限-维护与查询参考.md)。 |
| 业务地点找不到 | 是否误在部门树查找工厂/仓库。 | DBC/WMS 对应页。 |

## 验收检查清单

- [ ] 部门树层级与现场责任结构一致。
- [ ] 验收账号均已挂正确部门。
- [ ] “本部门”与“本部门及下级”在非超管下结果可区分。
- [ ] 未把工厂/仓库误建为部门树唯一隔离手段。
