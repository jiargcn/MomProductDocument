# 消息通知

> 状态：BATCH-02 首轮占位页。当前页面描述 System 侧消息通知入口；公共消息能力的底层模型与可靠性治理归入基础设施“消息、通知与待办”页面。

## 1. 页面定位

消息通知用于让用户及时获得待办、审批、异常、任务、告警和系统公告。本文档需要区分三层内容：

| 层级 | 说明 |
| --- | --- |
| 业务触发 | WMS、MES、QMS、EAM、ANDON、SCP 等业务在何时产生通知。 |
| 平台能力 | 消息模板、渠道、发送记录、失败重试、已读未读、待办归集。 |
| 用户入口 | 用户在 System 中如何查看、订阅、处理或关闭消息。 |

## 2. 后续取证范围

| 主题 | 需要确认的内容 |
| --- | --- |
| 消息类型 | 待办、审批、异常、告警、公告、系统提醒等类型是否存在。 |
| 模板与变量 | 模板配置入口、变量来源、业务链接和多语言/多租户支持。 |
| 发送渠道 | 站内信、短信、邮件、企业微信/钉钉等渠道是否实现。 |
| 处理闭环 | 消息是否能跳转到业务对象，已读、处理、关闭如何记录。 |
| 权限与数据范围 | 用户是否只能看到自己有权处理或查看的消息。 |
