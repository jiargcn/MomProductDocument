# 岗位、任务分配与审批主体

> 状态：占位页已建立。旧版规划书提出“任务优先分配给岗位而非具体用户”；当前是否实现及匹配规则待核验。

## 1. 设计目的

将“谁有权限”与“谁应处理某项工作”分开说明：RBAC 解决可访问资源，岗位和审批主体解决待办归属、执行责任与组织替换。

## 2. 参考设计

| 主题 | 参考原则 |
| --- | --- |
| 任务归属 | 任务可按岗位分配，人员变动时由岗位成员承接，不应只绑定个人。 |
| 匹配条件 | 岗位可与地点、物料/物料组、项目、分类等业务条件关联。 |
| 审批主体 | 可按用户、角色、部门负责人、申请人等规则确定。 |
| 冲突处理 | 多条匹配规则需有优先级、默认规则与可解释的命中结果。 |

## 3. 待核验内容

- 当前岗位、角色、用户、部门之间的模型与维护入口。
- 任务派发、转派、委托、超时升级与关闭的实际行为。
- 审批流配置、审批意见、撤回及审计记录。
