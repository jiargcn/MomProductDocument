# 数据权限与决策权限

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：系统管理员、实施顾问及需要区分“功能权限”与“数据范围/高风险动作”的人员。

## 业务目的与适用范围

数据权限回答：在已经能够进入某功能的前提下，用户能看到或操作哪些数据范围。决策权限回答：哪些高风险动作（如审批、关闭、撤销、人工干预）还需要额外约束。

本页与[RBAC 权限模型](01-RBAC权限模型.md)互补：RBAC 解决功能入口，本页解决范围与高风险动作边界。当前第二阶段优先完成了 RBAC 主链；本页先给出可依赖的边界说明，细项规则与维护步骤作为下一刀回填。

## 与 RBAC 的分工

| 问题 | 先看哪里 |
| --- | --- |
| 为什么没有某菜单/按钮 | [RBAC 权限模型](01-RBAC权限模型.md) |
| 为什么能进页面但看不到某些单据/组织数据 | 本页（数据范围） |
| 为什么按钮在、状态也对，仍不能审批/撤销 | 本页（决策动作）+ 对应业务页状态规则 |

## 当前可确认的边界

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 给了菜单仍看不到数据 | 检查数据范围，不要只加菜单。 |
| 想用超管验证业务规则 | 超管可能绕过范围限制，验收业务规则应用专用角色。 |
| 审批按钮在但不能过 | 同时核验决策权限、单据状态和业务页规则。 |
