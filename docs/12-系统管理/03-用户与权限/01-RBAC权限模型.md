# RBAC 权限模型

> 状态：占位结构已建立。资源类型与授权原则来自旧版规划书；当前 System 模块实现待核验。

## 1. 设计目的

RBAC 解决“主体能访问什么资源和动作”。它不替代数据权限、岗位派工和审批规则，三者应分别说明后再组合使用。

## 2. 参考设计

| 主题 | 参考原则 |
| --- | --- |
| 授权主体 | 用户可关联多个角色，角色与个人授权可合并计算。 |
| 作用域 | 用户、角色与资源可按站点/工厂等范围生效。 |
| 资源类型 | 可区分功能（菜单/页面）、动作（按钮/API）、属性（字段）和记录（数据对象）权限。 |
| 最终权限 | 最终可用权限由角色、个人授权、数据范围和业务前置条件共同决定。 |

## 3. 待填写任务
