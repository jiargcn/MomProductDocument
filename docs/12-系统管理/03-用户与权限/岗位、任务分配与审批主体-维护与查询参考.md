# 岗位、任务分配与审批主体 - 维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 配套主文档：[岗位、任务分配与审批主体](03-岗位、任务分配与审批主体.md)。本页只保留维护入口、配置项和查询联查。

## 维护入口

| 入口 | 作用 | 说明 |
| --- | --- | --- |
| 岗位管理 | 维护岗位主数据，配置与用户、库位、物料等关联。 | System 主入口。 |
| 用户管理 | 为用户分配岗位。 | 与角色分配相互独立。 |
| 对应 WMS 业务页 | 查看任务领取人、按区域过滤后的任务列表。 | 派发/领取细则以业务页为准。 |

## 配置项说明

| 配置项 | 业务含义 | 配置时注意 |
| --- | --- | --- |
| 岗位基本信息 | 岗位代码、名称、状态等。 | 命名与现场班组习惯对齐，便于交接。 |
| 用户—岗位 | 哪些人属于该岗位。 | 调动人员时同步调整，避免旧区域权限残留。 |
| 岗位—库位 | 岗位可作业/可见的库位范围。 | 与角色部门数据范围不是同一机制。 |
| 岗位—物料 | 岗位关联的物料范围。 | 是否被某业务消费，回业务页确认。 |
| 任务领取人 | 业务任务上的承接人。 | 在业务单据上维护/产生，不是岗位页通用派工字段。 |

## 建议配置顺序

1. 先维护库位等主数据。
2. 再建立岗位并配置库位（及按需物料）。
3. 再把用户挂到岗位。
4. 确认用户角色菜单已具备对应业务入口。
5. 用非超管现场账号验证任务可见范围。

## 查询与联查

| 场景 | 建议看什么 | 联查建议 |
| --- | --- | --- |
| 用户进不了某库区任务 | 用户岗位、岗位库位是否覆盖。 | 岗位页、库位主数据、WMS 任务页。 |
| 人员调岗后仍看到旧区域 | 旧岗位是否未解除。 | 用户—岗位关系。 |
| 有岗位仍无菜单 | 角色菜单是否缺失。 | [RBAC 权限模型-维护与查询参考](RBAC权限模型-维护与查询参考.md)。 |
| 部门范围与库位范围混淆 | 分别检查角色数据范围与岗位库位。 | [数据权限与决策权限-维护与查询参考](数据权限与决策权限-维护与查询参考.md)。 |

## 验收检查清单

- [ ] 用户已分配目标岗位，且岗位库位覆盖验收库区。
- [ ] 非目标库区任务在非超管账号下不可见或不可作业（按业务实际规则）。
- [ ] 调整岗位库位后，可见性变化符合预期。
- [ ] 未把“通用审批主体配置”写成必配步骤（除非目标环境已启用对应能力）。
