# 数据权限与决策权限 - 维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 配套主文档：[数据权限与决策权限](02-数据权限与决策权限.md)。本页只保留维护入口、配置项和查询联查。

## 维护入口

| 入口 | 作用 | 说明 |
| --- | --- | --- |
| 角色管理 → 数据权限 | 为角色设置数据范围类型，自定义时勾选部门。 | 主配置入口。 |
| 用户管理 | 确认用户所属部门与启用角色。 | 范围计算依赖用户组织与角色。 |
| 部门管理 | 维护组织树。 | 本部门及下级、指定部门都依赖树结构。 |

## 配置项说明

| 配置项 | 业务含义 | 配置时注意 |
| --- | --- | --- |
| 数据范围类型 | 全部 / 指定部门 / 本部门 / 本部门及下级 / 仅本人。 | 先选类型，再决定是否需要部门树。 |
| 指定部门列表 | 自定义范围时生效的部门集合。 | 规则会并入用户本部门；仍应显式确认组织归属。 |
| 用户所属部门 | 本部门、本部门及下级、仅本人的计算基础。 | 部门为空时范围可能异常收窄。 |
| 用户启用角色 | 多角色范围取并集。 | 排查“看得过多”时逐角色检查。 |

## 建议配置顺序

1. 先维护部门树与用户归属。
2. 再配置角色功能菜单（见 RBAC）。
3. 再为角色设置数据范围。
4. 最后用非超管业务账号验证目标页面可见范围。

## 查询与联查

| 场景 | 建议看什么 | 联查建议 |
| --- | --- | --- |
| 看不见某部门数据 | 角色范围类型、自定义部门、用户部门。 | 角色页、用户页。 |
| 多角色范围过大 | 各角色是否存在“全部”或过宽自定义。 | 用户角色列表。 |
| 业务单据不受影响 | 该业务是否接入部门数据权限。 | 业务页、问题总账。 |
| 库位相关仍不对 | 是否应查岗位区域权限。 | [岗位、任务分配与审批主体](03-岗位、任务分配与审批主体.md)。 |

## 验收检查清单

- [ ] 非超管账号下，“本部门”与“全部”可见结果有明显差异。
- [ ] 自定义部门范围下，目标部门数据可见，非目标部门不可见。
- [ ] 多角色并集行为符合预期。
- [ ] 未把“仓库/库位隔离”误配成仅靠角色部门范围解决。
