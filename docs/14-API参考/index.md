# API 参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；集成开发（顺带）。

## 业务目的与适用范围

联调卡在「路径找不到、鉴权失败、业务页说有动作但 Swagger 对不上」时，先来本组：讲清**如何发现接口、如何按约定调用、失败时联查哪里**。业务「为什么有这个动作」仍以各业务模块页为准；平台调用轨迹与重试见[数据交换与集成可靠性](../03-基础设施/09-数据交换与集成可靠性.md)。

读完本页，应能：按「约定 → 模块索引 → 场景矩阵 → 业务页相关接口」顺序定位；并知道本站**不**提供全量 OpenAPI 抄本——完整字段以环境 Knife4j/Swagger 为准。

本轮 **不** 把全站 Controller 逐条抄成手册（体量过大且易过期）。此处沉淀已证实的组织方式、抽样路径与挂接矩阵。

!!! example "写实示例：发现一条接口"
    **给定：** 要确认「拉取当前用户权限信息」路径。
    **做法：** 打开调用约定页确认认证边界 → 在模块索引「System」节找到 `GET /system/auth/get-permission-info` → 到环境 Knife4j 核字段 → 业务含义回 [租户与认证](../12-系统管理/01-租户与认证/index.md)。不要在本站臆造 Header 名或网关前缀的唯一真值。

### 建议验证点

- 能在目标环境打开对应服务的 Knife4j/Swagger（可能按服务分别入口）。
- 能从模块索引落到一条抽样路径，并在 Swagger 核到同名方法。
- 权限失败时能联查 RBAC / 权限标识，而不是只看前端按钮。
