# 模块接口索引

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；集成开发（顺带）。
> 本页按模块给出**代表性**管理端路径与业务文档入口；**非全量清单**。字段级定义以环境 Knife4j 为准。

## 业务目的与适用范围

知道业务在哪个模块，但不确定管理端路径落在哪——用本页做**抽样索引**。它帮助快速跳到业务文档，并给出已取证的代表性路径；不替代 Swagger，也不编造未证实模块（如 PS）的伪路径。

读完本页，应能：按模块找到入口与抽样路径；知道补全量目录的正确做法是导出/浏览 Knife4j 后回填业务页。

## 如何使用本表

| 你要… | 做法 |
| --- | --- |
| 做页面联调 | 先打开业务模块页，再在本表找代表性路径，最后到 Knife4j 看字段。 |
| 做外部集成 | 先看[跨系统集成场景](03-跨系统集成场景.md)，再回本表与业务页。 |
| 补全量目录 | 按服务导出/浏览 Knife4j，把高频接口回填到业务页「相关接口」表。 |
| 找调用约定 | [调用约定与文档发现](01-调用约定与文档发现.md) |

路径均为控制器映射的**相对路径**（不含环境主机与可能的网关前缀）。

!!! example "写实示例"
    **给定：** 要联调 MES 工单下发。
    **做法：** 打开 [计划管理](../06-MES-生产管理/03-计划管理/index.md) 看业务规则 → 本表 MES 节见 `/mes/work-order` 族 → Knife4j 核 `publish` 等具体方法与入参。不要把本表当成 SDK 或全量目录。

### 建议验证点

- 任选本表一条抽样路径，能在对应服务 Knife4j 中找到同名或同前缀方法。
- PS 节保持「无伪路径」；需要执行层接口时转到 MES。
- 数采路径变更时，以环境 iotgw 标签为准并回写证据，不在本表臆造新族。

## 当前限制

- 本表是索引不是 SDK。
- 路径随版本变更时，以环境 Swagger 为准并回写证据。
- Finance 等未单列章节，有需要时按同样方式增补。
