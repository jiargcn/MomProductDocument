# 跨模块追溯与业务闭环

> 状态：占位页已建立。下列闭环来自旧版规划书的覆盖范围，实际单据关系待当前源码核验。

## 本页解决什么问题

排障与验收常卡在「从结果回不到来源」：库存变了找不到收货，检验结论对不上到货，安灯关了却不知道影响了哪张工单。本页强调以 **业务对象**（物料、批次、包装、单据、设备、异常）为主线做跨模块追溯，而不是按菜单页面拼接。

读完应能判断：某条端到端链路应覆盖哪些模块与关键事实；写业务页时应保留可追溯编号与撤销/冲销后的链保留说明；未核验的自动闭环不得写成现状。端到端主路径见[核心业务流程](11-核心业务流程.md)。

## 如何使用本页

| 你的目的 | 建议做法 |
| --- | --- |
| 理解应能追溯什么 | 读「参考闭环」表 |
| 写业务页时引用 | 点名本业务保留的追溯键（单号、批次、包装等）与上下游模块；细节写业务页 |
| 排障时对照 | 按闭环选模块入口，再沿单号/批次正向或反向联查 |
| 验收撤销/退货 | 确认冲销后追溯链是否仍可回查（规则待核验则标 `❓`） |

## 参考闭环

| 业务链路 | 涉及模块 | 应能追溯的关键事实 |
| --- | --- | --- |
| 采购至入库 | SCP、QMS、WMS | 需求/订单、供应商发货、到货检验、收货、上架、库存。 |
| 生产执行 | MES、WMS、QMS、EAM、ANDON | 计划/工单、投料、报工、消耗、设备状态、质量与异常。 |
| 发运与结算 | WMS、SCP、财务/外部系统 | 发货计划、拣配、出库、客户签收、结算或接口状态。 |
| 异常闭环 | ANDON、MES、EAM、QMS | 异常来源、响应人、处置、复位/关闭、影响业务对象。 |

## 自检或验证点

1. 从库存结果能否回查到业务记录与来源单据（挂接见[库存数据挂接模型](02-库存数据挂接模型.md)）？
2. 跨模块是否写清共用编号（内部单号、外部单号、批次/包装/设备）？
3. 是否把规划书闭环直接写成已全部自动打通（如报工 NG→QMS 仍见 `GAP-071`）？
