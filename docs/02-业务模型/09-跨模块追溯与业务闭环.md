# 跨模块追溯与业务闭环

> 状态：占位页已建立。下列闭环来自旧版规划书的覆盖范围，实际单据关系待当前源码核验。

## 1. 设计目的

将业务对象而非页面作为追溯主线：可从物料、批次、包装、单据、设备或异常反查其来源、处理、质量结论、库存变化和对外交换记录。

## 2. 参考闭环

| 业务链路 | 涉及模块 | 应能追溯的关键事实 |
| --- | --- | --- |
| 采购至入库 | SCP、QMS、WMS | 需求/订单、供应商发货、到货检验、收货、上架、库存。 |
| 生产执行 | MES、WMS、QMS、EAM、ANDON | 计划/工单、投料、报工、消耗、设备状态、质量与异常。 |
| 发运与结算 | WMS、SCP、财务/外部系统 | 发货计划、拣配、出库、客户签收、结算或接口状态。 |
| 异常闭环 | ANDON、MES、EAM、QMS | 异常来源、响应人、处置、复位/关闭、影响业务对象。 |

## 3. 待核验清单

- 跨模块使用的业务编号、外部编号、批次号、包装号和设备编号。
- 正向追溯与反向追溯的查询入口、权限和性能限制。
- 撤销、冲销、退货、返工、报废时的追溯链保留规则。
