# 库存管理精度与唯一粒度

> 状态：关键业务键已取得用户确认；其余精度模式与实现待核验。旧版规划书的粒度设计不得直接覆盖当前定义。
> 字段行为模式：本页覆盖 **P3（库存/余额选择）** 与 **P8（管理精度/采集）** 的共享语义。写法见[页面数据字典规范](04-页面数据字典规范.md)。

## 1. 设计目的

库存管理精度决定“什么被视为同一笔库存”，进而影响库存余额唯一性、事务汇总、扫码、包装、盘点和追溯。

## 2. 当前已确认的余额业务键

当前确认的库存余额业务键由以下维度组成：

| 维度（业务名） | 在唯一粒度中的角色 |
| --- | --- |
| 物料 | 区分不同物料 |
| 批次 | 区分批属性 |
| 托盘号 | 区分托盘 |
| 包装号 | 区分包装 |
| 库存状态 | 区分可用/质检/冻结等状态口径 |
| 库位 | 区分存放地点 |

> 差距标记：该业务键已确认，但数据库是否存在等价的组合唯一约束仍待技术核验（`GAP-005`）。

## 3. 选择库存余额时必须写清什么（P3）

业务页的选择器范围表至少回答：

| 必须说明 | 示例问法 |
| --- | --- |
| 过滤粒度 | 按物料？是否还按批次/托盘/包装/库位/库存状态？ |
| 可用量口径 | 可用 = 余额 − 预留/预期？是否含在途？❓ 各业务可能不同 |
| 选中后锁定 | 是否锁定库位、批次、包装、数量上限？ |
| 选不到原因 | 无余额、状态不符、库位不在范围、权限外、粒度维度未采集 |

禁止只写“弹窗选择库存”。通例见[通用选择器过滤惯例](12-通用选择器过滤惯例.md)。

## 4. 现场采集与精度（P8）

| 维度 | 何时必须采集 | 缺失时的业务风险 |
| --- | --- | --- |
| 包装 / 批次 / 托盘 / 库位 / 库存状态 | 物料管理精度或任务配置要求时 | 无法完成收发、或入账粒度不足导致追溯混淆 |
| 数量 | 始终 | 无法形成库存事务与余额更新 |

采集入口可能是 Web 或 PDA；具体开关以任务/业务类型配置为准（见采购收货等业务页）。

## 5. 详情联查注意

余额详情联查库存事务时，若过滤条件未覆盖完整业务键，可能混入其它托盘/状态/库位的事务（`GAP-022`）。培训与排查应以完整粒度条件为准。

## 6. 参考设计与待核验边界

旧版规划书曾按数量、批次、包装和唯一码区分不同管理精度。当前系统是否支持这些模式、各模式是否切换，以及其与上述余额业务键的关系，均须依据 `dev` 基线核验。
