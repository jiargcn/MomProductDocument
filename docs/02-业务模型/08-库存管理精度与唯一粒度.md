# 库存管理精度与唯一粒度

> 状态：关键业务键已取得用户确认；其余精度模式与实现待核验。旧版规划书的粒度设计不得直接覆盖当前定义。

## 1. 设计目的

库存管理精度决定“什么被视为同一笔库存”，进而影响库存余额唯一性、事务汇总、扫码、包装、盘点和追溯。

## 2. 当前已确认的余额业务键

当前确认的库存余额业务键由以下字段组成：`batch_id`、`pallet_number`、`package_number`、`item_code`、`inventory_status`、`location_code`。

> 差距标记：该业务键已确认，但数据库是否存在等价的组合唯一约束仍待技术核验，详见产品差距总账中的 GAP-005。

## 3. 参考设计与待核验边界

旧版规划书曾按数量、批次、包装和唯一码区分不同管理精度。当前系统是否支持这些模式、各模式是否切换，以及其与上述余额业务键的关系，均须依据 `dev` 基线核验。
