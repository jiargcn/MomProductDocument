# 单据类型、业务类型与单据配置

> 状态：占位页已建立。以下是旧版规划书提出的设计方向；当前配置项、字段和生效逻辑待源码与测试环境核验。

## 1. 设计目的

将“同一单据模型在不同业务中的差异”收敛为可配置的业务类型、事务类型和单据设置，避免每个业务在代码或页面中重复实现编号、状态和自动动作。

## 2. 参考设计

| 配置主题 | 参考含义 |
| --- | --- |
| 业务类型 | 定义单据适用场景、来源、目标、状态及可执行动作。 |
| 事务类型 | 定义库存或业务事实的增减方向、原因和影响。 |
| 单据编号 | 定义编号规则、段号和生成时点。 |
| 自动动作 | 申请自动提交、自动审批、自动创建任务、直接生成记录等开关。 |
| 任务控制 | 库位、数量上下限、连续扫描、部分完成等现场控制规则。 |
| 接口设置 | 记录生成后是否产生对外交换数据及交换类型。 |
