# 单据类型、业务类型与单据配置

> 状态：共享语义页（2026-07-22 按共享语义补强）。配置项是否在每个消费业务生效，必须逐业务验证，不得写成全站已闭合矩阵。
> 相关正式页：DBC「业务类型 / 单据设置 / 单据开关 / 规则管理」。

## 本页解决什么问题

同一套「申请—任务—记录」在不同业务里，编号、自动审批、能否改数量/库位、库存方向往往不同。本页说明这些差异应收敛到 **业务类型、单据设置、单据开关、规则、事务类型** 等配置，而不是在每个业务页重复发明一套。

读完应能判断：现场行为异常时先查哪类配置、改配置可能影响什么、哪些结论已证实、哪些仍是 `❓` / GAP；写业务页时应点名「读了哪份配置」，不要假设全站同一默认值。

字段行为写法见[页面数据字典规范](04-页面数据字典规范.md)（本页配置对象多属 P1 / P12）。

## 如何使用本页

| 你的目的 | 建议做法 |
| --- | --- |
| 理解配置如何分流行为 | 读「配置对象如何分流行为」表 |
| 写业务页时引用 | 写明本业务读取的业务类型/开关/规则；任务控制类配置与现场门禁对照 |
| 排障时对照 | 先业务类型/任务控制 → 再业务页采集与动作门禁；勿默认 Web 与 PDA 一致 |
| 验收配置影响 | 用文末自检；缺失/停用开关的默认行为以 GAP 为准，勿脑补 |

## 配置对象如何分流行为（P1 / P12）

| 配置对象 | 已能说明的业务作用 | 改变后可能影响 | 证实边界 |
| --- | --- | --- | --- |
| 业务类型 | 场景分类；可约束物料/库存/地点范围、入出库与在途、自动处理、任务控制、编号与打印关联 | 申请/任务/记录路径、现场可否改量改位、库存事务方向、打印 | 字段是否被各 WMS/MES 页完整消费 ❓；见业务类型页与 `GAP-060` |
| 单据设置 | 单据编号规则（流水、日期格式等）及与业务类型的关联文本 | 单号生成 | 业务类型存在性校验不足（`GAP-061`）；导入入口不一致 |
| 单据开关 | 特定代码控制校验或自动动作 | 相关业务的校验/自动行为 | `available=false` 或缺失时各调用方默认行为矩阵未建（`GAP-062`） |
| 规则管理 | 按业务类型/分类/版本挂接条件与节点 | 自动化与库存精度相关行为 | 前后端模型字段不一致（`GAP-063`） |
| 事务类型 | 库存或业务事实的增减方向、原因 | 库存事务入账口径 | 编码与消费方 ❓ 待核验 |

## 与现场任务控制的关系

业务类型中的任务控制类配置（如是否允许改数量、改库位、连续扫描、部分完成）会被收货等现场任务读取。排查「现场能不能改」时：

1. 先看业务类型 / 任务配置；
2. 再看具体业务页的采集要求与状态—动作门禁（对应规范中的精度类与门禁类写法）；
3. 不要假设 Web 与 PDA 入口一致（例：采购收货 `GAP-010`）。

自动提交 / 自动通过 / 自动执行等策略与申请—任务链路的关系，见[申请、任务与记录模型](01-申请任务记录模型.md)。

## 参考设计（未逐项证实前仅作对照）

| 配置主题 | 参考含义 |
| --- | --- |
| 业务类型 | 定义单据适用场景、来源、目标、状态及可执行动作。 |
| 事务类型 | 定义库存或业务事实的增减方向、原因和影响。 |
| 单据编号 | 定义编号规则、段号和生成时点。 |
| 自动动作 | 申请自动提交、自动审批、自动创建任务、直接生成记录等开关。 |
| 任务控制 | 库位、数量上下限、连续扫描、部分完成等现场控制规则。 |
| 接口设置 | 记录生成后是否产生对外交换数据及交换类型。 |

上表是对照用参考设计，**不等于**当前基线已全部落地；落地结论以「配置对象如何分流行为」表的证实边界与各业务页为准。

## 自检或验证点

1. **能点名配置：** 业务页是否写清「本业务读哪份业务类型 / 开关 / 规则」？
2. **能说清影响：** 改任务控制（改量、改位、连续扫描等）后，现场期望如何变？
3. **边界诚实：** 是否把「可能影响」写成了「全站已证实」？`GAP-060`～`GAP-063` 相关结论是否仍保留？
4. **入口差异：** Web 与 PDA 行为不一致时，是否单独说明而非默认相同？
