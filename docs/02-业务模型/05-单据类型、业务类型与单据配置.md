# 单据类型、业务类型与单据配置

> 状态：共享语义页（2026-07-22 按字段行为模式补强）。配置项是否在每个消费业务生效，必须逐业务验证，不得写成全站已闭合矩阵。
> 相关正式页：DBC「业务类型 / 单据设置 / 单据开关 / 规则管理」。

## 1. 设计目的

将“同一单据模型在不同业务中的差异”收敛为可配置的业务类型、事务类型和单据设置，避免每个业务在代码或页面中重复实现编号、状态和自动动作。

## 2. 配置对象如何分流行为（P1 / P12）

| 配置对象 | 行为模式 | 已能说明的业务作用 | 改变后可能影响 | 证实边界 |
| --- | --- | --- | --- | --- |
| 业务类型 | P12 | 场景分类；可约束物料/库存/地点范围、入出库与在途、自动处理、任务控制、编号与打印关联 | 申请/任务/记录路径、现场可否改量改位、库存事务方向、打印 | 字段是否被各 WMS/MES 页完整消费 ❓；见业务类型页与 `GAP-060` |
| 单据设置 | P12 | 单据编号规则（流水、日期格式等）及与业务类型的关联文本 | 单号生成 | 业务类型存在性校验不足（`GAP-061`）；导入入口不一致 |
| 单据开关 | P1 / P12 | 特定代码控制校验或自动动作 | 相关业务的校验/自动行为 | `available=false` 或缺失时各调用方默认行为矩阵未建（`GAP-062`） |
| 规则管理 | P12 | 按业务类型/分类/版本挂接条件与节点 | 自动化与库存精度相关行为 | 前后端模型字段不一致（`GAP-063`） |
| 事务类型 | P12 | 库存或业务事实的增减方向、原因 | 库存事务入账口径 | 编码与消费方 ❓ 待核验 |

## 3. 与现场任务控制的关系

业务类型中的任务控制类配置（如是否允许改数量、改库位、连续扫描、部分完成）会被收货等现场任务读取。培训时应：

1. 先看业务类型/任务配置；
2. 再看具体业务页的 P8 采集表与 P9 动作门禁；
3. 不要假设 Web 与 PDA 入口一致（例：采购收货 `GAP-010`）。

## 4. 参考设计（未逐项证实前仅作对照）

| 配置主题 | 参考含义 |
| --- | --- |
| 业务类型 | 定义单据适用场景、来源、目标、状态及可执行动作。 |
| 事务类型 | 定义库存或业务事实的增减方向、原因和影响。 |
| 单据编号 | 定义编号规则、段号和生成时点。 |
| 自动动作 | 申请自动提交、自动审批、自动创建任务、直接生成记录等开关。 |
| 任务控制 | 库位、数量上下限、连续扫描、部分完成等现场控制规则。 |
| 接口设置 | 记录生成后是否产生对外交换数据及交换类型。 |
