## 台账（DBC）

| 应核对 | 提醒 |
| --- | --- |
| 设备编号、名称、类型、状态 | 编号被全模块引用，慎改。 |
| 工厂/车间/产线/工序/工位、责任人 | 影响派工与统计。 |
| 供应商/生产商、日期与寿命 | 主数据口径见 DBC 文档。 |

操作细则见 DBC [设备台账管理-维护与查询参考](../../04-DBC-主数据管理/07-设备管理/05-设备台账管理-维护与查询参考.md)。

## 报修与维修

| 应核对 | 提醒 |
| --- | --- |
| 设备、故障类型、描述、紧急度、是否停机 | 与现场一致；可附图。 |
| 来源工单（保养/巡检/点检/开拉） | 异常转报修时保留来源号。 |
| 报修状态推进 | 暂存→提交→审批→创建/确认。 |
| 维修：派发、接单、执行、验证 | 双验证（维修/报修）都可能出现。 |
| 自动派工/执行/验证 | 以单据与环境配置为准。 |

### 常用动作

报修：新增、暂存、提交、撤回、审批、确认、作废。  
维修：派发、转单、接单、执行/暂存、提交验证、完成、暂停、作废。

## 停机与关联

| 应核对 | 提醒 |
| --- | --- |
| 停机起止、原因 | 勿与 ANDON 呼叫时长混算，除非现场已规定同一口径。 |
| 是否同步开 ANDON | 未证实自动联动时，按组织流程手工协同。 |

## 最小验收清单

- [ ] 从 EAM 菜单打开设备台账，确认落地为 DBC 页且可改归属。
- [ ] 完成一笔报修审批并生成维修工单。
- [ ] 工单经历接单→执行→验证→完成。
- [ ] 来自巡检异常的报修能看到来源类型/单号（若启用）。
- [ ] 维修领用备件时能联查备件出库/WMS 同步状态（若启用同步）。
