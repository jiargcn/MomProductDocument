# 巡检保养-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[巡检保养](index.md)。计划→工单→记录主线、转报修边界及建议验证点以主文档为准；本页给出计划/工单动作入口。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 先理解计划如何出工单、异常如何出口 | 返回[巡检保养](index.md) |
| 配方案和检查项 | [基础数据-维护与查询参考](../01-基础数据/基础数据-维护与查询参考.md) |
| 建/审计划 | “计划” |
| 派工与执行 | “工单与记录” |
| 异常转报修（非 ANDON、非 QMS） | “异常出口” |
| PDA | [终端操作](../06-终端操作/终端操作-维护与查询参考.md) |
| 开验证或「不出工单」排障 | 返回[巡检保养 · 建议验证点](index.md#建议验证点) |

## 页面与业务对象

| 菜单入口 | 用途 |
| --- | --- |
| 保养计划 / 保养工单 / 保养记录 | 保养链。 |
| 巡检计划 / 巡检工单 / 巡检记录 | 巡检链。 |
| 点检计划 / 点检工单 / 点检记录 | 点检链。 |
| 开拉计划/工单/记录（若启用） | 开拉链。 |
| EAM工作日历 | 排程。 |

## 计划

| 应维护/确认 | 提醒 |
| --- | --- |
| 编号、名称、设备、类型、方案编码 | 方案必须存在且适用。 |
| 周期 / cron、提前生成天数、生成周期 | 决定出单节奏。 |
| 班组、组织位置 | 派工范围。 |
| 审批与自动策略 | 未通过不应假定已排程。 |

### 常用动作

新增/修改、提交审批、更新计划、按环境启用定时生成。

## 工单与记录

| 应核对 | 提醒 |
| --- | --- |
| 状态：待接单→待执行→执行中→待验证→已完成 | 超期会到已逾期。 |
| 计划号、方案、计划/实际时间 | 与现场一致。 |
| 是否要求拍照 | 工单字段可要求图片。 |
| 验证人与验证意见 | 未验证勿当完成。 |

### 常用动作

派发、接单、开始执行、提交、验证、退单、作废。

## 异常出口

| 场景 | 建议操作 |
| --- | --- |
| 检查项不合格需维修 | 按组织流程转报修；来源类型可选保养/巡检/点检/开拉。 |
| 需产线响应 | 同步 ANDON 呼叫（若流程要求）；EAM 不替代响应链。 |
| 与 MES 开工点检冲突 | 分清：开工卡点配置在 MES，设备点检执行在 EAM。 |

## 最小验收清单

- [ ] 方案+计划审批后能生成至少一张工单。
- [ ] 工单可接单、执行、验证并产生记录。
- [ ] 逾期/作废路径可演示（权限允许时）。
- [ ] 异常样例可转到报修且带来源。
- [ ] PDA 与 Web 状态一致（抽查一条）。
