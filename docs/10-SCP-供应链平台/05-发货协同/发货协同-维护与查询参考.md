# 发货协同-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[发货协同](index.md)。主线判断（仓库无单先查推送等）见主文档；本页偏操作与字段细节。

## 快速定位

| 你的目的 | 先看哪里 |
| --- | --- |
| 弄清来源选择、数量上限与推送相关字段 | 「字段说明」 |
| 新建发货申请并形成 ASN | 「发货申请」→「发货记录」 |
| 查已发在途 / 某 ASN 是否已推仓 | 「发货记录」；失败再看「接口与收货申请标志」 |

| 核对是否已在下游建收货申请 | 「发货记录」中的已建标志与申请号 |
| 包装打印 / 标签 | 基础数据或标签与包装相关页（若启用） |

## 字段说明

### 字段说明总表

| 字段 | 业务作用 | 谁在何时维护/选择 | 业务约束摘要 | 影响哪些功能 | 变更或错选风险 |
| --- | --- | --- | --- | --- | --- |
| 订单号 / 计划号 | 可发来源 | 建申请时选择 | 已发布/可发且有剩余 | 申请明细、可发上限 | 来源错→超未交或选不到 |
| 申请状态 | 提交/处理门禁 | 状态机推进 | 通用请求状态 | 能否提交、完成 | 非法状态强行提交 |
| 到仓 / 月台 / 时间窗 | 到货安排 | 建申请 | 仓→月台级联 | 仓库收货安排 | 到错仓 |
| 明细量 / 包装 / 批次 | 发运内容 | 建行；包装/批次按物料 | ≤未交量 | ASN、收货核对 | 超发、漏采核对维度 |
| ASN | 仓库识别键 | 记录形成后生成/带入 | 有记录才有 ASN | WMS 识别到货 | 有 ASN 仍无仓单→先查推送 |
| 已建收货申请标志 / 号 | 防重复建单 | 下游建单后回写（启用时） | 标志 + 申请号；启用条件 ❓ | 建收货申请 | 重复建单；空标志≠推送失败 |
| 推送结果 | 协同是否落地 | 接口登记 | 成功才有仓库可收对象 | WMS 可见性 | 假定已入库或盲目重提申请 |

### 选择器可选范围

| 选择字段 | 选择对象 | 可选范围 | 范围依赖 | 选中后带回 | 选不到时通常原因 |
| --- | --- | --- | --- | --- | --- |
| 采购订单 / 行 | 订单行 | 宜 PUBLISHED 且剩余可发；精确状态集 ❓（`FSEM-006`） | 供应商、头状态、行累计 | 物料、可发量 | 未发布、已发完、已关闭 |
| 要货计划 / 行 | 计划行 | 宜 ACCEPTED（或可发）且有剩余；精确状态集 ❓（`FSEM-006`） | 供应商、计划状态、已发量 | 物料、可发量 | 未接受、已发完、已关闭 |
| 到仓 / 月台 | 仓库 / 月台 | 有效仓储主数据；仓→月台 | 上游仓库建模 | 名称等 | 未建模、级联断 |

## 发货申请

| 应核对 | 提醒 |
| --- | --- |
| 订单号 / 计划号、供应商、到仓 / 月台、计划到货 | 与计划 / 订单一致；到错仓仓库按错误落点安排。 |
| 明细数量、包装、批次 | 不超过可发量；包装/批次影响收货核对。 |
| 承运、车牌、运输方式 | 入厂核对。 |

### 常用动作

新增、提交 / 同意 / 处理（按状态机）、关闭；已生成记录后勿随意删改物料行。

## 发货记录

| 应核对 | 提醒 |
| --- | --- |
| ASN、申请号、执行时间、状态 | 跟踪主键；先确认 ASN 是否形成。 |
| 是否已建收货申请标志 / 申请号 | 是否已在下游建收货申请；启用条件见 `GAP-017`。 |
| 撤销 | 可能触发重新推送 WMS，需查接口；勿只撤 SCP 侧却假定仓侧已消。 |

## 接口与收货申请标志

| 步骤 | 关注点 |
| --- | --- |
| 推送后 | 接口信息中 SCP-WMS、发货主 / 明细是否成功。 |
| 失败 | 修数据后按环境提供的重推 / 撤销重发路径处理；勿直接改 WMS 库存。仓库无单时优先查本表，而不是立刻再建一张申请。 |
| 收货 | 到 WMS 采购收货执行；结果回看采购跟踪。 |
| 已建收货申请 | 标志有值时避免重复建单；标志为空先区分「未启用自动建单」与「推送/建单失败」。 |

## 最小验收清单

- [ ] 基于已接受计划或已发布订单成功创建发货申请。
- [ ] 形成发货记录并产生 ASN。
- [ ] 接口登记成功；WMS 侧可见对应到货 / 收货来源（按环境）。
- [ ] 推送失败场景可定位接口信息，并按重推 / 撤销重发恢复可见性。
- [ ] 计划 / 订单已发数量增加。
- [ ] 超可发量被拒绝或提示。
