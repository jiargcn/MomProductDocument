# 发货协同-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[发货协同](index.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 新建 ASN | “发货申请” |
| 查已发在途 | “发货记录” |
| 推送失败 | “接口与收货申请标志” |
| 包装打印 | 基础数据/标签与包装相关页 |

## 页面与业务对象

| 菜单入口 | 用途 |
| --- | --- |
| 供应商发货申请 | 创建/提交发货。 |
| 供应商发货记录 | 已形成 ASN 的执行结果。 |
| 供应商包装（若启用） | 包装辅助。 |

## 发货申请

| 应核对 | 提醒 |
| --- | --- |
| poNumber / ppNumber、供应商、到仓/月台、计划到货 | 与计划/订单一致。 |
| 明细 qty、包装、批次 | 不超过可发量。 |
| 承运、车牌、运输方式 | 入厂核对。 |

### 常用动作

新增、提交/同意/处理（按状态机）、关闭；已生成记录后勿随意删改物料行。

## 发货记录

| 应核对 | 提醒 |
| --- | --- |
| asnNumber、requestNumber、执行时间、状态 | 跟踪主键。 |
| purchasereceiptRequestFlag / Number | 是否已在下游建收货申请。 |
| 撤销 | 可能触发重新推送 WMS，需查接口。 |

## 接口与收货申请标志

| 步骤 | 关注点 |
| --- | --- |
| 推送后 | 接口信息中 SCP-WMS、发货主/明细是否成功。 |
| 失败 | 修数据后按环境提供的重推/撤销重发路径处理；勿直接改 WMS 库存。 |
| 收货 | 到 WMS 采购收货执行；结果回看采购跟踪。 |

## 最小验收清单

- [ ] 基于已接受计划或已发布订单成功创建发货申请。
- [ ] 形成发货记录并产生 ASN。
- [ ] 接口登记成功；WMS 侧可见对应到货/收货来源（按环境）。
- [ ] 计划/订单已发数量增加。
- [ ] 超可发量被拒绝或提示。
