# 要货计划

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；计划员、供应商（顺带）。操作细节见[要货计划-维护与查询参考](要货计划-维护与查询参考.md)。

## 业务目的与适用范围

计划已经「接受」了却发不了货——大概率是状态没走到 ACCEPTED，或已发量已经用满。要货计划（采购计划）处在协同链中段：承接预测（及订单/导入），把可执行供应窗口落到供应商可接受的计划单；其后由[发货协同](../05-发货协同/index.md)扣减已发量。计划含仓库/月台、时间窗、交期、关联采购订单号等；状态含 NEW、PUBLISHED、ACCEPTED、REJECTED、CLOSED、COMPLETED，支持自动发布/自动接受开关；客户长期计划作为另一类计划输入；计划主/明细可通过 SCP↔WMS 接口推送。

读完本页，应能判断：

1. 「能否发货」先看计划是否 ACCEPTED，再看行已发量是否用满；
2. 推送 WMS 成功≠库存已变；
3. 本页负责**可发计划与数量窗口**，ASN 物流与仓内收货不在此展开。

## 一笔计划如何协同

```mermaid
flowchart LR
    A["预测/订单/导入"] --> B["采购计划 NEW"]
    B --> C["发布 PUBLISHED"]
    C --> D["供应商接受 ACCEPTED"]
    D --> E["发货扣减 shippedQty"]
    E --> F["完成/关闭"]
```

新建（可来自预测汇总、关联 PO 或导入）→ 发布给供应商 → 接受（或自动接受）→ 发货引用并回写已发 → 完成/关闭。未接受前不宜当作可发来源（精确状态集见 `FSEM-006`）。

!!! example "写实示例：给定 → 期望"
    **给定：** 计划 PP-3001，供应商 V-01，关联 PO-2001；头 NEW→发布→供应商接受；行物料 A 计划量 100，已发 0；autoAccept 关闭。
    **期望：**

    1. PUBLISHED 后供应商可见；接受后状态 ACCEPTED，发货侧可引用该行（精确可选集 ❓ `FSEM-006`）。
    2. 发货 40 成功后行 shippedQty 增加；再超剩余可发量应被拒绝或提示。
    3. 若误开 autoAccept，发布后可能直接 ACCEPTED——培训须说明开关影响，改前确认在用单据。
    4. 接口推送计划主/明细成功只表示 WMS 侧可登记，不代表收货或库存已变（同实例 ❓ `GAP-017`）。

## 主对象

| 对象 | 业务含义 |
| --- | --- |
| 采购计划头 | 单号、供应商、时间窗、仓/月台、状态、交期、PO、联系人、自动发布/接受等。 |
| 采购计划行 | 物料、计划量、接受量、订购量、已发/未收/已收、行状态、交期排序等。 |
| 客户长期计划 | 消息号、视界起止、供应商/买方卖方编码、状态 NEW/CLOSED 等。 |

### 关键字段业务角色

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 警惕什么 |
| --- | --- | --- | --- |
| 计划状态 | 发布/接受/关闭门禁 | NEW→PUBLISHED→ACCEPTED→CLOSED/COMPLETED；可 REJECTED | 未接受不可当可发 |
| 自动发布/自动接受 | 跳过人工环节 | 头开关 | 误开导致未审即协同 |
| 供应商 / 仓库 / 月台 | 到货安排 | 有效主数据；仓→月台级联 | 月台错影响到货 |
| 时间窗 / 交期 | 供应窗口 | 窗口外计划需重排 | SLA 失真 |
| 关联采购订单号 | 与订单未交衔接 | 可选关联 | 无 PO 时发货约束不同 |
| 行计划量 / 已发量 | 可发上限 | 发货回写 shippedQty | 超发失败 |
| 推送 WMS | 计划同步 | 主/明细可推送；同实例 ❓（`GAP-017`） | 勿当库存已变 |

完整表见[维护与查询参考](要货计划-维护与查询参考.md)。

## 与发货 / WMS / MES 边界

本页负责**可发计划号与可发量窗口**；其后发货、仓内收货、结算各有专页。

| 协同方 | 本页负责 | 不在本页展开 |
| --- | --- | --- |
| 要货预测 | 消费已发布预测（按配置） | 预测版本协同细节 |
| 采购订单 | 关联 poNumber | 订单头状态机 |
| 发货协同 | 提供可发计划号 ppNumber 与可发量 | ASN 物流与推送细节 |
| WMS | 计划主/明细可推送 | 收货执行与库存事务 |
| 采购跟踪 / 发票结算 | — | 收货跟踪与开票 |
| MES | — | 是否下发生产：**未证实** |

## 建议验证点

- 发布后供应商可见；接受后状态 ACCEPTED，发货侧能引用该行。
- 未接受（或已关闭/完成）时，发货侧不宜再选为可发来源（精确集 ❓ `FSEM-006`）。
- 发货成功后计划行 shippedQty 增加；超剩余可发被拒绝或提示。
- autoPublish / autoAccept 开/关两种路径各走一单，核对是否跳过人工环节。
- 需要推送 WMS 时接口登记成功；勿用「推送成功」反证库存已变。

## 限制与待确认

- 驳回后重发流程与权限：待环境确认（`GAP-017`）。
- 客户长期计划与采购计划的强制汇总规则：待确认（`GAP-017`）。
- 下游发货可选计划精确状态集待测（`FSEM-006`）。
