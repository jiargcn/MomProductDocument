# 要货计划-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[要货计划](index.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 发布计划给供应商 | “采购计划” |
| 供应商接受/驳回 | “接受与数量” |
| 维护客户长期计划 | “客户长期计划” |
| 查开放计划数 | 供应商首页 openPurchasePlanCount |

## 页面与业务对象

| 菜单入口 | 用途 |
| --- | --- |
| 采购计划（purchasePlanMain） | 主计划协同。 |
| 客户长期计划（customerLongtermPlanMain） | 长期视界计划。 |

## 采购计划

| 应核对 | 提醒 |
| --- | --- |
| 供应商、时间窗、仓库、月台、交期 | 影响发货到仓/月台。 |
| 状态 NEW→PUBLISHED→ACCEPTED | 未接受前发货可能受限。 |
| autoPublish / autoAccept | 开启后减少人工步骤，培训需说明。 |
| 明细 planQty / acceptQty / shippedQty / notReceiveQty / receivedQty | 进度以累计为准。 |

### 常用动作

保存、发布、接受、驳回、关闭；发货后关注 shippedQty 回写。

## 客户长期计划

| 应核对 | 提醒 |
| --- | --- |
| 视界起止、供应商账号、买方/卖方编码 | 消息类计划输入。 |
| 状态 NEW / CLOSED | 关闭后勿再纳入新计划。 |

## 最小验收清单

- [ ] 发布计划后供应商可见；接受后状态 ACCEPTED。
- [ ] 发货后计划行 shippedQty 增加。
- [ ] 首页开放/全部计划计数与列表一致。
- [ ] 需要推送 WMS 时接口登记成功（查接口信息）。
