# 采购订单-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[采购订单](index.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 查开放订单 | “查询” |
| 发布给供应商 | “头与发布” |
| 核对行数量累计 | “明细行” |
| 离散订单入口 | 供应商管理下离散订单页面（若启用） |

## 页面与业务对象

| 菜单入口 | 用途 |
| --- | --- |
| 采购订单（supplierdeliver/purchaseMain） | 主协同订单。 |
| 离散采购订单（若启用） | 离散类型入口。 |

## 查询

| 要找什么 | 推荐条件 |
| --- | --- |
| 开放订单 | 状态非关闭/完成；供应商首页 openPurchaseCount。 |
| 某供应商全部订单 | 供应商编码 + 单号。 |
| 某物料未交 | 单号进详情看行 shipped/received。 |

## 头与发布

| 应核对 | 提醒 |
| --- | --- |
| 单号、供应商、类型 SCHEDULE/DISCRETE、交期 | 类型影响后续路径。 |
| 状态 READY→PUBLISHED | 未发布供应商通常不可见。 |
| 寄售、税率、业务类型、数据来源 | 结算与接口线索。 |

### 常用动作

保存、发布、关闭；删除仅在未产生下游发货/收货时谨慎操作。

## 明细行

| 应核对 | 提醒 |
| --- | --- |
| 行号、物料、订购量、单位、包装 | 与供应商物料一致。 |
| 单价、金额、超收比例 | 对账与超收校验。 |
| shippedQty / receivedQty / returnedQty / putawayQty | 只读累计，勿手工当库存改。 |
| 行状态 detailStatus | 控制是否可继续发货。 |

!!! example "📷 截图占位"
    订单详情头+行，标出已发/已收列。

## 最小验收清单

- [ ] 创建并发布一张订单，供应商门户可见。
- [ ] 行上订购量正确；发货后 shippedQty 增加。
- [ ] 首页开放/全部订单计数与列表过滤一致。
- [ ] 关闭后不再计入开放订单。
