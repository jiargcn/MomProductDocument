# 采购订单

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；采购员、供应商门户用户（顺带）。操作细节见[采购订单-维护与查询参考](采购订单-维护与查询参考.md)。

## 业务目的与适用范围

采购订单是 SCP 里企业与供应商之间的协同起点：采购方建单、发布后，供应商在门户查看并确认（或按计划策略自动接受）；订单头/行带着订购量与已发、已收、已退等累计，作为后续发货与跟踪的上限线索，也可向 WMS 登记推送。读完本页，应能判断：卡在「未发布、门户不可见」，还是「行已发完/已关闭，不能再发」。

订单类型含排程类与离散类；头状态见下文。旧稿「24 小时强制确认」等**未证实**，不以之为培训硬规则。

## 一笔订单如何协同

```mermaid
flowchart LR
    A["创建/导入订单"] --> B["发布（已发布）"]
    B --> C["供应商确认或策略自动接受"]
    C --> D["发货申请/记录"]
    D --> E["收货累计回写行"]
    E --> F["关闭/完成"]
```

!!! example "写实示例：给定 → 期望"
    **给定：** 订单 PO-SCP-1001，供应商 V-01，排程类；头为就绪；行 10 物料 A 订购 100。发布后供应商确认；随后发货 60、WMS 侧实收 58（脱敏数量，仅示意累计关系）。
    **期望：**

    1. 发布前供应商门户通常看不到该单；发布后进入可见/可确认范围。
    2. 确认（或策略自动接受）后，发货侧可引用该行；可发上限受订购量与已发累计约束。
    3. 发货推进后，行上已发累计增加；收货回写后已收累计增加（具体接口时序见边界与待确认）。
    4. 头关闭或完成后，一般不再计入「开放订单」，也不应再作为可发来源（精确状态集见 `FSEM-006`）。

### 主对象

| 对象 | 业务含义 | 使用者关心 |
| --- | --- | --- |
| 订单头 | 单号、供应商、类型、状态、下单/交期、税率、寄售、业务类型、数据来源等。 | 是否已发布、能否进入发货协同。 |
| 订单行 | 行号、物料、订购量、包装、单价/金额、已发/已收/已退/已上架、超收比例、行状态等。 | 未交数量、行是否仍可发。 |

## 状态与关键动作

头状态常见为：就绪、已发布、关闭、完成（英文码见维护参考，培训以中文业务结果为准）。

| 常见动作 | 业务结果 |
| --- | --- |
| 新增 / 修改 / 删除（受限） | 维护草稿或就绪数据。 |
| 发布 | 进入供应商可见 / 可确认范围。 |
| 确认 / 接受（供应商侧或自动） | 进入可发货协同。 |
| 关闭 / 完成 | 结束开放订单；首页「开放订单」统计据此聚合。 |

## 与计划 / 发货 / WMS / ERP 边界

一张订单会被计划、发货、WMS、ERP 各拿走一部分数据；下表说清本页负责什么：

| 协同方 | 本页负责 | 不在本页展开 |
| --- | --- | --- |
| 要货计划 | 可被计划引用订单号等 | 计划发布与接受状态机 |
| 发货协同 | 提供可发订单行与数量上限线索 | ASN 物流与推送细节 |
| WMS | 可通过接口推送订单主/明细 | 库内收货执行 |
| ERP | 保留 ERP 订单类型、行税码等线索 | ERP 主单权威与回写协议 |
| DBC | 引用供应商 / 物料编码 | 主数据治理 |

## 关键判断

| 判断点 | 应先确认什么 | 影响 |
| --- | --- | --- |
| 供应商看不到订单 | 头是否已发布、供应商用户是否绑定 | 门户无单 |
| 无法发货 | 行剩余可发量、行状态、计划关联 | 发货校验失败 |
| 数量对不上 | 行上已发 / 已收 / 已退累计 | 跟踪与对账偏差 |

### 关键字段业务角色

完整状态与选择器见[维护与查询参考](采购订单-维护与查询参考.md)。「可用供应商 / 可用物料」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)；本页只保留订单类型、发布状态与行累计等差异。

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 警惕什么 |
| --- | --- | --- | --- |
| 订单类型（排程 / 离散） | 协同路径分流 | 建单时选定；入口与类型需一致 | 类型错 → 后续入口不符 |
| 头状态 | 发布 / 关闭门禁 | 就绪 → 已发布 → 关闭 / 完成 | 未发布则供应商通常不可见 |
| 供应商 | 协同对象 | 可用供应商；门户绑定另论 | 绑错则门户无单 |
| 行物料 / 订购量 | 可发上限来源 | 与行累计对照 | 超发会被校验拦住 |
| 超收比例 | 收货容差线索 | 行级百分比 | 容差过宽或过严 |
| 寄售 / 业务类型 | 结算与策略线索 | 头字段 | 结算口径错 |
| 推送 WMS | 订单同步意图 | 接口类型已证实；是否同实例 ❓（`GAP-017`） | 勿把 WMS 菜单当成同一张单 |

| 选择字段 | 选择对象 | 可选范围（当前可写） | 范围依赖 | 选不到时通常原因 |
| --- | --- | --- | --- | --- |

## 建议验证点

- 就绪订单发布后，供应商门户可见；未发布时门户不可见。
- 确认（或自动接受）后，发货侧能引用该行；未确认策略下的行为以环境配置为准。
- 发货后行已发累计增加；关闭 / 完成后不再计入开放订单，且不宜再被选为可发来源。
- 排程类与离散类分别建一单，核对入口与后续发货路径是否一致。
- 行物料选不到时：先查物料可用与可采购用途（`FSEM-001`），再查供应商物料关系（`GAP-045`）。
- 推送 WMS 后，勿默认 WMS「采购订单」与 SCP 订单为同一实例（`GAP-017`）。

## 限制与待确认

- 发布后变更是否需供应商再确认：依赖环境配置，待联调。
- 与 ERP 双向同步时机与失败补偿：待确认（`GAP-017`）。
- WMS 侧「采购订单」与 SCP 订单是否同一实例：**未证实**（`GAP-017`），文档按 SCP 侧对象描述。
- 下游发货可选订单的精确状态集、以及权限裁剪后的选择器范围，待测（`FSEM-006`）。
- 正文中的状态名（就绪 / 已发布等）与行累计量线索（已发 / 已收等）来自已取证枚举与字段，**不是**由中文标签反译。
