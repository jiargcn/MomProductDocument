# SCP 供应链平台

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。

## 模块职责

SCP 是面向**供应商协同**的门户与协同单据域：采购订单确认、要货预测/计划协同、供应商发货（ASN）、到货/退货跟踪视图、发票与索赔/对账。独立前端 `scp-ui`、独立后端模块；通过 **SCP↔WMS 接口登记**推送发货记录、采购计划/订单、包装与条码等。

**库存事务、库内收货执行、余额以 WMS 为准。** SCP 侧亦有采购收货/退货/上架等对象（多从 WMS 迁移或供供应商侧查询），培训时勿把 SCP 收货页写成仓库库存权威。

旧概述中虚构指标样例、未经取证的「24 小时确认」等规则不再作为培训事实。

## 建议学习顺序

1. [基础数据](01-基础数据/index.md) — 供应商/物料/价格/仓库与单据开关（SCP 侧副本）。
2. [要货预测](03-要货预测/index.md) → [要货计划](04-要货计划/index.md) — 需求对齐到可发布计划。

4. [发货协同](05-发货协同/index.md) — ASN/发货申请与记录，推送 WMS。
5. [采购跟踪](06-采购跟踪/index.md) — 收货/退货/短缺跟踪视图。
6. [发票结算](07-发票结算/index.md) — 发票、索赔、对账日历。

## 核心流程（总览）

```mermaid
flowchart LR
    BASE["基础数据"] --> FC["要货预测"]
    FC --> PLAN["要货/采购计划"]
    PLAN --> PO["采购订单"]
    PO --> ASN["发货申请/记录"]
    ASN --> WMS["WMS 收货执行"]
    WMS --> TRK["采购跟踪视图"]
    TRK --> INV["发票/索赔/对账"]
```

供应商首页看板指标（开放订单/计划、已取/未取发货等）由 SCP 首页接口聚合统计，具体数值以环境数据为准，文档不固化样例数字。

## 与其它模块边界

| 模块 | SCP 负责 | 不在 SCP 展开 |
| --- | --- | --- |
| DBC | 协同侧供应商/物料/仓库等副本维护与引用 | 企业主数据治理权威、工厂建模主维护 |
| WMS | 发货 ASN/计划/订单/包装条码等推送；收货跟踪查询 | 库内收货任务执行、库存事务与余额 |
| QMS | 发货检验明细等协同线索（若启用） | 来料检验 ATR 权威与判定 |
| MES | — | 生产计划/工单；与 SCP 直接联动**未证实** |
| ERP/财务 | 订单类型、发票已开票视图、成本中心只读同步等线索 | 付款与总账规则 |
