## 计划方预测

| 应核对 | 提醒 |
| --- | --- |
| 供应商、单号、版本、窗口 begin/end | 窗口外明细勿当有效需求。 |
| 状态、发布时间、计划员 | 发布后供应商才应看到。 |
| 明细：物料、planQty、dueDate、明细版本 | 与活动版本一致。 |
| 汇总 totalPlan/Deliver/Receipt | 只读观察进度。 |

### 常用动作

新增/修改明细、发布版本；关闭过期版本前确认无下游计划依赖。

## 供应商侧预测

| 应核对 | 提醒 |
| --- | --- |
| 是否已读、最后阅读时间 | 协同催办。 |
| 可确认范围 | 以环境权限与状态为准，勿假设可改计划总量。 |

## 最小验收清单

- [ ] 计划方发布后，绑定该供应商的用户可见对应版本。
- [ ] 明细物料与数量正确；版本切换不会误用旧版。
- [ ] 已读标记可更新（若启用）。
- [ ] 可从预测追溯到后续采购计划（按配置）；未开自动汇总时不报「系统坏了」。
