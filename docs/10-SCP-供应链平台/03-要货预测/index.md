# 要货预测

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：计划员、供应商门户；操作见[要货预测-维护与查询参考](要货预测-维护与查询参考.md)。

## 业务目的与适用范围

要货预测用于中长期需求发布与供应商对齐。计划方维护预测主单与明细（版本、窗口、计划量等），供应商侧有独立查看/确认页面。预测可为后续采购计划提供输入；接口类型枚举含预测主/明细，可登记对外同步。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 供应商、物料、周期窗口 | 预测范围。 |
| 版本号与是否当前活动版本 | 避免对错版本。 |
| 供应商是否已读（isRead） | 协同催办线索。 |

## 主对象

| 对象 | 业务含义 |
| --- | --- |
| 预测主 | 单号、供应商、版本、发布/窗口时间、状态、计划总量/已发/已收汇总、计划员等。 |
| 预测明细 | 物料、计划量、交期、在途量、版本、是否已读、状态等。 |

## 与计划 / 订单边界

| 协同方 | 本页负责 | 不在本页展开 |
| --- | --- | --- |
| 要货计划 | 提供可汇总的已发布预测 | 计划接受/驳回与推送 WMS |
| 采购订单 | 可带主 PO 线索字段 | 订单状态机 |
| MES | — | 生产计划消费预测：**未证实** |

## 限制与待确认

- 供应商确认数量字段与驳回路径的完整按钮集：以测试环境为准。
- 预测→计划的自动/手工规则：依赖策略配置，待联调短句补充。
