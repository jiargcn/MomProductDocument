# 追溯管理-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[追溯管理](index.md)做排查与规则维护。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 物料不良波及哪些产品 | “正向追溯” |
| 成品/码反查过程 | “逆向追溯” |
| 赋码规则不对 | “追溯码规则” |
| 追溯空结果 | “排空顺序” |

## 页面与入口

| 菜单 | 用途 |
| --- | --- |
| 正向追溯 | 物料代码 / 物料 SN / 批次 → 产品过程。 |
| 逆向追溯 | 码/SN/物料/批次/工单/日期 → 任务与明细页签。 |
| 追溯码规则 | 按物料维护规则。 |

## 正向追溯

| 入口条件 | 期望结果 | 操作提醒 |
| --- | --- | --- |
| 物料代码 | 该物料参与的产品生产过程集合。 | 代码须与主数据一致。 |
| 物料 SN | 该 SN 用到的产品。 | SN 唯一；无绑定则空。 |
| 批次号 | 批次相关产品过程。 | 批次粒度可能一到多产品。 |

## 逆向追溯

| 步骤 | 说明 |
| --- | --- |
| 1. 条件分页查追溯任务 | 可用唯一追溯码、产品 SN、物料、批次、工单、生产日期等。 |
| 2. 打开明细 | 人员信息（报工日志）、工序加工（工位作业）、工艺/参数、物料消耗；消耗另有树形图。 |

条件尽量带工单号或唯一码，避免过大结果集。

## 追溯码规则

| 应维护 | 关键限制 |
| --- | --- |
| 物料编码/名称、规则信息。 | 规则变更影响新赋码；历史码仍按当时规则理解。 |

改规则后用新工单样例验证解析，再推广。

## 排空顺序

1. 报表中是否有该工单/码的报工或完工记录。  
2. 终端是否按正确模式报工、是否扫物料 SN。  
3. 追溯码规则是否覆盖物料。  
4. 正向/逆向入口是否用对起点。  
5. 仍空则登记数据缺失（采集缺口），不要改追溯页“凑结果”。

## 最小验收清单

- [ ] 正向：物料代码、物料 SN、批次各一笔有结果样例。
- [ ] 逆向：唯一码进入后，人员/工序/消耗至少一个页签有数。
- [ ] 消耗树可展开且节点可解释。
- [ ] 新物料配置追溯码规则后，新报工可被逆向查到。
