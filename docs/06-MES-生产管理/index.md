# MES 生产管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：工艺/计划/线边执行相关业务与实施人员；BATCH-03 起按分组逐步业务化。

## 这一组业务解决什么问题

MES 承接生产从工艺与计划到现场执行、报工完工与追溯的运行过程，并为质量、仓储、设备协同提供生产侧事实。库存余额与出入库事务以 WMS 为准；共享工厂/物料主数据以 DBC 为准。

## 关键边界

| 边界 | 口径 |
| --- | --- |
| vs DBC | 物料、工厂建模、工序字典等主数据在 DBC；工艺路线维护页虽可出现在主数据目录，**当前实现维护主体在 MES**。 |
| vs WMS | 投料/完工引起的库存变动以 WMS 事务与余额为准。 |
| vs QMS | 过程检验、不良判定细则回 QMS；工艺可配置检验相关节点，但不替代检验方案。 |
| vs PS/ERP | ERP 订单可转换入 MES；排程算法与结果对比仍归 PS，本模块不展开。 |

```mermaid
flowchart LR
    A["工艺路线"] --> B["生产订单/工单"]

    E --> F["追溯/质量/入库协同"]
```

## 当前限制

- 菜单分组不能穷尽全部后端接口；未覆盖能力记入差距或后续叶页。
- 追溯/终端/基础建模等旧稿含未核验字段与状态，**不得直接当培训事实**；工艺与计划以第二轮回填页为准。
