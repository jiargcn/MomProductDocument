# MES 生产管理

> 基线状态：已完成 `dev` 后端首轮取证。菜单分组与控制器覆盖范围存在差异；本页只列已确认能力，叶页字段、状态流与端侧交互待逐步核验。

## 已核验的模块职责

MES 负责生产执行过程中的运行数据与现场协同。当前后端存在工艺路线及其运行版本、生产订单/工单及其运行数据、工序与工位任务、线边客户端、投料/线边库、报工/完工、生产追溯、质量与不良记录、点检、设备参数追溯、标签打印记录和终端日志等接口。

| 能力域 | 当前已核验的服务对象 | 文档边界 |
| --- | --- | --- |
| 工艺与执行建模 | 工艺路线、工序、工位、产品/产线/工位关系及运行配置。 | DBC 负责共享主数据；MES 负责生产执行所需的运行配置。 |
| 生产计划与执行 | 生产订单、工单、工位任务、工序步骤、报工/完工及其日志。 | 计划来源与排程协同待与 PS、ERP 接口核验。 |
| 线边与物料 | 线边端、线边库、投料口、自动叫料/投料、物料接收和消耗日志。 | 实际库存变动及余额以 WMS 为准。 |
| 追溯与质量 | 正向/逆向追溯、工单追溯任务、条码扫描、质量日志、不良记录及返修配置。 | 质量判定与检验配置同 QMS 的边界待核验。 |
| 设备与数据采集 | 工位设备、扭矩、点检、设备参数追溯、ThingsBoard 接口。 | 设备资产主数据归 EAM/DBC；采集接入归数采管理。 |
| 终端与输出 | 线边客户端、SSE、终端日志、标签打印及打印日志。 | MES 定义业务触发和记录；通用标签/打印能力归平台。 |

## 关键边界

1. MES 的“生产执行事实”不得与 WMS 的库存事务/余额混写；涉及出入库必须链接到库存数据挂接模型。
2. MES 的标签打印接口和打印日志表明其存在业务侧触发能力，不改变标签、条码和打印作为平台能力的长期归属。
3. 当前菜单中的“基础建模、工艺管理、计划管理、追溯管理、报表统计、终端操作”不足以穷尽后端控制器；后续需在页面覆盖总账中将未覆盖接口落为页面或差距项。

## 模块概述

本模块用于承接生产从订单/工单到工序、工位与现场终端执行的运行过程，并为追溯、质量、设备与仓储协同提供生产侧事实数据。
