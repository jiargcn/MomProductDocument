# 工艺管理-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[工艺管理](index.md)使用。本页按路线头、图形节点、产品关联、工位配置与运行快照给出维护与查询要点；计划/工单如何绑定路线与下发运行快照见[计划管理](../03-计划管理/index.md)及其[维护与查询参考](../03-计划管理/计划管理-维护与查询参考.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 新建或改版工艺路线 | “路线头与版本” |
| 画路线图、配工序节点 | “图形与工序节点” |
| 配置转序、并行、抢单/派单 | “执行控制（节点级）” |
| 绑产品或配工位 | “产品关联与工位配置” |
| 排查现场仍走旧工艺 | “运行快照与联查” |
| 批量导入/导出 | “导入导出” |

## 页面与业务对象

| 菜单入口 | 当前用途 |
| --- | --- |
| 工艺路线管理（`mes/processroute/index`） | 维护路线头、图形、节点与相关配置的主入口。 |
| （能力接口）工序—工位配置 | 维护路线工序与工位对应；支持导入导出。菜单名以环境为准。 |
| （能力接口）运行路线/快照 | 执行侧使用的路线快照查询与维护能力；生成时机待计划管理页闭合。 |

DBC「工艺建模」目录下的工艺路线页说明维护主体在 MES，日常改版请以本菜单为准。

## 路线头与版本

| 应维护/确认 | 关键限制 |
| --- | --- |
| 路线编码、名称、类型、版本。 | 编码冲突会报错；改工艺优先升版，避免覆盖在用版本。 |
| 物料编码/名称/类型、BOM、BOM 版本。 | 与产品关联、计划选路线强相关。 |
| 状态。 | 文案与启停规则以页面字典为准，勿臆造英文状态当培训事实。 |
| 是否前置校验、校验规则编码。 | 仅在明确规则已配置时开启。 |
| 备注。 | 写清版本变更原因便于审计。 |

### 常用动作

| 动作 | 目的 | 操作提醒 |
| --- | --- | --- |
| 新增 | 建立路线头。 | 先准备物料、BOM、工序主数据。 |
| 修改 | 更正头信息或状态。 | 在用版本变更需评估在制工单。 |
| 删除 | 移除未使用路线。 | 已被计划/快照引用时勿强删；先确认无引用。 |
| 按产品查路线 | 选料时定位可用路线。 | 核对版本与状态是否可被计划引用。 |

## 图形与工序节点

保存图形时，系统会按图解析并落工序节点明细（顺序、工序、前后关系、节拍、节点扩展数据等）。

| 应维护/确认 | 关键限制 |
| --- | --- |
| 起止与工序节点、连线顺序。 | 图与明细必须一致；只改图未保存成功会导致执行异常。 |
| 工序编码/名称（来自主数据）。 | 节点须对应已存在工序。 |
| 节拍、前后工序。 | 影响流转与产能估算。 |
| 节点扩展配置。 | 可能含检验/设备等扩展线索，细则待联调，不在本页写成独立质检点三表模型。 |

### 常用动作

| 动作 | 目的 | 操作提醒 |
| --- | --- | --- |
| 更新图形 | 保存路线图并生成/刷新节点。 | 保存后回看节点列表是否齐全。 |
| 保存节点明细 | 落执行控制字段。 | 并行、分派等配置见下一节。 |

## 执行控制（节点级）

| 配置项 | 业务含义 | 操作提醒 |
| --- | --- | --- |
| 最小合格转序数量 | 本工序合格累计达到阈值后才允许转入下一工序（批量场景）。 | 过严会堵线，过松会提前流转。 |
| 批量转序策略 | 工单维度累计达标后统一放行，或按任务立即转序。 | 空配置按系统默认（工单累计达标）；变更须用样例工单验证。 |
| 是否并行 / 并行上限 | 允许同工序多工位并行作业及上限。 | 并行开启时上限须合法，否则保存失败。 |
| 作业分派模式 | 抢单（现场认领）或派单（系统指定工位）；空表示未配置。 | 与线边作业领取方式直接相关，须与工位配置一起验收。 |

## 产品关联与工位配置

| 对象 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 路线—产品 | 路线编码/版本与物料的适用关系。 | 编辑路线时用于回显适用产品；优先级规则待环境核对。 |
| 工序—工位 | 路线、工序、工位、是否主工位、工位类型等。 | 多工位时须明确主工位；支持导入导出。 |

### 常用动作

| 动作 | 目的 | 操作提醒 |
| --- | --- | --- |
| 维护产品关联 | 明确路线适用哪些物料/产品。 | 与路线头上的物料/BOM 信息对照。 |
| 维护工位配置 | 决定现场可在哪些工位执行该工序。 | 改后应用样例工单验证抢单/派单。 |
| 导入工位配置 | 批量建立工序—工位。 | 使用模板；失败看错误文件，不导入系统编号与审计列。 |

## 运行快照与联查

| 场景 | 建议查什么 | 说明 |
| --- | --- | --- |
| 改了主数据，现场仍旧 | 工单绑定的路线版本与运行快照。 | 执行侧可能冻结快照，不只读最新主数据。 |
| 工位对不上 | 工序—工位配置、是否主工位。 | 联查工位主数据与终端。 |
| 转序被挡 | 最小合格转序数量、批量转序策略、报工合格累计。 | 联查工单执行与报工记录。 |
| 分派异常 | 节点分派模式是否为抢单/派单。 | 联查线边作业领取入口。 |

## 导入导出

| 能力 | 范围 | 提醒 |
| --- | --- | --- |
| 路线头导入/导出 | 路线主信息批量处理。 | 图形与复杂节点关系不宜只靠一行 Excel 完整表达；导入后仍须校验图与节点。 |
| 工位配置导入/导出 | 工序—工位关系。 | 模板字段以系统下载为准。 |
| 高级查询 | 列表条件组合检索。 | 与统一查询口径一致，见基础设施查询说明。 |

## 最小验收清单

- [ ] 新建路线：头信息 + 图形保存后节点齐全，可按产品查到。
- [ ] 升版：旧版在制不受错误覆盖；新版可被新计划引用（规则以实测为准）。
- [ ] 转序：最小合格数量与批量策略各有命中/不命中样例。
- [ ] 并行与分派：抢单认领、派单指定工位各验证一单。
- [ ] 工位配置导入：成功行生效，失败行有错误回执。
- [ ] 改主数据后：通过运行快照/工单版本解释现场行为。
