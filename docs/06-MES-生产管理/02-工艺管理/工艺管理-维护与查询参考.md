# 工艺管理-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[工艺管理](index.md)使用。本页按路线头、图形节点、产品关联、工位配置与运行快照给出维护与查询要点；计划/工单如何绑定路线与下发运行快照见[计划管理](../03-计划管理/index.md)及其[维护与查询参考](../03-计划管理/计划管理-维护与查询参考.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 理解版本/转序/分派如何改行为 | “字段说明” |
| 新建或改版工艺路线 | “路线头与版本” |
| 画路线图、配工序节点 | “图形与工序节点” |
| 配置转序、并行、抢单/派单 | “执行控制（节点级）” |
| 绑产品或配工位 | “产品关联与工位配置” |
| 排查现场仍走旧工艺 | “运行快照与联查” |
| 批量导入/导出 | “导入导出” |

## 字段说明
主数据选择通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)。

### 字段说明总表
| 字段 | 业务作用 | 谁在何时维护/选择 | 业务约束摘要 | 影响哪些功能 | 变更或错选风险 |
| --- | --- | --- | --- | --- | --- |
| 路线编码 | 路线身份 | 新建时录入 | 冲突报错；引用后慎删 | 计划选路线、追溯 | 改码破坏引用 |
| 路线版本 | 改版边界 | 升版时维护 | 在用版勿覆盖 | 在制快照、新单引用 | 覆盖在用版 |
| 物料 / BOM / 版本 | 适用对象 | 建路线时选择 | 物料可用；BOM 与物料匹配 | 产品关联、计划绑定 | 错料错 BOM |
| 路线状态 | 可否引用 | 启停/字典 | 字典文案 ❓ | 计划过滤 | 停用后新单选不到 |
| 工序节点 | 执行路径 | 图形保存后落明细 | 工序须存在 | 转序、报工 | 图表明细不一致 |
| 最小合格转序数量 | 转序门槛 | 节点配置 | ≥0；过严堵线 | 线边转序 | 阈值与批量不匹配 |
| 批量转序策略 | 放行口径 | 节点配置 | 工单累计 / 按任务立即；空=工单累计 | 转序时机 | 策略选错卡线 |
| 是否并行 / 上限 | 多工位并行 | 节点配置 | 并行时上限合法 | 工位任务并发 | 非法配置保存失败 |
| 作业分派模式 | 领取方式 | 节点配置 | 抢单 / 派单 / 空 | 线边作业 | 与工位配置冲突 |
| 工序—工位 / 主工位 | 可执行工位 | 工位配置维护 | 多工位须有主工位 | 抢单/派单 | 无主工位执行乱 |
| 运行快照 | 执行冻结 | 下发时系统生成 | 与工单绑定 | 在制执行 | 误以为改主数据即改在制 |
| 创建人/时间等 | 审计 | 系统写入 | 勿当可维护项 | 追溯 | — |

### 选择器可选范围
| 选择字段 | 选择对象 | 可选范围 | 范围依赖 | 选中后带回或锁定 | 选不到时通常原因 |
| --- | --- | --- | --- | --- | --- |
| 物料 | 物料主数据 | 可用物料（用途过滤 ❓） | 工厂/权限 ❓ | 名称、类型 | 停用、权限外 |
| BOM / BOM 版本 | BOM 关系 | 与已选物料匹配的 BOM 版本 | 已选物料 | 版本号 | 无 BOM、版本不符 |
| 工序（节点） | 工序主数据 | 已建工序 | DBC 工序 | 工序名称/类型 | 未建模 |
| 工位 | 工位主数据 | 工厂建模下有效工位 | 车间/产线 | 工位名称、类型 | 停用、未建 |
| 按产品查路线 | 工艺路线 | 与产品/物料关联且状态可引用的路线（启停规则 ❓） | 物料/产品 | 路线编码、版本 | 无关联、已停用 |

### 取值影响矩阵（P1：转序策略 / 分派）

| 取值 | 业务含义 | 影响的功能或流程 | 系统行为变化 | 错选/漏选后果 |
| --- | --- | --- | --- | --- |
| 批量转序：工单累计达标 | 整单合格累计够才统一放行 | 批量转序 | 未达标不可转下序 | 过严则堵在本序 |
| 批量转序：按任务立即转 | 单任务达标即可转 | 批量转序 | 更快流转 | 过松则半成品提前下序 |
| 分派：抢单 | 现场认领 | 线边领取 | 工位主动抢 | 无人抢则任务积压 |
| 分派：派单 | 系统指定工位 | 线边领取 | 指定工位执行 | 工位配错无人可做 |
| 分派：空 | 未配置 | 领取方式 | 按运行时默认/其它配置（❓） | 现场规则不清 |

### 联动 / 回填摘要（P4 / P5）

| 上游变化 | 下游行为 | 业务原因 |
| --- | --- | --- |
| 更换路线头物料 | 应重核 BOM、产品关联 | 适用对象变了 |
| 保存图形 | 刷新/生成工序节点明细 | 图是明细来源 |
| 变更并行开关 | 校验并行上限；非法则保存失败 | 防止无效并行 |
| 工单下发 | 生成运行快照并锁定执行用工艺 | 在制不受主数据随意改写 |

## 路线头与版本

| 应维护/确认 | 关键限制 |
| --- | --- |
| 路线编码、名称、类型、版本。 | 编码冲突会报错；改工艺优先升版，避免覆盖在用版本。 |
| 物料编码/名称/类型、BOM、BOM 版本。 | 与产品关联、计划选路线强相关。 |
| 状态。 | 文案与启停规则以页面字典为准，勿臆造英文状态当培训事实。 |
| 是否前置校验、校验规则编码。 | 仅在明确规则已配置时开启。 |
| 备注。 | 写清版本变更原因便于审计。 |

## 图形与工序节点

保存图形时，系统会按图解析并落工序节点明细（顺序、工序、前后关系、节拍、节点扩展数据等）。

| 应维护/确认 | 关键限制 |
| --- | --- |
| 起止与工序节点、连线顺序。 | 图与明细必须一致；只改图未保存成功会导致执行异常。 |
| 工序编码/名称（来自主数据）。 | 节点须对应已存在工序。 |
| 节拍、前后工序。 | 影响流转与产能估算。 |
| 节点扩展配置。 | 可能含检验/设备等扩展线索，细则待联调，不在本页写成独立质检点三表模型。 |

### 常用动作

| 动作 | 目的 | 操作提醒 |
| --- | --- | --- |
| 更新图形 | 保存路线图并生成/刷新节点。 | 保存后回看节点列表是否齐全。 |
| 保存节点明细 | 落执行控制字段。 | 并行、分派等配置见下一节。 |

## 执行控制（节点级）

| 配置项 | 业务含义 | 操作提醒 |
| --- | --- | --- |
| 最小合格转序数量 | 本工序合格累计达到阈值后才允许转入下一工序（批量场景）。 | 过严会堵线，过松会提前流转。 |
| 批量转序策略 | 工单维度累计达标后统一放行，或按任务立即转序。 | 空配置按系统默认（工单累计达标）；变更须用样例工单验证。 |
| 是否并行 / 并行上限 | 允许同工序多工位并行作业及上限。 | 并行开启时上限须合法，否则保存失败。 |
| 作业分派模式 | 抢单（现场认领）或派单（系统指定工位）；空表示未配置。 | 与线边作业领取方式直接相关，须与工位配置一起验收。 |

## 产品关联与工位配置

| 对象 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 路线—产品 | 路线编码/版本与物料的适用关系。 | 编辑路线时用于回显适用产品；优先级规则待环境核对。 |
| 工序—工位 | 路线、工序、工位、是否主工位、工位类型等。 | 多工位时须明确主工位；支持导入导出。 |

### 常用动作

| 动作 | 目的 | 操作提醒 |
| --- | --- | --- |
| 维护产品关联 | 明确路线适用哪些物料/产品。 | 与路线头上的物料/BOM 信息对照。 |
| 维护工位配置 | 决定现场可在哪些工位执行该工序。 | 改后应用样例工单验证抢单/派单。 |
| 导入工位配置 | 批量建立工序—工位。 | 使用模板；失败看错误文件，不导入系统编号与审计列。 |

## 运行快照与联查

| 场景 | 建议查什么 | 说明 |
| --- | --- | --- |
| 改了主数据，现场仍旧 | 工单绑定的路线版本与运行快照。 | 执行侧可能冻结快照，不只读最新主数据。 |
| 工位对不上 | 工序—工位配置、是否主工位。 | 联查工位主数据与终端。 |
| 转序被挡 | 最小合格转序数量、批量转序策略、报工合格累计。 | 联查工单执行与报工记录。 |
| 分派异常 | 节点分派模式是否为抢单/派单。 | 联查线边作业领取入口。 |

## 导入导出

| 能力 | 范围 | 提醒 |
| --- | --- | --- |
| 路线头导入/导出 | 路线主信息批量处理。 | 图形与复杂节点关系不宜只靠一行 Excel 完整表达；导入后仍须校验图与节点。 |
| 工位配置导入/导出 | 工序—工位关系。 | 模板字段以系统下载为准。 |
| 高级查询 | 列表条件组合检索。 | 与统一查询口径一致，见基础设施查询说明。 |

## 最小验收清单

- [ ] 新建路线：头信息 + 图形保存后节点齐全，可按产品查到。
- [ ] 升版：旧版在制不受错误覆盖；新版可被新计划引用（规则以实测为准）。
- [ ] 转序：最小合格数量与批量策略各有命中/不命中样例。
- [ ] 并行与分派：抢单认领、派单指定工位各验证一单。
- [ ] 工位配置导入：成功行生效，失败行有错误回执。
- [ ] 改主数据后：通过运行快照/工单版本解释现场行为。
