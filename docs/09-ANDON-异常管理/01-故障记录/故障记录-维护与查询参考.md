# 故障记录-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[故障记录](index.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 登记/关闭呼叫 | “问题主记录” |
| 查通知与到岗 | “过程记录” |
| 维护停机原因 | “停机项目” |
| 配响应与消息 | [问题响应](../02-问题响应/问题响应-维护与查询参考.md) |

## 页面与业务对象

| 菜单入口 | 用途 |
| --- | --- |
| 安东故障记录 | 过程/协同视图（前端 processRecord）。 |
| 停机项目表 | 停机原因主数据。 |
| （若环境另有问题主单页） | 问题主记录 CRUD。 |

## 问题主记录

| 应维护/确认 | 提醒 |
| --- | --- |
| 问题用户、岗位、起止时间 | 呼叫中→结束时补结束时间与时长。 |
| 设备/模具/零件/工单 | 能填则填，便于跨模块联查。 |
| 项目分类、问题分类、来源、类型 | 来源影响责任与报表。 |
| 描述、措施、重要度、紧急度 | 紧急度含在机整改等口径。 |
| 附件、分析登记、维修单状态 | 按组织是否必填。 |

### 常用动作

新增、修改、结束呼叫、审结（总体状态）；删除需谨慎。

## 过程记录

| 应核对 | 提醒 |
| --- | --- |
| 关联问题 ID、通知类型与内容 | 与响应链触发一致。 |
| 到岗人、到岗时间、到岗状态、时长 | 考核响应 SLA。 |

## 停机项目

| 应维护/确认 | 提醒 |
| --- | --- |
| 停机项目名称/类型、计划时长、呼叫岗位 | 与 EAM「设备停机」单据区分。 |
| 设备、班次（若使用） | 统计维度。 |

## 最小验收清单

- [ ] 能建立一条问题主记录并结束呼叫。
- [ ] 过程记录能看到通知与到岗。
- [ ] 来源分别选安东/巡检/质量的样例可区分。
- [ ] 停机项目可被业务选用（若流程需要）。
- [ ] 需要维修时能手工关联或新开 EAM 报修（当前无强制自动）。
