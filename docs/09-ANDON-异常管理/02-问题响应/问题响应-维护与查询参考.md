# 问题响应-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[问题响应](index.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 配岗位顺序与升级 | “响应链” |
| 配通知文案 | “消息配置” |
| 做整改跟踪 | “整改（若启用）” |
| 查到岗是否达标 | [故障记录](../01-故障记录/故障记录-维护与查询参考.md) |

## 页面与业务对象

| 菜单入口 | 用途 |
| --- | --- |
| 问题响应链配置 | 分类-岗位-时长-上级-间隔-消息。 |
| 消息配置表 | 消息编号与内容。 |
| 整改相关（若挂出） | 整改与跟踪。 |

## 响应链

| 应维护/确认 | 提醒 |
| --- | --- |
| 问题分类 | 与呼叫时选择一致。 |
| 问题岗位与编码、序号 | 序号即通知顺序。 |
| 标准时长、通知间隔 | 分钟口径。 |
| 上级岗位与编码 | 超时升级对象。 |
| 消息编号 | 必须在消息配置中存在。 |

### 常用动作

按分类维护多行节点；调整顺序前先确认现场值班表。

## 整改（若启用）

| 应维护/确认 | 提醒 |
| --- | --- |
| 责任人、计划起止、说明、状态 | 与问题关闭策略对齐。 |
| 跟踪记录 | 防措施悬空。 |

## 最小验收清单

- [ ] 至少一类问题分类配置完整响应链（含上级与消息）。
- [ ] 发起呼叫后目标岗位能收到通知（环境联调）。
- [ ] 过程到岗能登记且时长可统计。
- [ ] 超时场景下上级是否收到：记录实际结果（启用/未启用）。
- [ ] 不期望自动开 EAM 维修时，流程说明为手工报修。
