# MOM 制造运营管理系统

## 产品简介

MOM（Manufacturing Operations Management）制造运营管理系统，是一套面向离散制造业的全流程数字化管理平台，涵盖从主数据、库房、生产、质量、设备、异常、供应链到排程的核心业务闭环。

## 目标受众与阅读路径

正式业务页按 **测试人员、实施人员、运维人员**（三者并列）的「业务行为说明书」来写：功能范围清楚、逻辑可讲、配置可落地、验证可开单；运维同样能据此**日常答疑、系统维护与排障**，对系统细节的了解需求与测试、实施同级。主路径为：业务模块首页 → 分组 → **主文档** → 需要时再进同组「维护与查询参考」。

其余角色可按下列路径阅读：

| 角色 | 怎么读 |
|------|--------|
| 新加入开发 | 模块首页 → 主文档 → 侧栏「认识系统」下的[业务模型](02-业务模型/10-领域模型.md)（共享约定） |
| 了解产品全景 | 可从侧栏「认识系统」读起：见下方入口；业务细节与操作步骤见各模块主文档及维护与查询参考 |

### 了解产品全景（认识系统）

按顺序扫读，可建立「MOM 管什么、不管什么、各模块各管一段」的整体印象：

1. [系统概述](01-总体框架/05-系统概述.md) — 解决什么问题、从哪读起  
2. [系统边界与产品全景](01-总体框架/01-系统边界与产品全景.md) — 覆盖范围与外部边界  
3. [模块边界与系统上下文](01-总体框架/04-模块边界与系统上下文.md) — 各模块职责一览  
4. 侧栏「业务应用」任选 2～3 个模块首页，先读「解决什么问题」（或等价开篇边界）

需要部署形态、终端分工或技术选型时，再打开[多组织、多工厂与数据隔离](01-总体框架/02-多组织、多工厂与数据隔离.md)、[多端与终端架构](01-总体框架/03-多端与终端架构.md)、[技术架构](01-总体框架/06-技术架构.md) / [技术栈](01-总体框架/07-技术栈.md)。

### 测试 / 实施 / 运维怎么读

从目标业务的**模块首页**进入 → 分组 → **主文档**（配置如何起作用、建议验证点、状态与跨模块影响）→ 需要操作细节时再进「维护与查询参考」。

- **测试**：据此设计验证场景（含配置差异、异常与上下游）。
- **实施**：据此落地配置并说清行为变化。
- **运维**：据此**答疑、日常维护与排障**——查清配置/状态含义、卡点与改动影响；**业务叶页是主入口**，平台「基础设施 / 系统管理」仅补充部署监控等，**不是**运维唯一阅读路径。

产品全景入口用于建立整体印象；配置差异、验证场景与字段操作细节以各业务主文档及维护与查询参考为准。

## 文档结构

侧栏按四层组织；**维护与查询参考**不单独占菜单，从各业务主文档内链进入。

| 分区 | 内容 |
|------|------|
| 认识系统 | 总体框架（概述/边界/架构）与业务模型（领域、流程、共享业务规则）——了解产品全景优先从这里起步 |
| 业务应用 | DBC / WMS / MES / QMS / EAM / ANDON / SCP；PS 排程为**规划中**入口（不可按已上线对外承诺）——测试 / 实施 / 运维的主要入口 |
| 平台与集成 | 基础设施、系统管理、数采、API 参考——部署监控等补充，**不替代**业务叶页 |
| 版本路线图 | 当前版本功能清单与未来迭代方向 |

阅读提示：设备**身份台账**看 DBC「设备台账」；报修/巡检/保养等**运维执行**看 EAM。报表模板与嵌入方式见平台「报表看板与数据输出」；菜单侧配置入口见「报表管理」。
