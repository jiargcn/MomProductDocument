# 基础服务

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）。

## 本页解决什么问题

基础服务承接跨模块的**可观测与后台运行**能力：日志与审计、定时调度、对外数据交换的成败与重试。读完本页，应能按「查人 / 查作业 / 查外部回写」三选一进入正确叶页，并与业务单据审计、业务「任务单」区分开。

**何时用本页 / 不归本页**

| 何时进来 | 不归本页 |
| --- | --- |
| 不知道该查操作日志、调度控制台还是接口调用记录 | 单据状态为何变更、数量为何对不上 → 业务页与业务审计字段 |
| 区分「平台后台作业」与「仓储收货/发料任务」 | 列表筛选、导入模板字段、附件必传规则 → [公共能力](公共能力.md)各叶页 |
| 排障前先选对入口类型 | 权限/认证配置 → [系统管理](../12-系统管理/index.md)；基础设施 APM/告警大盘 → [监控运维](12-监控运维.md) |

## 如何使用

| 你的目的 | 建议怎么做 |
| --- | --- |
| 查谁登录、谁操作、接口报错 | [日志、审计与运行监控](07-日志、审计与运行监控.md) |
| 启停/补跑后台定时作业 | [定时任务与后台作业](08-定时任务与后台作业.md) |

## 服务地图

| 能力 | 说明页面 | 当前可确认结论 | 业务侧责任 |
| --- | --- | --- | --- |
| 日志与审计 | [日志、审计与运行监控](07-日志、审计与运行监控.md) | System 操作/登录 + Infra API 访问/错误 + WMS 导入导出日志分层 | 单据状态、数量、审批意见等业务语义审计写在业务页 |
| 定时任务与后台作业 | [定时任务与后台作业](08-定时任务与后台作业.md) | 以 XXL-Job 调度控制台为主；与业务「任务单」不同 | 说明某 Handler 的业务条件与开关 |
| 数据交换可靠性 | [数据交换与集成可靠性](09-数据交换与集成可靠性.md) | 接口调用信息、异步成功/失败与人工重试 | 字段映射、单据挂接见业务模型与业务页 |

## 与公共能力的边界

| 主题 | 归哪里 |
| --- | --- |
| 查询筛选、导入导出、文件、消息、报表模板 | [公共能力](公共能力.md) |
| 事后定位、无人值守作业、对外调用可恢复 | 本页及下方三叶页 |
| 存储介质、保留周期、脱敏、生产告警订阅 | 叶页「待确认」+ 运维规范；产品页不编造 |

## 短自检

- 现象是「人/菜单操作」→ 日志页；「到点没跑」→ 定时任务页；「外部没入账」→ 数据交换页。
- 不会把 WMS 收货任务/发料任务说成「定时任务」。
- 不会用平台操作日志替代业务单据上的状态与数量审计。
