# 查询、筛选与用户视图

> 状态：占位页已建立。当前前端实现与默认列表规则待逐页核验。

## 1. 设计目的

形成统一的数据列表与详情页规范，使用户能快速看到关键字段、按稳定条件筛选、保存个人或角色视图，并从详情页跳转至关联业务。

## 2. 参考设计线索

旧版规划书提出可保存自定义视图、提供默认视图，以及可配置字段显示权限。当前实现状态待核验。

## 3. 统一待定义内容

| 场景 | 页面必须说明 |
| --- | --- |
| 数据列表 | 默认字段、显示顺序、默认排序、常用筛选、权限/脱敏和导出范围。 |
| 详情页 | 信息分组、关键状态、操作区、关联数据 Tab/链接和审计信息。 |
| 用户视图 | 保存范围、共享范围、默认视图、字段权限冲突处理。 |
| 查询性能 | 分页、最大时间范围、模糊查询限制和异步导出策略。 |
