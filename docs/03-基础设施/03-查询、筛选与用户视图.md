# 查询、筛选与用户视图

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：实施顾问、前端/业务管理员，以及需要统一列表页“怎么查、怎么看”的培训人员。

## 业务目的与适用范围

查询与筛选决定用户能否在列表中快速定位单据；详情与关联跳转决定能否完成联查。本页给出**跨模块列表页的平台口径**；各业务默认列、必筛条件与高级查询字段仍以业务页为准。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 默认时间范围 | 无范围时结果过多或过慢。 |
| 账号数据范围 | 筛选条件再宽，也出不了范围外数据。 |
| 是否有高级查询 | 许多仓储页支持自定义条件组合。 |
| 详情联查路径 | 培训需要知道从哪跳到关联对象。 |

## 平台通用口径

```mermaid
flowchart LR
    A["进入业务列表"] --> B["常用筛选 / 高级查询"]
    B --> C["分页结果"]
    C --> D["打开详情 / 关联跳转"]
    D --> E["按需导出当前条件结果"]
```

| 主题 | 当前可确认结论 |
| --- | --- |
| 列表 + 分页 | 业务列表普遍采用分页查询，避免一次拉全量。 |
| 常用筛选 | 页头提供高频字段筛选（单号、状态、时间、组织/地点等，因页而异）。 |
| 高级查询 | 多业务支持自定义条件组合（条件列、运算符、值），用于复杂检索。 |
| 详情与联查 | 详情分组、状态、操作区、关联 Tab/链接由业务页定义。 |
| 导出范围 | 导出通常受当前筛选/权限影响；以业务导出实现为准。 |
| 用户视图 | **未证实**存在全站统一的“保存并共享角色默认视图中心”；列显示偏好若存在，多为页面级或本地行为，需逐页确认。 |

## 业务页必须写清什么

| 场景 | 建议写清 |
| --- | --- |
| 数据列表 | 默认列、默认排序、常用筛选、权限/脱敏、导出是否跟随筛选。 |
| 高级查询 | 可用条件字段、限制（如最大条件数，若有）。 |
| 详情页 | 信息分组、关键状态、操作、关联跳转与审计信息。 |
| 性能提示 | 建议时间范围、避免无条件模糊扫全表（若业务有约束）。 |

## 关键判断

| 判断点 | 应先确认什么 | 影响 |
| --- | --- | --- |
| 查不到数据 | 筛选过严、时间范围、数据权限、岗位库位。 | 先放宽筛选再用非超管验证范围。 |
| 高级查询无结果 | 条件列是否选错、运算符是否匹配类型。 | 对照样例条件。 |
| 两人同角色列表不同 | 部门/岗位范围或个人筛选残留。 | 清筛选并核对组织岗位。 |
| 想要统一保存视图 | 确认目标页是否真有保存视图；无则登记需求，不写进现状。 |

## 查询与联查

| 场景 | 建议看什么 | 联查 |
| --- | --- | --- |
| 列表慢 | 时间范围、是否无索引条件、是否导出全量。 | 运维/日志。 |
| 详情缺关联 | 业务是否提供跳转，还是需手工按单号查。 | 业务页。 |
| 筛选字典不对 | 字典是否启用、是否租户内数据。 | 系统字典（System）。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 把“高级查询”当成独立模块菜单 | 它是列表能力，入口在各业务页。 |
| 培训“人人可保存共享视图” | 仅在已证实的页面这么说；否则改为“按筛选条件查询”。 |
| 导出与屏幕不一致 | 检查是否未带上筛选条件或权限截断。 |
