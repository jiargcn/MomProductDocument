# 消息、通知与待办

> 状态：占位页已建立。旧版规划书提出站内、邮件、短信与 ANDON 消息等方向；当前渠道与配置待核验。

## 1. 设计目的

将业务事件转化为可处理、可追踪的通知和待办，统一说明消息来源、接收主体、升级、已读/完成状态及发送失败处理。

## 2. 参考设计

旧版规划书建议通知可面向岗位发送，并由 ANDON 等模块调用统一消息服务；展示端可包含 WEB、PDA、邮件或短信。该设计待当前基线确认。

## 3. 待核验清单

- 消息模板、渠道、接收人/岗位解析、发送频率和免打扰规则。
- 待办的来源单据、跳转地址、完成/关闭条件和超时升级。
- 发送日志、失败重试、去重、撤回与审计。
