# 日志、审计与运行监控

> 状态：占位页已建立。当前日志表、监控平台和告警链路待 Infra/System 源码核验。

## 1. 设计目的

让业务操作、权限决策、接口交换、后台任务和系统故障均可被定位和审计；业务日志与技术日志应能通过业务编号或追踪标识关联。

## 2. 待核验清单

| 类型 | 最低应记录内容 |
| --- | --- |
| 业务审计 | 操作者、时间、前后状态、原因、关联单据和附件。 |
| 操作与登录 | 用户、终端、来源、动作、结果和失败原因。 |
| 接口与任务 | 请求/响应摘要、幂等键、状态、重试、耗时和异常。 |
| 运行监控 | 服务可用性、错误率、队列积压、任务失败、资源指标和告警。 |
