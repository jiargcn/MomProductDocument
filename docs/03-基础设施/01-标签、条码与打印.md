# 标签、条码与打印

> 状态：占位页已建立。旧版规划书的标签模型仅作设计参考；当前菜单配置在 DBC 的事实不改变本能力的长期平台归属。

## 1. 设计目的

为包装、器具、产品、库存等业务对象提供统一的编码、标签模板、打印、补打、作废和审计能力，避免各模块自行拼接条码或直接控制打印设备。

## 2. 参考设计

| 主题 | 参考设计 |
| --- | --- |
| 标签类型 | 标签类型定义业务对象和可用模板。 |
| 条码编码 | 条码由可配置编码段组成，明确每段来源、长度、校验与解析规则。 |
| 输出触发 | 包装、器具创建或拆并包等业务动作可触发打印；触发条件必须可追溯。 |
| 补打与作废 | 补打需记录原因、操作者、次数；作废标签不可再次作为有效业务标识。 |

## 3. 分工边界

| 范围 | 责任 |
| --- | --- |
| DBC | 提供物料、包装规格、业务类型等可引用的业务主数据；当前配置入口可保留。 |
| 平台能力与基础设施 | 标签/编码规则、模板渲染、打印队列、设备适配、补打和审计。 |
| 业务模块 | 定义何时、为何业务对象、以哪些业务变量请求标签或打印。 |

## 4. 待核验清单

- 标签类型、条码段、模板、打印机和打印记录的真实实体与配置入口。
- 标签唯一性、重复打印、作废、重打和扫码校验规则。
- WEB、PDA、线边端的打印触发链路与失败补偿。
