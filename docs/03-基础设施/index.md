# 平台能力与基础设施

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；平台实施与各业务模块运维协同（顺带）。

## 模块解决什么问题

本目录承载**跨业务复用**的平台能力与运行底座（标签、报表、查询视图、导入导出、文件、消息、日志、定时任务、数据交换等）。读完本页，应能：按能力名找到对应页；分清「能力怎么配」在本目录、「业务怎么用」在 WMS/MES 等业务页；权限/认证问题去系统管理。

不重复描述各业务模块的业务规则。公共能力总览见[公共能力](公共能力.md)；日志/调度/交换总览见[基础服务](基础服务.md)。

| 权限 / 认证 | [RBAC](../12-系统管理/03-用户与权限/01-RBAC权限模型.md)、[租户与认证](../12-系统管理/01-租户与认证/index.md) |

## 编写边界

1. 本模块描述“能力如何提供、如何配置、如何治理”。
2. 具体业务模块描述“业务在何处使用该能力、产生什么数据”。
3. 涉及权限时链接 RBAC；涉及认证时链接租户与认证。

## 长期归属约定

标签、条码和打印属于平台能力与基础设施。即使当前配置入口位于 WMS/DBC，也只表示业务配置入口现状；模板、编码、渲染、输出、设备适配与审计按平台能力治理。
