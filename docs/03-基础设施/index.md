# 平台能力与基础设施

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：平台/基础设施实施顾问、各业务模块运维协同；具体业务如何使用这些能力见对应业务模块页。
> 定位：承载跨业务模块复用的平台能力与运行底座；不重复描述 WMS、MES、QMS 等模块的具体业务规则。读完本页，你能知道标签、报表、查询、导入导出、消息、日志、任务调度等通用能力分别在哪一页，以及权限、认证类问题该去系统管理哪个页面。

## 编写边界

1. 本模块描述“能力如何提供、如何配置、如何治理”。
2. 具体业务模块描述“业务在何处使用该能力、产生什么数据”。
3. 涉及权限时，链接到[RBAC 权限模型](../12-系统管理/03-用户与权限/01-RBAC权限模型.md)；涉及认证时链接到[租户与认证](../12-系统管理/01-租户与认证/index.md)。

## 长期归属约定

标签、条码和打印属于平台能力与基础设施。即使当前配置入口位于 WMS/DBC，也只表示业务配置入口现状；模板、编码、渲染、输出、设备适配与审计按平台能力治理。
