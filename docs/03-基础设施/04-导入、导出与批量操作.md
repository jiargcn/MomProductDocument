# 导入、导出与批量操作

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：实施顾问、业务管理员，以及需要统一解释“模板怎么填、失败怎么查”的支持人员。

## 业务目的与适用范围

导入、导出与批量操作用于把大批量主数据或业务数据安全地送入/带出系统。本页给出**跨模块可复用的平台口径**；具体模板字段、必填与校验仍以各业务页为准，不得用本页替代物料、收货等专项导入说明。

| 你的目的 | 建议阅读 |

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 业务是否提供下载模板 | 只能按当前页模板填写，不可沿用历史或其它对象模板。 |
| 导入模式（如新增/更新） | 许多入口带模式参数，选错会覆盖或跳过数据。 |
| 账号是否有导入/导出权限 | 前端有按钮不等于接口一定强制鉴权（见 GAP-014）。 |
| 主数据是否已就绪 | 导入常依赖物料、包装、仓库等已存在且可用。 |

## 平台通用口径

```mermaid
flowchart LR
    A["下载业务模板"] --> B["按模板填业务字段"]
    B --> C["上传并选择模式"]
    C --> D["行校验 / 主数据校验"]
    D --> E["成功写入或返回错误"]
    E --> F["必要时查看导入日志/错误文件"]
```

| 主题 | 当前可确认结论 |
| --- | --- |
| 入口形态 | 导入/导出挂在各业务页面，不是单一“全站导入中心”替代业务入口。 |
| 模板原则 | 模板应只暴露需人工填写的业务字段；系统生成、审计、选择器回填字段不应依赖用户手填。 |
| 常见参数 | 多处导入支持模式（如覆盖/更新相关参数）；具体枚举以业务页为准。 |
| 结果回执 | 常见返回成功/失败统计类结果；部分提供错误文件或异常信息。 |
| 导入/导出日志 | WMS 侧提供导入日志、导出日志菜单，可按业务名、状态、文件名等查询进度与错误线索。 |
| 批量操作 | 列表批量启用/禁用/删除等属业务页动作；状态与关联保护以业务规则为准。 |

## 业务页必须写清什么

每一个提供导入的业务页，至少应说明：

1. 模板字段、必填、类型/长度与样例；
2. 不应出现在模板中的字段（由系统或上下游生成）；
3. 选择器字段的数据来源；
4. 行内/跨行/主数据/重复判定规则；
5. 部分成功、错误回执、重试与权限要求。

本页不展开各对象字段表；发现模板与保存校验不一致时，登记问题总账后继续其它对象。

## 查询与联查

| 想解决的问题 | 推荐定位方式 | 建议联查 |
| --- | --- | --- |
| 导入失败找不到原因 | 页面提示 → 导入日志 → 错误文件 URL/异常信息。 | WMS 导入日志菜单。 |
| 导出无文件或按钮灰 | 权限标识、筛选条件、数据是否为空。 | RBAC、业务列表。 |
| 同字段在导入与页面规则不一致 | 对照业务页“导入专项”说明与问题总账。 | 对应 DBC/WMS 业务页。 |
| 大批量超时 | 分批导入、查导入日志进度。 | 运维/任务（若后续有异步专项）。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 用旧模板导入 | 重新下载当前页模板。 |
| 导入“成功”但业务数据不对 | 核对模式参数、更新范围与主数据引用，不只看成功条数。 |
| 把平台页当成可操作导入台 | 回到具体业务页执行导入。 |
| 导出按钮可见但接口失败 | 按 RBAC 分层排查，不单改前端。 |

## 当前限制与待确认事项

- 是否所有模块导入都写入同一套导入日志，待建立覆盖清单；
- 预校验与正式导入两段式是否为全站统一能力，**未证实**；
- 部分业务导入按钮/接口被注释或路径不一致，以各业务页与问题总账为准；
- 统一错误码体系仍属目标口径，当前以各接口返回与错误文件为准。
