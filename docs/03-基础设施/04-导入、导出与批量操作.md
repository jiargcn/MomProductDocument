# 导入、导出与批量操作

> 状态：占位页已建立。当前导入模板和接口待逐页取证。

## 1. 设计目的

以统一规则控制批量数据进入和离开系统：模板只暴露应由用户填写的业务字段，其余字段由系统、选择器、默认值或上下游单据填充；错误必须可定位、可修正、可重试。

## 2. 参考设计线索

旧版规划书提出批量操作及统一错误代码/错误信息需求。当前导入任务、错误回执和重复处理策略待核验。

## 3. 页面输出要求

每一个业务导入页必须给出：

1. 模板字段、必填条件、类型/长度、样例和不应出现在模板中的字段。
2. 选择器字段的数据源与筛选条件。
3. 行内校验、跨行校验、主数据校验、权限校验和重复导入判定。
4. 预校验、正式导入、错误回执、部分成功、重试与审计规则。
