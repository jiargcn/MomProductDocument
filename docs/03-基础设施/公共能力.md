# 公共能力

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）。
> 状态：公共能力主页面已回填；部署/安全等运行专项见本目录「运行与安全」分组。

## 本页解决什么问题

公共能力是各业务模块**共用**的运行服务与统一治理口径（标签、报表、查询、导入导出、文件、消息等）。读完本页，应能：按能力名进到正确叶页；分清「平台怎么提供」与「业务何时用」；不在业务页重复发明平台规则。

**何时用本页 / 不归本页**

| 何时进来 | 不归本页 |
| --- | --- |
| 只知能力名，不知该读哪一篇 | 某张单据的字段、状态机、扫码强制规则 → 对应业务页 |
| 判断「该在平台配还是业务里配」 | 权限角色、租户认证 → [系统管理](../12-系统管理/index.md) |
| 培训前先建立跨模块复用边界 | 部署拓扑、安全基线、APM → [部署](10-部署架构.md) / [安全](11-安全体系.md) / [监控](12-监控运维.md) |

## 归属说明

标签、条码和打印长期归属平台；当前主配置入口在 WMS（详见标签页）。DBC 提供被引用的主数据；业务模块负责触发与扫码校验。

## 短自检

- 能按「能力名」从本页地图落到唯一叶页，而不是在业务模块里另起一套平台说明。
- 能向同事说清：业务页只写触发/口径/变量；模板、存储、渠道、通用校验口径回本目录叶页。
- 标签争议时，能先指向 WMS 现入口，同时说明平台治理目标（不假装已有独立 Infra 标签中心）。
