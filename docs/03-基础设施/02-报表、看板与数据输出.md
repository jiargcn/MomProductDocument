# 报表、看板与数据输出

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：实施顾问、报表管理员，以及需要解释“报表在哪配、业务页如何打开报表”的支持人员。

## 业务目的与适用范围

报表与看板用于按既定口径展示业务结果、支持打印/导出与管理审视。本页说明**当前平台级报表入口与嵌入方式**；具体指标口径、筛选参数与业务状态过滤以各报表/业务页为准。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 报表服务地址是否已配置 | 设计器与预览依赖报表基地址与登录令牌。 |
| 账号是否有报表菜单权限 | 设计器与业务报表入口受 RBAC 控制。 |
| 指标口径是否已评审 | 报表好看不等于口径正确。 |
| 租户上下文 | 多租户下报表数据需在正确租户中验证。 |

## 当前已证实的形态

```mermaid
flowchart LR
    A["报表设计器维护模板"] --> B["得到报表标识"]
    B --> C["业务页或报表菜单打开预览"]
    C --> D["用户查看 / 打印 / 导出（以模板能力为准）"]
```

| 形态 | 当前结论 |
| --- | --- |
| 报表设计器 | 菜单提供积木报表（JimuReport）设计入口，用于维护报表模板。 |
| 业务内嵌预览 | 多业务页通过报表预览地址打开指定模板，并携带登录令牌（及租户信息，视页面实现）。 |
| 报表菜单聚合 | 存在“报表 / 报表列表”类菜单，可挂具体业务报表入口。 |
| 模块专项报表 | MES/EAM/WMS 等另有报表统计、OEE 等业务报表页，不一定全部来自同一设计器清单。 |
| 列表导出 | Excel 导出属于导入导出能力，不等于报表模板输出。 |

## 分工边界

| 角色 | 负责什么 |
| --- | --- |
| 平台 / 报表管理员 | 设计器权限、模板发布、服务地址与通用嵌入约定。 |
| 业务模块 | 何时打开哪张模板、参数如何传入、口径与状态过滤。 |
| 数据权限 | 报表是否受部门范围约束需逐报表验证，不能默认“与列表一致”。 |

## 关键判断

| 判断点 | 应先确认什么 | 影响 |
| --- | --- | --- |
| 设计器打不开 | 报表基地址、令牌、菜单权限。 | 先修环境再改模板。 |
| 业务页报表空白 | 模板标识、令牌过期、租户参数。 | 对照同模块其它可打开报表。 |
| 数字与列表不一致 | 时间范围、状态过滤、组织范围。 | 回口径评审，不先改样式。 |
| 只要 Excel | 走列表导出，不一定要报表模板。 | 见导入导出页。 |

## 查询与联查

| 场景 | 建议看什么 | 联查 |
| --- | --- | --- |
| 找不到模板 | 报表设计器列表、业务页写死的模板标识。 | 业务页说明。 |
| 打印效果不对 | 模板设计与打印机/浏览器打印设置。 | 标签打印页（若是标签场景）。 |
| 权限不足 | 角色菜单是否含设计器/报表入口。 | RBAC。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 把看板大屏当成已有统一 BI | 当前以积木报表 + 模块报表页为主；独立实时看板平台待证实。 |
| 改模板影响多业务 | 先确认模板是否被多处引用，再变更。 |
| 用超管验收数据范围 | 换业务角色验证报表可见数据。 |

## 当前限制与待确认事项

- 统一指标字典/语义层未证实；
- 报表与角色数据范围的逐表矩阵未建立；
- 实时刷新看板、钻取闭环待按具体报表核验；
- 异步大批量报表任务与水印脱敏策略待专项。
