# 采集点-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[采集点](index.md)。主线关系与排障判断见主文档；本页给操作定位与字段核对。

读完本页，应能：在「采集点信息」与「设备参数配置」两页完成新增/导出，并按分区编码核对参数是否挂对采集点。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 配目标 IP/端口 | “采集点信息” |
| 打开某参数采集 | “设备参数配置” |
| 核对分区与采集点 | 参数表「分区编码」列 |
| 理解为何无日志 | 回主文档「关键判断」；再查[设备管理](../02-设备管理/index.md) 日志/MQTT |

## 页面与业务对象

| 菜单入口 | 组件路径线索 | 用途 |
| --- | --- | --- |
| 采集点管理 | `iot/collectpointinfo` | 采集点 CRUD / 导出 |
| 设备参数配置 | `iot/assetparamconfig` | 参数项 CRUD / 导出 |

## 采集点信息

### 查询

| 要找什么 | 推荐条件 |
| --- | --- |
| 某编码采集点 | 采集点编码 |
| 某 IP 目标 | 目标 IP（若页面提供筛选） |

### 应核对字段

| 字段 | 提醒 |
| --- | --- |
| 采集点编码 / 名称 | 编码供参数分区与边缘引用 |
| 目标 IP / 端口 | 与现场网络一致 |
| 目标参数地址 | 寄存器/点位地址类线索 |
| 目标类型 | 字典 `iot_target_type` |
| 扩展参数 1 / 2 | 按协议补充；含义随类型变化 |

### 常用动作

新增、编辑、删除、导出。前端权限码形如 `iot:collect-point-info:*`（接口层以网关服务为准）。

## 设备参数配置

### 查询

| 要找什么 | 推荐条件 |
| --- | --- |
| 某设备全部参数 | 设备编码 |
| 仅采集项 | 是否采集 = 采集 |

### 应核对字段

| 字段 | 提醒 |
| --- | --- |
| 设备编码 / 名称 | 须先有采集设备或约定编码 |
| 参数编码 / 名称 / 类型 | 类型字典 `data_type` |
| 读写方式 | 字典 `iot_param_rw` |
| 是否采集 | 不采集则边缘通常不上报该项 |
| 分区编码 | 与采集点规则对齐；界面亦称分区编码 |
| 参考值 | 参考/阈值线索，非库存数量 |

## 最小验收清单

- [ ] 新建一条采集点，列表可见编码与目标 IP。
- [ ] 为设备新增参数项，分区编码指向该采集点，打开采集。
- [ ] 导出采集点 / 参数配置 Excel 成功（有权限时）。
- [ ] 边缘上报后，在「采集日志」能按设备/参数/采集点查到值。
