# NodeRed

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> **资料状态：薄弱。** 本组只写清菜单意图、与 MES 的残留线索及缺口；不能当作已齐套的 Node-RED 运维手册。

## 业务目的（产品意图）

在数采根菜单下提供 Node-RED 相关入口，用于边缘流程编排、与产线校验/报工类动作协同（历史设计方向）。菜单上可见：

| 菜单 | path / 组件线索 | 盘点结论 |
| --- | --- | --- |
| NodeRed管理 | 父菜单 `nodered`（无独立 component） | 仅目录级入口 |
| NodeRed日志管理 | `mes/noderedlog/index`，路由名 `noderedlog` | **本仓库前端无** `views/mes/noderedlog` 目录 |

## 当前可确认状态

| 检查项 | 结论 |
| --- | --- |
| `reference/menu.csv` | 有上述两项菜单 |
| `sourcecode/frontend/ui/src/views` | **无** Node-RED 管理/日志页面实现 |
| MES 表 `mes_node_red_log` | 出现在遗留表下线脚本/审计清单中；现行 Java **无**稳定 CRUD 命中 |
| MES 错误码 / i18n | 仍保留 Node-RED 前序校验类文案（产线码空、请求体空、自动线料口等） |

**因此本轮不做**字段级维护与查询参考，避免编造不存在的表单字段。

## 与数采主链、MES 的边界

| 协同方 | 本页可写 | 不要写成事实 |
| --- | --- | --- |
| 数采采集点/日志 | Node-RED 可能作为边缘编排消费采集或触发业务 | 已证实的固定流程拓扑 |
| MES | 历史/残留「前序校验」错误码说明曾有 Node-RED 入参校验 | 现行 Controller 与日志表完整可用 |
