# NodeRed

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> **规划中 / 能力未落地。** 本组只写清菜单意图、与 MES 的残留线索及缺口；不能当作已覆盖的 Node-RED 运维手册。

## 业务目的（产品意图）

在数采根菜单下提供 Node-RED 相关入口，用于边缘流程编排、与产线校验/报工类动作协同（历史设计方向）。菜单上可见：

| 菜单 | path / 组件线索 | 盘点结论 |
| --- | --- | --- |
| NodeRed管理 | 父菜单 `nodered`（无独立 component） | 仅目录级入口 |
| NodeRed日志管理 | `mes/noderedlog/index`，路由名 `noderedlog` | **当前无**可用的日志管理页面实现 |

## 当前可确认状态

| 能力点 | 结论 |
| --- | --- |
| 菜单入口 | 有「NodeRed管理」「NodeRed日志管理」两项 |
| 管理/日志页面 | 当前无可用的管理台与日志查询页面实现 |
| Node-RED 日志落库 | 历史曾有相关日志对象线索；现行**无**稳定的维护/查询入口 |
| 前序校验提示 | MES 仍保留 Node-RED 相关前序校验类提示（如产线码空、请求体空、自动线料口等） |
| 结果枚举 | 相关枚举仍在，**不等于**管理台页面已可用 |

因此本页**不提供**字段级维护与查询参考：当前无可对应的表单字段可据实描述。

## 与数采主链、MES 的边界

| 协同方 | 本页可写 | 不要写成事实 |
| --- | --- | --- |
| 数采采集点/日志 | Node-RED 可能作为边缘编排消费采集或触发业务 | 已证实的固定流程拓扑 |
| MES | 历史/残留「前序校验」提示说明曾有 Node-RED 入参校验 | 现行管理台与日志查询完整可用 |
| 终端报工 | 线边报工以 MES 终端为准 | Node-RED 日志页可查全部报工 |
