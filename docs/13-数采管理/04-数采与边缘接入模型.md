# 数采与边缘接入模型

> 状态：占位页已建立。旧版规划书提出 PLC、设备监控、接口表监控与数采插件方向；当前架构待源码和现场环境核验。

## 1. 设计目的

统一说明设备、采集点、协议、边缘程序和业务模块之间的职责，避免将采集原始数据、设备状态和业务事件混为同一类数据。

## 2. 参考设计线索

旧版规划书提出可通过设备/PLC、ANDON 硬件或接口表获取事件，并由后台程序处理监控、消息推送和业务逻辑。该设计应作为核验清单，不应直接认定为现状。

## 3. 待核验清单

| 主题 | 需要确认的内容 |
| --- | --- |
| 采集对象 | 设备、采集点、信号、协议、采样频率和单位。 |
| 接入方式 | Node-RED、MQTT、PLC、HTTP、文件或数据库接口等实际方式。 |
| 数据质量 | 离线缓存、断线补传、去重、时钟、异常值和数据保留。 |
| 业务消费 | MES、EAM、ANDON 对采集数据的转换、触发和追溯关系。 |
