# 数采与边缘接入模型

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 统一说明设备、采集点、边缘程序、MQTT/网关与业务模块之间的职责，避免把采集原始值、设备台账和业务事件混为一谈。

## 1. 设计目的

- 分清**配置**（采哪里、采什么）、**通道**（边缘/MQTT）、**时序事实**（采集日志）与**业务消费**（MES 点检、WMS 读参、AGV 回调等）。
- 标明本仓库已证实的 Web 管理面，与仓外 IoT 网关服务的边界。
- 旧版规划书中的 PLC/插件方向只作核验清单，不直接认定为现状。

## 2. 分层模型（已证实 + 未证实）

```mermaid
flowchart TB
    subgraph conf ["配置层（Web /iot 已证实）"]
      CP["采集点"]
      ASSET["采集设备"]
      PARAM["参数配置"]
    end
    subgraph edge ["边缘与通道（部分待联调）"]
      NR["Node-RED / 其它边缘"]
      MQTT["MQTT"]
      GW["IoT 网关 /iotgw"]
    end
    subgraph fact ["事实层"]
      LOG["参数采集日志"]
      ST["MQTT 客户端状态"]
    end
    subgraph biz ["业务消费"]
      MES["MES 点检等"]
      WMS["WMS 部分作业"]
      AGV["AGV 回调服务"]
    end
    CP --> PARAM
    ASSET --> PARAM
    PARAM --> NR
    NR --> MQTT
    MQTT --> GW
    GW --> LOG
    GW --> ST
    GW --> MES
    GW --> WMS
    AGV -->|"iotCallback 等"| AGV
```

| 层级 | 已证实 | 未证实 / 薄弱 |
| --- | --- | --- |
| 配置层 | 五类 Web 页 + `/iotgw` CRUD/导出路径族 | 网关 DDL、唯一约束、删除级联 |
| 边缘与通道 | 菜单与 MQTT 监控页存在；MES/WMS 依赖 `iotgw` API 包 | Node-RED 页面实现；补传/时钟/去重策略 |
| 事实层 | 采集日志、MQTT 状态页面字段 | 时序存储引擎与保留期 |
| 业务消费 | MES 点检服务引用网关读参/发消息；WMS 部分服务引用读参；AGV 提供 `iotCallback` | 各场景字段映射与失败补偿全量矩阵 |

## 3. 与邻近模块的权威边界

| 主题 | 权威落点 | 说明 |
| --- | --- | --- |
| 设备身份与工厂归属 | **DBC** 设备/工装台账、车间产线工位 | 数采「采集设备」是采集侧资产 |
| 维修 / 巡检工单 | **EAM** | 不读采集日志当工单 |
| 异常呼叫 | **ANDON** | 事件响应 ≠ 参数时序 |
| 库存事务 | **WMS** | 即使作业读了 IoT 值，库存仍以 WMS 为准 |
| AGV 点位地图 | **DBC** 点位 / 潜伏式 AGV 点位 | AGV 模块侧重任务回调，不作数采配置台 |
| 线边报工 | **MES 终端** | Node-RED 不得替代终端状态机文档 |

## 4. 待核验清单（承接旧稿）

| 主题 | 需要确认的内容 |
| --- | --- |
| 采集对象 | 信号、协议、采样频率、工程单位与字典取值全集。 |
| 接入方式 | Node-RED、MQTT、PLC、HTTP 等在目标环境中的实际组合。 |
| 数据质量 | 离线缓存、断线补传、去重、时钟、异常值、保留周期。 |
| 业务消费 | 点检、叫料/容器、AGV 触发等与采集编码的映射及失败处理。 |
| 安全 | 集成账号权限、MQTT 认证、回调接口暴露面。 |
