# DBC 主数据管理

> 基线状态：已完成首轮源码取证（`dev`）。本页的“已核验事实”可作为当前文档入口；其余既有字段表、流程图仍需按页面继续核验。

## 已核验事实与职责边界

DBC 是制造业务共享主数据和业务基础配置的提供方。当前后端模块已存在物料、供应商/客户、仓库—库区—库位、车间—产线—工位、工序/工艺、货主/承运商、设备/工装等相关控制器与服务；同时提供业务类型、单据设置、单据开关和规则管理。

| 领域 | 当前已核验能力 | 文档边界 |
| --- | --- | --- |
| 主数据 | 物料、供应商、客户、仓库/库区/库位、生产组织、工艺、物流与设备/工装等维护服务。 | 定义业务对象及其引用规则；具体业务单据由各业务模块维护。 |
| 业务基础配置 | 业务类型、单据设置、单据开关和规则管理。 | 为单据、库存与业务执行提供配置；通用模型见[业务模型](../02-业务模型/05-单据类型、业务类型与单据配置.md)。 |
| 管理精度 | 规则服务提供按物料、库位查询库存管理精度的接口。 | 实际库存余额键和事务写入规则由 WMS 说明。 |
| 导入导出 | 控制器基类、导入日志、导出日志等实现存在于 DBC 模块。 | 通用治理规则见[导入、导出与批量操作](../03-基础设施/04-导入、导出与批量操作.md)。 |

> 标签、条码与打印不再作为 DBC 的长期平台职责。本模块可继续提供物料、包装及当前配置入口所需的业务数据；模板、编码、渲染、打印设备与审计以[平台能力与基础设施](../03-基础设施/01-标签、条码与打印.md)为准。

## 当前规则实现要点

1. 新建规则默认处于草稿状态，并自动生成版本号。
2. 规则按“业务类型 + 规则分类”管理发布版本；发布一条规则时，既有发布版本会转为失效状态。
3. 发布和发布态更新会校验规则脚本；规则页不能再将“单纯按数值优先级排序”写为已确认事实。
4. 当前存在按物料号集合、以及按物料号和库位号查询管理精度的接口；具体返回值和业务使用点待后续取证。

## 模块概述

主数据管理（Data Base Core，DBC）是 MOM 系统的数据基石，统一管理和维护各业务模块共享的核心基础数据，确保数据的一致性和准确性。

## 业务分组

| 分组 | 主数据 |
|------|--------|
| 01-物料管理 | 物料基本信息、BOM、物料包装信息、包装规格、物料库区配置管理、生产线物料关系管理、标准成本价格单管理 |
| 02-供应商管理 | 供应商、供应商物料 |
| 03-客户管理 | 客户、客户月台、客户物料 |
| 04-工厂建模 | 仓库管理、库区管理、库位管理、库位组管理、月台管理、车间管理、生产线管理、工位管理、工序管理、工艺路线、模具类型管理、模具信息管理、点位管理、潜伏式AGV点位配置表 |
| 05-策略设置 | 班次管理、班组管理、业务类型、单据设置、单据开关、规则管理 |
| 06-物流配置 | 货主、承运商 |
| 07-设备管理 | 设备台账管理、工装台账管理、生产商管理 |

## 主数据维护顺序

主数据之间存在层层依赖，维护时必须遵循"先基础后业务"的顺序。以下按批次划分依赖关系。

### 前置依赖（不在 DBC 范围内）

以下数据由系统管理模块提供，DBC 维护前需确认已存在：

| 前置数据 | 说明 |
|---------|------|
| 组织架构 | 仓库、车间、生产线等主数据需要关联组织，用于权限隔离和数据归属 |
| 用户与权限 | 主数据维护人员需要对应角色权限 |
| 数据字典 | 物料类型、状态等枚举值由系统字典定义，DBC 引用字典项 |

### 第一批：底层基础数据（无外部依赖）

最先录入，建立企业物理结构和人员组织的基础框架：

| 主数据 | 依赖关系 | 维护要点 |
|--------|---------|---------|
| 仓库管理 | 无 | 企业所有仓储实体的物理位置入口 |
| 车间管理 | 无 | 定义生产组织的物理区域 |
| 货主 | 无 | 库存所有权归属的最顶层实体 |
| 承运商 | 无 | 物流履约的承运方基础数据 |
| 供应商 | 无 | 采购供应链的最前端 |
| 客户 | 无 | 销售和发货的目标实体 |
| 生产商管理 | 无 | 设备/工装的生产厂家基础数据 |

### 第二批：依赖第一批的工厂结构数据

在仓库和车间基础上，建立精细化物理位置：

| 主数据 | 依赖关系 | 维护要点 |
|--------|---------|---------|
| 库区管理 | 仓库 | 同一仓库下按类型（原料区/成品区/退货区等）划分 |
| 库位组管理 | 库区 | 库区内按物理或逻辑分组的中间层 |
| 库位管理 | 库位组 | 最小存储单元，须先确认库位组结构 |
| 月台管理 | 仓库 | 仓库的装卸货出入口 |
| 班次管理 | 车间 | 定义车间维度的排班时间模板 |
| 班组管理 | 车间、班次 | 班次的具体执行班组 |
| 业务类型 | 无 | 所有业务单据的事务归类基础 |

### 第三批：依赖前两批的产品与工艺数据

| 主数据 | 依赖关系 | 维护要点 |
|--------|---------|---------|
| 物料基本信息 | 无 | 最核心基础数据，其他所有模块都引用物料 |
| 物料包装信息 | 物料基本信息 | 物料的包装规格说明 |
| 包装规格 | 物料基本信息 | 定义物料的包装层级和换算关系 |
| 工序管理 | 无 | 工序主数据，被工艺路线引用 |
| 工艺路线 | 物料基本信息、工序 | 定义产品制造过程，按物料维度维护 |
| 设备台账管理 | 组织架构 | 设备资产台账 |
| 工装台账管理 | 组织架构 | 工装模具台账 |

### 第四批：依赖前三批的关联配置数据

| 主数据 | 依赖关系 | 维护要点 |
|--------|---------|---------|
| BOM | 物料基本信息 | 产品结构树，定义父项→子项的用量关系 |
| 物料库区配置 | 物料基本信息、库区 | 物料默认存放库区，指定默认仓位 |
| 生产线管理 | 车间 | 车间内的产线定义 |
| 工位管理 | 生产线、工序 | 产线上的作业工位，须先维护工序 |
| 供应商物料 | 物料基本信息、供应商 | 供应商对物料的报价关系 |
| 客户物料 | 物料基本信息、客户 | 客户对物料的规格要求 |
| 客户月台 | 客户 | 客户收货的月台信息 |

### 第五批：依赖 BOM 和工艺的深度配置

| 主数据 | 依赖关系 | 维护要点 |
|--------|---------|---------|
| 物料库区配置 | 物料基本信息、库区 | 补充物料与库位的默认对应关系 |
| 生产线物料关系 | 物料基本信息、生产线 | 产线专属物料配置 |
| 标准成本价格单 | 物料基本信息 | 物料的标准成本价，用于成本核算 |
| 模具类型管理 | 无 | 模具分类管理 |
| 模具信息管理 | 模具类型、工装台账 | 具体模具资产信息 |
| 点位管理 | 车间 | AGV 导航点位 |
| 潜伏式AGV点位配置 | 点位 | AGV 潜伏式点位参数 |

### 第六批：运营策略与规则配置

| 主数据 | 依赖关系 | 维护要点 |
|--------|---------|---------|
| 单据设置 | 业务类型 | 定义单据编号规则 |
| 单据开关 | 单据设置 | 控制单据类型的启用/禁用 |
| 规则管理 | 物料基本信息、库位等 | 业务规则的条件和优先级配置 |
