## 当前页面事实卡（第二轮源码、DDL 已证实）

> 适用基线：测试环境对应的 `dev` 分支，2026-07-15。本节优先于后文历史草稿。当前一条 `basic_itempackage` 记录定义“一个物料使用一种包装规格时的标准包装数量及有效性”；物料与包装规格均以编码保存。它不是库存中的实物包装、包装条码或标签记录。

### 数据对象、字段与业务键

| 对象/字段 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 数据表 | `basic_itempackage`，主键 `id`。 | 物料包装主数据为独立配置表。 |
| 核心业务字段 | `item_code`、`uom`、`package_unit` 均为 `varchar(64) NOT NULL`；`package_std_qty` 为 `numeric(18,6) NOT NULL`。 | 真实技术字段应使用上述名称；后文的 `materialCode`、`packSpec`、`packQty` 等是历史草稿伪字段，不能用于接口、SQL 或导入。 |
| 可用性与有效期 | `available` 为非空 `boolean`、默认 `true`；`active_time`、`expire_time` 为可空时间戳。 | 可用性是布尔值；有效期允许为空，不能把“必须填写生效日期”写成当前接口规则。 |
| 默认包装 | `default_packageunit` 为可空 `varchar(64)`；Web 使用字符串 `TRUE` / `FALSE`。 | **不是 Boolean 列**。同一物料的默认包装由服务层维护；当前 DDL 未强制该字段非空。 |
| 审计与系统字段 | `remark` 为 `text`；另有创建/更新/删除人和时间、`extra_properties`、`concurrency_stamp`、`site_id`、`tenant_id`、`deleted`。 | 审计字段和系统字段由服务/框架维护，不应作为人工导入列。 |
| 业务键 | 创建服务按 `itemCode + packageUnit` 查询重复；DDL 仅见主键，未见该组合的复合唯一约束。 | 此组合是当前服务层业务键；数据库并发或旁路写入保护不足，已登记为差距。 |

### 新增约束与选择器

| 字段/动作 | 当前证据 | 当前结论 |
| --- | --- | --- |
| 物料代码 `itemCode` | 前端选择器以物料基础信息页面和 `getItembasicPage` 作为数据源，筛选 `available = true`；选中后写入物料 `code`，并带出 `uom`。 | 日常新增应从当前可用的[物料基本信息](01-物料基本信息.md)选择，计量单位随物料带出且前端禁用编辑。服务直接创建仅检查 `itemCode` 非空，未在此方法中查询物料是否真实存在，不能把选择器约束误写成接口强制约束。 |
| 包装规格 `packageUnit` | 前端选择器以[包装规格](04-包装规格.md)页面和 `getPackageunitPage` 为数据源，筛选 `available = true`；选中后写入包装规格 `code`，并带出包装类型供显示。 | 日常新增应选择当前可用包装规格。服务创建只检查非空；是否在其他层校验包装规格真实存在，待测试环境与接口专项核验。 |
| 包装数量 `packageStdQty` | DDL 非空、精度 6；前端数值输入配置 `min: 0`、`precision: 6`；导入通用校验将含“数量”的必填列要求为 `> 0`。 | 培训与导入按“必须大于 0、最多 6 位小数”执行。普通新增服务仅检查非空，前端/导入与服务边界存在差异。 |
| 是否可用 `available` | DDL 非空默认 `true`；前端新增 API 强制赋值为 `true`，表单中不显示该字段。 | Web 新增默认可用；启用/禁用应通过独立动作维护。 |
| 默认包装 `defaultPackageunit` | 前端开关默认 `FALSE`；创建时若选 `TRUE`，服务将同物料其他包装规格改为 `FALSE`。若选 `FALSE`，服务要求该物料仍保有其它 `TRUE` 的默认包装。 | 同一物料的日常维护应始终保留一个默认包装；默认项切换会联动修改同物料其它记录。直接接口缺少默认值可能触发空值异常，不能省略该字段。 |
| 有效期 | 前端提交前仅在两个时间都存在时校验 `expireTime > activeTime`。 | 两个时间可空；若同时填写，失效时间必须晚于生效时间。服务层是否有同等创建/更新校验待核验。 |

### 编辑、删除与接口挂接

| 动作 | 当前入口与实现 | 当前结论 |
| --- | --- | --- |
| 编辑 | Web 编辑禁用 `itemCode`、`packageUnit` 和 `packType`，并关闭两类选择器。服务更新仅写入包装数量、可用性、有效期、包装规格和默认包装；未写入 `uom`、备注、包装名称/类型。 | 培训按“物料代码和包装规格不可编辑”执行。服务仍接收包装规格并写回，且更新前的完整唯一性/引用校验被注释，接口绕过页面的边界已登记为差距。 |
| 启用/禁用 | Web 行操作权限标识为 `dbc:itempackage:enable` / `dbc:itempackage:disable`；后端接口实际启用 `wms:itempackage:enable` / `wms:itempackage:disable`。 | 已能证明前后端权限命名不一致，不能据此证明某角色一定可执行。须按 RBAC 模板和测试环境核验。 |
| 删除 | Web 支持删除，默认包装仅给出更强确认提示；服务先记录删除人/删除时间，再执行框架删除并登记一条 WMS-SCP 接口待发送记录。默认包装“不得删除”的服务校验已被注释。 | 当前代码允许删除默认包装；是否存在库存、单据、标签等引用拦截未见，不能作为安全维护规则。 |
| WMS-SCP 接口记录 | 新增、更新、删除均写入 `InterfaceInfo`：类型 `WMS-SCP`、名称“推送物料包装信息”、地址 `/dbc/itempackage/create`、业务号为物料代码。默认包装变更时会将该物料的包装记录组装为列表。 | 已证实系统会登记接口数据；实际传输时机、第三方接收结果、重试和删除语义仍待 Infra/接口专项取证。 |

### 导入规则

| 主题 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 模板应由用户填写 | `ItempackageImportExcelVO` 读取：物料代码、包装规格、包装数量、是否可用、生效时间、失效时间、默认包装规格。其中物料代码、包装规格、包装数量、是否可用、默认包装规格标为必填。 | 模板仅保留这些业务字段；计量单位应由物料带出，包装名称/类型应由包装规格解析，审计字段不得由用户填写。 |
| 导出列与导入列 | `ItempackageExcelVO` 导出还包含计量单位、包装名称、包装类型、创建/更新人及时间；导入读取 VO 不含这些字段。 | **不能将导出文件直接当作人工导入模板。** 导入、保存、导出使用不同 VO，模板以读取 VO 为准。 |
| 重复与模式 | 导入文件内按 `itemCode + packageUnit` 查重；保存时也以该组合查现存记录。模式：`1` 更新（不存在则新增、存在则更新）、`2` 追加（只新增，存在报错）、`3` 覆盖（只更新，不存在报错）。 | 同一文件不可重复同一物料/包装规格；导入前需选择正确模式。 |
| 业务校验 | 导入会校验物料、单位、包装规格、数量、可用性、默认包装规则；物料存在时将其 `uom` 覆盖到导入对象。默认包装为 `FALSE` 时，系统检查该物料是否已有/本次导入是否包含 `TRUE` 默认项。 | 物料代码、包装规格、数量和默认包装是导入关键控制点；具体错误文案、字典转换和跨行默认项处理须在测试环境回归。 |
| 错误处理 | `updatePart = false` 时存在任一错误会回滚整批；`true` 时允许成功行继续写入并返回行错误。 | 导入前应明确是否允许部分成功，避免误将部分更新当作整批成功。 |

## 概述

物料包装信息记录物料与包装规格之间的标准数量关系，是收货、发货、扫码换算和标签相关能力可引用的主数据之一。具体下游使用点和接口传输结果仍需逐页取证。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| `itemCode` | 物料代码 | `varchar(64)` | 非空 | 收发货、包装换算等（待逐页核验） | 持久化字段。 |
| `uom` | 计量单位 | `varchar(64)` | 非空 | 同上 | 由物料选择器/导入物料带出。 |
| `packageUnit` | 包装规格 | `varchar(64)` | 非空 | 同上 | 持久化字段，关联包装规格编码。 |
| `packageStdQty` | 包装数量 | `numeric(18,6)` | 非空 | 数量换算 | 导入最终要求大于 0。 |
| `available` | 是否可用 | `boolean` | 非空，默认 `true` | 可用配置过滤 | 前端新增默认传 `true`。 |
| `defaultPackageunit` | 默认包装规格 | `varchar(64)` | DDL 可空 | 默认换算选择 | 前端使用 `TRUE` / `FALSE`。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)` | 可空 | 有效性判断 | 两者都有值时前端要求失效晚于生效。 |
