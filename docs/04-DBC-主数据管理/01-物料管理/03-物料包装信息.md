# 物料包装信息

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：主数据维护人员、仓库收发人员、物流协同人员及需要管理包装追溯的业务人员。

## 业务目的与适用范围

物料包装信息用于说明一种物料在收货、存储、搬运和发运时采用什么包装形式，以及包装与物料、数量和追溯信息如何对应。它帮助仓库把“按件、箱、托盘或其它容器操作”的现场动作，与物料主数据和库存记录保持一致。

本页不替代实际包装标签、托盘号或库存包装记录；它描述的是可复用的包装资料和业务规则。

## 何时需要维护

在新物料启用、包装规格变更、物流方式调整、客户/供应商对包装提出要求，或现场扫码与数量换算出现问题时，应检查或维护物料包装信息。

## 一项包装信息如何被使用

```mermaid
flowchart LR
    A["物料基本信息"] --> B["维护适用包装信息"]
    B --> C["收货/上架采集包装"]
    C --> D["库存按包装追溯"]
    D --> E["发料/发运按包装执行"]
```

包装信息需要与物料单位、包装规格和现场标签口径一致。若一个物料存在多种包装形式，应明确各自适用的收发、存储或交付场景。

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 适用物料 | 包装资料属于哪种物料。 | 先确认物料编码和基本单位。 |
| 包装形式与层级 | 箱、托、容器等是否符合现场实际。 | 与包装规格、标签和搬运方式保持一致。 |
| 数量换算 | 一包装对应多少物料数量。 | 单位和换算变更需评估库存、收发和标签影响。 |
| 启停或替换 | 旧包装是否仍在现场或客户交付中使用。 | 不建议直接删除已用于库存追溯的包装资料。 |

## 维护与查询说明

物料包装信息采用单文档：维护和查询规则与包装业务说明合并在本页。

| 操作 | 需要核对什么 | 当前限制或建议 |
| --- | --- | --- |
| 新增包装关系 | 物料、基本单位、包装规格、包装数量、可用状态和默认包装标识。 | 物料与包装规格均从可用主数据中选择；包装数量必填且必须大于 0。 |
| 设置默认包装 | 同一物料在当前收发场景下应优先使用哪一种包装。 | 每个物料的默认包装应保持唯一；新增或导入前先检查已有默认关系。 |
| 修改或停用 | 换算、默认包装、有效期是否影响在途收货、库存包装、发料或交付。 | 先评估实际库存包装和未完成作业；不以直接删除代替切换。 |
| 批量导入 | 物料、包装规格、包装数量、默认标识、可用状态和有效期。 | 导入会核对物料与包装规格的对应关系、数量和默认标识；处理模式与错误回执待测试确认。 |

建议列表优先显示物料、单位、包装规格/名称/类型、包装数量、默认包装、可用状态和有效期；常用筛选为物料、包装规格、包装类型和状态。详情建议按“物料与单位、包装与换算、默认与有效期、库存/收发引用、审计信息”分组。

## 查询、详情与联查

| 查询目标 | 建议入口 | 应继续联查 |
| --- | --- | --- |
| 某物料可用哪些包装 | 按物料查询包装信息。 | 包装规格、物料单位。 |
| 某包装为何不能收发 | 包装状态、换算和适用物料。 | 收货/发料任务、标签规则。 |
| 某托/箱中的物料如何追溯 | 包装信息与实际库存包装记录。 | 库存余额、库存事务。 |

详情页建议按“适用物料、包装层级与换算、标签/追溯、状态与变更、相关库存业务”分组。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 现场包装与主数据不一致 | 暂停使用旧口径，核对包装规格、标签和物料单位后再维护。 |
| 收货数量换算错误 | 同时核对物料基本单位、包装数量和任务单位。 |
| 已有库存包装需要变更 | 评估库存、批次/包装追溯和未完成收发后再处理。 |

## 当前限制与待确认事项

- 默认包装的唯一性、处理模式和库存/标签影响待测试确认；
- 包装层级、托盘/容器关系和标签生成方式需结合测试环境确认；
- 包装变更对现存库存、扫描和客户交付的影响需补实际场景。
