# 物料库区配置管理

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；仓储主数据维护、库房主管、采购与线边物流相关人员（顺带）。

## 业务目的与适用范围

当前菜单名沿用「物料库区配置管理」，实际维护的是**物料在指定仓库中的默认管理配置**：物料、仓库、管理精度、入库包装及有效状态。读完本页，应能判断：收货包装口径该回查本页还是物料包装；「选不到库区/库位」为何不能指望本页；模板里「区域代码」为何应按仓库理解。

本页不是库区、库位或上架策略页（`GAP-032`），也不能代替实际库存记录。具体新增、编辑、导入与查询见[物料库区配置管理-维护与查询参考](09-物料库区配置管理-维护与查询参考.md)。

## 什么时候需要维护

| 业务事件 | 应做什么 | 维护前要确认 |
| --- | --- | --- |
| 新物料准备在某仓库收发或存放 | 新增该物料与仓库的默认配置。 | 物料已启用、仓库已启用，包装口径已确认。 |
| 物料改用新的收发包装或管理粒度 | 调整配置前先评估在途收发和现场执行口径。 | 新旧包装的换算、标签和库存管理影响。 |
| 仓库启停、物料停用或业务范围变化 | 停用不再适用的配置；不要直接把历史配置改成另一业务含义。 | 是否仍有未完成收货、上架、发料或跨系统同步。 |
| 需要为外部供应链平台提供基础口径 | 维护后关注接口记录和异常反馈。 | 数据是否已通过本页维护而非仅通过批量导入。 |

## 它与仓储业务的关系

```mermaid
flowchart LR
    A["物料基本信息"] --> C["物料—仓库默认配置"]
    B["仓库与包装资料"] --> C
    C --> D["收货、库存与线边物流的基础口径"]
    C --> E["供应链平台的主数据同步"]
    D --> F["实际库存与作业记录"]
```

该配置为后续作业提供基础条件，但不表示系统已按此配置自动推荐库区或库位，也不表示它会自动生成收货、上架或发料任务。实际结果仍以业务单据、任务和库存查询为准。存放地点约束若指库区/库位，须到仓储策略或作业页处理，不要在本页找「默认库位」。

!!! example "写实示例：给定配置 → 期望行为"
    **给定：** 物料 RM-001、仓库 WH-A、管理精度按现场约定、入库包装选自该物料可用包装（每箱 50 PCS）；配置可用。
    **期望：**

    1. 可按物料查到 WH-A 下的默认配置；包装口径应与[物料包装信息](03-物料包装信息.md)一致。
    2. 页面保存时包装对象与后端校验对象可能不一致（`GAP-031`）——保存失败时先核两端口径，勿只改显示值。
    3. 在本页找不到「默认库区/库位」字段是当前能力边界（`GAP-032`），不是操作遗漏。
    4. 导入模板「区域代码」填库区编码会导致错仓（`GAP-034`）；培训一律按仓库填写。

## 维护时最重要的判断

### 关键字段业务角色

完整语义与选择器见[维护与查询参考](09-物料库区配置管理-维护与查询参考.md)。**当前页面不是库区/库位策略页**（`GAP-032`）。选择器通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)；管理精度与余额粒度通例见[库存管理精度与唯一粒度](../../02-业务模型/08-库存管理精度与唯一粒度.md)。仓→区→位级联勿套用本页（见[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md) 差异示例）。

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- |
| 物料 | 配置归属物料 | 选自可用物料；编辑锁定；与仓库组成关系键 | 无收发场景勿提前配置 |
| 仓库 | 适用仓储范围 | 选自可用仓库；编辑锁定；模板「区域代码」实为仓库（`GAP-034`） | 勿填成库区代码 |
| 管理精度 | 该仓内管理粒度口径 | 必选；对各作业强制门禁 ❓ | 与实物管理方式不符 |
| 入库包装 | 收货包装口径 | 随物料过滤包装候选；Web 偏物料包装、服务偏包装规格（`GAP-031`） | 前后端口径不一致会导致保存失败 |
| 是否可用 / 有效期 | 配置是否生效 | 可用必填；物料+仓库唯一性 ❓ | 在途业务切换前先评估 |
| 库区 / 库位 / 默认 / 优先级 | 历史/目标能力 | **当前主表单未接入**（`GAP-032`） | 培训不得写成已实现库位推荐 |

| 需要判断什么 | 业务含义 | 建议做法 |
| --- | --- | --- |
| 物料与仓库是否确有业务关系 | 避免为尚未启用的仓库或无收发场景的物料提前配置。 | 以实际收发、保管或线边供应范围为准。 |
| 管理精度是否适合 | 影响物料在该仓库内的管理口径。 | 由仓储负责人按物料特性和作业方式确认，不能只按历史习惯填写。 |
| 入库包装是否一致 | 影响收货时使用的包装口径。 | 与物料包装信息、包装规格和现场标签口径一并核对。 |
| 变更是否会影响在途业务 | 避免新旧口径同时生效造成现场困惑。 | 先查询未完成收货、库存及待处理任务，再安排切换时间。 |

### 选不到配置对象与存放约束

| 现场现象 | 先回查本页什么 | 再联查什么 |
| --- | --- | --- |
| 物料或仓库选不到 | 对象是否存在、是否可用 | 物料基本信息、仓库资料、权限范围 |
| 入库包装选不到/保存失败 | 物料包装是否已建；前后端包装对象是否一致 | [物料包装信息](03-物料包装信息.md)、包装规格（`GAP-031`） |
| 要指定默认库区/库位 | **本页当前无此能力** | 仓储策略或作业规则专项确认 |
| 收货包装与现场不一致 | 本页入库包装 + 是否可用 | 收货单据、标签、物料包装数量 |

### 建议验证点

- 未先维护物料包装时，入库包装候选是否为空或保存失败（`GAP-031`）。
- 导入「区域代码」填仓库码 vs 库区码的结果差异（`GAP-034`）。
- 停用配置后，新收货是否仍引用旧包装口径（时点 ❓）。
- 页面是否不出现库区/库位/优先级字段（`GAP-032` 能力边界确认）。

## 使用边界与风险提示

- 本页当前不能作为“物料可放在哪个库区/库位”的权威依据（`GAP-032`）；需要此类规则时，应另行确认仓储策略页面及现场执行逻辑。
- 页面中的“区域代码”字样与实际选择的仓库对象存在术语不一致（`GAP-034`）。培训、模板和业务沟通一律以“仓库”为准。
- 普通页面维护和批量导入在包装对象的选择口径上存在差异风险（`GAP-031`）；正式批量维护前应先用测试数据验证。
- 删除未见完整下游引用保护：❓ 待确认。

| 想回答的问题 | 建议先查什么 | 再联查什么 |
| --- | --- | --- |
| 某物料在哪些仓库建立了默认配置 | 本页按物料查询。 | 物料基本信息、仓库资料。 |

| 配置变更是否影响供应链协同 | 本页的维护记录和接口异常。 | 供应链平台对应主数据的同步结果。 |

## 当前限制与待确认事项

- `GAP-031`：前端包装选择对象与后端校验对象不一致，可能导致保存失败或空包装。
- `GAP-032`：菜单名/历史草稿含库区库位策略，当前仅物料—仓库—精度—包装。
- `GAP-034`：导入“区域代码”实为仓库；导入包装与同步行为需专项回归。
- 物料+仓库组合唯一性、停用后收货引用、管理精度对各作业的强制程度：❓ 待确认。
