# 物料库区配置管理

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓储主数据维护人员、库房主管、采购与线边物流相关人员。
> 本页先讲清业务用途；具体新增、编辑、导入和查询操作见[物料库区配置管理-维护与查询参考](09-物料库区配置管理-维护与查询参考.md)。

## 这项配置实际解决什么问题

当前页面名称沿用“物料库区配置管理”，但已启用的维护内容实际是**物料在指定仓库中的默认管理配置**：为一项物料选择适用仓库、管理精度和入库包装规格，并维护其有效状态。它不是库区、库位或上架策略维护页面，也不能代替实际库存记录。

因此，读者应把它理解为“物料—仓库默认配置”。当需要维护具体库区、库位、上架推荐或临时存放地点时，应在对应的仓储配置或作业页面处理；当前页面尚不能证明这些能力已经实现。

## 什么时候需要维护

| 业务事件 | 应做什么 | 维护前要确认 |
| --- | --- | --- |
| 新物料准备在某仓库收发或存放 | 新增该物料与仓库的默认配置。 | 物料已启用、仓库已启用，包装口径已确认。 |
| 物料改用新的收发包装或管理粒度 | 调整配置前先评估在途收发和现场执行口径。 | 新旧包装的换算、标签和库存管理影响。 |
| 仓库启停、物料停用或业务范围变化 | 停用不再适用的配置；不要直接把历史配置改成另一业务含义。 | 是否仍有未完成收货、上架、发料或跨系统同步。 |
| 需要为外部供应链平台提供基础口径 | 维护后关注接口记录和异常反馈。 | 数据是否已通过本页维护而非仅通过批量导入。 |

## 它与仓储业务的关系

```mermaid
flowchart LR
    A["物料基本信息"] --> C["物料—仓库默认配置"]
    B["仓库与包装资料"] --> C
    C --> D["收货、库存与线边物流的基础口径"]
    C --> E["供应链平台的主数据同步"]
    D --> F["实际库存与作业记录"]
```

该配置为后续作业提供基础条件，但不表示系统已按此配置自动推荐库区或库位，也不表示它会自动生成收货、上架或发料任务。实际结果仍以业务单据、任务和库存查询为准。

## 维护时最重要的判断

| 需要判断什么 | 业务含义 | 建议做法 |
| --- | --- | --- |
| 物料与仓库是否确有业务关系 | 避免为尚未启用的仓库或无收发场景的物料提前配置。 | 以实际收发、保管或线边供应范围为准。 |
| 管理精度是否适合 | 影响物料在该仓库内的管理口径。 | 由仓储负责人按物料特性和作业方式确认，不能只按历史习惯填写。 |
| 入库包装是否一致 | 影响收货时使用的包装口径。 | 与物料包装信息、包装规格和现场标签口径一并核对。 |
| 变更是否会影响在途业务 | 避免新旧口径同时生效造成现场困惑。 | 先查询未完成收货、库存及待处理任务，再安排切换时间。 |

## 使用边界与风险提示

- 本页当前不能作为“物料可放在哪个库区/库位”的权威依据；需要此类规则时，应另行确认仓储策略页面及现场执行逻辑。
- 页面中的“区域代码”字样与实际选择的仓库对象存在术语不一致。培训、模板和业务沟通一律以“仓库”为准，并将该术语差异作为改进项跟踪。
- 普通页面维护和批量导入在包装对象的选择口径上存在差异风险；正式批量维护前应先用测试数据验证。

## 查询与联查

| 想回答的问题 | 建议先查什么 | 再联查什么 |
| --- | --- | --- |
| 某物料在哪些仓库建立了默认配置 | 本页按物料查询。 | 物料基本信息、仓库资料。 |
| 某仓库有哪些物料配置 | 本页按仓库查询。 | 物料包装信息、实际库存余额。 |
| 收货包装为什么与现场不一致 | 本页的入库包装配置。 | 物料包装信息、包装规格、收货单据和标签。 |
| 配置变更是否影响供应链协同 | 本页的维护记录和接口异常。 | 供应链平台对应主数据的同步结果。 |
