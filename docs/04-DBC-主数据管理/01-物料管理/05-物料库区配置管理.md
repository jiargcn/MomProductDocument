## 当前页面事实卡（第二轮源码、DDL 已证实）

> 适用基线：测试环境对应的 `dev` 分支，2026-07-15。页面名称为“物料库区配置管理”，但当前主对象实际是 `basic_itemwarehouse`（物料仓库默认配置）：核心关系为“物料 + 仓库 + 管理精度 + 入库包装规格”。本节优先于后文历史草稿；不要把尚未在当前表单中实现的库区、库位、默认标记或优先级写成当前功能。

### 数据对象、字段与真实边界

| 对象/字段 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 核心表 | `basic_itemwarehouse`，主键 `id`；`item_code`、`warehouse_code`、`manage_mode`、`package_unit` 均为 `varchar(64) NOT NULL`。 | 当前配置的粒度至少为物料与仓库，不是历史草稿中的“物料 + 仓库 + 库区 + 库位”。 |
| 核心配置 | `itemCode`、`warehouseCode`、`manageMode`、`packUnit`；`available` 非空、默认 `true`；有效期可空。 | `packUnit` 是后端 VO/DO 的入库包装规格字段；真实 DDL 列为 `package_unit`。 |
| DDL 扩展配置 | DDL 另有 `area_code`、`area_type`、溢出库位、入/出库包装、库存上下限、安全库存、补料/混放/排序等列。 | 这些字段多数未出现在当前页面 Base VO 和表单中，不能据此宣称页面已支持库区策略、补料策略或库位推荐。 |
| 唯一性 | DDL 未见复合唯一约束；导入与查询按 `itemCode + warehouseCode + available = true` 查一条。普通创建未见同等重复校验。 | 当前“物料 + 仓库”仅是导入/查询使用的候选业务键，不能当作已由数据库或创建服务强制保证的唯一规则。 |

### 新增约束、选择器与编辑限制

| 字段/动作 | 当前证据 | 当前结论 |
| --- | --- | --- |
| 物料代码 `itemCode` | Web 选择器使用物料基础信息与 `getItembasicPage`，筛选 `available = true`；服务以物料代码查询物料服务。 | 日常新增应选择可用[物料基本信息](01-物料基本信息.md)。服务查询到空结果时是否抛错依赖下游服务实现，待测试环境确认。 |
| 仓库代码 `warehouseCode` | Web 选择器使用仓库基础信息与 `getWarehousePage`，筛选 `available = true`；服务调用 `warehouseExist`，未找到即报错。 | 日常新增必须选择存在的可用仓库。页面字段名称是仓库，不应写成库区。 |
| 管理精度 `manageMode` | 前端为管理精度字典必填；服务仅检查非空。 | 需要填写字典值；服务是否校验字典合法性待核验。 |
| 入库包装规格 | 后端字段是 `packUnit`；服务会调用包装规格服务确认其存在且可用。前端字段却配置为 `packageUnit`，选择器从[物料包装信息](03-物料包装信息.md)选择其 `packageUnit`。 | 已确认前后端字段命名不一致：正常 Web 新增/编辑可能无法把选择值绑定到后端必填的 `packUnit`。该问题必须作为前端缺陷处理，不能在培训中把当前 Web 流程描述为可用。 |
| 有效期 | Web 在两个时间均有值时校验失效晚于生效；DDL 允许为空。 | 两个时间可空；同时填写时按前端规则执行，服务同等校验待核验。 |
| 编辑 | Web 编辑禁用物料和仓库选择器；服务更新整体写回请求对象，没有同等锁定或重复校验。 | 培训按“物料、仓库不可编辑”执行；接口仍可能绕过页面改变关键关联，已登记差距。 |
| 删除 | 服务只校验记录存在，然后删除并登记 SCP 接口；未见库存、收货、上架或默认库位策略的引用检查。 | 当前不能确认删除是否会影响下游业务，应由测试环境和引用专项补证。 |

### 导入、列表、详情与接口挂接

| 主题 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 导入模板列 | `ItemwarehouseImportVO` 读取：物料号、区域代码、管理精度、入库包装规格、是否可用、生效/失效时间、备注；物料号、区域代码、管理精度、入库包装规格为必填，是否可用未标必填。 | 当前模板字段实际称“区域代码”，但 Web/DDL/后端核心字段为仓库代码，存在术语冲突；模板不得加入库区、库位、默认标识和优先级等未实现表单字段。 |
| 导入校验 | 服务检查物料代码、仓库存在、管理精度、入库包装规格存在且可用、可用性；导入时将 `inPackUnit` 同时写入 `packUnit` 与 `outPackUnit`。 | 导入的包装规格校验对象是包装规格主数据，不等同于 Web 选择的物料包装关系；两条路径的业务语义不一致。 |
| 导入模式与错误 | 模式 `1` 不存在插入、存在更新；模式 `2` 存在报错；模式 `3` 不存在时不写入也未见错误记录。此服务在错误后未见整批回滚处理。 | 覆盖模式与“部分更新”行为须测试确认；不能套用其它页面的通用回滚结论。 |

| 默认列与详情 | 当前列顺序为物料、仓库、管理精度、入库包装规格、可用性、有效期、创建/更新信息、备注、操作；物料代码可打开当前记录详情。详情未见分组或关联 Tab。 | 列顺序可作当前 Web 配置事实；详情分组建议为基本关联、包装/精度、有效性、库存策略扩展、接口与审计。 |
| 建议快速跳转 | 建议物料详情（`itemCode`）、仓库详情（`warehouseCode`）、物料包装与包装规格（包装字段）、WMS 预计入/库存余额（物料+仓库过滤）。 | 当前未见这些实际跳转入口；建议不等于已实现能力。 |
| SCP 接口记录 | 新增/更新/删除均登记 `InterfaceInfo`，类型 `dbc-SCP`、名称“推送物料仓库默认配置”、地址分别为 `/create`、`/update`、`/delete`，业务号为物料代码。导入写入未见同等接口登记。 | 已证实普通维护会登记待发接口；导入同步、实际发送、失败重试和第三方反馈待接口专项取证。 |
| RBAC | Web 使用 `wms:itemwarehouse:*`；Controller 的创建、更新、删除、导入、导出 `@PreAuthorize` 注解被注释。 | 当前只能确认前端候选权限，不能确认接口层授权。 |

## 概述

当前页面实际维护物料的仓库默认配置及管理精度、包装规格。它可为后续仓储策略提供数据基础，但目前不能证明已经实现库区/库位推荐或上架策略执行。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| `itemCode` | 物料代码 | `varchar(64)` | 非空 | 物料默认配置 | 应从物料选择器选择。 |
| `warehouseCode` | 仓库代码 | `varchar(64)` | 非空 | 仓库默认配置 | 历史“库区代码”说法不适用于当前主表单。 |
| `manageMode` | 管理精度 | `varchar(64)` | 非空 | 库存管理口径 | 当前仅见非空校验。 |
| `packUnit` | 入库包装规格 | `varchar(64)` | 非空 | 包装配置 | 后端真实字段；前端错误配置为 `packageUnit`。 |
| `available` | 是否可用 | `boolean` | 非空，默认 true | 配置过滤 | 导入未标必填但服务要求非空。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)` | 可空 | 有效性 | 前端时间先后校验。 |
