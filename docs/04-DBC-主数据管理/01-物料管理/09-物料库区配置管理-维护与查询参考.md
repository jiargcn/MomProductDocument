# 物料库区配置管理-维护与查询参考

> 对应业务说明：[物料库区配置管理](05-物料库区配置管理.md)。
> 本页的“库区”沿用菜单名称；实际维护对象是物料—仓库默认配置。

## 维护前检查清单

1. 物料、仓库均已建立且处于可用状态。
2. 已确认该物料确实会在该仓库收发、保管或为线边供应服务。
3. 已确认管理精度和入库包装与仓储现场口径一致。
4. 如为变更而非新增，已查询库存、在途收发和未完成作业，明确切换时点。

## 新增与编辑

| 维护项 | 新增要求 | 编辑限制与建议 |
| --- | --- | --- |
| 物料 | 从可用物料中选择。 | 页面通常不应改为另一物料；如业务关系变了，建议停用旧配置后新建。 |
| 仓库 | 从可用仓库中选择。 | 页面通常不应改为另一仓库；避免把历史配置改作新仓库用途。 |
| 管理精度 | 必须选择，且应符合该仓库的管理方式。 | 变更前应由仓储负责人确认对库存和作业的影响。 |
| 入库包装 | 必须填写；应与物料包装、包装规格和收货现场一致。 | 变更前应检查在途收货、标签和库存处理口径。 |
| 是否可用与有效期 | 启用状态不可为空；有效期可按业务切换计划填写。 | 停用优先于删除；失效时间应晚于生效时间。 |
| 备注 | 用于写清适用范围、切换原因或现场说明。 | 不用备注替代应维护的主数据和规则。 |

当前页面会锁定物料和仓库这两个关系对象，培训按“关联关系不直接改写”的方式操作。接口层是否能绕过该限制、删除前是否会检查下游引用，尚待测试验证。

## 批量导入规则

| 模板应包含 | 不应由模板承担 | 导入前校验 |
| --- | --- | --- |
| 物料、仓库、管理精度、入库包装、是否可用、生效时间、失效时间、备注。 | 库区、库位、默认库位、上架优先级等当前页面未确认已启用的能力；创建/修改人等系统审计信息。 | 物料存在、仓库存在、管理精度和包装口径有效、可用状态不为空、有效期合理。 |

- 模板中的“区域代码”实际对应仓库。发布模板或培训材料时，应在表头说明中改写为“仓库”，避免使用者误填库区。
- 导入可按不同模式新增、更新或只更新已有数据；“只更新”遇到不存在记录时的反馈表现待测试，不应把无错误文件当作必然成功。
- 批量导入会使用与页面维护不同的包装校验路径。首次导入或包装变更前，必须在测试环境用小批量数据验证保存结果。

## 列表、详情与联查建议

| 位置 | 推荐显示或筛选 | 用途 |
| --- | --- | --- |
| 列表首屏 | 物料、仓库、管理精度、入库包装、可用状态、有效期。 | 快速识别当前适用关系。 |
| 常用筛选 | 物料、仓库、管理精度、可用状态、有效期。 | 查找某物料或某仓库的配置范围。 |
| 详情分组 | 基本关系；管理与包装；有效期；变更与审计。 | 降低所有字段平铺带来的阅读负担。 |
| 建议联查 | 物料基本信息、仓库、物料包装信息、包装规格、实际库存、收货记录。 | 解释包装口径、实际使用和变更影响。 |

当前详情未确认已有分组或快捷跳转；以上为后续页面改造的目标结构，不应理解为已上线功能。

## 常见问题

| 现象 | 优先检查 | 处理建议 |
| --- | --- | --- |
| 物料或仓库无法选择 | 对象是否存在、是否可用，以及当前用户的维护范围。 | 先补齐或启用基础资料，再重新维护。 |
| 页面保存失败且与包装有关 | 当前物料包装关系、包装规格和填写口径。 | 不要只改页面显示值；先确认两个包装主数据是否一致。 |
| 导入后收货口径不符 | 模板的仓库、管理精度、入库包装和导入结果。 | 先停止继续导入，在测试环境复现后再批量修正。 |
| 需要指定库区或库位 | 是否存在相应仓储策略或作业规则。 | 不要将本页当作库位推荐配置；转由仓储配置专项确认。 |
