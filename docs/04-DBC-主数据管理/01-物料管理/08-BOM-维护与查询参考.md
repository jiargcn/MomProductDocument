# BOM-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[BOM](02-BOM.md)使用；本页用于维护物料结构、查询版本与反查使用关系。

## 快速定位

| 你要做什么 | 建议先看 |
| --- | --- |
| 为产品建立一套结构 | “新增 BOM 明细”。 |
| 调整用量、版本或有效期 | “修改、停用与版本切换”。 |
| 批量初始化结构 | “导入或批量处理”。 |
| 排查备料或发料数量 | “列表、详情与反查”。 |

## 新增 BOM 明细

一条 BOM 明细描述“某父项在某版本下需要一个子项及其用量”。建立前应先确认父项、子项均为可用物料，且工程、计划和仓库对单位与版本口径达成一致。

| 需要准备或选择什么 | 为什么 | 当前限制或建议 |
| --- | --- | --- |
| BOM 编号与版本 | 将属于同一结构版本的明细组织在一起。 | 两项页面必填；版本最长 50 个字符。同一结构的子项应保持相同编号和版本口径。 |
| 父项物料 | 明确被生产或装配的对象。 | 从成品、半成品范围内选择；不要用原材料或临时编码作为父项。 |
| 子项物料 | 明确所需组成物料。 | 从可用物料中选择，并避免把父项自身作为子项。循环结构校验需测试确认。 |
| 数量与单位 | 决定计划、备料和发料需求。 | 数量页面必填，输入不得为负数；单位由子项物料带入，不应手工改写。 |
| 生效/失效时间、发料方式、关键物料标识 | 说明结构适用期及现场发料关注点。 | 生效时间页面必填；字段对计划、工单的实际强制作用待测试验证。 |
| 是否可用、备注 | 控制结构使用范围并保留变更原因。 | 可用状态页面必填；重大变更应在备注或变更记录中说明原因。 |

## 修改、停用与版本切换

| 操作 | 何时使用 | 先检查什么 | 结果或风险 |
| --- | --- | --- | --- |
| 修改用量或子项 | 工程确认结构或消耗发生调整。 | 未完成计划、工单、备料、在制品和已发料差异。 | 可能改变后续需求，不应直接改写正在执行的业务口径。 |
| 新增版本 | 新旧结构需要在明确时间点切换。 | 新版本完整性、生效时间、旧版本的停用安排。 | 建议保留旧版本用于历史追溯，不以覆盖原明细代替版本管理。 |
| 停用版本或明细 | 该结构不再用于新生产。 | 是否仍有在途计划、工单或返工业务。 | 停用后的选择器和下游拦截效果待测试。 |

## 导入或批量处理

导入适合一次性建立稳定的产品结构或进行统一版本切换。导入前应由工程、计划和仓库共同确认父项、子项、单位、数量和生效时间；不要将临时替代料或现场缺料处理直接导入 BOM。

| 模板内容 | 用户填写/系统填充 | 当前注意事项 |
| --- | --- | --- |
| BOM 编号、版本、父项物料、子项物料 | 用户填写 | 是结构识别与归属依据；导入前先统一编码口径。 |
| 子项数量、有效期、发料方式、关键物料、可用状态 | 用户填写 | 数量、状态和有效期直接影响业务理解；实际导入必填和模式需用测试环境确认。 |
| 子项单位、物料名称、审计信息 | 系统带入或展示 | 不应作为人工关系键；如模板出现，以系统当前导出模板为准。 |

## 列表、详情与反查

| 查询目标 | 建议默认看什么 | 常用筛选 | 详情/联查 |
| --- | --- | --- | --- |
| 某父项当前用哪些子项 | BOM 编号、版本、父项、子项、数量、单位、可用状态和有效期。 | 父项、版本、可用状态。 | 联查父项/子项物料、工艺路线和计划。 |
| 某子项被哪些产品使用 | 子项、父项、版本、数量和有效期。 | 子项、可用状态、时间范围。 | 联查备料、发料、工单或计划。 |
| 为什么需求数量异常 | 父项、子项、数量、单位、版本和有效期。 | 父项 + 版本。 | 对比计划需求与实际发料记录。 |

当前详情缺少 BOM 专属分组和快速跳转。建议按“结构身份、父项信息、子项明细、版本与有效期、发料与关键物料、工艺/计划引用、审计信息”分组，并提供父项、子项、工艺路线、计划和发料的入口。

## 操作前快速核对

1. 父项、子项和单位是否均为当前可用且符合工程口径的物料？
2. 同一版本下的 BOM 编号、版本和明细是否完整一致？
3. 数量、损耗、发料方式和有效期是否已由工程、计划、仓库共同确认？
4. 变更是否会影响已下达计划、在制品、备料或历史追溯？
