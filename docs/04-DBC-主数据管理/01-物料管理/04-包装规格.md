## 当前页面事实卡（第二轮源码、DDL 已证实）

> 适用基线：测试环境对应的 `dev` 分支，2026-07-15。本节优先于后文历史草稿。`basic_packageunit` 是可被物料包装信息引用的包装规格主数据，并通过 `parent_code` 组织包装层级；它不是库存实物包装或条码实例。

### 数据对象、字段与业务键

| 对象/字段 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 数据表与业务键 | 表为 `basic_packageunit`，主键 `id`；服务以 `code` 检查重复。DDL 未见 `code` 的唯一约束。 | 包装代码 `code` 是当前服务层业务键，数据库并发/旁路保护待补。 |
| 标识字段 | `code`、`name`、`descr` 为 `varchar(64) NOT NULL`；`parent_code` 为可空 `varchar(64)`；`type` 为可空 `varchar(255)`。 | 真实技术名为 `code`、`name`、`descr`、`parentCode`、`type`；历史表中的 `packCode`、`packName`、`packDesc` 不可作为接口字段。 |
| 尺寸与体积 | 长、宽、高、重量为 `numeric(18,6)`；`standard_volume` 为 `numeric(18,2) NOT NULL DEFAULT 0`。 | 虽然前端/DO 对尺寸重量采用整数输入或 `Integer`，DDL 精度更高，数据精度与 UI/DO 的不一致需专项确认。 |
| 器具与管理模式 | `is_container`、`pack_unit_management_mode` 均为可空 `varchar(64)`；前端器具开关使用字符串 `TRUE` / `FALSE`。 | **“是否器具”不是 Boolean 字段**；管理模式只在器具场景显示/填写。 |
| 有效性与审计 | `available` 为 `boolean NOT NULL DEFAULT true`；有效期可空；另有备注、审计、扩展、地点、租户和逻辑删除字段。 | 系统/审计字段不应进入人工导入模板。 |

### 新增约束、层级与编辑限制

| 字段/动作 | 当前证据 | 当前结论 |
| --- | --- | --- |
| 包装代码、名称、类型、描述 | Web 均为必填；描述最大 50 字符。基础 VO 标为非空；DDL 对 `code`、`name`、`descr` 非空，`type` 在 DDL 可空但服务要求非空。 | 新增时按“代码、名称、类型、描述必填，描述不超过 50 字”执行；服务和 DDL 对类型的约束存在层间差异。 |
| 包装代码唯一 | 创建/更新服务以 `code` 查询重复并排除自身。 | 当前服务层禁止重复代码；DDL 没有唯一约束，已登记差距。 |
| 父包装 `parentCode` | 前端选择器数据源为包装规格 `getPackageunitPageParent`，固定筛选 `available = true`；选择父包装时会将父项 `type` 带入当前表单。更新服务仅拒绝 `code = parentCode`。 | 日常新增应从可用的父包装选择；当前只证实直接自引用检查，未证实父包装真实存在、创建时自引用、跨层循环和父子类型一致性均受服务强制校验。 |
| 器具与管理模式 | Web 中 `isContainer = TRUE` 时显示管理模式选项，并默认选取字典首项；为 `FALSE` 时清空管理模式。导入中仅在器具为 `TRUE` 时强制管理模式非空且必须为“批量化管理”或“精细化管理”对应字典值。 | 现场维护按前端联动填写；普通创建/更新服务没有同等的器具/管理模式校验，不能将导入规则直接外推为接口硬约束。 |
| 尺寸、重量、体积 | Web 以数值输入，标准体积最小 0、精度 2，其他尺寸字段默认 0；未见长度、重量等必须大于零的服务校验。 | 仅标准体积有前端非负约束；其它数值边界、单位与实际业务含义待测试环境确认。 |
| 有效期与可用性 | Web 在两个时间均存在时校验失效晚于生效；新增 API 强制 `available = true`，新增表单禁用可用性，编辑时允许调整。 | Web 新建默认可用，启用/禁用也有独立动作；服务端有效期校验未见。 |
| 编辑 | Web 编辑锁定代码、名称、类型；更新服务却将请求对象整体转换后更新，并允许同 ID 代码通过唯一校验。 | 培训按“代码、名称、类型不可编辑”执行；直接接口仍可能绕过页面改变关键标识，已登记差距。 |
| 删除 | 服务删除前检查是否存在 `basic_itempackage.package_unit = code` 的物料包装引用；存在则拒绝，否者记录删除时间并删除。 | 已证实“被物料包装引用时不可删除”。对子包装、库存、标签、单据等其它引用是否受限，待继续核验。 |

### 导入、列表、详情与接口挂接

| 主题 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 实际导入模板 | Controller 实际读取 `PackageunitimportNewExcelVO`，列为包装代码、父包装代码、包装规格名称、包装类型、长宽高重量、标准体积、单位、是否器具、是否可用、包装描述、管理模式、生效/失效时间、备注。代码、名称、类型、单位、是否器具、是否可用、描述、管理模式标为必填。 | 此为当前人工导入模板依据；不要使用名称相近但字段明显为物料主数据的 `PackageunitImportExcelVo`。 |
| 导入校验与模式 | 服务校验代码、父子不相同、名称、类型、描述、器具、可用性；器具为 `TRUE` 时校验管理模式。模式 `1` 更新（不存在插入、存在更新）、`2` 追加（存在报错）、`3` 覆盖（不存在既不写入也未登记错误）。 | 覆盖模式对不存在记录的处理与通用导入约定不同，应以当前实际服务为准并登记测试项。任一错误且不允许部分更新时会回滚。 |
| 模板/VO 差异 | `PackageunitExcelVO` 与实际导入 VO 的名称列分别为“包装名称”/“包装规格名称”；旧 `PackageunitImportExcelVo` 是物料字段集合，Controller 未使用。 | 导出、模板和代码类名存在历史混乱，模板标题和字段字典必须以 Controller 实际读取类为准。 |
| 默认列表与查询 | Web 初始筛选 `available = true`，树形分页接口为 `/dbc/packageunit/pageTree`；查询项包含代码、名称、类型、可用性、创建时间。后端普通分页默认按创建时间倒序。 | 树形结构来自 `parentCode`；默认查询会隐藏禁用项。 |
| 默认列顺序 | 当前前端顺序为代码、父包装代码（表格隐藏）、名称、描述、类型、单位、是否器具、管理模式、长宽高重量、标准体积、可用性、生效/失效、创建/更新信息、备注、操作。 | 列顺序可作当前 Web 配置事实；最终显示列仍允许用户设置。 |
| 当前详情与建议跳转 | 代码可打开当前记录的通用详情组件；前端未见关联 Tab 或跨业务跳转。建议后续补：父/子包装树、引用该规格的[物料包装信息](03-物料包装信息.md)、标签/器具与 WMS 使用记录。 | 建议结构不是当前 UI 事实，跳转路由和过滤参数待具体页面取证。 |
| WMS-SCP 接口记录 | 新增、更新和导入成功均会登记 `InterfaceInfo`，类型 `WMS-SCP`，名称“推送包装规格信息”，业务号为包装代码。新增地址为 `/dbc/packageunit/create`，更新为 `/dbc/packageunit/update`；导入更新仍写入 `/create`。 | 已证实系统登记接口数据；第三方传输、重试、导入更新地址语义仍待接口专项核验。 |
| 动作权限 | Web 使用 `dbc:packageunit:*`；后端启用/禁用使用 `wms:packageunit:*`，创建/更新/删除/导入/导出权限注解被注释。 | 前端按钮权限不能证明接口鉴权，纳入 RBAC 实测矩阵。 |

### 当前待确认差异

| 编号 | 差异 | 影响 | 后续动作 |
| --- | --- | --- | --- |
| PU-FIELD-001 | DDL 的尺寸/重量为 `numeric(18,6)`，DO/VO/前端却按 `Integer` 整数输入；`isContainer` 又是字符串标记。 | 尺寸精度、接口类型与导入数据可能产生截断或字段口径错误。 | 以测试环境接口响应和实际数据复核后，确定业务字典精度。 |
| PU-RULE-001 | 代码唯一性只在服务层；Web 锁定代码/名称/类型，但更新 API 仍可提交这些字段；创建未见父子同码校验。 | 并发、旁路调用和层级循环可能破坏主数据规则。 | 已登记产品差距，待技术评估和测试验证。 |
| PU-IMPORT-001 | 旧导入类与实际 Controller 读取类不一致；覆盖模式下不存在代码未见明确错误；导入更新接口记录仍指向 `/create`。 | 模板维护、导入结果和第三方接口状态可能被误解。 | 纳入导入与接口专项回归。 |
| PU-RBAC-001 | 前后端权限前缀不一致，且主要接口权限注解被注释。 | 无法仅凭页面确认角色权限和接口保护。 | 纳入逐页 RBAC 实测矩阵。 |

## 概述

包装规格定义可供物料包装关系引用的包装代码、层级、属性和有效性，可用于包装换算、标签/器具管理及 WMS 收发货场景。具体下游使用点须逐页取证。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| `code` | 包装代码 | `varchar(64)` | 非空，服务层查重 | 物料包装关系 | 真实业务键。 |
| `parentCode` | 父包装代码 | `varchar(64)` | 可空 | 包装树 | 当前只证实禁止更新时自引用。 |
| `name` / `descr` | 包装名称/描述 | `varchar(64)` | 非空 | 页面显示与选择 | 描述 Web 最大 50 字符。 |
| `type` | 包装类型 | `varchar(255)` | DDL 可空、服务要求非空 | 收发货过滤 | 字典值待测试确认。 |
| `length` / `width` / `height` / `weight` | 尺寸/重量 | `numeric(18,6)` | 可空 | 待逐页核验 | DO/前端按整数处理，存在精度差异。 |
| `standardVolume` | 标准体积 | `numeric(18,2)` | 非空，默认 0 | 待逐页核验 | 前端最小 0、精度 2。 |
| `isContainer` | 是否器具 | `varchar(64)` | 可空 | 器具管理模式 | Web 使用 `TRUE` / `FALSE`。 |
| `available` | 是否可用 | `boolean` | 非空，默认 true | 选择器过滤 | 新建 Web 默认可用。 |
