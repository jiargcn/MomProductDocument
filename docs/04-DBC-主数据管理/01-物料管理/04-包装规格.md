# 包装规格

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：主数据维护人员、物流/仓储人员、标签与包装管理人员。

## 业务目的与适用范围

包装规格用于定义可复用的包装标准，例如包装类型、外形尺寸、承载能力或其它现场约束。物料包装信息引用包装规格后，仓库和物流人员才能以统一口径理解“使用什么容器、可装多少、如何搬运或存放”。

包装规格是标准资料；实际库存中的某一个箱、托或容器仍应由收发和库存业务记录追溯。

## 何时需要维护

新包装投入使用、包装尺寸/承载条件变化、库位与搬运设备需要适配，或物料包装信息无法满足现场操作时，应新增或调整包装规格。

## 包装规格如何服务业务

```mermaid
flowchart LR
    A["定义包装规格"] --> B["物料包装信息引用"]
    B --> C["标签与现场包装使用"]
    C --> D["收货、上架、搬运、发运"]
    D --> E["库存与物流追溯"]
```

规格应表达现场真正需要判断的信息，而不是为了完整而维护无用途的尺寸或属性。涉及库位容量、AGV/叉车搬运或客户交付时，应与相应业务责任人共同确认口径。

【示例数据占位：标准周转箱和标准托盘两种规格，展示它们被哪些物料包装信息引用，以及对上架/搬运的影响。】

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 规格身份 | 名称/编码是否能让现场区分包装标准。 | 使用稳定、可沟通的命名规则。 |
| 尺寸与承载条件 | 是否满足库位、搬运和运输要求。 | 变更前评估现有库存、设备和作业方式。 |
| 适用范围 | 哪些物料、包装信息或业务场景引用该规格。 | 不将客户临时要求直接改成通用标准。 |
| 启停与替换 | 旧规格是否仍被物料包装或库存使用。 | 优先通过停用/替代完成切换。 |

## 查询、详情与联查

| 查询目标 | 建议入口 | 应继续联查 |
| --- | --- | --- |
| 某规格用于哪些物料 | 包装规格及引用关系。 | 物料包装信息、物料基本信息。 |
| 为什么现场不能使用某包装 | 规格状态、尺寸/承载信息和现场限制。 | 库位、搬运设备、任务配置。 |
| 变更会影响什么 | 当前引用、库存包装和未完成收发。 | 物料包装、标签、库存查询。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 规格名称相近难以区分 | 统一编码/命名并补充可识别的业务描述。 |
| 新规格无法被物料包装选择 | 核对状态、适用范围和维护权限。 |
| 修改后影响现场搬运 | 先在测试或现场确认库位、设备与标签适配，再发布。 |

## 当前限制与待确认事项

- 实际字段、尺寸/承载校验、选择器过滤与导入规则待继续核验；
- 与库位容量、AGV/叉车、标签模板和客户包装要求的自动校验边界待确认；
- 列表默认字段、详情分组和引用跳转需补测试环境截图。
