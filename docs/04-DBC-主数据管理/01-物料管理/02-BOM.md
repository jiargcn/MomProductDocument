# BOM

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：工艺/生产计划人员、主数据维护人员、仓库备料人员及需要查询用料关系的业务人员。

## 业务目的与适用范围

BOM 用于说明生产一个父项物料需要哪些子项物料、各需要多少，以及这些关系在什么范围内适用。它把“要生产什么”与“需要准备什么料”连接起来，是生产计划、备料、发料和成本分析理解物料结构的重要依据。

BOM 不等同于库存余额或实际领料记录：它表达的是应使用的结构和数量，实际用料仍应以生产、发料和收料等业务记录为准。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 父项物料和子项物料 | 确认要生产的对象及其组成。 |
| 用量、单位和适用范围 | 决定计划和备料时如何换算需求。 |
| 生效/失效或版本口径 | 避免不同版本的结构同时被误用。 |
| 工艺路线和生产地点 | 确认物料结构与实际生产方式是否匹配。 |

## 一套 BOM 如何被业务使用

```mermaid
flowchart LR
    A["维护父项与子项关系"] --> B["确认用量与适用范围"]
    B --> C["生产计划/工单引用"]
    C --> D["生成备料或发料需求"]
    D --> E["仓库按实际业务发料"]
    E --> F["生产与成本追溯"]
```

维护完成后，BOM 应帮助计划和仓库理解需要准备哪些物料；当工程变更、替代料或版本变化发生时，应先评估未完成计划、在制品和现有备料，再决定是否直接变更或启用新版本。

## 关键字段业务角色

下表只列影响结构识别、选料范围与用量的关键项；完整语义与选择器范围见[维护与查询参考](08-BOM-维护与查询参考.md)。写法约定见[页面数据字典规范](../../02-业务模型/04-页面数据字典规范.md)。

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- | --- |
| BOM 编号 + 版本 | P6 | 将同版结构明细组织在一起 | 两项页面必填；同版明细应共用编号与版本口径 | 勿用覆盖原明细代替版本管理 |
| 父项物料 | P2 / P6 | 被生产/装配的对象 | 从**成品/半成品**范围选择；Web 编辑锁定 | 勿把原材料当父项；旁路改关联见 `GAP-023` |
| 子项物料 | P2 / P6 | 组成物料 | 从可用物料范围选择；勿选父项自身；编辑锁定 | 循环结构拦截 ❓；重复组合见 `GAP-023` |
| 子项单位 | P5 / P7 | 用量计量单位 | 选子项后**带回**，不应手工改写 | 单位错导致备料数量不可比 |
| 数量 | P7 | 单位产品/批次消耗 | 必填、不得为负 | 变更影响计划与备料需求 |
| 生效时间 / 是否可用 | P10 / P1 | 结构适用窗与可用性 | 生效时间页面必填；对计划强制作用 ❓ | 多版本并存时勿凭名称随意选用 |
| 发料方式 / 关键物料 | P1 | 现场发料关注点标识 | 辅助说明；对工单强制 ❓ | 勿当成已证实的自动发料策略 |

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 父项/子项关系 | 是否确为生产或装配所需的组成关系。 | 不要把临时替代、库存地点或客户别名写进 BOM。见上表 P2/P6。 |
| 用量与单位 | 是否能正确反映一个生产批次或单位产品的消耗。 | 变更前核对单位换算与损耗口径。见上表 P7。 |
| 版本与有效期 | 新旧结构能否在正确时间切换。 | 已有未完成工单时先确认采用哪一版本。 |
| 启停或删除 | 是否仍有计划、工单、备料或历史追溯需要引用。 | 优先使用受控变更和停用，不建议直接删除。 |

## 查询、详情与联查

| 想解决的问题 | 推荐查看 | 建议联查 |
| --- | --- | --- |
| 一个成品需要什么料 | 父项、BOM 版本和子项清单。 | 物料基本信息、工艺路线。 |
| 某物料用于哪些产品 | 子项反查及适用状态。 | 生产计划、工单或备料需求。 |
| 为什么备料数量异常 | BOM 用量、单位、版本和实际需求。 | 发料任务、生产记录。 |
| 工程变更影响什么 | 生效时间、未完成工单和库存准备。 | 工艺路线、生产计划。 |

详情页建议按“基本身份、组成明细、版本/适用范围、变更与状态、相关工艺与生产引用、系统信息”分组。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 子项物料无法选择 | 核对物料是否已建立、可用、单位和生产用途是否正确。 |
| 同一产品出现多个结构 | 先确认版本、有效期和生产场景，不要凭名称随意选用。 |
| 修改后影响正在生产 | 暂停直接修改，评估工单、备料和在制品后走变更流程。 |
| 实际领料与 BOM 不一致 | 分别核对工程结构、损耗、替代料和现场异常记录。 |
