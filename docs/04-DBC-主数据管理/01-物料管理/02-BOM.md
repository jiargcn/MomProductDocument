# BOM（物料清单）

## 概述

BOM（Bill of Materials）描述产品结构中父项物料与子项物料的层级用量关系，是 MES 工单展开、MES 齐套检查、WMS 发料的核心依据。

## 字段说明

### BOM表头

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| bomCode | BOM编码 | VARCHAR(50) | 必填、唯一 | MES工单展开 | BOM的唯一标识，工单通过BOM编码引用具体版本的物料清单，系统主键 |
| bomName | BOM名称 | VARCHAR(200) | 必填 | MES工单展开 | BOM的显示名称，通常与产品/半成品名称保持一致，便于识别 |
| parentMaterial | 父项物料 | VARCHAR(50) | 必填，来源表：物料 | MES工单展开（按父项展开子项）、工序跳阶（跨工序发料） | 定义BOM所属的产品/半成品物料，所有子项物料的用量都相对于此父项展开 |
| version | 版本 | VARCHAR(20) | 必填 | MES工单展开（版本决定引用哪份BOM数据） | BOM的版本号，用于区分同一父项物料的不同版本，支持变更历史追溯 |
| effectiveDate | 生效日期 | DATE | 必填 | MES工单展开（生效日期前工单不引用此BOM） | BOM启用的时间节点，工单展开时根据生效日期匹配对应版本，生效前为草稿状态 |
| invalidDate | 失效日期 | DATE | 非必填 | MES工单展开（失效后工单不再引用） | BOM失效的时间节点，填写后此BOM在失效日期之后不可被新工单引用，空值表示永久有效 |
| status | 状态 | ENUM | 字典项：草稿/生效/禁用 | MES工单展开（草稿/禁用状态不可被工单引用） | BOM的审核状态，草稿BOM仅保存不参与运算，禁用BOM不可被新建工单引用 |
| bomLevel | BOM层级 | INT | 必填，默认0 | MES齐套展开（决定BOM展开的深度） | BOM在多层结构中的层级，0表示顶层BOM（直接对应工单完工对象），子BOM层级>0 |
| standardWorkTime | 标准工时 | DECIMAL(18,4) | 非必填 | MES报工（工序级报工参考工时）、成本核算（标准人工成本计算） | BOM父项物料的标准总工时，单位小时，用于产能计算和标准成本估算 |
| remark | 备注 | VARCHAR(500) | 非必填 | | BOM的补充说明，如变更原因、适用范围等 |

### BOM子项（BOMLine）

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| sequence | 序号 | INT | 必填 | WMS发料顺序（按序号从小到大发料） | 子项物料在BOM中的排列顺序，影响发料优先级，序号小的先发料 |
| childMaterial | 子项物料 | VARCHAR(50) | 必填，来源表：物料 | WMS发料（发料对象）、MES工序用料（工序级物料需求） | BOM子项的物料编码，定义父项物料生产时需要的具体原材料/辅料 |
| quantity | 用量 | DECIMAL(18,4) | 必填 | WMS发料数量计算（用量×工单数量=发料量）、MES齐套（判断是否满足工单需求） | 生产1个父项物料所需的子项物料数量，是发料数量核算和齐套判断的核心数据 |
| scrapRate | 报废率 | DECIMAL(5,2) | 非必填，默认0 | WMS发料数量（实际发料=用量×工单数量×(1+报废率)） | 子项物料在生产过程中的预计报废比例，影响发料数量的增加，发料员按实际报废投料 |
| stationNo | 工位 | VARCHAR(50) | 非必填 | MES工序发料（发料到指定工位） | 子项物料发放到的具体工位编号，不填则由系统按工序自动分配 |
| isKeyItem | 关键件 | BOOLEAN | 非必填 | MES齐套检查（关键件未齐套不允许开工） | 标识子项是否为关键件，关键件库存未齐套时工单不允许开工，确保关键物料优先到位 |
| isOptional | 选配件 | BOOLEAN | 非必填 | MES工单选配（选配件由工单选择是否参与） | 标识子项是否为选配件，工单下达时可选择是否包含此子项，适用于产品配置多样性场景 |
| backFlush | 倒冲料 | BOOLEAN | 非必填 | MES报工倒冲（报工完成后自动触发WMS出库） | 标识子项是否执行倒冲作业，工单报工后系统自动按用量触发WMS出库，无需人工发料 |
| substituteMaterial | 替代物料 | VARCHAR(50) | 非必填，来源表：物料 | MES齐套替代（主料不足时自动替换替代料） | 当子项物料库存不足时，系统自动切换到替代物料进行齐套，替代料同样消耗BOM用量 |

## 字段级业务影响

### backFlush（倒冲料）

| 场景 | 对WMS的影响 | 对MES的影响 | 典型应用 |
|------|------------|-------------|----------|
| 开启倒冲 | 报工后自动生成出库单，不经过人工发料环节 | 工序报工后物料自动扣减库存，工单用料实时同步 | 连续型生产线、自动化产线 |
| 关闭倒冲 | 需人工在WMS发起发料单，经确认后出库 | 工单需等待发料齐套后才可开工 | 离散型生产、工序间有在制品的场景 |

### isKeyItem（关键件）

| 场景 | 对MES的影响 |
|------|-------------|
| 关键件未齐套 | 工单状态锁定不允许开工，系统提示关键件缺料 |
| 关键件已齐套 | 工单解除锁定，允许进入生产环节 |

### substituteMaterial（替代物料）

| 场景 | 对齐套的影响 |
|------|-------------|
| 主料有库存 | 优先使用主料，不触发替代 |
| 主料库存不足 | 自动匹配替代物料进行齐套，替代料的齐套量=主料不足量 |
| 替代料也无库存 | 工单继续待料，关键件未齐套不允许开工 |

## 关联关系

```
父项物料（成品/半成品）← BOM → 子项物料（原材料/辅料）
BOM → BOMLine（多层嵌套，可支持6层以上）
物料 → BOM（产品配置用料）
替代物料 → 物料（BOM Line级替代关系）
```

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 字典项 | status（草稿/生效/禁用） |
| 来源表 | parentMaterial→物料表，childMaterial→物料表，substituteMaterial→物料表 |
| 联动影响 | 替代物料配置→齐套不足时自动替换；报废率→影响发料数量；倒冲→报工自动触发WMS出库 |
| 版本管理 | BOM变更应记录变更原因和生效日期，工单引用时按生效日期使用对应版本；同一父项物料同时只能有一个生效版本 |
