## 当前页面事实卡（第二轮源码、DDL 已证实）

> 适用基线：测试环境对应的 `dev` 分支，2026-07-15。本节优先于后文历史草稿。当前实现中的一条 `basic_bom` 记录表示“父项物料 + 子项物料 + 版本”的一条组成关系，**不是独立 BOM 表头与 BOM 明细两张表**。

### 对象、字段与业务键

| 对象/字段 | 当前实现 | 文档结论 |
| --- | --- | --- |
| 数据表 | `basic_bom`，主键为 `id`。 | BOM 采用单表的父子项关系记录模型。 |
| 父项、子项、单位、用量 | `product_item_code`、`component_item_code`、`component_uom`、`component_qty`；DDL 均为非空，数量为 `numeric(18,6)`。 | 真实技术名以此为准，不使用历史草稿的 `parentMaterial`、`childMaterial`、`quantity` 等伪字段名。 |
| 版本与 BOM 编号 | `version` 为 `varchar(64)` 可空，`bom_code` 为 `varchar(255)` 可空。 | 页面和导入均要求版本；BOM 编号在页面也要求，但接口基础 VO 未作非空注解，存在层间差异。 |
| 状态与时间 | `available` 非空默认 `true`；`active_time`、`expire_time` 可空。 | 当前没有“草稿/生效/禁用”状态字段，不能沿用历史草稿的状态机说法。 |
| 业务唯一性 | 创建服务按 `product_item_code + component_item_code + version` 查询重复。DDL 仅有主键和普通索引，未见对应复合唯一约束。 | 该组合是当前服务层业务键；数据库层并发/旁路写入保护不足，已登记产品差距。 |

### 新增约束、选择器与编辑限制

| 字段/动作 | 当前证据 | 当前结论 |
| --- | --- | --- |
| 父项物料 `productItemCode` | 前端选择器调用 `selectItembasicPageToFgAndSemibasicPage`；服务要求物料存在且 `available = true`。 | 父项必须从物料选择器选择并且当前可用；具体“成品/半成品”筛选配置由后端选择器接口决定，待测试环境确认。 |
| 子项物料 `componentItemCode` | 前端选择器调用 `selectTypeToItembasic`；服务要求物料存在且 `available = true`。 | 子项必须是当前可用物料；选择器与父项使用不同后端查询接口，不能假定两者筛选规则相同。 |
| 子项单位 `componentUom` | 后端要求非空；前端表单禁用人工编辑，在选择子项时将所选物料的 `uom` 带入。 | 日常新增应由子项物料带出单位，不建议人工修改。 |
| 子项用量 `componentQty` | 后端要求非空且创建时必须大于 0；前端数值输入框最小值为 0、精度 6。 | 后端“> 0”是最终约束，不能只按前端的 `min: 0` 写规则。 |
| 父子不能相同 | 前端提交前比较父/子物料号。 | 已见前端限制；后端是否存在同等校验未见，直接接口调用仍待核验。 |
| 有效期 | 页面要求生效时间；提交前校验失效时间晚于生效时间。服务按 `available`、生效/失效时间筛选有效 BOM。 | 前端与服务均有时间口径，但 DDL 允许为空，历史草稿“生效日期必填”不能直接作为接口结论。 |
| 编辑 | Web 编辑表单禁用父项和子项物料，并关闭其选择器；服务未覆盖 `beforeUpdate` 进行同等锁定。 | 日常维护按“父/子物料不可编辑”培训；接口层是否可绕过页面改动、是否会触发唯一性冲突，已登记差距。 |
| 关键物料 | 创建服务在关键物料为 `true` 时，按父项和 BOM 编号检查是否已有其它关键子项。 | 当前是服务层约束，关键物料与层级/版本的完整产品规则仍待测试核验。 |

## 概述

BOM（Bill of Materials）描述产品结构中父项物料与子项物料的层级用量关系，是 MES 工单展开和 WMS 发料的核心依据。

## 字段说明

### BOM表头

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| bomCode | BOM编码 | VARCHAR(50) | 必填、唯一 | MES工单展开 | BOM的唯一标识，工单通过BOM编码引用具体版本，系统主键 |
| bomName | BOM名称 | VARCHAR(200) | 必填 | MES工单展开 | BOM的显示名称，通常与产品/半成品名称保持一致 |
| parentMaterial | 父项物料 | VARCHAR(50) | 必填，来源表：物料 | MES工单展开（按父项展开子项） | 定义BOM所属的产品/半成品物料，所有子项物料的用量都相对于此父项展开 |
| version | 版本 | VARCHAR(20) | 必填 | MES工单展开（版本决定引用哪份BOM数据） | BOM的版本号，用于区分同一父项物料的不同版本，支持变更历史追溯 |
| effectiveDate | 生效日期 | DATE | 必填 | MES工单展开（生效日期前工单不引用此BOM） | BOM启用的时间节点，工单展开时根据生效日期匹配对应版本 |
| invalidDate | 失效日期 | DATE | 非必填 | MES工单展开（失效后工单不再引用） | BOM失效时间节点，失效后不可被新工单引用，空值表示永久有效 |
| status | 状态 | ENUM | 字典项：草稿/生效/禁用 | MES工单展开（草稿/禁用状态不可被工单引用） | BOM的审核状态，草稿BOM仅保存不参与运算，禁用BOM不可被新建工单引用 |
| bomLevel | BOM层级 | INT | 必填，默认0 | MES工单展开（用于多层BOM的层级展开，记录BOM变更的层级位置） | BOM在多层结构中的层级，0表示顶层BOM（直接对应工单完工对象），子BOM层级>0 |
| standardWorkTime | 标准工时 | DECIMAL(18,4) | 非必填 | 产能计算（汇总BOM各工序标准工时，估算工单完工周期）、标准成本估算 | BOM父项物料的标准总工时，单位小时，用于产能计算和标准成本估算 |
| remark | 备注 | VARCHAR(500) | 非必填 | | BOM的补充说明，如变更原因、适用范围等 |

### BOM子项（BOMLine）

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| sequence | 序号 | INT | 必填 | WMS发料顺序（按序号从小到大发料） | 子项物料在BOM中的排列顺序，影响发料优先级，序号小的先发料 |
| childMaterial | 子项物料 | VARCHAR(50) | 必填，来源表：物料 | WMS发料（发料对象） | BOM子项的物料编码，定义父项物料生产时需要的具体原材料/辅料 |
| quantity | 用量 | DECIMAL(18,4) | 必填 | WMS发料数量计算（用量×工单数量=发料量） | 生产1个父项物料所需的子项物料数量，是发料数量核算的核心数据 |
| scrapRate | 报废率 | DECIMAL(5,2) | 非必填，默认0 | WMS发料数量（实际发料=用量×(1+报废率)） | 子项物料在生产过程中的预计报废比例，影响发料数量的增加 |
| stationNo | 工位 | VARCHAR(50) | 非必填 | WMS发料（指定物料投料的目的工位，影响发料后的物料流向） | 子项物料发放到的具体工位编号，不填则由系统按工序自动分配 |
| isKeyItem | 关键件 | BOOLEAN | 非必填 | MES齐套检查（开启后工单齐套检查时必须确认此物料到位，否则不允许开工） | 标识子项是否为关键件，关键件库存未齐套时工单不允许开工 |
| isOptional | 选配件 | BOOLEAN | 非必填 | MES工单下达（选配物料在工单下达时可选是否包含，不选则不参与齐套和发料） | 标识子项是否为选配件，工单下达时可选择是否包含此子项 |
| backFlush | 倒冲料 | BOOLEAN | 非必填 | MES报工倒冲（报工完成后自动触发WMS出库） | 标识子项是否执行倒冲作业，工单报工后系统自动按用量触发WMS出库，无需人工发料 |
| substituteMaterial | 替代物料 | VARCHAR(50) | 非必填，来源表：物料 | WMS发料（当子项物料库存不足时，系统自动切换到替代物料进行发料） | 当子项物料库存不足时，可切换到替代物料进行发料 |

## 字段级业务影响

### backFlush（倒冲料）

| 场景 | 对WMS的影响 | 对MES的影响 | 典型应用 |
|------|------------|-------------|----------|
| 开启倒冲 | 报工后自动生成出库单，不经过人工发料环节 | 工序报工后物料自动扣减库存 | 连续型生产线、自动化产线 |
| 关闭倒冲 | 需人工在WMS发起发料单，经确认后出库 | 工单需等待人工发料完成后才可报工 | 离散型生产、工序间有在制品的场景 |

## 关联关系

```
父项物料（成品/半成品）← BOM → 子项物料（原材料/辅料）
BOM → BOMLine（多层嵌套）
物料 → BOM（产品配置用料）
替代物料 → 物料（BOM Line级替代关系）
```

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 字典项 | status（草稿/生效/禁用） |
| 来源表 | parentMaterial→物料表，childMaterial→物料表，substituteMaterial→物料表 |
| 联动影响 | 倒冲→报工自动触发WMS出库；报废率→影响发料数量；替代物料→主料不足时自动替换 |
| 版本管理 | BOM变更应记录变更原因和生效日期，工单引用时按生效日期使用对应版本；同一父项物料同时只能有一个生效版本 |
