# BOM

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；工艺/计划、主数据维护、仓库备料等业务人员（顺带）。

## 业务目的与适用范围

BOM 说明生产一个父项需要哪些子项、各用多少，以及这套结构在什么范围适用。读完本页，应能判断：备料/发料数量对不上时先查用量与单位还是查版本；子项选不到时是物料不可用还是不在可选范围；改结构前要先挡住哪些在途计划。

BOM 不是库存余额，也不是实际领料记录：它表达应使用的结构与数量；实际用料以生产、发料和收料等业务记录为准。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 父项物料和子项物料 | 确认要生产的对象及其组成；父项须在成品/半成品范围，子项须为可用物料。 |
| 用量、单位和适用范围 | 决定计划和备料时如何换算需求；单位随子项带回，勿手工改写。 |
| 生效/失效或版本口径 | 避免不同版本的结构同时被误用。 |
| 工艺路线和生产地点 | 确认物料结构与实际生产方式是否匹配。 |

## 一套 BOM 如何被业务使用

```mermaid
flowchart LR
    A["维护父项与子项关系"] --> B["确认用量与适用范围"]
    B --> C["生产计划/工单引用"]
    C --> D["生成备料或发料需求"]
    D --> E["仓库按实际业务发料"]
    E --> F["生产与成本追溯"]
```

维护完成后，BOM 应帮助计划和仓库理解需要准备哪些物料。工程变更、替代料或版本变化时，应先评估未完成计划、在制品和现有备料，再决定直接变更还是启用新版本。

    以「成品 A」由「部件 B、部件 C、紧固件 D」组成的 BOM 为例，展示一次生产需求如何转为备料需求（脱敏示例数据）。

!!! example "写实示例：给定配置 → 期望行为"
    **给定：** 父项成品 FG-100（成品）、子项 RM-螺栓（可用、基本单位 PCS）；BOM 版本 V1，用量 4 PCS/件；生效时间已到、是否可用=是。
    **期望：**

    1. 计划按 FG-100 引用 V1 时，备料需求含 RM-螺栓 × 4 × 计划数量。
    2. 若把子项单位改成与物料主数据不一致的理解（或旁路改单位），备料数量与现场按 PCS 发料会对不上。
    3. 若子项停用或不可用，BOM 新增时通常选不到该子项；已保存明细的下游拦截时点 ❓。
    4. 另建 V2 并停用 V1 前，须先确认未完成工单仍引用哪一版，避免「改了主数据、在途计划仍按旧量」。

## 维护时的关键判断

下表只列影响结构识别、选料范围与用量的关键项；完整语义与选择器范围见[维护与查询参考](08-BOM-维护与查询参考.md)。写法约定见[页面数据字典规范](../../02-业务模型/04-页面数据字典规范.md)。父项/子项「仅可选适用物料范围」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)；成品/半成品父项范围与循环拦截等差异见本页。

### 关键字段业务角色

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- |
| BOM 编号 + 版本 | 将同版结构明细组织在一起 | 两项页面必填；同版明细应共用编号与版本口径 | 勿用覆盖原明细代替版本管理 |
| 父项物料 | 被生产/装配的对象 | 从**成品/半成品**范围选择；Web 编辑锁定 | 勿把原材料当父项；旁路改关联见 `GAP-023` |
| 子项物料 | 组成物料 | 从可用物料范围选择；勿选父项自身；编辑锁定 | 循环结构拦截 ❓；重复组合见 `GAP-023` |
| 子项单位 | 用量计量单位 | 选子项后**带回**，不应手工改写 | 单位错导致备料数量不可比 |
| 数量 | 单位产品/批次消耗 | 必填、不得为负 | 变更影响计划与备料需求 |
| 生效时间 / 是否可用 | 结构适用窗与可用性 | 生效时间页面必填；对计划强制作用 ❓ | 多版本并存时勿凭名称随意选用 |
| 发料方式 / 关键物料 | 现场发料关注点标识 | 辅助说明；对工单强制 ❓ | 勿当成已证实的自动发料策略 |

### 选不到料、单位与下游约束

| 现场现象 | 先回查本页什么 | 再联查什么 |
| --- | --- | --- |
| BOM 里选不到父项 | 物料是否成品/半成品、是否可用 | [物料基本信息](01-物料基本信息.md) 类型与启停 |
| BOM 里选不到子项 | 子项是否可用、是否等于父项自身 | 物料基本信息；重复/旁路改键见 `GAP-023` |
| 备料数量与发料对不上 | 用量、子项单位（须与物料单位一致） | 发料任务、实际发料记录 |
| 计划仍按旧结构算 | 版本、生效时间、是否可用 | 未完成计划/工单引用的版本 |

### 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 父项/子项关系 | 是否确为生产或装配所需的组成关系。 | 不要把临时替代、库存地点或客户别名写进 BOM。 |
| 用量与单位 | 是否能正确反映一个生产批次或单位产品的消耗。 | 变更前核对单位换算与损耗口径；单位以子项物料带回为准。 |
| 版本与有效期 | 新旧结构能否在正确时间切换。 | 已有未完成工单时先确认采用哪一版本。 |
| 启停或删除 | 是否仍有计划、工单、备料或历史追溯需要引用。 | 优先使用受控变更和停用，不建议直接删除。 |

### 建议验证点

- 父项不在成品/半成品范围时，是否无法选为父项。
- 子项停用后，新增明细是否选不到；已引用明细的下游拦截时点。
- 用量变更后，新计划备料数量是否按新用量计算；在途工单是否仍按旧量。
- 子项单位是否只读带回；旁路改单位后备料数量是否与现场不可比（`GAP-023` 旁路风险）。
- 导入必填与保存模型是否一致（`GAP-024`）；小样导入后再批量。

## 查询、详情与联查

| 想解决的问题 | 推荐查看 | 建议联查 |
| --- | --- | --- |
| 一个成品需要什么料 | 父项、BOM 版本和子项清单。 | 物料基本信息、工艺路线。 |
| 某物料用于哪些产品 | 子项反查及适用状态。 | 生产计划、工单或备料需求。 |
| 为什么备料数量异常 | BOM 用量、单位、版本和实际需求。 | 发料任务、生产记录。 |
| 工程变更影响什么 | 生效时间、未完成工单和库存准备。 | 工艺路线、生产计划。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 子项物料无法选择 | 核对物料是否已建立、可用、单位和生产用途是否正确。 |
| 同一产品出现多个结构 | 先确认版本、有效期和生产场景，不要凭名称随意选用。 |
| 修改后影响正在生产 | 暂停直接修改，评估工单、备料和在制品后走变更流程。 |
| 实际领料与 BOM 不一致 | 分别核对工程结构、损耗、替代料和现场异常记录。 |
