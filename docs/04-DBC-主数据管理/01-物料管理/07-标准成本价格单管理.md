# 标准成本价格单管理

> 当前事实基线：`dev` 分支的 DBC 前端、后端与 `reference/dbddl/base.sql`，取证日期为 2026-07-16。本页可用于当前实现培训和测试设计；涉及跨模块成本核算效果与测试环境数据的结论仍需复核。

## 页面定位

标准成本价格单维护物料的价格、币种、可用状态和生效/失效时间。当前实现以单表 `basic_stdcostprice` 保存记录，并提供按物料查询价格的接口；不属于[申请、任务与记录模型](../../02-业务模型/01-申请任务记录模型.md)。

## 当前页面事实卡（第二轮源码、DDL 已证实）

### 实体与关键字段

| 技术字段 | 中文含义 | 类型与长度 | 当前约束 / 来源 |
| --- | --- | --- | --- |
| `id` | 主键 | `bigint` | 数据库主键、自增序列。 |
| `supplier_code` | 供应商代码 | `varchar(64)`，可空 | DDL、DO、VO 均存在；当前 Web 表单和导入模板未配置该字段，服务层也未校验供应商存在性。 |
| `item_code` | 物料代码 | `varchar(64)`，非空 | VO 必填；服务层按该字段查重；前端选择可用物料。 |
| `currency` | 货币 | `varchar(64)`，非空 | VO 必填，前端与导入使用 `CURRENCY` 字典；服务层仅校验非空。 |
| `price` | 价格 | `numeric(18,6)`，非空 | VO 必填；前端最小值为 `0`、精度为 6 位。服务层仅校验非空。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 前端默认筛选为可用；提供启用、禁用操作。 |
| `active_time` / `expire_time` | 生效时间 / 失效时间 | `timestamp(6)`，可空 | 前端以日期时间控件维护；当前服务未见生效区间、先后关系或自动失效校验。 |
| `remark` | 备注 | `text`，可空 | Web 前端规则限制为 50 个字符；数据库未设长度限制。 |
| `concurrency_stamp` | 乐观锁版本 | `integer`，非空，默认 `1` | DO 标记为乐观锁；不是业务录入字段。 |

### 新增约束与选择器

| 项目 | 当前实现 | 培训 / 测试要点 |
| --- | --- | --- |
| 物料选择器 | 数据源为物料基本信息分页接口；选择器筛选 `available = true`，回填物料 `code`。 | 必须选择当前可用物料；失焦校验也按物料代码精确查询。 |
| 币种 | `CURRENCY` 字典下拉，可搜索、可清空。 | 提交前须有值；当前后端只校验非空，未发现字典有效性复核。 |
| 价格 | 数字框，最小 `0`、精度 6 位。 | Web 阻止负数；接口层和导入服务未见“不得为负”的独立校验，旁路调用需回归。 |
| 是否可用 | 新增表单默认 `true`；后端必填。 | 可用状态决定部分按物料取价接口是否返回该记录。 |
| 供应商 | 当前 Web 的供应商选择器代码整体被注释，实际表单未录入。 | 不应将供应商列为当前人工新增的必填字段；VO 的 `required` 描述与实际前端、服务校验不一致。 |

### 唯一性、编辑、删除与状态动作

当前服务及导入都通过 `selectByStdcostpriceCode(itemCode)` 查找既有记录，即以 **`item_code` 单字段**作为业务唯一键；DDL 只有 `id` 主键，未见 `item_code` 唯一索引。创建与更新时均执行该查重，因此同一物料不能通过服务层创建两条标准成本价格记录。

| 操作 | 当前页面行为 | 服务 / 接口事实 |
| --- | --- | --- |
| 编辑 | Web 编辑时锁定 `itemCode` 并关闭选择器；其它表单字段可编辑。 | 更新服务仍接收 `itemCode` 并按它进行查重，因此非 Web 调用理论上可以改码。 |
| 启用 / 禁用 | 可用状态为 `false` 显示“启用”，为 `true` 显示“禁用”。 | 仅按 `id` 更新 `available`，并写趋势记录；接口使用的是 `wms:bom:enable` / `wms:bom:disable` 权限，和前端 `dbc:stdcostprice:*` 不一致。 |
| 删除 | Web 有删除入口。 | 仅校验记录存在后按 `id` 删除，并写趋势记录；未见被业务单据引用时的拦截。 |
| 创建 / 更新 | 前端显示 `dbc:stdcostprice:create/update` 按钮权限。 | 控制器的创建、更新、删除权限注解当前被注释；不能把前端隐藏当作接口访问控制结论。 |

### 导入模板与校验

当前导入模板字段顺序为：**物料代码、币种、价格、是否可用、生效时间、失效时间、备注**。`supplier_code`、主键、审计字段、乐观锁不应出现在模板中。

| 校验或行为 | 当前实现 |
| --- | --- |
| 必填 | 物料代码、币种、价格、是否可用。 |
| 字典下拉 | 币种、是否可用；模板分别在第 2、4 列配置下拉。 |
| 值校验 | 校验物料代码非空、币种非空、价格非空、可用状态非空；未见导入时校验物料主数据存在或可用。 |
| 重复识别 | 只按物料代码查找既有记录，不考虑币种、供应商或生效期间。 |
| 导入模式 | 模式 1：存在则更新、不存在则新增；模式 2：只新增不存在记录；模式 3：只更新已存在记录。当前模式 2 / 3 对“不应新增 / 不应更新”的行未见错误回执。 |
| 错误处理 | 校验失败生成错误明细；`updatePart` 参数传入服务但当前实现未使用，未见全量回滚分支。 |

### 列表、详情与快速跳转

前端配置的业务字段顺序为：物料代码、币种、价格、是否可用、生效时间、失效时间；随后是创建时间、创建者、最后更新时间、最后更新者、备注和操作列。默认查询条件为 `available = true`，后端默认按创建时间倒序。

建议详情按以下分组呈现，作为后续详情组件改造样板：

1. **成本主体**：物料代码、供应商代码、币种、价格、是否可用。
2. **有效期间**：生效时间、失效时间。
3. **审计与说明**：备注、创建/更新人、创建/更新时间、趋势记录。
4. **关联查询**：跳转至[物料基本信息](01-物料基本信息.md)，并提供“按该物料查询标准成本价格”的入口；跨模块成本使用情况待后续逐模块取证后再增加。

### 当前实现的调用边界

服务提供 `queryStdcostpriceByItemCode`：按物料代码查询时，仅在记录存在且 `available = true` 时返回价格；查不到或不可用时，单条查询方法返回价格为 `0` 的临时对象。批量查询接口则返回该物料的所有价格记录，并补充物料单位。该差异需要调用方和测试用例显式处理。

系统在生产线物料关系查询中已读取可用标准成本价格以补充展示数据；其他页面虽存在调用引用，但本轮尚未逐一验证其是否用于实际成本核算，不能据此写成“WMS 出库自动计价”或“MES 工单自动核算”的当前事实。

## 历史草稿校正说明

早期文档使用了 `materialCode`、`standardCost`、`effectiveDate`、`invalidDate`、`isEnabled` 等非当前后端字段名，并将“物料 + 生效时间唯一”“新成本生效后旧成本自动失效”“WMS / MES 自动成本核算”写为既成事实。当前 DDL / DO / VO 的真实字段为 `item_code`、`price`、`active_time`、`expire_time`、`available`；上述业务规则尚未从本轮源码证实，故不再作为当前实现说明。
