# 生产线物料关系管理

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；生产主数据维护、工艺/计划、线边物流人员（顺带）。

## 业务目的与适用范围

生产线物料关系建立「某条产线与某项物料长期适用」的共同口径，支撑按产线查物料、查 BOM 版本，以及计划/线边物流判断「这料能不能上这条线」。读完本页，应能判断：按产线查不到料是关系未建、停用还是入口物料范围不对；本关系与 BOM、实际工单/发料各管什么；原料/完工库位为何不能按已上线功能培训。

它不等于 BOM（BOM 管构成；本页管产线适用范围），也不会因维护一条关系就自动创建工单、发料或库存变动。具体操作见[生产线物料关系管理-维护与查询参考](10-生产线物料关系管理-维护与查询参考.md)。

## 什么时候需要维护

| 业务事件 | 应做什么 | 维护前要确认 |
| --- | --- | --- |
| 新产线投产或新物料切入 | 新增产线与物料的适用关系。 | 工艺、BOM和生产责任人是否认可该适用范围。 |
| 工艺改造、产线切换或物料替代 | 评估已有关系，再新增、停用或调整有效期。 | 未完成计划、线边库存和在途发料是否会受影响。 |
| 需要按车间或产线查询物料 | 先核对关系是否完整、状态是否可用。 | 物料类型和车间—产线归属是否正确。 |
| 一次性导入多个产线物料关系 | 使用经验证的模板分批导入。 | 重复关系、物料范围和导入模式是否已确认。 |

## 关系如何服务生产物流

```mermaid
flowchart LR
    A["物料基本信息"] --> C["生产线物料关系"]
    B["车间与生产线资料"] --> C
    C --> D["按产线查询物料与 BOM 版本"]
    D --> E["计划、工艺与线边物流判断"]
    E --> F["实际工单、发料和追溯记录"]
```

同一物料可关联多条生产线，同一生产线也可关联多项物料。若业务存在「主用线、备用线、临时替代」等差别，应在工程或调度规则中明确；当前关系本身未确认支持这些优先级语义（`GAP-035`）。

!!! example "写实示例：给定配置 → 期望行为"
    **给定：** 产线 LINE-01、物料 FG-100（主入口允许的 CCP/BCP 等范围）、关系可用、生效时间已到。
    **期望：**

    1. 按 LINE-01 可查到 FG-100；再联查该物料的 [BOM](02-BOM.md) 版本做结构核对。
    2. 若在原材料/产成品（PC/YCL）入口维护，可选物料范围可能更窄（`GAP-037`）——选不到时先换对入口，再查启停。
    3. 停用关系后，按产线查询通常不再把该料当适用物料；对已下达计划的拦截时点 ❓。
    4. 页面上没有原料/完工库位可维护入口（`GAP-035`）；现场库位问题转到仓储/作业配置，不要在本页找。

## 维护时最重要的判断

### 关键字段业务角色

完整语义见[维护与查询参考](10-生产线物料关系管理-维护与查询参考.md)。**原料/完工库位等历史字段当前主 Web 未开放**（`GAP-035`），勿按库位策略页培训。选择器通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)；本页物料范围按入口过滤（`GAP-037`），与「仅可用主数据」通例并存。库位级联差异见[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md)。

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- |
| 生产线 | 关系一端 | 选自已存在产线；编辑锁定；与物料组成业务键 | 临时试产勿固化为长期关系（`GAP-036`） |
| 物料 | 关系另一端 | 按入口筛选允许的物料范围（主入口常见 CCP/BCP）；编辑锁定 | PC/YCL 视图物料范围可能不同（`GAP-037`） |
| 物料类型 | 视图区分辅助 | 多由物料带回 | 勿当人工改分类入口 |
| 是否可用 / 有效期 | 关系是否仍适用 | 停用优先；失效晚于生效 | 停用前查计划与线边库存 |
| 原料/完工库位 | 目标能力/历史字段 | **当前主页面未确认可维护**；库位校验未接入保存（`GAP-036`） | 勿写成已上线功能（`GAP-035`） |
| 主用线 / 默认用量 | 历史草稿字段 | **不在当前表**（`GAP-035`） | 优先级语义未证实，勿培训为已实现 |

| 需要判断什么 | 业务含义 | 建议做法 |
| --- | --- | --- |
| 产线与物料是否真实适用 | 防止将一次临时试产固化为长期主数据。 | 与工艺、生产和物料责任人共同确认。 |
| 关系是否已存在 | 同一产线与同一物料不应重复维护。 | 先按产线和物料查询，再决定新增或变更。 |
| 是否仍在使用 | 无效关系会干扰查询和业务判断。 | 优先停用，保留历史追溯；删除前先评估引用。 |
| 有效期是否需要控制 | 用于配合分阶段投产或切换。 | 同时填写时确保失效时间晚于生效时间。 |

### 选不到料与产线约束

| 现场现象 | 先回查本页什么 | 再联查什么 |
| --- | --- | --- |
| 新增时物料选不到 | 是否在当前入口允许范围；物料是否可用 | 物料基本信息；换入口再试（`GAP-037`） |
| 按产线查不到某料 | 关系是否存在、是否可用、有效期 | 车间—产线归属、BOM/计划 |
| 同一关系无法新增 | 是否已有产线+物料记录 | 更新有效期/状态，或确认确需新关系（`GAP-036`） |
| 现场发料/工单未按预期 | 勿假定本关系自动驱动执行 | 工单、发料任务、线边库存；再核 BOM |

### 建议验证点

- 主入口与 PC/YCL 入口可选物料范围是否符合 `GAP-037` 描述。
- 重复「产线+物料」新增是否被拦截；旁路改键风险（`GAP-036`）。
- 停用后按产线查询是否不再展示该料；对已下达计划的影响时点。
- 确认主 Web 无原料/完工库位维护入口（`GAP-035` 能力边界）。

## 查询与联查

| 想回答的问题 | 建议先查什么 | 再联查什么 |
| --- | --- | --- |
| 某条产线涉及哪些物料 | 本页按产线或车间查询。 | 物料基本信息、BOM、工艺路线。 |
| 某物料可在哪些产线使用 | 本页按物料查询。 | 生产线、车间和相关计划。 |
| 某产线可用物料的产品结构 | 产线物料关系。 | BOM 版本与物料信息。 |
| 为什么现场无法按预期执行 | 实际工单、发料任务、线边库存和本关系。 | 先确认是主数据缺失，还是实际单据/库存条件不满足。 |
