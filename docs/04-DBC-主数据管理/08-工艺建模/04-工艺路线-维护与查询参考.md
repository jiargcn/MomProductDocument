# 工艺路线-维护与查询参考

> 本页是[工艺路线](02-工艺路线.md)的操作参考。实际维护能力主要在 MES，路线变更须按生产变更流程受控执行。

## 新增与编辑约束

| 信息组 | 应维护的信息 | 维护限制 |
| --- | --- | --- |
| 路线识别 | 路线编码、名称、类别、版本、状态、备注。 | 编码、名称、类别、版本为当前输入模型必填；唯一性及编辑锁定待验证。 |
| 适用对象 | 物料、BOM及其版本。 | 选择条件、可否一对多和路线选择优先级待确认。 |
| 路线节点 | 工序、顺序、前后关系、节拍、节点配置。 | 修改前必须评估在途工单与追溯影响。 |
| 执行控制 | 前置校验、校验规则、并行作业、分批/最小放行条件、分派方式。 | 仅由理解现场执行规则的人员配置，并以生产样例验证。 |

## 导入与批量处理

当前资料未确认工艺路线存在可用于正式维护的导入模板。路线含节点关系和图形/配置内容，若后续提供导入能力，应至少定义路线主信息、版本、工序顺序、前后关系、执行控制、校验方式和错误回执，不能仅按一行普通主数据处理。

## 最小验收清单

- [ ] 正常路线能从首工序按预期顺序执行到末工序。
- [ ] 节点关系变更后，至少验证一笔新工单和一笔在途/历史工单表现。
- [ ] 并行、放行或分派配置有命中与不命中样例。
- [ ] 停用/切换版本后，新业务选择与历史追溯符合批准口径。
