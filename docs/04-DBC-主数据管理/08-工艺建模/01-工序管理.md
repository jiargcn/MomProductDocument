# 工序管理

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。

## 这项主数据解决什么问题

工序管理定义制造过程中的基本作业步骤，并将步骤与车间、生产线、工序类型、节拍和有效期关联。它是工艺路线、生产现场和生产追溯共同使用的步骤口径，而不是实际报工或工单任务。

## 维护与查询说明

| 维护事项 | 当前可确认做法 | 操作提示 |
| --- | --- | --- |
| 新增 | 代码为必填；维护名称、类型、车间、生产线、节拍、有效期、备注和可用状态。 | 先确认步骤在什么车间/产线执行，避免只按名称创建。 |
| 编辑 | 可调整名称、类型、节拍、归属和有效期。 | 已被工艺路线或生产业务引用时，应先评估顺序、节拍和现场影响。 |
| 停用 | 用可用状态控制后续选择。 | 不删除仍需用于历史路线或追溯的工序。 |
| 导入 | 模板包含代码、名称、类型、车间、生产线、节拍、可用状态、有效期和备注。 | 代码、车间、生产线和可用状态为当前模板必填；先核对现场归属。 |
| 查询 | 列表优先显示代码、名称、类型、车间、生产线、节拍、可用状态和有效期。 | 常用筛选为代码/名称、类型、车间/产线与可用状态。 |

建议详情按“基本识别、现场归属、工艺参数、生效控制、引用与变更记录”分组；从工序跳转至工艺路线、工位和生产执行的入口需要补充截图验证。

## 它与工艺和生产的关系

```mermaid
flowchart LR
    A["车间、生产线"] --> B["工序"]
    B --> C["工艺路线"]
    C --> D["生产计划与现场执行"]
    D --> E["生产追溯"]
```

工序是路线的组成基础。路线、工单和追溯的实际读取时点、停用保护和版本选择规则需在 MES 场景中继续验证。

## 当前边界与待确认事项

- 页面输入模型与导入模板对车间、生产线、可用状态的必填表达存在差异，需以实际页面/接口验证为准。
- 当前尚未确认工序代码唯一性、车间与产线从属校验、节拍合法范围及已引用后的编辑/停用保护。
- 工序与工位、工艺路线、MES 实际执行的完整挂接范围仍待补充测试。
