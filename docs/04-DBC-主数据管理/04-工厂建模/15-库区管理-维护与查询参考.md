# 库区管理-维护与查询参考

> 对应业务说明：[库区管理](02-库区管理.md)。

## 新增与编辑

| 维护项 | 新增要求 | 编辑与变更建议 |
| --- | --- | --- |
| 库区编码和名称 | 编码必填，名称应体现作业用途。 | 已引用后的改码影响待确认，建议以停用旧库区和新增替代为主。 |
| 所属仓库 | 必填，必须先有仓库。 | 不建议跨仓直接改挂；如调整布局，先迁移库存和库位。 |
| 类型、管理精度与部门 | 类型为必填导入口径；管理精度和部门按现场职责维护。 | 变更前评估对库存管理与作业分工的影响。 |
| 是否可用与有效期 | 导入时必填；用于控制当前适用范围。 | 停用前先检查下属库位、库存与任务。 |

## 批量导入规则

模板应包含：编码、名称、仓库、类型、是否可用、所属部门、生效时间、失效时间、备注。

- 不应导入库位、库存、物料限制或实际任务数据。
- 编码、仓库、类型和可用状态为必填；仓库须已存在。
- 按仓库→库区→库位组→库位顺序导入；导入模式和错误处理需用小批量数据先验证。

## 查询与详情建议

列表优先展示编码、名称、所属仓库、类型、管理精度、可用状态、有效期；常用筛选为仓库、类型、部门、可用状态。详情建议分为“基本关系、用途与管理、下属地点、状态与有效期、审计与备注”，并链接仓库、库位、库存余额和作业任务。

## 常见问题

| 现象 | 处理建议 |
| --- | --- |
| 库区无法被上架或拣选使用 | 核对仓库归属、库区状态、下属库位和实际任务规则。 |
| 库区用途与实际库存不符 | 先处理库存和在途作业，再调整库区用途或状态。 |
| 导入后库位无法关联 | 核对库区编码、仓库归属与导入顺序。 |
