## 当前实现摘要

内部月台维护类型、所属仓库和默认库位，供仓储业务选择可用月台。页面通过通用 CRUD 入口调用 `DockServiceImpl` 的 `beforeCreate` / `beforeUpdate` 钩子，因此类型、仓库和默认库位关系校验可作为当前保存路径证据。

月台代码仅校验非空，DDL 未见业务唯一约束；编辑允许改变仓库、默认库位等关键归属，删除未见引用拦截，详见产品差距总账 GAP-052。

## 字段与新增约束

| 技术字段 | 中文名 | DDL 类型/约束 | Web 规则 | 服务校验 |
| --- | --- | --- | --- | --- |
| `code` | 代码 | `varchar(64)`，非空 | 必填、最多 50 字符、`validateYS` 校验。 | 非空；未见重码校验。 |
| `name` | 名称 | `varchar(64)`，可空 | 必填、最多 50 字符。 | VO 与 DDL均未见服务必填校验。 |
| `type` | 类型 | `varchar(64)`，非空 | `DOCK_TYPE` 字典，必填。 | 非空。 |
| `warehouseCode` | 仓库代码 | `varchar(64)`，非空 | 必填；仓库分页选择器，筛选 `available=true`。 | 校验仓库存在。 |
| `defaultLocationCode` | 默认库位代码 | `varchar(64)`，非空 | 必填；库位选择器筛选 `available=true` 且与当前仓库相同。 | 校验库位存在并属于该仓库。 |
| `city` / `address` | 城市 / 地址 | `varchar(64)`，可空 | 最多 50 字符。 | 无额外校验。 |
| `contactPerson` / `contactPhone` | 联系人 / 联系电话 | `varchar(64)`，可空 | 最多 50 字符；电话使用前端手机号格式校验。 | 未见服务端格式校验。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 开关控件，默认 `true`，必填。 | 非空。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)`，可空 | 日期时间选择器。 | 未见区间校验。 |
| `remark` | 备注 | `text`，可空 | 最多 50 字符。 | 无额外校验。 |

`description`、审计字段、删除信息、扩展属性、并发戳、站点和租户属于系统/持久化字段，不进入人工模板。

## 编辑、启停与删除

| 动作 | 当前可证实行为 | 使用口径 |
| --- | --- | --- |
| 编辑 | Web 锁定 `code`，其余字段包括类型、仓库和默认库位可编辑；服务再次校验默认库位归属。 | 正常页面不可改代码；改仓库时必须同时选择该仓库中的默认库位。 |
| 启用/停用 | 列表提供启停，服务的月台查询会筛选 `available=true`。 | 停用前需核查采购收货、销售出库和在途任务引用；当前未见统一拦截。 |
| 删除 | 通用删除仅校验记录存在后删除。 | 已被业务单据引用的月台不得按“可直接删除”培训。 |

前端按钮权限标识为 `dbc:dock:create/import/export/update/enable/disable/delete`。Controller 继承通用 Controller，需通过 RBAC 实测确认接口层授权，不以按钮显示作结论。

## 导入规则

`DockImportExcelVO` 模板字段依次为：代码、名称、城市、地址、类型、仓库代码、默认库位代码、联系人姓名、联系人电话、是否可用、生效时间、失效时间、备注。

通用导入会进入本服务覆写的 `validImport`，可确认：

| 校验 | 当前规则 |
| --- | --- |
| 必填 | 代码、名称、类型、仓库代码、默认库位代码、是否可用。 |
| 关联 | 校验仓库存在；校验默认库位存在且属于该仓库。 |
| 数据类型 | 类型使用 `DOCK_TYPE` 字典；是否可用使用布尔字典。 |
| 既有记录与模式 | 通用导入路径未见本页覆写既有记录匹配键；需继续确认其新增/覆盖时的业务键。 |
| 事务 | 出现错误且未选择部分更新时整批回滚。 |

模板不包含 `description`、ID、审计、删除、扩展、并发、站点及租户字段。

## 列表、详情与快速跳转

当前前端默认列顺序为：代码、名称、类型、仓库代码、默认库位代码、城市、地址、联系人姓名、联系人电话、是否可用、生效时间、失效时间、备注、审计列、操作。默认筛选为代码、名称、类型、是否可用（默认“可用”）。

建议主列表优先显示：代码、名称、类型、仓库、默认库位、是否可用；地址、联系人、有效期和审计列作为补充。

当前详情为通用组件，未见专用关联 Tab。后续建议：

| 分组 | 内容 | 建议跳转 |
| --- | --- | --- |
| 基本与归属 | 代码、名称、类型、仓库、默认库位、可用状态 | 仓库、默认库位 |
| 联系与有效期 | 城市、地址、联系人、有效期、备注 | 无 |
| 业务引用 | 采购收货、销售出库、发运/到货任务 | 对应单据查询并预填月台（待实现/验证） |
| 审计 | 创建/更新信息 | 操作日志（待公共能力取证） |

## 关联关系

```
内部月台（basic_dock）
  ├─ 所属仓库（basic_warehouse）
  ├─ 默认库位（basic_location，同一仓库）
  └─ 采购收货、销售出库等仓储业务的月台选择

客户月台（basic_customerdock）
  ├─ 所属客户
  └─ 可关联仓库和默认库位
```

> 两类月台可共享仓库/库位等引用，但当前未证实存在自动同步、自动映射或可互换使用关系。
