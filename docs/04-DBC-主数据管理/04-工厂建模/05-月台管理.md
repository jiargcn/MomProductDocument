# 月台管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓库收发人员、物流调度人员、仓储主数据维护人员。
> 具体新增、编辑、启停、导入和查询操作见[月台管理-维护与查询参考](18-月台管理-维护与查询参考.md)。

## 业务目的与适用范围

月台用于维护企业内部收货、发货、装卸或车辆交接的现场位置。采购收货、销售发运和物流协同可通过月台明确物料从哪里进出、由谁交接，减少“同一仓库但不同装卸点”带来的现场混乱。

## 何时需要维护

新增装卸区域、月台编号调整、收发货流程需要分流，或任务/车辆无法匹配交接地点时，应维护月台资料。

## 月台如何进入收发业务

```mermaid
flowchart LR
    A["仓库与现场地点"] --> B["月台"]
    B --> C["采购收货/销售发运任务"]
    C --> D["车辆、物料与交接记录"]
```

月台是交接地点，不等于库存库位。物料完成交接后仍需通过收货、上架或出库等业务确定实际库存变化。

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 归属仓库/地点 | 月台服务哪个仓库和收发区域。 | 与现场标识和车辆路线一致。 |
| 月台编号/名称 | 是否便于调度和司机/仓库人员识别。 | 避免与客户月台混淆。 |
| 状态与可用性 | 是否可用于当前收发任务。 | 维修或停用前处理在途预约/任务。 |

## 查询、详情与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 某收货/发运在哪交接 | 任务、来源单据、月台。 |
| 某月台有哪些在途工作 | 月台、收货/发运任务、车辆/预约信息。 |
| 为什么无法选择月台 | 月台状态、仓库归属和业务页面条件。 |

## 维护与查询边界

月台维护的是企业内部装卸交接点，需要同时明确所属仓库、默认库位、月台类型和联系人。默认库位只表达收发后的默认地点口径；实际库存变化仍需经过收货、上架或出库等业务过程。

页面支持批量导入。月台导入前必须确保仓库和默认库位已建成且关系正确；不要把客户月台、客户交付地址或车辆预约信息导入到本页。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 月台与仓库不匹配 | 核对归属、现场编号和任务来源。 |
| 月台停用但仍有任务 | 先协调转移或完成在途任务。 |
| 将客户月台当成企业月台 | 客户月台用于交付目的地，本页用于企业内部装卸点。 |

## 当前限制与待确认事项

- 月台与预约、车辆、收发任务的自动带入关系待验证；
- 容量、时间窗、导入和权限规则待核验；
- 需补收货/发运页面选择月台的截图与异常提示。
