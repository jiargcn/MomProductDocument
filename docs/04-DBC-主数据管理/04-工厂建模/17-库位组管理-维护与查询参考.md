# 库位组管理-维护与查询参考

> 对应业务说明：[库位组管理](04-库位组管理.md)。

## 新增与编辑

| 维护项 | 新增要求 | 编辑与变更建议 |
| --- | --- | --- |
| 编码、名称与描述 | 编码必填，名称应说明分组用途。 | 上线后避免随意改码，防止影响库位引用。 |
| 所属仓库、库区 | 均为必填，需与实际现场层级一致。 | 跨区域调整应先检查成员库位、库存和盘点任务。 |
| 是否可用与有效期 | 可用状态必填。 | 停用前确认是否仍被库位、任务或盘点范围引用。 |
| 备注 | 写清分组用途、范围或调整原因。 | 不用备注代替库位成员关系。 |

## 批量导入规则

模板应包含：编码、名称、仓库、库区、是否可用、生效时间、失效时间、备注。

- 不应导入库位成员、库存数量、盘点任务或审计字段；库位成员关系在库位资料中维护。
- 编码、名称、仓库、库区、可用状态为必填；仓库和库区必须已存在且归属正确。
- 导入完成后，按库位组查询库位是否均已正确归属；重复规则和错误回执待测试确认。

## 查询与详情建议

列表优先显示编码、名称、仓库、库区、可用状态、有效期；常用筛选为仓库、库区、状态。详情建议分为“基本关系、成员库位与库存、状态与有效期、审计与备注”，并联查成员库位、组内库存与盘点任务。

## 常见问题

| 现象 | 处理建议 |
| --- | --- |
| 库位不在预期组内 | 到库位资料核对所属库位组和上层地点。 |
| 盘点范围异常 | 核对库位组用途、成员库位和盘点任务范围。 |
| 导入后无法关联库位 | 核对仓库、库区和库位组编码是否一致。 |
