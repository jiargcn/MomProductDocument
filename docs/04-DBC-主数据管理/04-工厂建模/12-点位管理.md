## 当前实现摘要

持久化表为 `basic_point`，核心身份和归属为 `code`（点位）与 `locationCode`（库位）。当前 Web 页面维护库位、AGV 点位、支持 AGV 型号、点位类型、楼层、方向、动作类型、设备/参数编码、有效期和备注；历史草稿中的点位名称、坐标、车间归属不是当前模型字段。

服务创建时会按 `code` 查重，并再按 `code + locationCode` 查重；DDL 只有普通索引。Web 却把 `code` 字段整体注释，表单规则仍要求它，导入则仍将其作为必填和匹配键。此处以及导入更新未回写导入字段的问题，见产品差距总账 GAP-057。

## 字段与新增约束

| 技术字段 | 中文名 | DDL 类型/约束 | 当前页面规则 | 当前服务证据 |
| --- | --- | --- | --- | --- |
| `code` | 点位 | `varchar(64)`，非空 | 当前页面字段已注释；规则对象仍标必填。 | 创建按代码查重；导入必填。 |
| `locationCode` | 库位 | `varchar(64)`，非空 | 必填；库位选择器，数据源为库位分页接口，筛选 `available=true`。 | 创建按代码+库位查重；导入会查询库位并回填名称。 |
| `locationName` | 库位名称 | `varchar(64)`，可空 | 只读，由选中库位前端回填。 | 导入由库位查询回填。 |
| `agvCode` | AGV 点位 | `varchar(64)`，可空 | 可填写；列表可查询。 | 导入列存在但非必填；未见关联存在性校验。 |
| `supportedAgvModels` | 支持 AGV 型号 | `varchar(64)`，可空 | `SUPPORTED_AGV_MODELS` 字典，前端必填。 | 导入必填；普通创建未见服务必填。 |
| `pointType` | 点位类型 | `varchar(64)`，可空 | `POINT_TYPE` 字典，前端必填。 | 导入必填；普通创建未见服务必填。 |
| `floor` | 楼层 | `smallint`，可空 | 数字输入，最小值 1，前端必填。 | 导入必填；普通创建未见服务必填。 |
| `pointDirection` | 点位方向 | `varchar(64)`，可空 | `POINT_DEIRECTION` 字典；当支持型号为 `F4` 时前端改为必填。 | 导入非必填；服务未见同等条件校验。 |
| `actionType` | 动作类型 | `varchar(16)`，可空 | `ACTION_TYPE` 字典，前端必填。 | 导入必填；普通创建未见服务必填。 |
| `assetCode` / `paramCode` | 设备/参数编码 | `varchar(64)`，可空 | 当前普通输入字段。 | 未见设备、参数存在性校验。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 当前 Web 字段已注释。 | 导入无论模板值如何均强制写 `true`。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)`，可空 | 日期时间选择器。 | 将 `1970` 年时间归空；未见区间先后校验。 |
| `remark` | 备注 | `text`，可空 | 普通输入字段。 | 无额外校验。 |

## 编辑、删除与权限

| 动作 | 当前可证实行为 | 使用口径 |
| --- | --- | --- |
| 编辑 | Web 未锁定核心字段；服务仅校验记录存在后整体更新，并处理 1970 年时间。 | 不应假定点位、库位映射或调度属性不可改；变更前应核查 AGV 与现场调度引用。 |
| 删除 | 服务校验存在后即删除，并写趋势记录；未见下游引用检查。 | 被 AGV 调度、设备或业务配置引用的点位不应视为可安全删除。 |
| 权限 | 前端使用 `dbc:point:create/import/export/update/delete`。 | 后端 Controller 权限注解需 RBAC 实测，不能以按钮配置判定接口已受保护。 |

## 导入规则

`PointImportExcelVO` 的当前模板顺序为：点位、库位、AGV 点位、动作类型、支持 AGV 型号、点位类型、楼层、点位方向、设备编码、参数编码、生效时间、失效时间、备注。

| 校验/模式 | 当前规则 |
| --- | --- |
| 必填 | 点位、库位、动作类型、支持 AGV 型号、点位类型、楼层；AGV 点位、方向、设备/参数可空。 |
| 库位 | 导入通过 `locationApi.getOne` 验证并回填库位名称。 |
| 唯一性 | 导入先按代码与代码+库位查重；而实际既有记录按代码+库位查找。该组合与创建路径的“代码全局唯一”约束并存。 |
| 既有记录 | 不存在且非覆盖模式时插入；存在且非仅新增模式时更新。 |
| 更新风险 | 当前更新分支仅对旧对象写回库位名称和可用状态，未将导入行的动作类型、型号、类型、楼层、方向、设备/参数、时间和备注写入旧对象。 |
| 状态 | 导入插入/更新均强制可用，模板不含可用状态。 |
| 错误处理 | 返回逐行错误；当前实现未见“有错误且未选择部分更新时整批回滚”。 |

模板不应包含 `locationName`、系统审计字段；是否应补回可用状态，以及应以代码还是代码+库位作为唯一匹配键，需先完成产品与技术口径确认。

## 列表、详情与快速跳转

当前默认列顺序为：库位代码、库位名称、AGV 点位、支持 AGV 型号、点位类型、楼层、方向、动作类型、设备编码、参数编码、生效时间、失效时间、备注、审计列、操作。当前主表未显示点位代码，虽其仍为持久化与导入关键字段。

建议主列表优先显示：点位代码、库位代码/名称、AGV 点位、支持型号、点位类型、楼层/方向、可用状态；将动作、设备参数与审计字段放入补充列。

当前采用通用详情组件，未见专用关联页签。后续建议：
