# 库区管理

> 当前基线：`dev`（测试环境目标基线），源码与 DDL 取证日期：2026-07-16。
> 页面定位：维护仓库内的逻辑库区，是库位、上架、库存查询等对象的空间归属基础；不适用申请、任务与记录模型。

## 当前实现摘要

当前持久化表为 `basic_area`。页面支持新增、编辑、启停、删除、导入和导出；默认按代码、名称、类型、是否可用查询。库区关联一个可用仓库，库区类型取自字典。普通新增/编辑走通用 `WinBaseService` CRUD，当前库区服务未覆写其 `beforeCreate` 或 `beforeUpdate`；因此不能把类内未被该流程直接调用的私有校验方法写成已生效的普通保存规则。

代码唯一性、仓库关联变更、类型/状态服务端校验和下游删除保护尚未形成可证明的闭环，详见[产品差距总账](../../15-版本路线图/03-问题与待确认/产品差距总账.md) GAP-049。

## 字段与新增约束

| 技术字段 | 中文名 | DDL 类型/长度 | Web 表单/选择器 | 当前服务证据 |
| --- | --- | --- | --- | --- |
| `code` | 代码 | `varchar(64)`，非空 | 必填；最多 50 个字符；`validateYS` 格式校验。 | 通用新增直接持久化；本类私有代码校验方法未见被普通新增钩子调用。 |
| `name` | 名称 | `varchar(64)`，可空 | 最多 50 个字符。 | 未见普通新增必填校验。 |
| `description` | 描述 | `varchar(64)`，可空 | 当前主页面字段配置未见该字段。 | VO/DDL 存在；页面展示与维护入口待测试核验。 |
| `warehouseCode` | 仓库代码 | `varchar(64)`，非空 | 必填；仓库查询选择器，数据源为仓库分页接口，筛选 `available=true`。 | 私有校验会查询可用仓库；普通保存路径是否调用该校验不能由当前子类证明。 |
| `type` | 类型 | `varchar(64)`，可空 | `AREA_TYPE` 字典；页面和导入均要求选择。 | 普通保存 VO 未标注必填；导入路径显式校验非空。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 开关控件，默认 `true`；页面必填。 | 普通保存 VO 的 `@NotNull` 已注释；导入路径显式校验非空。 |
| `departmentCode` | 所属部门代码 | `varchar(64)`，可空 | 部门选择器，数据源为 System 部门分页接口；`departmentCodeName` 为只读显示字段。 | 导入时若填写，调用部门接口校验其存在。 |
| `manageMode` | 管理精度 | `varchar(64)`，可空 | DDL/VO 与前端规则中存在；主表单字段配置及关联逻辑当前被注释。 | 当前不能作为已启用的维护规则。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)`，可空 | 日期时间选择器。 | 未见时间区间校验。 |
| `remark` | 备注 | `text`，可空 | 最多 50 个字符。 | 未见额外服务校验。 |

`id`、审计字段、逻辑删除、并发戳、扩展属性和租户等为系统字段，不纳入人工新增或导入模板。

## 编辑、启停与删除

| 动作 | 当前可证实行为 | 使用口径 |
| --- | --- | --- |
| 编辑 | Web 编辑锁定 `code` 和 `warehouseCode`，并关闭仓库选择器；通用更新会把请求对象转换后整体更新。 | 正常 Web 操作不可改代码/仓库；不得据此推断接口层同样受保护。 |
| 启用/停用 | `available` 参与多个库区查询的可选条件；列表提供启用/停用动作。 | 停用前应核查库位、库存、上架及在途单据引用；尚未见统一拦截证据。 |
| 删除 | 通用删除仅检查记录存在后删除。 | 已被库位或库存业务使用的库区不应视为可安全删除，需先做引用核查。 |

前端按钮权限标识为 `wms:areabasic:create/import/export/update/enable/disable/delete`。该页面 Controller 继承通用 Controller，需在 System/RBAC 实测中确认接口层鉴权，不能仅以按钮标识推断。

## 导入规则

导入类 `AreabasicImportExcelVo` 的字段顺序为：代码、名称、仓库代码、类型、是否可用、所属部门代码、生效时间、失效时间、备注。

| 导入校验 | 当前可证实规则 |
| --- | --- |
| 必填 | 代码、仓库代码、类型、是否可用。 |
| 仓库 | 校验仓库代码对应可用仓库。 |
| 部门 | 填写部门代码时调用 System 部门接口校验存在。 |
| 重复与模式 | 通用导入按对象的既有记录判断新增/追加/覆盖；本页未见该既有记录匹配键的子类覆写证据，需继续核验。 |
| 错误回滚 | 出现错误且未选择部分更新时，通用导入事务回滚。 |

模板不应含 `id`、`departmentCodeName`、创建/更新人、创建/更新时间、删除信息、租户、并发戳、扩展属性等系统或展示字段。`description` 和 `manageMode` 虽在 DDL/VO 存在，但当前导入类不包含它们。

## 列表、详情与快速跳转

### 列表口径

当前前端默认列的前段为：代码、名称、仓库代码、类型、是否可用、所属部门代码、所属部门名称、生效时间、失效时间、备注，随后为审计列和操作。默认筛选为代码、名称、类型、是否可用（默认“可用”）。

建议培训主列表优先显示：代码、名称、仓库代码、类型、是否可用、所属部门代码；有效期、备注和审计列作为补充。

### 详情分组与跳转占位

当前使用通用详情组件，未见库区专用 Tab。后续应在测试环境验证并按以下分组补齐：

| 分组 | 内容 | 建议跳转 |
| --- | --- | --- |
| 基本与归属 | 代码、名称、仓库、类型、是否可用、部门 | 仓库、库位组、库位 |
| 管理规则 | 管理精度、有效期、备注 | 库存策略/上架策略（待确认） |
| 审计 | 创建/更新信息 | 操作日志（待公共能力取证） |
| 下游引用 | 库位、库存、上架、盘点、调拨 | 关联列表预填仓库和库区条件（待实现/验证） |
