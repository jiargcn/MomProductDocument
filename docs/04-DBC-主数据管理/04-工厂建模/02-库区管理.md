# 库区管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓储主数据维护人员、库房主管、上架/拣选人员。
> 具体新增、编辑、启停、导入和查询操作见[库区管理-维护与查询参考](15-库区管理-维护与查询参考.md)。

## 业务目的与适用范围

库区是仓库内部按用途、状态或作业方式划分的区域，例如收货区、存储区、拣选区、隔离区。它帮助业务在“同一仓库”内继续区分物料应在哪里处理、哪些库存可用以及作业应从哪个区域开始。

## 何时需要维护

仓库启用新功能区、收发流程调整、需要隔离/待检管理，或物料库区配置与实际作业地点不一致时，应维护库区资料。

## 库区如何服务作业

```mermaid
flowchart LR
    A["仓库"] --> B["按用途划分库区"]
    B --> C["库位与作业规则"]
    C --> D["收货、上架、拣选、盘点"]
```

库区体现业务用途，不等于某一条库存记录。实际可用性仍需结合库存状态、库位和质量结论判断。

## 关键字段业务角色

完整语义、选择器与联动见[维护与查询参考](15-库区管理-维护与查询参考.md)。通例见[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md)。本页模式以 **P1 / P2 / P4 / P6** 为主。

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- | --- |
| 库区编码 | P6 | 区域内地点识别 | 新增必填；Web 编辑通常锁定代码；唯一性 ❓（`GAP-049`） | 已挂库位后勿随意改码 |
| 所属仓库 | P2 / P4 | 库区的上级地点 | **仅可选已存在仓库**；编辑通常锁定仓库；变更仓库应先迁移下级 | 错挂仓库导致整区作业错位 |
| 库区类型 | P1 | 收货/存储/拣选/隔离等用途口径 | 字典；**不能假定**类型会自动限制库存状态或上架策略 | 勿只靠类型名称推断强制隔离 |
| 管理精度 / 部门 | P1 / P13 | 管理粒度与组织归属资料 | 部门不能代替仓库归属；精度与策略挂接 ❓ | 误把部门当地点过滤条件 |
| 是否可用 / 有效期 | P1 / P10 | 控制区域是否可被引用 | 停用前查下属库位、库存与任务 | 有库存仍停用 → 作业中断 |

### 关键维护与变更摘要

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 归属仓库 | 库区属于哪个仓库。 | 避免跨仓重复或错挂。 |
| 区域用途 | 是否用于收货、存储、拣选、隔离或其它作业。 | 用途应与实际流程和人员分工一致。 |
| 下属库位 | 是否能满足区域内的实际存放和扫码操作。 | 先建区域再按需要建库位。 |
| 启停/调整 | 是否仍有库存或在途任务。 | 调整前先迁移库存和作业。 |

## 查询、详情与联查

可从仓库查库区，从库区查库位、库存余额和当前作业；发现上架或拣选地点不合理时，应同时检查物料库区配置、库存状态和任务规则。

## 维护与查询边界

库区必须先明确所属仓库和用途，再建立库位组、库位和作业规则。当前支持批量导入，导入时应按“仓库先于库区”的顺序进行；部门归属可作为管理信息维护，但不能代替仓库归属。

当前资料不能证明库区类型会自动限制库存状态、物料范围或上架策略。需要隔离、待检、拣选等强制控制时，必须同时验证库存状态、任务规则与现场操作，不能只依赖库区名称。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 任务找不到目标区域 | 核对仓库归属、库区状态、物料配置和任务条件。 |
| 待检/隔离库存进入普通区 | 停止后续使用，核对库存状态、质量结论和库区用途。 |
| 调整库区影响库存 | 先安排调拨/转库并处理未完成任务。 |

## 当前限制与待确认事项

- 普通保存与导入校验路径不一致（`GAP-049`）；勿把导入规则写成页面保存的既成约束；
- 库区类型是否强制限制库存状态和物料范围需测试；
- 需补列表、详情和任务引用截图。
