# 生产线管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、启停、导入和查询操作见[生产线管理-维护与查询参考](20-生产线管理-维护与查询参考.md)。

## 业务目的与适用范围

生产线用于维护实际生产执行的线体，是工单分派、工艺路线、工位、线边物料和设备协同的共同地点。它帮助系统区分“同一车间内由哪条线生产、物料应送到哪里、设备与工位如何关联”。

## 何时需要维护

产线新建、停线、搬迁、产线改造、工艺或线边物料关系调整时，应维护生产线资料。

## 生产线如何被业务使用

```mermaid
flowchart LR
    A["车间"] --> B["生产线"]
    B --> C["工位、工艺路线、物料关系"]
    C --> D["工单执行与线边发料"]
```

生产线资料建立后，还需维护工位、物料关系和工艺路线；不应只因名称相同就假定不同业务页面自动关联。

## 关键字段业务角色

完整语义见[维护与查询参考](20-生产线管理-维护与查询参考.md)。本页模式：**P1 / P2 / P4 / P6**。

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- | --- |
| 产线编码 / 名称 | P6 | 线体身份 | 必填；查重与库约束 ❓（`GAP-054`） | 改码影响工位/工单引用 |
| 所属车间 | P2 / P4 | 上层组织归属 | 必填；须先有可用车间；跨车间改挂高风险 | 车间停用后选不到 |
| 类型 | P1 | 线体用途分类 | 字典；导入必填口径 ❓ | 类型与工艺场景不符 |
| 原料/成品库位 | — | 存储侧线索 | **主页面/模板通常未维护**；勿当当前自动收发规则（`GAP-054`） | 历史字段勿写入培训 |
| 是否可用 | P1 | 可否被工位/工单引用 | 停线前查工位与在途 | 停线仍有在制 |

### 关键维护与变更摘要

| 维护点 | 业务判断 |
| --- | --- |
| 归属车间 | 是否处于正确现场组织。 |
| 线体身份与状态 | 是否能被计划、工艺和现场人员稳定识别。 |
| 工位/物料/工艺关系 | 是否支持实际生产和线边供料。 |
| 停线或迁移 | 是否影响工单、设备和在途发料。 |

## 查询、详情与联查

可从生产线联查工位、物料关系、工艺路线、工单和线边任务；出现发料或工艺不匹配时，应优先核对这些关系而非直接修改任务结果。

## 维护与查询边界

生产线需要明确所属车间、线体类型、可用状态和有效期。原料/成品库位属于资料线索，**当前主页面不应按可维护自动规则培训**（`GAP-054`）。工位、产线物料关系和工艺路线应在各自页面完成。

页面支持批量导入。导入前必须先维护车间。

## 当前限制与待确认事项

- 车间存在性校验、库位配置入口与注释 UI 边界见 `GAP-054`；
- 与 MES 工单、设备、数采点和 AGV 的实际自动挂接需确认。
