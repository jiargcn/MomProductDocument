# 生产线管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、启停、导入和查询操作见[生产线管理-维护与查询参考](20-生产线管理-维护与查询参考.md)。

## 业务目的与适用范围

生产线用于维护实际生产执行的线体，是工单分派、工艺路线、工位、线边物料和设备协同的共同地点。它帮助系统区分“同一车间内由哪条线生产、物料应送到哪里、设备与工位如何关联”。

## 何时需要维护

产线新建、停线、搬迁、产线改造、工艺或线边物料关系调整时，应维护生产线资料。

## 生产线如何被业务使用

```mermaid
flowchart LR
    A["车间"] --> B["生产线"]
    B --> C["工位、工艺路线、物料关系"]
    C --> D["工单执行与线边发料"]
```

生产线资料建立后，还需维护工位、物料关系和工艺路线；不应只因名称相同就假定不同业务页面自动关联。

## 关键维护与变更

| 维护点 | 业务判断 |
| --- | --- |
| 归属车间 | 是否处于正确现场组织。 |
| 线体身份与状态 | 是否能被计划、工艺和现场人员稳定识别。 |
| 工位/物料/工艺关系 | 是否支持实际生产和线边供料。 |
| 停线或迁移 | 是否影响工单、设备和在途发料。 |

## 查询、详情与联查

可从生产线联查工位、物料关系、工艺路线、工单和线边任务；出现发料或工艺不匹配时，应优先核对这些关系而非直接修改任务结果。

## 维护与查询边界

生产线需要明确所属车间、线体类型、可用状态和有效期；原料/成品相关库位属于资料线索，但其是否在当前主页面参与维护或自动驱动作业需以测试环境为准。工位、产线物料关系和工艺路线应在各自页面完成，不能假定填写生产线后系统会自动补全它们。

页面支持批量导入。导入前必须先维护车间；如需配置线边地点或完工收货地点，应在测试环境验证当前入口和作业引用。

## 当前限制与待确认事项

- 产线字段、产能、班次、状态与导入规则待核验；
- 与 MES 工单、设备、数采点和 AGV 的实际自动挂接需确认；
- 需补产线详情、工位/物料/工艺页签截图。
