# 仓库管理

> 当前基线：`dev`（测试环境目标基线），源码与 DDL 取证日期：2026-07-16。
> 页面定位：维护 WMS 库区、库位、月台、物料仓库默认配置等对象所依赖的仓库主数据；它不是申请、任务或记录对象。

## 当前实现摘要

当前持久化表为 `basic_warehouse`。页面支持新增、编辑、启停、删除、导入和导出；列表可按代码、名称、类型、是否可用筛选。仓库编码是业务识别字段，但其唯一性、编辑保护与下游引用删除保护尚未在各层闭合，详见产品差距总账 GAP-048。

## 字段与新增约束

| 技术字段 | 中文名 | DDL 类型/长度 | 当前新增规则 | 页面输入规则 |
| --- | --- | --- | --- | --- |
| `code` | 代码 | `varchar(64)`，非空 | 服务校验非空；未见普通新增的重码查询。 | 必填；最多 50 个字符；使用 `validateYS` 编码格式校验。 |
| `name` | 名称 | `varchar(64)`，可空 | 未见服务必填校验。 | 最多 50 个字符。 |
| `type` | 类型 | `varchar(64)`，可空 | 未见服务必填校验。 | 使用 `WAREHOUSE_TYPE` 字典；当前无必填规则。 |
| `description` | 描述 | `varchar(64)`，可空 | 无额外服务校验。 | 最多 50 个字符。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 服务和导入均校验非空。 | 开关控件；默认 `true`；必填。 |
| `activeTime` | 生效时间 | `timestamp(6)`，可空 | 无额外校验。 | 日期时间选择器。 |
| `expireTime` | 失效时间 | `timestamp(6)`，可空 | 无额外校验。 | 日期时间选择器。 |
| `remark` | 备注 | `text`，可空 | 无额外校验。 | 最多 50 个字符；前端限制小于 DDL 容量。 |

系统审计字段包括 `id`、`creator`、`create_time`、`updater`、`update_time`、`deleted` 与 `tenant_id`，不属于人工新增或导入字段。

### 选择器与引用边界

仓库自身新增表单未使用外部业务选择器；类型取自仓库类型字典。服务提供的仓库查询会按 `available = true` 过滤，供库区、库位组、库位等后续对象的可选仓库范围使用。具体业务页面还应以其自身选择器条件为准。

## 编辑、启停与删除

| 动作 | 当前可证实行为 | 培训与操作口径 |
| --- | --- | --- |
| 编辑 | Web 编辑时禁用 `code`；其余表单字段按新增表单提交。服务端更新接口仍接收并写回 `code`。 | 正常用户操作不修改代码；不得据此推断接口层也禁止改码。 |
| 启用/停用 | 列表提供启用、停用动作；`available` 是仓库可选查询的共同过滤条件。 | 停用前应先确认库区、库位、月台、库存及在途业务的引用影响；当前服务未见统一引用拦截。 |
| 删除 | 服务仅校验记录存在后直接删除。 | 已被下游对象引用的仓库不应视为可安全删除；需先做引用核查。 |

前端按钮使用 `dbc:warehouse:create/import/export/update/enable/disable/delete` 标识。后端导入接口的 `@PreAuthorize` 注解处于注释状态；因此页面按钮可见性不能作为服务端强制授权的结论。

## 导入与导出

### 导入模板字段

导入类 `WarehouseImportExcelVo` 的字段顺序为：代码、名称、类型、描述、是否可用、生效时间、失效时间、备注。代码和是否可用为必填；类型使用仓库类型字典，是否可用使用布尔字典。审计字段不在模板中。

导入先逐行校验代码非空、是否可用非空，再按代码查找已有记录：

| 模式 | 当前处理 |
| --- | --- |
| 非“仅更新”模式 | 未找到同代码记录时插入。 |
| 非“仅新增”模式 | 找到同代码记录时按导入数据更新。 |
| “仅新增”遇到已存在记录 | 生成错误记录。 |
| 存在错误且未选择部分更新 | 整批回滚。 |

导入更新与普通编辑一样可写入代码以外的输入字段；模板不应包含 ID、创建/更新人、创建/更新时间、逻辑删除和租户等系统字段。

## 列表、详情与快速跳转

### 列表建议口径

当前前端默认列顺序为：代码、名称、类型、描述、是否可用、生效时间、失效时间、备注、创建时间、创建者、最后更新时间、最后更新者、操作。默认筛选为代码、名称、类型、是否可用（默认“可用”）。

培训和日常查询建议优先显示：代码、名称、类型、是否可用、生效/失效时间；描述、备注和审计字段作为补充列，避免主列表被低频字段占据。

### 详情分组与跳转占位

当前通用详情组件未见针对仓库的专用分组或跳转 Tab。后续建议按以下分组补齐，并在测试环境验证实际可跳转入口：

| 分组 | 字段/内容 | 建议跳转 |
| --- | --- | --- |
| 基本信息 | 代码、名称、类型、描述、是否可用 | 库区、库位组、库位 |
| 有效期与备注 | 生效时间、失效时间、备注 | 无 |
| 审计信息 | 创建/更新人、创建/更新时间 | 操作日志（待公共日志能力取证） |
| 下游引用 | 库区、库位、客户月台、物料仓库配置、库存查询 | 各关联列表并预填仓库条件（待页面逐项实现/验证） |

## 关联关系

```
仓库（basic_warehouse）
  ├─ 库区 / 库位组 / 库位
  ├─ 客户月台、物料仓库默认配置等 DBC 配置
  └─ WMS 库存、上架、盘点、调拨等业务的仓储范围
```

> 关联是业务与数据模型层面的依赖，不表示当前每个列表或详情页均已实现快捷跳转。
