# 仓库管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓储主数据维护人员、库房主管、收发与库存管理人员。
> 具体新增、编辑、启停、导入和查询操作见[仓库管理-维护与查询参考](14-仓库管理-维护与查询参考.md)。

## 业务目的与适用范围

仓库是企业管理实物库存的一级地点。收货、上架、发料、出库、库内作业和盘点都需要先明确物料属于哪个仓库，仓库再向下组织库区、库位和现场作业地点。

仓库主数据描述的是地点和管理边界，不直接表示某种物料的实际数量；实际库存应在库存余额和事务中查询。

## 何时需要维护

新建库房、仓储组织调整、启用外部仓/线边仓/隔离仓，或需要在业务中区分库存归属时，应新增或维护仓库资料。

## 仓库如何服务业务

```mermaid
flowchart LR
    A["仓库"] --> B["库区"] --> C["库位"]
    C --> D["收货、上架、发料、出库"]
    D --> E["库存余额、事务与盘点"]
```

维护完成后，仓库应成为库存查询和作业任务的共同地点口径。仓库停用或调整前必须评估现有库存、在途任务和下属库区/库位。

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 身份与名称 | 能否清楚区分不同实体或管理范围的仓库。 | 使用稳定编码和现场可理解名称。 |
| 仓库属性 | 是否为常规、隔离、线边或其它业务用途。 | 属性变化需评估收发与库存选择。 |
| 下属地点 | 是否已建立适用库区、库位和月台。 | 不在没有作业地点的仓库直接开展业务。 |
| 启停/调整 | 是否仍有库存、任务和未完成单据。 | 优先分阶段迁移库存与作业，再停用。 |

## 查询、详情与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 仓库有哪些作业地点 | 库区、库位、月台。 |
| 仓库内有什么库存 | 库存余额、库存事务。 |
| 某任务为何在该仓执行 | 收发任务、来源单据、库位。 |

## 维护与查询边界

仓库的编码、名称、类型和可用状态构成基础维护内容；仓库下的库区、库位和月台应在各自页面维护。当前支持批量导入，因此新仓库上线前应先导入或维护仓库，再依次维护下层地点，避免下游选择器找不到归属。

仓库停用或删除前，不能只看本页状态；应联查库存余额、未完成收发和盘点任务。删除保护、状态变更是否会拦截在途业务尚待测试，正式操作以停用和分阶段迁移为优先策略。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 业务页面无法选择仓库 | 核对仓库状态、权限、下属地点和业务配置。 |
| 仓库要停用但仍有库存 | 先完成调拨、出库或盘点处理，不直接停用。 |
| 同一地点建了多个仓库 | 核对组织边界，避免库存被拆分到重复地点。 |

## 当前限制与待确认事项

- 仓库字段、类型、导入与唯一性规则待继续核验；
- 仓库停用对任务和选择器的实际影响需测试；
- 详情分组、库存/任务跳转和数据权限范围待补截图与实测。
