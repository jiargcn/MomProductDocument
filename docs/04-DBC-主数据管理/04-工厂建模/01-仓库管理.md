# 仓库管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；仓储主数据与库房主管（顺带）。
> 具体新增、编辑、启停、导入和查询操作见[仓库管理-维护与查询参考](14-仓库管理-维护与查询参考.md)。

## 业务目的与适用范围

仓库是企业管理实物库存的一级地点。收货、上架、发料、出库、库内作业和盘点都要先明确物料属于哪个仓库；仓库再向下组织库区、库位和现场作业地点。

读完本页，应能判断：何时必须先建仓库、停用前要清什么，以及下游收货/上架**选不到仓库**时先查本页状态还是继续查库区/库位。仓库描述的是地点和管理边界，不直接表示物料数量——数量在库存余额与事务中查询。

## 何时需要维护

新建库房、仓储组织调整、启用外部仓/线边仓/隔离仓，或需要在业务中区分库存归属时，应新增或维护仓库资料。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 仓库编码与名称 | 一级地点识别；名称应能区分管理范围。 |
| 仓库类型 | 常规/隔离/线边等用途口径；是否自动限制收发路径 ❓ |
| 是否可用与有效期 | 控制是否可供业务引用。 |
| 下属地点计划 | 无库区/库位时，收货与上架往往无法落到可执行地点。 |

## 仓库如何服务业务

```mermaid
flowchart LR
    A["仓库"] --> B["库区"] --> C["库位"]
    C --> D["收货、上架、发料、出库"]
    D --> E["库存余额、事务与盘点"]
```

维护完成后，仓库应成为库存查询和作业任务的共同地点口径。仓库停用或调整前必须评估现有库存、在途任务和下属库区/库位。

!!! example "写实示例：给定配置 → 期望行为"
    **给定：** 新建仓库 `WH-RM`（原料仓），是否可用 = 是；已建库区 `RCV`、库位 `RCV-01` 并挂在该仓下。
    **期望：**

    1. 收货/上架等业务选仓库时，在「仅可选可用仓库」通例下应能选到 `WH-RM`（权限裁剪 ❓）。
    2. 选定该仓后，库区/库位选择器应按级联只出示该仓下属地点（通例见[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md)）。
    3. 若仓库可用但未建库区/库位，可能仍能选到仓，但无法完成落到库位的收货/上架——继续查下级地点。
    4. 停用 `WH-RM` 后，新单原则上不应再选到该仓；过滤时点 ❓。有库存或在途任务时不应直接停用。

## 关键字段业务角色

完整语义与选择器说明见[维护与查询参考](14-仓库管理-维护与查询参考.md)。仓→区→位通例见[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md)。「仅可选可用仓库 / 权限裁剪」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)。仓库本身无下游级联字段（级联在库区/库位/月台页与业务页）。

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- |
| 仓库编码 | 仓储地点的一级业务识别 | 新增必填；日常 Web 编辑通常锁定；唯一性与旁路改码保护 ❓（`GAP-048`） | 被库区/库位/库存引用后勿改码 |
| 名称与描述 | 现场与单据展示 | 名称应能区分管理范围 | 名称含糊导致错选仓库 |
| 仓库类型 | 区分常规、隔离、线边等用途口径 | 字典选择；❓ 是否自动限制收发路径待确认 | 类型变化前评估收发与库存选择 |
| 是否可用 / 有效期 | 控制是否可供业务引用 | 可用状态必填；停用后选择器过滤时点 ❓ | 有库存或在途任务时勿直接停用 |
| 组织/数据权限投影 | 裁剪可见仓库范围 | ❓ 各业务页权限裁剪矩阵待确认 | 选不到仓库时先查权限与状态 |

### 关键维护与变更摘要

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 身份与名称 | 能否清楚区分不同实体或管理范围的仓库。 | 使用稳定编码和现场可理解名称。 |
| 仓库属性 | 是否为常规、隔离、线边或其它业务用途。 | 属性变化需评估收发与库存选择。 |
| 下属地点 | 是否已建立适用库区、库位和月台。 | 不在没有作业地点的仓库直接开展业务。 |
| 启停/调整 | 是否仍有库存、任务和未完成单据。 | 优先分阶段迁移库存与作业，再停用。 |

### 可用性与类型如何起作用

仓库页没有独立策略配置。对下游「能不能选到仓、再选到区/位」主要靠：

| 维护点 | 起作用的方式 | 排障时先看 |
| --- | --- | --- |
| 是否可用 / 有效期 | 停用后原则上不应再被新单选中；过滤时点 ❓ | 仓库是否可用、是否在有效期内 |
| 仓库类型 | 表达用途口径；是否自动限制收发路径 ❓ | 类型是否与现场场景匹配 |
| 下属库区/库位 | 选到仓后仍可能因无下级地点无法落位 | 是否已按仓→区→位建齐 |

现场说「收货/上架选不到仓库」：先查本页可用与权限；能选仓但选不到库区/库位：转到[库区管理](02-库区管理.md)、[库位管理](03-库位管理.md) 并对照级联通例。

### 建议验证点

- 新建可用仓库后，收货/上架选仓是否可见（有无权限裁剪）。
- 有仓无库位时，业务能否完成落到库位的操作（期望：不能闭环）。
- 停用仓库后各选择器是否立即过滤（时点 ❓）。
- 变更仓库类型前后，收发路径是否变化（自动限制 ❓）。

## 维护与查询边界

仓库的编码、名称、类型和可用状态构成基础维护内容；仓库下的库区、库位和月台应在各自页面维护。当前支持批量导入，因此新仓库上线前应先导入或维护仓库，再依次维护下层地点，避免下游选择器找不到归属。

仓库停用或删除前，不能只看本页状态；应联查库存余额、未完成收发和盘点任务。删除保护、状态变更是否会拦截在途业务尚待测试，正式操作以停用和分阶段迁移为优先策略。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 业务页面无法选择仓库 | **排障入口：** ① 本页是否可用、有效期；② 用户权限/组织范围；③ 业务页是否还有任务配置限制。能选仓但选不到区/位 → 查库区/库位归属与级联。 |
| 仓库要停用但仍有库存 | 先完成调拨、出库或盘点处理，不直接停用。 |
| 同一地点建了多个仓库 | 核对组织边界，避免库存被拆分到重复地点。 |

## 当前限制与待确认事项

- 编码唯一性、改码与删除引用保护未闭合（`GAP-048`）；
- 仓库停用对任务和选择器的实际影响需测试；
- 详情实际 Tab、跳转过滤与数据权限范围待补截图与实测。
