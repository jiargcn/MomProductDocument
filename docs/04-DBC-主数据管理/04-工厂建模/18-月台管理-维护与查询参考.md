# 月台管理-维护与查询参考

> 对应业务说明：[月台管理](05-月台管理.md)。

## 新增与编辑

| 维护项 | 新增要求 | 编辑与变更建议 |
| --- | --- | --- |
| 月台编码、名称 | 编码和名称必填，应与现场标识一致。 | 改码或改名时同步检查车辆调度、收发任务和现场标牌。 |
| 类型 | 必填，按收货、发货等实际用途选择。 | 类型变更前评估已有预约或在途交接。 |
| 所属仓库、默认库位 | 均为必填，仓库和库位应已建立且关系正确。 | 默认库位变更前确认在途任务和实际落位规则。 |
| 地址与联系人 | 按调度和现场交接需要填写。 | 联系方式变更后及时通知相关岗位。 |
| 是否可用与有效期 | 可用状态必填。 | 停用前先转移或完成在途收发与预约。 |

## 批量导入规则

模板应包含：编码、名称、城市、地址、类型、仓库、默认库位、联系人、联系电话、是否可用、有效期、备注。

- 不应导入客户月台、车辆、预约单、收发任务或实际库存。
- 编码、名称、类型、仓库、默认库位、可用状态为必填；仓库和默认库位必须预先维护。
- 导入后应抽查默认库位是否属于所填仓库；跨仓默认库位是否被系统拦截需测试确认。

## 查询与详情建议

列表首屏建议展示编码、名称、类型、所属仓库、默认库位、可用状态；常用筛选为仓库、类型、状态。详情建议分为“基本地点、收发与默认库位、联系信息、状态与有效期、审计”，并联查采购收货、销售发运、预约和车辆交接。

## 常见问题

| 现象 | 处理建议 |
| --- | --- |
| 收发任务无法选择月台 | 核对月台类型、状态、仓库归属和任务条件。 |
| 月台默认库位不正确 | 先查在途收发，再受控变更并验证后续任务。 |
| 导入后月台与库位不一致 | 核对仓库和默认库位编码，必要时在测试环境复现。 |
