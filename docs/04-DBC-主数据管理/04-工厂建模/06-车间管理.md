# 车间管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、启停、导入和查询操作见[车间管理-维护与查询参考](19-车间管理-维护与查询参考.md)。

## 业务目的与适用范围

车间用于维护生产现场的组织与地点边界，是生产线、工位、工艺执行和线边物流的上层归属。它帮助计划、生产、仓储和设备相关业务以同一现场口径定位作业范围。

## 何时需要维护

新建生产区域、车间合并/拆分、产线归属调整，或生产和仓储无法按现场地点协同时，应维护车间资料。

## 车间如何服务生产

```mermaid
flowchart LR
    A["车间"] --> B["生产线"] --> C["工位"]
    C --> D["工艺、工单与线边物流"]
```

车间是现场组织资料，不表示具体产能或实际产出；实际计划、工单和生产记录应在相应业务中查询。

## 关键字段业务角色

完整语义见[维护与查询参考](19-车间管理-维护与查询参考.md)。本页模式：**P1 / P6 / P10**。

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- | --- |
| 车间编码 | P6 | 现场组织业务键 | 必填；服务层查重，库约束 ❓（`GAP-053`） | 被产线/工位引用后勿改码 |
| 名称 / 类型 | P6 / P1 | 识别与分类 | 类型字典选择 | 类型与现场用途不符 |
| 是否可用 / 有效期 | P1 / P10 | 可否被产线挂接 | 停用前查产线、工位与在途业务 | 停用导致产线无法归属 |

> 工厂/组织、负责人、父车间等为**历史字段，非当前模型**，勿按旧稿培训（`GAP-053`）。

### 关键维护与变更摘要

| 维护点 | 业务判断 |
| --- | --- |
| 名称与编码 | 能否清楚区分生产现场。 |
| 下属产线与工位 | 是否与实际生产布局一致。 |
| 启停/调整 | 是否仍有在途工单、产线或设备引用。 |

## 查询、详情与联查

可从车间查看下属产线、工位及相关生产任务；发生线边发料或工艺匹配问题时，应联查生产线、工位、工艺路线和物料关系。

## 维护与查询边界

当前车间对象核心是编码、名称、类型、状态与有效期，**不含**工厂/组织、负责人、父车间等历史字段（`GAP-053`）。生产线、工位、人员、设备和工艺关系分别由对应对象维护。

页面支持批量导入。新现场上线建议按车间、生产线、工位的顺序维护，再配置工艺、物料和终端关系。导入与普通保存的校验/状态口径可能不一致，勿混写。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 产线无法归属车间 | 核对车间状态、编码和维护权限。 |
| 车间调整影响在制业务 | 先评估工单、产线和线边库存，再分阶段切换。 |

## 当前限制与待确认事项

- 编码唯一性、删除保护、导入回执待核验（`GAP-053`）；
- 与 MES 组织、设备、工艺和数据采集点的自动关联需跨模块确认。
