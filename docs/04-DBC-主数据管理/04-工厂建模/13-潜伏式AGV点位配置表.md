# 潜伏式 AGV 点位配置

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：自动化/物流配置人员、仓储现场人员、AGV 调度协同人员。
> 具体新增、编辑、导入和查询操作见[潜伏式AGV点位配置-维护与查询参考](23-潜伏式AGV点位配置-维护与查询参考.md)。

## 业务目的与适用范围

潜伏式 AGV 点位配置用于将 AGV 可到达、取货、放货、充电或等待的现场点位与业务场景对应起来，使自动搬运任务知道从哪里取、送到哪里、在哪个点完成交接。它是自动化配置资料，不替代普通库位、工位或库存余额。

## 何时需要维护

AGV 新线路启用、站点迁移、库位/工位布局调整、自动搬运任务失败或现场交接方式变化时，应维护本配置。

## 配置如何服务自动搬运

```mermaid
flowchart LR
    A["库位/工位/现场点位"] --> B["AGV 点位配置"]
    B --> C["自动搬运任务"]
    C --> D["取放货、交接与状态反馈"]
```

配置需要与现场地图、AGV 能力、容器/包装方式和业务任务保持一致。变更前应确认是否存在在途任务，避免车辆到达旧点位或无法完成交接。

## 关键维护与变更

| 维护点 | 业务判断 |
| --- | --- |
| AGV 点位身份 | 是否对应真实、可到达的现场位置。 |
| 业务用途 | 是取货、放货、等待、充电还是其它用途。 |
| 关联地点与容器要求 | 是否与库位/工位、托盘/容器和搬运方式一致。 |
| 启停/迁移 | 是否影响正在执行或已下发的搬运任务。 |

## 查询、详情与联查

发生 AGV 任务异常时，应联查 AGV 点位、关联库位/工位、任务状态和现场实际标签；库存变化仍以业务记录和库存事务为准。

## 维护与查询边界

当前配置将一个“配置点位”与一个“点位代码”建立 AGV 用途和优先级关系，两个点位均从可用点位资料中选择。它是潜伏式 AGV 的基础配置，不表示已完成地图下发、任务派发或车辆状态回传；这些自动化执行结果应在 AGV 模块和现场系统中确认。

页面支持批量导入。应先维护普通点位，再维护 AGV 点位配置；导入前确认点位可达性、业务用途和现场标签一致，不导入车辆、地图、容器或实际任务信息。

## 当前限制与待确认事项

- 点位属性、地图坐标、任务映射、状态回传和错误恢复规则待从 AGV 模块/现场测试确认；
- 与 WMS 收发、库内作业、MES 线边物流的实际调用关系需验证；
- 需补地图、点位配置、任务下发和异常提示截图。
