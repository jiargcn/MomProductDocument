# 库位管理-维护与查询参考

> 对应业务说明：[库位管理](03-库位管理.md)。

## 新增与编辑

| 维护项 | 新增要求 | 编辑与变更建议 |
| --- | --- | --- |
| 库位编码、名称 | 编码和名称必填，应与现场标签一致。 | 改码前评估标签、终端扫描、库存追溯与外部系统映射。 |
| 所属仓库、库区、库位组 | 均为必填，必须先建立上层地点资料。 | 不建议有库存时直接改挂；优先走受控库存迁移。 |
| 外部系统库位口径 | 需填写外部系统识别信息。 | 与外部系统的字段含义和同步结果需专项确认。 |
| 备货优先级、容量和位置属性 | 备货优先级必填；容量、巷道、货架、行列按现场规划维护。 | 变更前核对推荐、盘点和设备/AGV 使用情况。 |
| 是否可用与有效期 | 可用状态必填。 | 停用前清空或迁移库存，处理未完成上架/拣选任务。 |

## 批量导入规则

模板应包含：编码、名称、仓库、库区、库位组、外部系统库位信息、巷道、货架、行列、备货优先级、容量、是否可用、有效期、备注。

- 不应导入实时已用体积、实际库存、批次、包装、审计信息或任务状态。
- 上层地点、库位组、备货优先级和可用状态为关键校验项；应先导入仓库、库区、库位组。
- 容量数值的单位、上限校验和导入模式需通过测试环境确认，避免把空值或文本错误写入地点属性。

## 查询与详情建议

列表首屏建议显示编码、名称、仓库、库区、库位组、备货优先级、可用状态；按仓库/库区/库位组/状态筛选。详情分为“地点层级、现场属性、容量与优先级、状态与有效期、审计”，并增加库存余额、库存事务、上架/拣选任务和标签的快捷联查。

## 常见问题

| 现象 | 处理建议 |
| --- | --- |
| 扫码找不到库位 | 核对编码、标签、可用状态及终端使用的识别码。 |
| 库位有库存不能停用 | 先执行调拨/转库或其他受控作业，不能直接改库位资料。 |
| 导入后库位层级错误 | 回查仓库、库区和库位组编码，核对导入顺序。 |
