# 模具信息管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。

## 业务目的与适用范围

模具信息用于维护生产现场使用的模具身份、类型、状态和适用范围，为工艺、生产执行和模具追溯提供基础资料。当前 DBC 中的模具资料应与 EAM 的设备/工装管理区分：本页描述生产建模所需的基础身份，维护、保养和资产全生命周期应以 EAM 边界为准。

## 何时需要维护

新模具投入、模具类型调整、模具停用、生产异常需追溯所用模具时，应维护或查询本资料。

## 关键维护与变更

| 维护点 | 业务判断 |
| --- | --- |
| 模具身份与类型 | 是否能区分具体模具与其类别。 |
| 适用工艺/产线 | 是否确实可用于目标生产场景。 |
| 状态 | 是否可投入生产，是否需要维修/停用。 |
| 变更影响 | 是否影响在制工单、工艺和质量追溯。 |

## 查询、详情与联查

应支持从模具查询类型、适用工艺/产线和相关生产记录；需要保养、检修、备件或资产信息时跳转 EAM 处理。

## 维护与查询说明

当前可查资料未找到可与本页一一对应的模具维护实现，因此不能输出“哪些字段可编辑、模板如何导入、状态如何流转”等操作结论。现有可用资料表明：模具的资产、寿命、入出库、验收、保养和维修应以 EAM 工装台账及其相关业务为准；本页仅保留生产建模中的对象说明和后续取证站位。

后续补充本页前，应先确认 DBC 菜单是否仍指向可用功能，或是否应将入口重定向到 EAM。确认后再补齐列表字段、详情分组、工艺/产线联查、PDA 查询和变更流程，避免从历史页面名称反推不存在的当前能力。
