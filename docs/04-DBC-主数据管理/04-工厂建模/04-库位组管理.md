# 库位组管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓储主数据维护人员、库存规划人员、库房主管。
> 具体新增、编辑、启停、导入和查询操作见[库位组管理-维护与查询参考](17-库位组管理-维护与查询参考.md)。

## 业务目的与适用范围

库位组用于把一组具有共同用途、区域或作业特征的库位组织起来，帮助库存查询、任务分配、库位推荐或盘点范围按组管理。它是对库位的业务分组，不替代库区或实际库存。

## 何时需要维护

需要按货架区、拣选区、线边区、隔离区或作业班组管理多个库位，或查询和盘点需要快速选择一批库位时，应维护库位组。

## 库位组如何被使用

```mermaid
flowchart LR
    A["库位"] --> B["库位组"]
    B --> C["查询、盘点、任务范围或推荐"]
    C --> D["库存与作业管理"]
```

库位组应反映稳定的管理边界。若只是一次临时作业范围，应在任务中选择范围，不建议反复修改长期库位组。

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 分组目的 | 组是按区域、用途、作业还是设备能力组织。 | 名称应能让使用者理解用途。 |
| 成员库位 | 哪些库位应纳入。 | 确认库位状态、库区和现场实际。 |
| 变更影响 | 是否影响查询、盘点或任务范围。 | 调整前检查未完成任务。 |

## 查询、详情与联查

可从库位组查询成员库位、组内库存和相关盘点/作业任务；需要定位具体实物时仍应下钻到库位、批次/包装和库存状态。

## 维护与查询边界

当前维护对象首先是“库位组与其所属仓库、库区”的基础关系。库位实际归入某个组时，应通过库位资料维护，不在本页用临时分组替代库存或任务范围。

页面支持批量导入，推荐按仓库、库区、库位组、库位的顺序维护。库位组是否直接参与任务推荐、盘点范围或数据权限过滤，尚需实测确认。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 库位未出现在分组中 | 核对成员关系、库位状态和维护权限。 |
| 任务范围过大或过小 | 检查库位组用途和成员是否与实际区域一致。 |
| 改组后影响盘点 | 先确认盘点计划和已下发任务，再进行调整。 |

## 当前限制与待确认事项

- 库位组是否直接参与推荐、盘点和权限过滤需验证；
- 成员维护、导入和删除保护待核验；
- 需补组成员维护、库存查询和盘点引用截图。
