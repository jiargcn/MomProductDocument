# 工位管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、启停、导入和查询操作见[工位管理-维护与查询参考](21-工位管理-维护与查询参考.md)。

## 业务目的与适用范围

工位是生产线中可执行具体工序、领取线边物料或采集现场数据的最小作业地点。它将工艺步骤、人员/设备、物料供给和生产执行定位到现场可识别的位置。

## 何时需要维护

新工位启用、工艺调整、设备迁移、线边物料地点变化或工位无法匹配生产任务时，应维护本资料。

## 工位如何服务执行

```mermaid
flowchart LR
    A["生产线"] --> B["工位"]
    B --> C["工序/设备/线边物料"]
    C --> D["生产执行与数据采集"]
```

工位不是人员账号或设备台账的替代品；它是现场作业位置，相关人员、设备和工艺需通过各自业务对象关联。

## 关键字段业务角色

完整语义见[维护与查询参考](21-工位管理-维护与查询参考.md)。本页模式：**P2 / P4 / P6 / P1**。

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- | --- |
| 工位编码 / 名称 | P6 | 现场位置身份 | 必填；服务查重，库唯一 ❓（`GAP-055`） | 改码影响终端与任务 |
| 车间 / 生产线 | P2 / P4 | 层级归属 | **先选已启用车间，再选该车间下产线** | 跨线改挂影响在途报工 |
| 工序 / 班组 | P2 | 工艺与组织关联资料 | Web 类型必填；导入缺类型/含历史工序列（`GAP-055`） | 勿把关联当强制执行门禁 |
| 地点（库位等） | P2 / P4 | 线边地点资料 | 前端可按库区类型筛库位；服务关联校验多已注释 ❓ | 旁路可能写入不一致归属 |
| 是否可用 | P1 | 可否被任务选用 | 停用前查报工/发料/采集 | 终端找不到工位 |

### 关键维护与变更摘要

| 维护点 | 业务判断 |
| --- | --- |
| 所属产线 | 是否位于正确线体。 |
| 工位身份与现场标签 | 是否可被任务、扫码和人员识别。 |
| 工序/设备/物料关系 | 是否符合实际作业方式。 |
| 停用/调整 | 是否仍有在途任务、设备或数据采集引用。 |

## 查询、详情与联查

可从工位联查生产线、工艺路线、线边物料、设备与生产任务；无法报工或无法发料时，应按来源关系逐项核对。

## 维护与查询边界

工位需要明确车间、生产线、名称和可用状态。前端选择器、普通保存校验与导入模板字段未闭合（`GAP-055`）：培训以**已启用页面字段**为准，不把注释校验或历史列写成当前能力。

页面支持批量导入，应按车间、生产线、工位的顺序进行。

## 当前限制与待确认事项

- 关联存在性、库位校验与导入模板差异见 `GAP-055`；
- 与 MES 工位设备/技能等页是否自动联动待确认。
