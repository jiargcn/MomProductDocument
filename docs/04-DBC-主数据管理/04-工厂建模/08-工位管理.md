# 工位管理

> 当前基线：`dev`（测试环境目标基线），源码与 DDL 取证日期：2026-07-16。
> 页面定位：维护生产现场工位及其车间、生产线、线边库位等归属，是 MES 现场执行、线边端操作、QMS 检验和 ANDON 定位可引用的主数据。本页不直接适用[申请、任务与记录模型](../../02-业务模型/01-申请任务记录模型.md)。

## 当前实现摘要

持久化表为 `basic_workstation`。当前 Web 页面维护代码、名称、类型、车间、生产线、原料库位、成品库位、可用状态、有效期和备注；`teamCode`、`processCode` 虽在表和 VO 中存在，但当前主表单已注释，不能作为当前人工维护能力。

新增专用服务会按工位代码查重；但 DDL 的代码索引不是唯一约束。车间、生产线、原料/成品库位在前端均通过选择器构造，后端普通保存对车间/生产线仅检查空值，关联存在性校验已注释，库位校验亦未接入。详见[产品差距总账](../../15-版本路线图/产品差距总账.md) GAP-055。

## 字段与新增约束

| 技术字段 | 中文名 | DDL 类型/约束 | 当前页面规则 | 当前服务证据 |
| --- | --- | --- | --- | --- |
| `code` | 代码 | `varchar(64)`，非空 | 必填、最多 50 字符、`validateYSK`；编辑锁定。 | 专用新增按代码查重；普通校验要求非空。 |
| `name` | 名称 | `varchar(64)`，可空 | 必填、最多 50 字符。 | 导入显式要求；普通保存未见服务必填。 |
| `type` | 类型 | `varchar(64)`，可空 | `WORK_STATION_TYPE` 字典，前端必选。 | 服务/导入未见必填或字典存在性校验。 |
| `workshopCode` | 车间代码 | `varchar(64)`，非空 | 必填；车间选择器，数据源为车间分页接口，筛选 `available=true`。 | 仅校验非空；查询车间存在性的代码已注释。 |
| `productionLineCode` | 生产线代码 | `varchar(64)`，非空 | 必填；生产线选择器，需先填车间，筛选同 `workshopCode` 且 `available=true`。 | 仅校验非 `null`；查询生产线存在性的代码已注释。 |
| `teamCode` | 班组代码 | `varchar(64)`，可空 | 当前主表单未配置。 | 未见本页新增/编辑校验。 |
| `rawLocationCode` | 原料库位 | `varchar(64)`，可空 | 库位选择器，筛选可用且库区类型来自配置项 `WorkStation-IN`。 | 库位存在性校验已注释。 |
| `fgLocationCode` | 成品库位 | `varchar(64)`，可空 | 库位选择器，筛选可用且库区类型来自配置项 `WorkStation-OUT`。 | 库位存在性校验已注释。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 开关控件，默认 `true`，必填。 | 非空。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)`，可空 | 日期时间选择器；前端提交前要求失效时间晚于生效时间。 | 未见同等服务端区间校验。 |
| `remark` | 备注 | `text`，可空 | 最多 50 字符。 | 无额外校验。 |
| `processCode` | 工序编码 | `varchar(64)`，可空 | 当前工序选择器和列已注释。 | 导入模型仍包含该列，未见关联校验。 |

`description`、审计、删除、扩展属性、并发戳、站点和租户为系统/持久化字段。历史草稿中的 `stationCode`、`stationName`、`deviceCode`、`workerCapacity`、设备强制点检等字段或规则，不是当前 `basic_workstation` 已证实模型，不能作为当前业务字典。

## 编辑、启停与删除

| 动作 | 当前可证实行为 | 使用口径 |
| --- | --- | --- |
| 编辑 | Web 锁定代码、车间和生产线；原料/成品库位、类型、名称、状态、有效期和备注可编辑。服务更新仍接收完整对象，仅重新做空值校验。 | 不应把前端锁定等同于接口不可改；变更归属前应核查 MES、线边端、QMS、ANDON 引用。 |
| 启用/停用 | 列表提供启停操作，默认列表筛选可用记录。 | 停用后将不再进入前端选择器；对存量执行、任务和终端影响待测试。 |
| 删除 | 服务只校验记录存在后删除。 | 被生产执行、工序、质量或异常记录引用的工位不应视为可安全删除。 |

前端按钮权限标识为 `dbc:workstation:create/import/export/update/enable/disable/delete`。接口层鉴权需 RBAC 实测，不以按钮配置下结论。

## 导入规则

`WorkstationImportExcelVo` 当前模板字段顺序为：代码、名称、车间代码、生产线代码、工序、是否可用、生效时间、失效时间、备注。

| 校验/模式 | 当前规则 |
| --- | --- |
| 必填 | 代码、名称、车间代码、生产线代码、是否可用。`processCode` 可空。 |
| 关联字段 | 当前导入校验只检查车间/生产线代码非空；未见车间、生产线、工序实际存在性或层级一致性校验。 |
| 库位字段 | 原料/成品库位在导入类中已注释，不应出现在当前模板。 |
| 既有记录 | 按代码查找；不存在且非覆盖模式时插入，存在且非仅新增模式时更新。 |
| 错误处理 | 有错误且未选择部分更新时整批回滚；否则生成逐行错误文件并允许有效行处理。 |

模板不含类型、班组、原料/成品库位、生产线名称、描述以及系统审计字段。类型在 Web 必填但模板未提供，且导入模型仍包含当前 Web 已注释的工序列，属于应优先统一的模板边界。

## 列表、详情与快速跳转

当前默认列顺序为：代码、名称、类型、车间代码、生产线代码、生产线名称、原料库位、成品库位、是否可用、生效时间、失效时间、备注、审计列、操作。默认筛选为代码、名称、类型、车间代码、生产线代码、生产线名称、是否可用（默认“可用”）。

建议主列表优先显示：代码、名称、类型、车间、生产线、是否可用；库位、有效期与审计列作为补充。

当前采用通用详情组件，页面另配置“库位信息”子表，但该子表提交字段 `subList` 与 `basic_workstation` 的真实持久化字段/服务处理关系尚未证实，不将其写成已实现关系。后续建议：

| 分组 | 内容 | 建议跳转 |
| --- | --- | --- |
| 基本与组织归属 | 代码、名称、类型、车间、生产线、可用状态 | 车间、生产线 |
| 现场物流配置 | 原料库位、成品库位、线边库位关系 | 库位、线边库存查询（待验证） |
| 工艺与执行 | 工序、班组、MES 报工/工单/终端执行 | 工序、生产执行查询（待实现/验证） |
| 质量与异常 | QMS 检验、ANDON 异常、设备/工装引用 | 对应关联查询预填工位条件（待实现/验证） |
| 审计 | 有效期、备注、审计字段 | 操作日志（待公共能力取证） |

## 关联关系

```
车间（basic_workshop）← 工位（basic_workstation）→ 生产线（basic_productionline）
  ├─ 原料/成品库位（basic_location，当前后端关联校验未接入）
  ├─ 工序、班组（字段存在，当前页面维护边界不完整）
  └─ MES、QMS、ANDON、线边端等业务引用（待逐页验证）
```
