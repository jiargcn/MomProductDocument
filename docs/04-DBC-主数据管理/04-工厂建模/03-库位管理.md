# 库位管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；仓储主数据、上架/拣选与盘点人员（顺带）。
> 具体新增、编辑、启停、导入和查询操作见[库位管理-维护与查询参考](16-库位管理-维护与查询参考.md)。

## 业务目的与适用范围

库位是仓储现场最细的可执行存放地点。上架要确认放到哪个库位，拣选/发料要确认从哪个库位取出，库存余额和盘点也要按库位定位实物。

读完本页，应能判断：库位如何挂在仓/区下、级联变更会清空什么，以及收货/上架**选不到或扫不到库位**时先查层级归属、可用状态还是标签/任务配置。本页维护的是地点身份与能力；数量、批次/包装和状态仍在库存业务中查询。

## 使用前准备

新增库位前应先确认：

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 所属仓库与库区 | 库位从属于仓/区；跨仓或跨用途错挂会导致作业跑偏。 |
| 现场编码 / 标签 | 扫码与系统须同一口径。 |
| 用途与可存放范围 | 是否适合存放、拣选、隔离等；容量/混放强制 ❓ |
| 设备 / AGV / 扫码规则 | 需要配合时一并确认，避免只建资料无法作业。 |

## 库位如何进入库存作业

```mermaid
flowchart LR
    A["仓库/库区"] --> B["库位资料与现场标签"]
    B --> C["上架、拣选、调拨、盘点任务"]
    C --> D["库存余额按库位查询"]
```

库位编码应让现场人员和系统使用同一口径。新增、停用或改名时，应先确认是否已有库存、未完成任务或打印标签在使用。

!!! example "写实示例：给定配置 → 期望行为"
    **给定：** 仓库 `WH-RM`、库区 `STG`（存储区）均可用；新建库位 `STG-A-01`，挂在该仓该区，是否可用 = 是，标签与编码一致。
    **期望：**

    1. 收货/上架在已选 `WH-RM` + `STG` 后，库位选择器应能选到 `STG-A-01`；未选仓/区或区不属于该仓时，通常选不到（通例）。
    2. 业务页变更仓库时，已选库区、库位应按通例**清空**后重选；变更库区时清空库位。
    3. 扫描标签 `STG-A-01` 应能命中该库位；编码与标签不一致时扫码失败。
    4. 停用库位后，新上架原则上不应再选到该位（过滤时点 ❓）；仍有库存时不得直接停用，应先受控迁移。
    5. 任务配置「禁止改库位」时，现场级联选择被锁定，以任务配置为准（见收货/上架等业务页）。

## 关键维护与变更

### 关键字段业务角色

库位级联通例见[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md)（先仓再区再组、变更清空下游）。选择器「仅可选当前仓/区下可用对象」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)。本页差异与边界见维护参考及 `GAP-050`。

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护时要警惕什么 |
| --- | --- | --- | --- |
| 库位编码 | 现场与系统共同识别的地点身份 | 新增必填；应与标签一致；唯一性/专用规则 ❓（`GAP-050`） | 改码影响标签、库存与在途任务 |
| 所属仓库 / 库区 / 库位组 | 层级归属（本页级联重点） | **先仓再区再组**；变更仓库清空区/组，变更库区清空组 | 跨仓库挂接导致作业跑偏 |
| 备货优先级 | 推荐优先资料 | 必填；是否进入推荐 ❓ | 勿写成已强制推荐 |
| 是否可用 | 可否被新业务选用 | 停用过滤时点 ❓ | 仍有库存时停用 |
| 外部系统库位口径 | 外部地点映射 | 页面标签与导入列含义易混（`GAP-050`） | 勿按错误列标题对接集成 |

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 所属仓库/库区 | 是否属于正确的现场层级。 | 不跨仓库或跨用途错误挂接。 |
| 地点编码与标签 | 现场扫描能否唯一识别。 | 标签变更前评估终端和库存追溯。 |
| 用途与能力 | 是否适合存放、拣选、隔离或其它作业。 | 与物料、包装和设备要求共同确认。 |
| 启停/迁移 | 是否仍有库存和在途任务。 | 先清空或转移库存，再停用。 |

### 级联与可用性如何起作用

| 维护点 / 通例 | 起作用的方式 | 排障时先看 |
| --- | --- | --- |
| 先仓再区再位 | 选库位前必须有正确仓/区；变更上游清空下游 | 业务页是否已选仓/区；库位是否挂在该区 |
| 是否可用 | 停用后原则上不可再被新业务选用；过滤时点 ❓ | 库位是否停用 |
| 标签与编码一致 | 扫码命中依赖同一识别口径 | 标签、编码、终端识别码是否一致 |
| 任务禁止改库位 | 现场级联被锁定 | 任务所属业务类型/现场控制 |

现场说「上架选不到库位」：① 仓/区是否已选且匹配；② 库位是否属于该区且可用；③ 任务是否禁止改位或扫码不匹配。

### 建议验证点

- 未选仓库/库区时，库位选择器是否为空。
- 换仓/换区后，下游已选值是否被清空并重过滤。
- 库位挂在错误库区时，正确区下是否选不到。
- 扫码标签与编码不一致时是否拦截。
- 停用库位后，收货/上架选位是否立即过滤（时点 ❓）。
- 有库存时改挂仓/区是否被禁止或提示高风险。

## 维护与查询边界

库位需要同时维护仓库、库区、库位组、外部系统识别码和备货优先级等地点口径；容量、巷道、货架、行列等属性服务于现场规划和后续规则。批量建库位前，应先完成仓库、库区和库位组，避免导入形成无归属或错误层级。

本页维护的是地点资料，不允许以直接修改库位资料的方式修正实际库存。库存错放、上架错误或盘点差异应通过受控仓内作业或盘点流程处理。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 收货/上架选不到库位 | **排障入口：** ① 是否已选仓库与库区；② 本页核对库位归属与可用；③ 任务是否禁止改位；④ 扫码是否与标签/编码一致。 |
| 扫描不到库位 | 核对标签、编码、状态和终端网络/权限。 |
| 库位有库存却要停用 | 先调拨或处理库存，不直接停用。 |
| 物料被放错库位 | 保留原因并走受控库内作业，避免直接改余额。 |

## 当前限制与待确认事项

- 通用保存入口与专用库位规则分离，ERP/外部字段口径不一致（`GAP-050`）；
- 容量、混放、物料限制与备货优先级是否进入推荐规则待确认；
- 详情布局、删除保护和扫描校验需测试。
