# 库位管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓储主数据维护人员、上架/拣选人员、库存与盘点人员。
> 具体新增、编辑、启停、导入和查询操作见[库位管理-维护与查询参考](16-库位管理-维护与查询参考.md)。

## 业务目的与适用范围

库位是仓储现场最细的可执行存放地点。上架需要确认物料放到哪个库位，拣选/发料需要确认从哪个库位取出，库存余额和盘点也需要按库位定位实物。

库位资料维护的是“地点能力和身份”；实际库存数量、批次/包装和状态仍应在库存业务中查询。

## 使用前准备

新增库位前应先确认其所属仓库和库区、现场编码/标签、用途、可承载或可存放范围，以及是否需要与设备、AGV 或扫码规则配合。

## 库位如何进入库存作业

```mermaid
flowchart LR
    A["仓库/库区"] --> B["库位资料与现场标签"]
    B --> C["上架、拣选、调拨、盘点任务"]
    C --> D["库存余额按库位查询"]
```

库位编码应让现场人员和系统使用同一口径。新增、停用或改名时，应先确认是否已有库存、未完成任务或打印标签在使用。

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 所属仓库/库区 | 是否属于正确的现场层级。 | 不跨仓库或跨用途错误挂接。 |
| 地点编码与标签 | 现场扫描能否唯一识别。 | 标签变更前评估终端和库存追溯。 |
| 用途与能力 | 是否适合存放、拣选、隔离或其它作业。 | 与物料、包装和设备要求共同确认。 |
| 启停/迁移 | 是否仍有库存和在途任务。 | 先清空或转移库存，再停用。 |

## 查询、详情与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 某物料在哪 | 库存余额、库位、批次/包装和状态。 |
| 某库位存什么 | 库位、库存余额、最近事务。 |
| 为什么不能上架/拣选 | 任务、库位状态、库区用途和物料配置。 |

## 维护与查询边界

库位需要同时维护仓库、库区、库位组、外部系统识别码和备货优先级等地点口径；容量、巷道、货架、行列等属性服务于现场规划和后续规则。批量建库位前，应先完成仓库、库区和库位组，避免导入形成无归属或错误层级。

本页维护的是地点资料，不允许以直接修改库位资料的方式修正实际库存。库存错放、上架错误或盘点差异应通过受控仓内作业或盘点流程处理。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 扫描不到库位 | 核对标签、编码、状态和终端网络/权限。 |
| 库位有库存却要停用 | 先调拨或处理库存，不直接停用。 |
| 物料被放错库位 | 保留原因并走受控库内作业，避免直接改余额。 |

## 当前限制与待确认事项

- 容量、混放、物料限制、库位组和推荐规则待继续核验；
- 详情布局、导入、删除保护和扫描校验需测试；
- 与 AGV、叉车、标签打印的实际集成边界待确认。
