# 库位管理

> 当前基线：`dev`（测试环境目标基线），源码与 DDL 取证日期：2026-07-16。
> 页面定位：维护库存存放的最小空间单元。库位代码是库存余额已确认业务粒度中的位置维度，并被 WMS 上架、发料、盘点、调拨及终端扫码使用；它不是申请、任务或记录对象。

## 当前实现摘要

持久化表为 `basic_location`。页面提供新增、编辑、启停、删除、导入、导出，并通过仓库→库区→库位组的级联选择器维护空间归属。

当前 Controller 继承通用 `WinBaseController`：`/create`、`/update`、`/delete` 分别直接调用通用 `WinBaseService.create/update/delete`。服务类虽另有 `createLocation`、`updateLocation`、`deleteLocation` 和 `importLocationList` 等更完整方法，但其自定义 Controller 路由已注释。因此，只有已被通用钩子调用的导入校验和新增后标签创建可作为当前通用入口的证据；不能把自定义方法里的去重、容量、归属和删除清理规则直接写成 Web 保存的已生效规则。

详见[产品差距总账](../../15-版本路线图/产品差距总账.md) GAP-050。

## 字段与新增约束

| 技术字段 | 中文名/当前页面标签 | DDL 类型/约束 | 当前 Web 规则 |
| --- | --- | --- | --- |
| `code` | 代码 | `varchar(64)`，非空 | 必填、最多 50 字符、`validateYS` 格式校验。 |
| `name` | 名称 | `varchar(64)`，可空 | 页面必填、最多 50 字符；服务 VO 不标注必填。 |
| `warehouseCode` | 仓库代码 | `varchar(64)`，非空 | 必填；仓库分页选择器，筛选 `available=true`。 |
| `areaCode` | 库区代码 | `varchar(64)`，非空 | 必填；库区选择器依赖已填仓库代码，筛选同仓库且 `available=true`。 |
| `locationGroupCode` | 库位组代码 | `varchar(64)`，非空 | 必填；库位组选择器依赖已填库区代码，筛选同库区且 `available=true`。 |
| `erpLocationCode` | 页面称“ERP库位类型” | `varchar(64)`，非空 | 必填；使用 `ERP_LOCATION` 字典。VO 注释称“ERP库位代码”，名称待统一。 |
| `qadLocationCode` | QAD 库位代码 | `varchar(64)`，非空 | 页面和导入均要求；服务自定义创建分支会在空值时回填库位代码，但该分支不是当前通用新增路径。 |
| `aisle` / `shelf` | 巷道 / 货架 | `varchar(250)`，可空 | 最多 50 字符，前端限制小于 DDL 容量。 |
| `locationRow` / `locationColum` | 行 / 列 | `integer`，可空 | 数字控件，最小值 1；`locationColum` 为当前真实拼写。 |
| `pickPriority` | 备货优先级 | `integer`，非空 | 页面必填，数字控件最小值 1。 |
| `maxWeight` / `maxArea` / `maxVolume` | 最大承重 / 面积 / 体积 | `numeric(18,6)`，可空 | 数字控件最小值 1，精度 6。 |
| `standardVolume` | 标准体积 | `numeric(18,2)`，非空，默认 0 | 表单默认 0，最小值 0，精度 2；导入模板未包含。 |
| `available` | 是否可用 | `boolean`，非空，默认 `true` | 开关控件，默认 `true`，页面和 VO 必填。 |
| `activeTime` / `expireTime` | 生效/失效时间 | `timestamp(6)`，可空 | 日期时间选择器；未见时间区间校验。 |
| `remark` | 备注 | `text`，可空 | 最多 50 字符。 |

`description`、`type`、`userGroupCode`、`inventoryVolume`、`usableVolume` 等字段在 DDL、VO 或服务中可见，但当前主页面表单或导入模板并未形成一致的维护入口，不能视为已完整可维护能力。

## 选择器与空间一致性

| 选择器字段 | 数据源与筛选 | 当前保证范围 |
| --- | --- | --- |
| 仓库 | DBC 仓库分页接口；`available=true` | 仅前端选择范围。 |
| 库区 | DBC 库区分页接口；当前表单的 `warehouseCode` + `available=true` | 形成仓库到库区的前端级联。 |
| 库位组 | DBC 库位组分页接口；当前表单的 `areaCode` + `available=true` | 形成库区到库位组的前端级联。 |
| ERP 库位 | 字典 `ERP_LOCATION` | 为字典选择，不代表 ERP 主数据接口校验。 |

服务自定义校验分支还会核验仓库存在、库区属于仓库、库位组属于库区、备货优先级和可用状态，但当前通用新增/更新入口未调用该分支；其接口生效性待测试环境验证。

## 编辑、启停与删除

| 动作 | 当前可证实行为 | 使用口径 |
| --- | --- | --- |
| 编辑 | Web 锁定 `code`；`warehouseCode`、`areaCode`、`locationGroupCode` 仍允许重新选择。通用更新整体写入请求字段。 | 正常页面不能改代码，但可改空间归属；改归属前应先核查库存、预计库存与未完成任务。 |
| 启用/停用 | 列表提供启停动作；多个下游查询按 `available=true` 取库位。 | 停用影响后续可选范围，实际对存量库存及未完成任务的拦截待验证。 |
| 删除 | 通用删除仅检查记录存在后删除；自定义删除方法中才包含标签删除，但未能证明为当前入口路径。 | 有库存/任务/标签关联的库位不得按“可直接删除”培训。 |

前端权限标识为 `dbc:location:create/import/export/update/enable/disable/delete`。后端自定义接口权限注解位于已注释代码中；需要 RBAC 实测，不以按钮标识推断服务端授权。

## 导入规则

导入类 `LocationImportExcelVO` 的字段顺序为：代码、名称、仓库代码、库区代码、库位组代码、ERP库位类型、ERP库位代码（实际映射 `qadLocationCode`）、巷道、货架、行、列、备货优先级、最大承重、最大面积、最大体积、是否可用、生效时间、失效时间、备注。

通用导入会调用本服务覆写的 `validImport`，因此以下规则可视为当前导入路径已接入：

| 校验 | 当前规则 |
| --- | --- |
| 必填 | 代码、名称、仓库、库区、库位组、ERP 值、QAD 库位代码、备货优先级、是否可用。 |
| 空间归属 | 校验库区属于仓库；库位组属于库区。 |
| 选择值 | 校验仓库存在、ERP 值非空、备货优先级与可用状态；库位组重复会转为单行错误。 |
| 既有记录 | 按库位代码查询；追加/覆盖模式依通用规则插入或更新，错误且未选择部分更新时整批回滚。 |

`standardVolume`、`description`、`type`、`userGroupCode`、库存体积/可用体积及审计、租户、逻辑删除、并发字段不在导入模板中。

## 关联关系

```
仓库 → 库区 → 库位组 → 库位（basic_location）
                              ├─ 库存余额与库存事务的位置维度
                              ├─ 上架、发料、盘点、调拨等 WMS 任务执行
                              └─ 条码/标签关联（当前入口与目标 Infra 归属待分层说明）
```
