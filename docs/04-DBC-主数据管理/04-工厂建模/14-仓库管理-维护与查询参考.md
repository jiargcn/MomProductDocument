# 仓库管理-维护与查询参考

> 对应业务说明：[仓库管理](01-仓库管理.md)。

## 新增与编辑

| 维护项 | 新增要求 | 编辑与变更建议 |
| --- | --- | --- |
| 仓库编码 | 必填，使用稳定且现场可识别的编码。 | 编码重复保护和改码影响待测试；上线后不建议随意改码。 |
| 名称、类型与描述 | 名称应清楚表达管理范围；类型按字典选择。 | 类型变化前评估收发、库存和下属地点的使用场景。 |
| 是否可用与有效期 | 可用状态必填；可按启用计划填写有效期。 | 停用前清理库存和在途作业；时间自动控制规则待确认。 |
| 备注 | 记录仓库职责、启停原因或迁移说明。 | 不用备注替代库区、库位和权限配置。 |

## 批量导入规则

模板应包含：编码、名称、类型、描述、是否可用、生效时间、失效时间、备注。

- 不应导入库存数量、库区、库位、创建/修改人等其它对象或系统审计信息。
- 编码和可用状态为必填；类型应使用模板提供的字典值。
- 导入前检查编码重复、名称可识别性和类型口径；导入模式、部分成功和错误回执以测试环境结果为准。

## 查询与详情建议

列表优先展示编码、名称、类型、可用状态、有效期；常用筛选为编码、名称、类型、可用状态。详情建议分为“基本信息、地点层级、状态与有效期、审计与备注”，并增加库区、库位、月台、库存余额和在途任务的联查入口。

## 常见问题

| 现象 | 处理建议 |
| --- | --- |
| 收发页面无法选择仓库 | 检查仓库是否可用、是否已建立下属作业地点，以及当前用户范围。 |
| 准备停用但仍有库存 | 先完成库存迁移、调拨或清理，再停用。 |
| 导入后地点无法挂接 | 核对仓库编码与下层资料使用的编码是否一致。 |
