# 工艺路线

## 概述

工艺路线定义产品制造的工序顺序和作业参数，是 MES 工单展开和工序流转的核心依据。

## 字段说明

### 工艺路线表头

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| routeCode | 工艺编码 | VARCHAR(50) | 必填、唯一 | MES工单展开（工单通过工艺编码引用） | 工艺路线的唯一身份标识，用于引用具体版本的工艺，系统主键 |
| routeName | 工艺名称 | VARCHAR(200) | 必填 | MES显示（工艺名称展示） | 工艺路线的显示名称，通常与适用产品名称一致 |
| materialCode | 适用物料 | VARCHAR(50) | 必填，来源表：物料 | MES工单匹配（工单物料匹配对应工艺） | 工艺路线适用的产品物料编码，同一产品可能有多个版本的工艺路线 |
| version | 版本 | VARCHAR(20) | 必填 | 工艺变更管理（区分不同版本的工艺） | 工艺路线的版本号，用于区分同一产品的不同工艺版本，支持变更历史追溯 |
| effectiveDate | 生效日期 | DATE | 必填 | MES工单展开（生效日期前工单不引用此工艺） | 工艺启用的时间节点，工单展开时根据生效日期匹配对应版本 |
| invalidDate | 失效日期 | DATE | 非必填 | MES工单展开（失效后工单不再引用） | 工艺失效的时间节点，失效后不可被新工单引用，空值表示永久有效 |
| status | 状态 | ENUM | 字典项：草稿/生效/禁用 | MES工单展开（草稿/禁用状态不可被工单引用） | 工艺的审核状态，草稿工艺仅保存不参与运算 |
| remark | 备注 | VARCHAR(500) | 非必填 | | 工艺的补充说明，如变更原因、适用范围说明等 |

### 工艺路线明细（RouteLine）

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| sequence | 工序序号 | INT | 必填 | 工序执行顺序（按序号从小到大执行）、工序流转方向 | 工序在工艺路线中的排列顺序，序号决定工序的执行先后顺序 |
| processCode | 工序编码 | VARCHAR(50) | 必填，来源表：工序 | MES工序执行（工单按工序顺序展开任务） | 工序的唯一标识，引用工序主数据中定义的工序 |
| productionLineCode | 生产线 | VARCHAR(50) | 非必填，来源表：生产线 | MES派工（工序分配到指定产线） | 工序执行所属的生产线，不填则由系统根据产品默认分配 |
| stationCode | 工位 | VARCHAR(50) | 非必填，来源表：工位 | MES报工点（报工时归属的工位） | 工序执行所在的工位，不填则由系统根据产线和工序自动匹配 |
| standardTime | 标准工时 | DECIMAL(18,4) | 非必填 | 产能计算（覆盖工序主数据的默认工时） | 此工序在此工艺路线中的标准工时，不填时使用工序主数据中定义的默认值 |
| isKeyProcess | 关键工序 | BOOLEAN | 默认否 | MES齐套检查（关键工序未完工不可流转到下一工序） | 标识工序是否为关键工序，关键工序未报工完工不允许物料流转到下一工序 |
| isInspectProcess | 检验工序 | BOOLEAN | 默认否 | QMS触发（检验工序报工后自动触发质量检验） | 标识是否为检验工序，检验工序报工完成后自动在QMS生成检验申请单 |
| isTransferProcess | 流转工序 | BOOLEAN | 默认否 | WMS触发（流转工序报工后自动触发库位间物料移转） | 标识是否为流转工序，流转工序报工完成后自动在WMS生成库位间移转单 |

## 字段级业务影响

### isKeyProcess（关键工序）

| 场景 | 对MES的影响 |
|------|-------------|
| 关键工序未完工 | 工单物料锁定，不允许流转到下一工序，系统告警提示关键工序未完成 |
| 关键工序已完工 | 解除锁定，可正常流转到下一工序 |

### isInspectProcess（检验工序）

| 场景 | 对QMS的影响 | 对MES的影响 |
|------|-------------|-------------|
| 开启 | 报工后自动在QMS生成检验申请单，需检验完成才可继续 | 工序完工后流程暂停，等待QMS检验结果回写 |
| 关闭 | 不触发QMS检验 | 工序完工后直接流转，不暂停 |

### isTransferProcess（流转工序）

| 场景 | 对WMS的影响 |
|------|-------------|
| 开启 | 报工完成后自动生成库位间移转单，物料从当前工位移转到下一工序的待领料区 |
| 关闭 | 不自动触发移转，需人工在WMS发起移转单 |

## 关联关系

```
物料（1）←→（N）工艺路线
工艺路线（1）←→（N）工艺路线明细 → 工序
工艺路线明细.productionLineCode → 生产线
工艺路线明细.stationCode → 工位
```

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 字典项 | status（草稿/生效/禁用） |
| 来源表 | materialCode→物料表，processCode→工序表，productionLineCode→生产线表，stationCode→工位表 |
| 联动影响 | 关键工序→报工未完工锁定物料流转；检验工序→报工触发QMS检验；流转工序→报工触发WMS移转 |
| 版本管理 | 同一产品同一版本唯一，不可重复；工单引用时按生效日期匹配版本 |
