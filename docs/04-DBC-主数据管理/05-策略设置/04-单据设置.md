# 单据设置

## 概述

单据设置定义业务单据（采购订单、销售订单、入库单等）的编号规则和流程配置。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| docTypeCode | 单据类型编码 | VARCHAR(50) | 必填、唯一 | 所有业务单据（系统通过此编码匹配单据规则） | 业务单据类型的唯一标识，用于关联具体单据的编号规则，如PO代表采购订单，WI代表入库单，系统主键 |
| docTypeName | 单据类型名称 | VARCHAR(200) | 必填 | 单据显示（显示单据类型名称） | 单据类型的显示名称，如"采购订单"、"销售订单"、"采购入库单"等 |
| prefixRule | 编号前缀规则 | VARCHAR(50) | 必填 | 单据编号生成（前缀部分） | 单据编号的前缀部分，如"PO-"、"WI-"、"SO-"等，前缀通常包含业务含义的缩写，便于识别单据类型 |
| sequenceRule | 流水号规则 | VARCHAR(50) | 必填 | 单据编号生成（流水号格式和位数） | 流水号的生成规则，如"YYYYMMDD4"表示年月日+4位序号（每天从0001开始），流水号保证单据编号在同类型中的唯一性 |
| defaultStatus | 默认状态 | ENUM | 字典项：草稿/已提交/已审批 | 单据新建状态（新建单据的初始状态） | 新建业务单据时的默认状态，草稿状态单据可编辑但不可执行，已提交/已审批状态可直接执行 |
| needApprove | 需要审批 | BOOLEAN | 默认否 | 审批流触发（开启后新建单据进入审批流程） | 标识此类型单据是否需要走审批流程，开启后新建的单据自动进入审批流，审批通过后才可执行 |
| remark | 备注 | VARCHAR(500) | 非必填 | | 单据设置的补充说明，如适用范围、特殊流程说明等 |

## 字段级业务影响

### prefixRule + sequenceRule（编号规则）

| 规则示例 | 生成的单据编号 | 说明 |
|----------|---------------|------|
| PO- + YYYYMMDD4 | PO-202605190001 | 采购订单，每天从0001开始 |
| WI- + SN(10) | WI-0000000001 | 入库单，纯流水号，无日期 |
| SO- + YYYYMM + 6 | SO-202605 + 000001 | 销售订单，按月流水 |

### needApprove（需要审批）

| 值 | 对单据流程的影响 |
|----|-----------------|
| 否 | 新建单据后直接可执行（状态=已审批）或编辑后直接提交（状态=已提交） |
| 是 | 新建单据后进入审批流，审批通过后状态变为已审批，才可执行 |

### defaultStatus（默认状态）

| 状态 | 对业务的影响 |
|------|-------------|
| **草稿** | 新建单据为草稿状态，可任意编辑，需提交后才可流转 |
| **已提交** | 新建单据直接为已提交状态（需审批），等待审批 |
| **已审批** | 新建单据直接为已审批，可直接执行（无需审批的单据通常用此状态） |

## 关联关系

```
单据设置 → 业务单据（采购订单/入库单/出库单/调拨单）
单据设置 → 审批流配置（needApprove=true时触发）
```

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 字典项 | defaultStatus（草稿/已提交/已审批） |
| 联动影响 | needApprove开启→新建单据自动进审批流；defaultStatus决定新建单据的初始状态，影响单据的流转起点 |
| 唯一性 | 同一docTypeCode唯一，不可重复定义同一单据类型的规则 |
