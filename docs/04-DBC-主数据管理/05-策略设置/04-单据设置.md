> 具体新增、编辑、导入和查询操作见[单据设置-维护与查询参考](10-单据设置-维护与查询参考.md)。

## 这项配置解决什么问题

单据设置为某一类业务单据维护识别口径和编号组成方式，包括单据类型、适用业务类型、编号前缀、日期段、流水长度、分隔符和重置周期。它使申请、任务、记录等业务对象能够使用一致且可辨识的单据号；它不替代业务类型本身，也不直接创建业务单据。

读完本页，测试能据此设计编号唯一性与重置边界的验证场景，实施能说清改哪项编号组成会让单据号发生什么变化，运维能据此判断取号失败或号段冲突时该查[业务类型](03-业务类型.md)还是单据设置。

## 一笔典型配置业务

**场景：** 为采购收货申请启用新的编号规则 `PORECV-yyyyMMdd-####`，按日重置流水（配置关系图见下文「它与业务类型和单据的关系」）。

1. **触发**：业务批准单据类型、适用业务类型代码、编号口径（前缀、日期格式、流水位数、重置周期）。
2. **处理**：新增一条单据设置并置为可用；确认对应[业务类型](03-业务类型.md)已可用，且类型上的号段关联指向本规则（若该场景使用类型侧关联）。
3. **结果**：目标业务新建申请时取得符合规则的新单号；同日内连续开单流水递增；跨日后按重置周期重新起号（边界行为须实测，见「建议验证点」）。
4. **关键分支**：
   - 业务类型填了不存在的值 → 可能静默绑到错误策略（当前存在性校验不足，`GAP-061`）；
   - 直接改在用规则的前缀/流水 → 已发出单据、打印件、外部对账可能对不上；
   - 停用规则 → 新单可能无法取号（失败或回退表现 ❓）。

!!! example "写实示例（给定配置 → 期望行为）"
    **给定：** 单据类型=收货申请；业务类型=已存在的采购收货类型代码；前缀=`PR`；日期=`yyyyMMdd`；流水=4 位；按日重置；可用=是。
    **期望：** 当日连续新建两笔申请，单号形如 `PR202607230001`、`PR202607230002`（分隔符以实际配置为准）。
    **对照：** 若单号前缀不对，先查本条编号组成；若完全取不到号，查可用状态/有效期，再查业务类型侧号段是否启用及单据开关。

## 使用前准备

1. 目标[业务类型](03-业务类型.md)已建立且可用（代码以实际配置为准；页面侧可能无法阻止填无效业务类型）。
2. 单据类型与真实业务对象一致（申请 / 任务 / 记录等）。
3. 编号口径已批准：前缀、日期格式、流水长度、分隔符、重置周期；并估算一个重置周期内的峰值量，避免流水位不够。
4. 若生产中已有在用规则：优先**新建 → 切换引用 → 再停用旧规则**，避免直接覆盖。
5. 导入批量建立前，确认导入入口与模板口径（入口不一致风险见 `GAP-061`）。

## 业务逻辑要点

| 要点 | 说明 |
| --- | --- |
| 主对象 | 一条单据设置 = 一类单据在某业务类型下的取号规则。 |
| 取号时机 | 作用于**新生成**的单据号；已生成单据通常保留原号。 |
| 与业务类型 | 共同决定「哪类业务用哪条规则」；业务类型还管范围/自动/现场，本页不管，两者须成对验证。 |
| 唯一性与并发 | 服务层对配置代码查重；并发生成、跨天重置、流水为 0 等边界 ❓。 |
| 外部影响 | 单号常被报表、打印件、第三方报文引用——改在用规则等于改外部契约。 |

## 关键字段业务角色

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- |
| 代码 / 名称 | 编号规则身份 | 代码查重仅服务层（`GAP-061`） | 在用规则勿直接改码 |
| 单据类型 | 规则适用对象类 | 必填 | 与真实业务对象不一致 |
| 业务类型 | 关联策略场景 | 当前可为自由文本，存在性校验不足（`GAP-061`） | 填了不存在的业务类型 |
| 编号组成 | 前缀/日期/流水/重置 | 整体影响唯一性与可读性；日期格式服务端校验 ❓ | 改在用规则导致号段混乱 |
| 是否可用 / 有效期 | 新业务能否取得规则 | 导入可用必填口径不一致 ❓ | 停用影响补单/新单 |

完整语义见[维护与查询参考](10-单据设置-维护与查询参考.md)。

## 配置如何起作用

单据设置本身不创建业务单据，但它的编号规则会顺着[业务类型](03-业务类型.md)一路影响到具体单据：

- **对申请/任务/记录：** 一条单据设置与某个业务类型绑定后，凡是该业务类型创建的申请/任务/记录都会按这条规则取号；未绑定或绑定到不存在的业务类型（`GAP-061`）会导致取号失败或绑错策略。
- **对库存方向：** 单据设置不直接决定库存方向，但编号异常（重复号、断号）会让库存事务难以按单追溯，间接影响库存问题的排查效率。
- **对编号与打印：** 前缀、日期格式、流水长度、分隔符、重置周期共同决定单据号是否唯一、可读；单据号又是打印模板取号的依据，规则一变，新生成单据的号段与外观随之改变，但通常不影响已打印的历史单据。

因此改单据设置前，应同时核对其绑定的[业务类型](03-业务类型.md)是否仍然匹配、是否停用；两者须成对验证，详见下文「它与业务类型和单据的关系」。

## 建议验证点

变更单据设置前，建议在测试环境走通一次完整链路：

- **创建：** 用绑定的业务类型发起一笔申请/任务，确认取号成功且规则符合预期（前缀、日期段、流水长度）。
- **执行：** 连续生成多笔单据，确认同一重置周期内编号不重复；跨重置边界（如跨天、跨月）时编号行为符合预期。
- **库存/编号：** 核对该单据编号能否在下游库存事务、打印件和外部接口中正确关联；核对打印模板是否正确取号。
- **撤销/异常：** 停用该单据设置后，确认新单是否按预期失败或回退，在途单据与历史打印是否不受影响（`GAP-061`）。

## 什么时候需要维护

| 业务事件 | 应做什么 | 维护前要确认 |
| --- | --- | --- |
| 新业务需要启用一种单据 | 建立单据设置并与适用业务类型对应。 | 单据类型、业务范围和编号口径已批准。 |
| 编号规则需要调整 | 先评估已生成单据、接口报文和打印件的影响。 | 是否会产生重复号、识别歧义或外部系统不兼容。 |
| 某类单据停止使用 | 优先停用配置并保留历史查询依据。 | 在途业务、补单和历史打印是否仍需使用。 |
| 多种规则需要集中建立 | 使用导入模板分批处理并回看每行结果。 | 业务类型和编号规范均已准备完毕；导入入口边界见 `GAP-061`。 |

## 维护时最重要的判断

| 需要判断什么 | 业务含义 | 建议做法 |
| --- | --- | --- |
| 单据类型与业务类型是否匹配 | 共同决定这条编号规则在哪类业务中使用。 | 先由业务负责人确认场景，再配置。 |
| 编号组成是否可长期识别 | 前缀、日期、流水和重置周期共同影响唯一性与可读性。 | 用预计峰值量模拟一个重置周期内的编号数量。 |
| 是否可以修改正在使用的规则 | 已使用编号可能被报表、打印件或第三方系统引用。 | 对生产中的规则优先新建、切换、再停用旧规则。 |
| 可用状态是否应关闭 | 关闭会影响新业务取得配置。 | 先确认在途单据、补单和历史重打需求。 |

## 做完影响什么

- **新开单据**：单号结构与流水节奏按新规则生成。
- **已生成单据**：通常保留原号；报表与外部系统仍按旧号检索。
- **打印与对接**：依赖单号识别的标签、接口报文需同步评估。
- **不直接影响**：物料/库位可选范围、自动提交、现场改量改位——仍由[业务类型](03-业务类型.md)等策略决定。
