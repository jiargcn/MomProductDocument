> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、导入和查询操作见[单据设置-维护与查询参考](10-单据设置-维护与查询参考.md)。
> 共享分流语义见[单据类型、业务类型与单据配置](../../02-业务模型/05-单据类型、业务类型与单据配置.md)。

## 这项配置解决什么问题

单据设置为一类业务单据规定识别口径与编号组成（单据类型、适用业务类型、前缀、日期段、流水、分隔符、重置周期）。它让申请/任务/记录取得一致可辨的单据号；不替代[业务类型](03-业务类型.md)，也不直接创建业务单据。

读完本页，应能说清改哪项组成会改变新单号，以及取号失败时先查本页还是业务类型。

## 一笔典型配置（含准备）

**场景：** 为采购收货申请启用 `PR` + `yyyyMMdd` + 4 位流水、按日重置。

**准备：** 目标[业务类型](03-业务类型.md)已可用；单据类型与真实对象一致；编号口径已批准并估算峰值；生产中已有在用规则时优先新建→切换→停用旧规则；导入前确认模板口径（`GAP-061`）。

1. **触发**：批准单据类型、业务类型代码与编号组成。
2. **处理**：新增并置为可用；确认业务类型侧号段关联指向本规则（若使用类型侧关联）。
3. **结果**：新建申请取得符合规则的新单号；同周期内流水递增；跨重置边界行为须实测。
4. **关键分支**：业务类型填不存在值 → 可能绑错（`GAP-061`）；直接改在用规则 → 外部对账可能对不上；停用 → 新单可能无法取号 ❓。

!!! example "写实示例（给定配置 → 期望行为）"
    **给定：** 单据类型=收货申请；业务类型=已存在代码；前缀=`PR`；日期=`yyyyMMdd`；流水=4 位；按日重置；可用=是。  
    **期望：** 当日连续两笔申请形如 `PR202607230001`、`PR202607230002`（分隔符以实际为准）。  
    **对照：** 前缀不对先查本条组成；完全取不到号先查可用/有效期，再查业务类型号段与单据开关。

```mermaid
flowchart LR
    A["业务类型"] --> B["单据设置"]
    B --> C["申请、任务或记录"]
    C --> D["查询、打印与外部对接"]
```

取号作用于**新生成**单据；已生成单据通常保留原号。单号常被报表、打印、第三方引用——改在用规则等于改外部契约。

## 关键字段业务角色

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- |
| 代码 / 名称 | 编号规则身份 | 代码查重仅服务层（`GAP-061`） | 在用规则勿直接改码 |
| 单据类型 | 规则适用对象类 | 必填 | 与真实业务对象不一致 |
| 业务类型 | 关联策略场景 | 当前可为自由文本，存在性校验不足（`GAP-061`） | 填了不存在的业务类型 |
| 编号组成 | 前缀/日期/流水/重置 | 影响唯一性与可读性；日期格式服务端校验 ❓ | 改在用规则导致号段混乱 |
| 是否可用 / 有效期 | 新业务能否取得规则 | 导入可用必填口径不一致 ❓ | 停用影响补单/新单 |

完整语义见[维护与查询参考](10-单据设置-维护与查询参考.md)。

## 建议验证点

- 用绑定业务类型开单，确认取号成功且组成符合预期。
- 同重置周期内连续多笔不重复；跨天/跨月边界符合预期。
- 单号能在库存事务、打印件、外部接口中关联。
- 停用后新单失败或回退符合预期；在途与历史打印不受影响（`GAP-061`）。

## 什么时候维护

| 业务事件 | 应做什么 | 维护前要确认 |
| --- | --- | --- |
| 新业务启用一种单据 | 建立单据设置并对应业务类型 | 单据类型、范围、编号口径已批准 |
| 编号规则需调整 | 先评估已发单据、报文、打印件 | 重复号、识别歧义、外部不兼容风险 |
| 某类单据停用 | 优先停用并保留历史查询依据 | 在途、补单、历史重打需求 |
| 批量建立 | 导入分批并回看每行结果 | 业务类型与规范已齐；入口见 `GAP-061` |

| 需要判断什么 | 建议做法 |
| --- | --- |
| 单据类型与业务类型是否匹配 | 先由业务负责人确认场景再配 |
| 编号组成是否可长期识别 | 用峰值量模拟一个重置周期内的编号数量 |
| 是否可改正在使用的规则 | 生产中优先新建、切换、再停用旧规则 |
| 是否关闭可用 | 先确认在途、补单与历史重打 |

## 改完影响什么与排障

- **新开单据**：单号结构与流水按新规则生成。
- **已生成单据**：通常保留原号。
- **打印与对接**：依赖单号的标签/报文需评估。
- **不直接影响**：物料/库位范围、自动提交、现场改量改位——仍由[业务类型](03-业务类型.md)等决定。

| 想回答的问题 | 先查 | 再联查 |
| --- | --- | --- |
| 某类单据按什么规则编号 | 单据类型、业务类型、编号组成 | [业务类型](03-业务类型.md)、业务页实际单号 |
| 无法创建或编号异常 | 可用状态、有效期、业务类型文本 | 业务类型、单据开关、规则、报错 |
| 某编号属于什么业务 | 前缀、日期段、单据类型 | 申请/任务/记录详情 |

操作与验收细节见[维护与查询参考](10-单据设置-维护与查询参考.md)。

## 当前边界与待确认事项

- 业务类型存在性、在用改码/删除保护、导入入口不一致（`GAP-061`）。
- 编号变更后既有单据处理、并发生成、跨系统唯一性待测。
- 有效期与流水重置的生效时点需补样例。
