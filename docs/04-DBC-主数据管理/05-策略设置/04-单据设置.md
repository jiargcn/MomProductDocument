# 单据设置

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 共享分流语义见[单据类型、业务类型与单据配置](../../02-业务模型/05-单据类型、业务类型与单据配置.md)。

本页回答：单据设置如何决定「某类单据按什么规则取号」；改前缀、日期段、流水或重置周期后，新开单的号码会怎样变。读完应能配一条可验证的编号规则，并知道它与[业务类型](03-业务类型.md)如何配合、配错时怎么查。

单据设置是**编号规则档案**：它让申请、任务、记录等对象使用一致且可辨识的单据号。它**不**创建业务单据，也**不**替代业务类型上的范围、自动处理与现场约束。

## 这项配置解决什么问题

现场与系统对接都需要能读懂、能检索、尽量不重复的单据号。单据设置把「单据类型 + 适用业务类型 + 前缀 / 日期段 / 流水长度 / 分隔符 / 重置周期」收成一条规则，供目标业务取号。

常见误用：

- 只改单据设置，却期望改变「能不能选库位、会不会自动提交」——那些在[业务类型](03-业务类型.md)；
- 把单据设置当成业务分类字典——它解决的是**怎么编号**，不是业务场景本身。

## 一笔典型配置业务

**场景：** 为采购收货申请启用新的编号规则 `PORECV-yyyyMMdd-####`，按日重置流水。

1. **触发**：业务批准单据类型、适用业务类型代码、编号口径（前缀、日期格式、流水位数、重置周期）。
2. **处理**：新增一条单据设置并置为可用；确认对应[业务类型](03-业务类型.md)已可用，且类型上的号段关联指向本规则（若该场景使用类型侧关联）。
3. **结果**：目标业务新建申请时取得符合规则的新单号；同日内连续开单流水递增；跨日后按重置周期重新起号（边界行为须实测，见验证点）。
4. **关键分支**：
   - 业务类型填了不存在的值 → 可能静默绑到错误策略（当前存在性校验不足，`GAP-061`）；
   - 直接改在用规则的前缀/流水 → 已发出单据、打印件、外部对账可能对不上；
   - 停用规则 → 新单可能无法取号（失败或回退表现 ❓）。

## 使用前准备

1. 目标[业务类型](03-业务类型.md)已建立且可用（代码以实际配置为准；页面侧可能无法阻止填无效业务类型）。
2. 单据类型与真实业务对象一致（申请 / 任务 / 记录等）。
3. 编号口径已批准：前缀、日期格式、流水长度、分隔符、重置周期；并估算一个重置周期内的峰值量，避免流水位不够。
4. 若生产中已有在用规则：优先**新建 → 切换引用 → 再停用旧规则**，避免直接覆盖。
5. 导入批量建立前，确认导入入口与模板口径（入口不一致风险见 `GAP-061`）。

## 业务逻辑要点

| 要点 | 说明 |
| --- | --- |
| 主对象 | 一条单据设置 = 一类单据在某业务类型下的取号规则。 |
| 取号时机 | 作用于**新生成**的单据号；已生成单据通常保留原号。 |
| 与业务类型 | 共同决定「哪类业务用哪条规则」；业务类型还管范围/自动/现场，本页不管。 |
| 唯一性与并发 | 服务层对配置代码查重；并发生成、跨天重置、流水为 0 等边界 ❓。 |
| 外部影响 | 单号常被报表、打印件、第三方报文引用——改在用规则等于改外部契约。 |

## 配置如何起作用

| 你改了什么 | 下游通常会发生什么 | 证实程度 |
| --- | --- | --- |
| 前缀 / 日期格式 / 流水位数 / 分隔符 | 新单据号的可读结构变化 | 编号组成配置存在；日期格式服务端校验深度 ❓ |
| 重置周期 / 当前序号 | 流水起算与回零节奏变化 | 配置项存在；跨周期与并发生成边界 ❓ |
| 单据类型 | 规则归属到哪一类业务对象 | 必填；须与真实对象一致 |
| 业务类型（关联文本） | 与哪条场景策略绑在一起 | **未证实**选择器/后端强校验（`GAP-061`） |
| 是否可用 / 有效期 | 新业务能否取得该规则 | 停用后新单取号应失败或回退 ❓；旧单查询一般仍按历史号 |
| 代码 / 名称 | 规则身份与检索 | 在用改码保护未见闭合（`GAP-061`） |

**实施口诀：** 业务类型先可用，再配单据设置；生产规则只增不改（新建切换）；每次改号段都用目标业务连续开两单以上验证。

## 建议验证点

1. **新建取号**：配置可用后，目标业务开一单，单号前缀、日期段、流水位数符合规则。
2. **连续不重复**：同一重置周期内连续开两单以上，流水递增且不重复。
3. **重置边界**：跨日/跨月（按所配周期）后再开一单，流水是否按约定重置（须在测试环境做，勿默认）。
4. **停用**：停用后新开单是否无法取号或明确报错；历史单号查询与重打是否仍可用。
5. **业务类型绑错**：故意填不存在的业务类型代码，观察能否保存及开单表现（用于暴露 `GAP-061`，勿当正规流程）。
6. **切换策略**：新建规则并切换引用后，仅新单走新规则；旧单号不变。
7. **导入**（若启用）：导入成功后仍按上面 1～2 抽查每种单据类型至少一笔，不能只看「导入成功」。

完整验收清单见[维护与查询参考](10-单据设置-维护与查询参考.md)。

## 关键字段业务角色

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- |
| 代码 / 名称 | 编号规则身份 | 代码查重仅服务层（`GAP-061`） | 在用规则勿直接改码 |
| 单据类型 | 规则适用对象类 | 必填 | 与真实业务对象不一致 |
| 业务类型 | 关联策略场景 | 当前可为自由文本，存在性校验不足（`GAP-061`） | 填了不存在的业务类型 |
| 编号组成 | 前缀/日期/流水/重置 | 整体影响唯一性与可读性；日期格式服务端校验 ❓ | 改在用规则导致号段混乱 |
| 是否可用 / 有效期 | 新业务能否取得规则 | 导入可用必填口径不一致 ❓ | 停用影响补单/新单 |

完整语义见[维护与查询参考](10-单据设置-维护与查询参考.md)。

## 什么时候需要维护

| 业务事件 | 应做什么 | 维护前要确认 |
| --- | --- | --- |
| 新业务需要启用一种单据 | 建立单据设置并与适用业务类型对应 | 单据类型、业务范围和编号口径已批准 |
| 编号规则需要调整 | 优先新建规则并切换，再停用旧规则 | 是否会产生重复号、识别歧义或外部不兼容 |
| 某类单据停止使用 | 优先停用并保留历史查询依据 | 在途业务、补单和历史打印是否仍需使用 |
| 多种规则需要集中建立 | 使用导入模板分批处理并回看每行结果 | 业务类型真实存在；注意导入入口边界（`GAP-061`） |

## 维护时最重要的判断

| 需要判断什么 | 业务含义 | 建议做法 |
| --- | --- | --- |
| 单据类型与业务类型是否匹配 | 共同决定这条编号规则在哪类业务中使用。 | 先由业务负责人确认场景，再配置。 |
| 编号组成是否可长期识别 | 前缀、日期、流水和重置周期共同影响唯一性与可读性。 | 用预计峰值量模拟一个重置周期内的编号数量。 |
| 是否可以修改正在使用的规则 | 已使用编号可能被报表、打印件或第三方系统引用。 | 对生产中的规则优先新建、切换、再停用旧规则。 |
| 可用状态是否应关闭 | 关闭会影响新业务取得配置。 | 先确认在途单据、补单和历史重打需求。 |

## 做完影响什么

- **新开单据**：单号结构与流水节奏按新规则生成。
- **已生成单据**：通常保留原号；报表与外部系统仍按旧号检索。
- **打印与对接**：依赖单号识别的标签、接口报文需同步评估。
- **不直接影响**：物料/库位可选范围、自动提交、现场改量改位——仍由[业务类型](03-业务类型.md)等策略决定。

## 异常与查询入口

| 想回答的问题 | 建议先查什么 | 再联查什么 |
| --- | --- | --- |
| 某类单据按什么规则编号 | 单据类型、业务类型、编号组成 | 对应业务页实际生成的单据号 |
| 业务无法创建或编号异常 | 可用状态、有效期、业务类型文本 | [业务类型](03-业务类型.md)、单据开关、规则管理及报错信息 |
| 某编号属于什么业务 | 前缀、日期段、单据类型 | 申请/任务/记录详情 |

## 当前边界（不打断主线）

- 业务类型存在性、在用改码/删除保护、导入入口不一致：`GAP-061`。
- 编号变更后的既有单据处理、并发生成、跨系统唯一性：待测试环境验证。
- 有效期与流水重置的具体生效时点：需补充样例。
