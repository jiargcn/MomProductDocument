# 班次班组

## 概述

班次班组定义生产现场的人员组织结构，用于 MES 报工归属和 EAM 设备排班。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| shiftCode | 班次编码 | VARCHAR(50) | 必填、唯一 | MES排班（排班时选择班次）、考勤统计（按班次汇总工时） | 班次的唯一身份标识，用于标识具体班次（如早班/中班/晚班），系统主键 |
| shiftName | 班次名称 | VARCHAR(200) | 必填 | MES显示（显示班次名称） | 班次的显示名称，如"早班 06:00-14:00" |
| startTime | 开始时间 | TIME | 必填 | MES考勤统计（上班刷卡时间比对）、报工时间归属（报工记录归属哪个班次按时间判断） | 班次的上班时间，用于判断工人上下刷卡是否在正常时间范围内，报工时间会自动归属到对应班次 |
| endTime | 结束时间 | TIME | 必填 | MES考勤统计（下班刷卡时间比对）、工时计算（班次时长=结束-开始） | 班次的下班时间，与开始时间共同决定班次时长，用于工时统计 |
| teamCode | 班组编码 | VARCHAR(50) | 必填、唯一 | MES派工（派工时选择班组）、报工归属（报工记录归属班组） | 班组的唯一身份标识，用于标识具体班组（如"装配A组"），系统主键 |
| teamName | 班组名称 | VARCHAR(200) | 必填 | MES显示（显示班组名称）、报表（按班组汇总产量） | 班组的显示名称，如"装配A组"、"检测B组" |
| workshopCode | 所属车间 | VARCHAR(50) | 必填，来源表：车间 | 组织归属（班组归属哪个车间）、MES派工范围（只允许派工到本车间的班组） | 班组所属车间的编码，班组只能在所属车间内进行派工 |
| teamLeader | 班组长 | VARCHAR(50) | 非必填，来源表：用户 | 职责归属（班组异常问题找班组长）、MES审批流（班组级审批路由到班组长） | 班组长的用户账号，用于班组级异常问题联络和审批路由 |
| teamSize | 人数 | INT | 非必填 | MES派工（判断班组是否有空余人力）、派工饱和度统计 | 班组的标准人数，用于派工时判断班组人力是否充足，实际派工不可超过班组人数 |
| status | 状态 | ENUM | 字典项：启用/禁用 | MES排班（禁用班组不可排班）、报工（禁用班组的报工记录仍可查） | 班组的启用/禁用状态，禁用后不可在新建派工中被选择 |

## 字段级业务影响

### startTime / endTime（班次时间）

| 场景 | 对MES的影响 |
|------|-------------|
| 报工时间在班次时间范围内 | 报工记录归属到该班次 |
| 报工时间在班次时间范围外 | 报工记录归属到跨班的实际时段，可能不计入任何班次或计入加班 |
| 跨天班次（如22:00-06:00） | 跨天00:00后，报工记录归属于次日班次 |

### teamSize（人数限制）

| 场景 | 对MES派工的影响 |
|------|-----------------|
| 当前派工人数 < teamSize | 可继续向此班组派工 |
| 当前派工人数 >= teamSize | 提示班组已满员，但仍可强制派工（允许超员配置） |

## 关联关系

```
车间（1）←→（N）班组
班组 → MES报工记录
班组 → MES上下工记录
班组 → EAM设备点检记录
```

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 字典项 | status（启用/禁用） |
| 来源表 | workshopCode→车间表，teamLeader→用户表 |
| 联动影响 | teamSize配合当前已派工人数校验，超员时告警；班次时间用于报工时间自动归属班次 |
