# 单据设置-维护与查询参考

> 本页是[单据设置](04-单据设置.md)的操作参考。编号规则属于高风险配置；正式切换前必须在测试环境完成业务样例验证。

## 新增与编辑约束

| 信息组 | 应维护的信息 | 维护限制 |
| --- | --- | --- |
| 识别信息 | 代码、名称、描述。 | 代码应稳定且能识别配置用途；重复检查规则待补充测试。 |
| 适用范围 | 单据类型、业务类型。 | 两者须与真实业务场景一致；业务类型应先建立并处于可用状态。 |
| 编号组成 | 前缀、日期格式、流水长度、分隔符、重置周期。 | 应整体评审，不能只改其中一项；需评估最大业务量与外部接口格式。 |
| 生效控制 | 生效/失效时间、可用状态、备注。 | 正在使用的规则优先新建后切换，避免直接覆盖历史口径。 |

建议在详情页按“基本信息、适用范围、编号规则、生效控制、变更记录”分组；当前分组样式和编辑锁定规则需补充截图验证。

## 导入规则

| 模板应包含 | 不应由模板随意填充 | 导入前校验 |
| --- | --- | --- |
| 代码、名称、描述、单据类型、业务类型、前缀、日期格式、流水长度、分隔符、重置周期、可用状态、生效/失效时间、备注。 | 生成过的实际单据号、历史流水进度、维护人和审计信息。 | 业务类型存在且可用；编号组成完整；同一适用范围不存在未评估的冲突；日期格式和流水长度能够满足编号容量。 |

导入后应抽查每种单据类型至少一笔业务，核对申请、任务或记录的编号表现；不能仅以“导入成功”作为验收结论。

## 验收清单

- [ ] 新建配置后，目标业务能取得正确的单据号。
- [ ] 同一重置周期内连续生成的编号不重复。
- [ ] 到达重置边界时的编号行为符合已批准规则。
- [ ] 停用配置不会错误影响历史查询和已生成单据。
- [ ] 修改或导入失败时，能取得可定位的错误信息。
