# 策略设置

## 这一组业务解决什么问题

策略设置用于维护影响业务单据、任务和规则执行方式的受控配置，例如班次、班组、业务类型、单据设置、单据开关和规则管理。它们不是普通主数据：一次变更可能影响多类业务的创建、编号、可用范围或处理方式。

## 建议学习与操作顺序

| 顺序 | 页面/业务对象 | 先解决什么 | 与下一步怎样衔接 |
| --- | --- | --- |
| 1 | 班次、班组 | 建立现场作业的班次与人员组织口径。 | 支持生产、仓储和现场任务的组织表达。 |
| 2 | 业务类型 | 区分不同业务场景的共同分类和处理口径。 | 是单据设置、开关和规则引用的重要基础。 |
| 3 | 单据设置、单据开关 | 维护单据运行和启停相关配置。 | 影响业务页面的可用入口和单据行为。 |
| 4 | 规则管理 | 维护受控的业务判断规则。 | 仅在已明确的业务范围内配置和验证。 |

## 关键业务对象与关系

```mermaid
flowchart LR
    A["班次/班组"] --> D["现场业务组织"]
    B["业务类型"] --> C["单据设置、单据开关、规则"]
    C --> D
    D --> E["申请、任务、记录等业务执行"]
```

图中表达配置影响业务的方向，不表示所有配置均已接入每一类业务；实际影响范围必须在对应页面和测试环境中确认。

## 常见问题与相关分组

当业务页面的单据类型、状态入口或处理方式与预期不一致时，先确认业务类型、单据设置、开关和规则的适用范围；不要在不了解影响范围时直接修改生产/仓储正在使用的配置。
