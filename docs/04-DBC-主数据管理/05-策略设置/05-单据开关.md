# 单据开关

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。

## 这项配置解决什么问题

单据开关用于维护某项受控单据能力的启用或关闭状态。它适合处理明确、可回退的业务入口控制，不应用来替代权限配置、业务类型配置或单据编号规则。

当前可确认的信息只覆盖开关代码、说明、设定值和可用状态；每一个开关究竟影响哪些业务页面、是否即时生效，仍需在测试环境中逐项验证。因此，任何生产环境的开关变更都应按高风险配置处理。

## 何时可以维护

| 场景 | 可以做什么 | 不应直接做什么 |
| --- | --- | --- |
| 新能力准备上线 | 建立或启用经过验证的开关。 | 未确认作用范围就直接打开。 |
| 某能力需要临时限制 | 按已批准的应急方案关闭，并记录原因和恢复条件。 | 把开关当作清理历史数据的手段。 |
| 能力已下线 | 停用开关并保留变更记录。 | 直接删除仍可能被历史配置引用的记录。 |
| 用户无权访问某页面 | 先检查权限和角色。 | 仅通过单据开关解决个人访问问题。 |

## 维护与变更原则

```mermaid
flowchart LR
    A["提出启停需求"] --> B["确认开关作用范围"]
    B --> C["测试环境验证"]
    C --> D["受控变更"]
    D --> E["业务结果与恢复验证"]
```

- 开关代码应稳定、可追溯；说明应写清业务目的、影响对象和恢复条件。
- 设定值与“是否可用”需要同时检查：前者表达当前选择，后者决定该配置是否仍应被使用。
- 没有明确的回退路径、验证人员和验证结果时，不应变更生产中的开关。

## 维护与查询说明

| 维护事项 | 当前可确认做法 | 需要补充验证 |
| --- | --- | --- |
| 新增 | 维护开关代码、业务说明、设定值和可用状态。 | 代码重复校验与各开关的合法取值。 |
| 编辑 | 优先修改说明或经验证的设定值。 | 已生效业务是否实时受影响、是否允许编辑代码。 |
| 启停 | 停用前确认没有待处理变更或依赖。 | 停用与设定值之间的优先级。 |
| 导入 | 当前已配置导入模板，但导入校验和更新方式尚未验证。 | 应先用少量样例确认模板字段、重复处理和错误回执。 |
| 查询 | 优先显示代码、说明、设定值、可用状态和最近变更信息。 | 详情页分组、审计记录和受影响业务的快捷跳转。 |

## 快速排查与联查

| 现象 | 优先检查 | 后续联查 |
| --- | --- | --- |
| 某项单据能力不可用 | 开关的设定值和可用状态。 | 业务类型、单据设置、角色权限和业务页面提示。 |
| 变更后结果不符合预期 | 作用范围和生效时点是否已验证。 | 对应业务的申请、任务、记录及日志。 |
| 需要恢复能力 | 原设定值、变更原因和回退方案。 | 测试用例与相关业务的恢复结果。 |
