# 班组管理

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、导入和查询操作见[班组管理-维护与查询参考](08-班组管理-维护与查询参考.md)。

## 这项配置解决什么问题

班组管理维护现场协作组织，将班组与车间、生产线、负责人、班组类别和有效状态关联起来。它用于表达“哪一组人负责哪个现场范围”，不等同于人员主数据、考勤排班或实际工单派工记录。

## 关键字段业务角色

完整语义见[维护与查询参考](08-班组管理-维护与查询参考.md)。本页模式：**P2 / P4 / P6 / P1**。

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- | --- |
| 班组编码 / 名称 | P6 | 班组身份 | 编码必填 | 改码影响工位/记录引用 |
| 车间 / 生产线 | P2 / P4 | 现场范围 | 导入两者必填；产线应属车间 ❓ | 错配导致责任范围错误 |
| 负责人 | P2 | 责任人资料 | 存在性/选择限制 ❓ | 勿当权限主体替代 |
| 班组类别 | P1 | 分类 | 导入必填 | 分类与现场不符 |
| 成员 | P2 | 人员清单 | **不在当前导入模板** | 勿用本页替代人事维护 |
| 是否可用 | P1 | 可否被选用 | 停用保护 ❓ | 在用班组被停 |

## 维护与变更原则

班组新增前应先确认车间、生产线和负责人信息。现场组织调整时，应先评估正在执行的生产、报工和线边作业；不要通过直接改写既有班组关系来覆盖历史责任。

```mermaid
flowchart LR
    A["车间与生产线"] --> B["班组与负责人"]
    B --> C["现场生产和协作口径"]
    C --> D["按班组查询与分析"]
```

## 查询与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 某班组负责哪个现场范围 | 车间、生产线、班组负责人。 |
| 某工位的班组口径是否正确 | 工位、生产线、班组资料。 |
| 组织调整影响哪些业务 | 生产记录、报工、任务和班组分析。 |

## 当前边界

- 页面支持批量导入，需先维护车间和生产线。
- 成员字段没有形成当前导入维护链路；人员清单和排班不应通过本页替代维护。
- 负责人有效性、班组停用和历史业务的影响待测试确认。
