# 班次管理

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 具体新增、编辑、导入和查询操作见[班次管理-维护与查询参考](07-班次管理-维护与查询参考.md)。

## 这项配置解决什么问题

班次管理维护现场作业的时间口径，包括班次起止时间、是否跨天、休息安排、可用状态和有效期。它为生产、仓储和现场记录提供统一的“当前属于哪个班次”的基础条件，不等同于排班、人员考勤或工单执行记录。

## 关键字段业务角色

完整语义见[维护与查询参考](07-班次管理-维护与查询参考.md)。

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- |
| 班次编码 / 名称 | 班次身份 | 编码必填；唯一性 ❓ | 改码影响已引用记录理解 |
| 开始 / 结束时间 | 时间窗 | 必填；与跨天共同解释 | 时间错导致归属班次错误 |
| 是否跨天 | 跨日班次口径 | 必填；合法性校验 ❓ | 跨天与休息重叠未闭合 |
| 休息安排 | 休息时段 | 导入时休息起止必填 | 多项休息行组织 ❓ |
| 是否可用 / 有效期 | 可否被选用 | 停用引用保护 ❓ | 修改在用班次未评估影响 |

## 维护与变更原则

新建或调整班次前，先确认现场作息、跨天规则、休息安排以及已使用该班次的业务范围。跨天、开始/结束时间和休息时间必须共同校验；没有完成影响评估时，不应直接修改正在使用的班次。

```mermaid
flowchart LR
    A["现场作息口径"] --> B["班次与休息安排"]
    B --> C["班组、生产与仓储记录"]
    C --> D["按班次查询与分析"]
```

## 查询与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 某时间属于哪个班次 | 班次起止、跨天标识和休息安排。 |
| 班组为何无法选择班次 | 班次状态、有效期及班组关联规则。 |
| 班次变更影响哪些数据 | 班组、生产记录、仓储记录和报表口径。 |

## 当前边界

- 页面支持批量导入；导入班次及休息安排前应使用测试数据验证跨天和时间范围。
- 当前未确认班次变更是否会自动重算既有业务记录或拦截在途业务。
- 班次详情分组、班组/业务记录的快捷跳转需后续补充截图验证。
