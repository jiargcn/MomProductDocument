# 业务类型

> 取证状态：已完成 DDL、DO/VO、服务、导入 VO 与 Web 设置页首轮校正；申请/任务/记录的实际状态流转、扩展规则和第三方接口仍需测试环境验证。基线：`dev` 分支、测试环境目标版本，2026-07-16。

## 业务定位

业务类型是单据执行的**策略档案**，不是单纯的库存事务分类字典。一个业务类型以 `code` 标识，定义某类业务单据可处理的物料、库存状态和库区范围，以及出入库事务类型、在途库存、自动流转、现场可修改项、流水号和打印模板。

WMS、MES、SCP 等单据在创建或执行时按业务类型读取策略；申请、任务、记录三层的通用职责与库存挂接见[申请、任务与记录模型](../../02-业务模型/01-申请任务记录模型.md)。本页只说明“每类单据如何选择其策略”，不重复定义三层模型。

| 项目 | 当前实现 |
| --- | --- |
| 持久化对象 | `document_businesstype` |
| 业务标识 | `code`，`varchar(64)`、非空；服务层按代码查重，DDL 当前仅有普通索引。 |
| 适用对象 | WMS 等业务单据的范围校验、库存事务、自动动作与打印/流水号配置。 |
| 可用性 | `available=true` 的记录才可由按代码获取策略的服务返回。 |
| 当前配置入口 | DBC 业务类型列表；行级“设置”弹窗配置业务明细、物料/入库/出库属性、申请/任务/规则、流水号、打印模板。 |
| 当前打印归属 | 页面当前经 WMS 标签模板 API 选择模板；产品目标归属为 INFRA 平台能力，迁移前应按“当前入口、目标归属”分别说明。 |

## 数据模型与字段分组

`document_businesstype` 的核心字段均以当前 DDL/VO 为准。逗号分隔的范围字段最大长度为 `varchar(512)`；代码、名称、分组、在途库位和打印模板编码最大长度为 `varchar(64)`。

| 分组 | 字段 | 说明 |
| --- | --- | --- |
| 标识与启用 | `code`、`name`、`grouping`、`available` | 代码、名称、业务分组和是否可用。 |
| 物料适用范围 | `itemTypes`、`itemStatuses`、`itemCategories`、`eqLevels` | 限定可处理的物料类型、物料状态、物品种类和质量等级。 |
| 入库策略 | `inAreaTypes`、`inAreaCodes`、`inInventoryStatuses`、`inTransactionType`、`cancelInTransactionType` | 约束入库侧库区、库存状态和正向/撤销事务类型。 |
| 出库策略 | `outAreaTypes`、`outAreaCodes`、`outInventoryStatuses`、`outTransactionType`、`cancelOutTransactionType` | 约束出库侧库区、库存状态和正向/撤销事务类型。 |
| 在途与自动流转 | `useOnTheWay`、`onTheWayLocation`、`autoSubmit`、`autoAgree`、`autoHandle`、`directCreateRecord` | 是否使用在途库位，以及申请、任务、记录层的自动化策略。 |
| 现场执行边界 | `allowModifyLocation`、`allowModifyQty`、`allowModifyBatch`、`allowModifyPackingNumber`、`allowSmallerQty`、`allowBiggerQty`、`allowRepeatScanning`、`allowScanTargetLocation` | 控制任务/终端执行时允许修改或扫描的范围；具体状态前置条件另在业务单据页取证。 |
| 单据标识与输出 | `requestNumberId`、`jobNumberId`、`recordNumberId`、`requestPrintTemplateCode`、`jobPrintTemplateCode`、`recordPrintTemplateCode` | 分别配置申请、任务、记录的流水号与打印模板。 |
| 生命周期与补充 | `activeTime`、`expireTime`、`remark`、`description`、`extraProperties`、`concurrencyStamp` | 生失效时间、说明、扩展属性与乐观锁。`extraProperties` 为文本列，扩展规则应另行核验。 |

## 新增与选择器约束

### 当前可证实的保存校验

| 项目 | 当前规则 | 证据层级 |
| --- | --- | --- |
| 代码 | 必填；服务层按 `code` 查询并阻止重复。 | VO、`BusinesstypeServiceImpl` |
| 名称 | 必填；服务层检查非空，不按名称查重。 | VO、服务 |
| 物料类型、物料状态 | 必填；服务层检查非空。 | VO、服务、Web 表单 |
| 是否使用在途库、是否可用 | 必填。 | VO、服务、DDL |
| 分组 | VO 与 DDL 均要求非空，但服务层未见同等显式校验。 | VO、DDL、服务 |
| 出/入库事务类型 | VO 标为必填说明，但服务创建/更新校验未调用对应的非空方法；不可把注释或字段说明当成已生效校验。 | VO、服务 |
| 差异库区类型 | 更新时，除 `InventoryDiscrepancy` 外，服务会从入/出库区类型中剔除 `DIFFERENT`。 | 服务 |

### 选择器与数据源

| 配置项 | 页面输入方式 | 数据源及筛选条件 |
| --- | --- | --- |
| 物料类型、物料状态、物品种类、质量等级 | 多选字典。 | 对应 DBC 字典；保存为逗号分隔编码。 |
| 入/出库区类型、入/出库库存状态 | 多选字典。 | 库区类型、库存状态字典；保存为逗号分隔编码。 |
| 入/出库库区范围 | 搜索选择或手工输入后失焦校验。 | 库区列表，`available=true`，且库区 `type` 必须属于已选择的入/出库区类型；多值以逗号保存。 |
| 在途库位 | 搜索选择或失焦校验。 | 库位列表，`available=true`、代码匹配；当前页面未见库区/仓库的额外联动约束。 |
| 打印模板 | 搜索选择或按代码/名称回填。 | 当前调用 WMS 标签模板选项接口；分别回填申请、任务、记录模板编码。 |
| 流水号 | 设置弹窗维护。 | 当前使用 System 流水号配置组件；具体可选范围和引用保护待专项取证。 |

## 编辑与执行限制

### 配置编辑

Web 主列表当前只展示代码、名称、分组、入库库存事务、出库库存事务和操作列；行级“设置”入口要求前端权限标识 `dbc:businesstype:update`。设置弹窗按以下九个分组组织，而非普通详情抽屉：

1. 业务明细
2. 物料属性
3. 入库属性
4. 出库属性
5. 申请设置
6. 任务设置
7. 规则设置
8. 流水号设置
9. 打印模板设置

当前服务更新允许接收策略字段；未发现“被单据引用后禁止改码、禁用或删除”的统一引用校验。代码的服务层查重并不等于数据库级保护，且普通新增/修改、导入、通用 Controller 路由是否完全同一路径，需以测试环境进一步确认。

### 业务执行

业务类型本身不直接执行收货、发料或盘点；它把可修改库位/数量/批次/包装、允许部分或超量完成、重复扫描和目标库位扫描等规则传递给各业务的申请、任务、记录与终端入口。实际“什么状态能做什么动作”必须由对应业务页的按钮、服务状态机和终端页面共同取证，不能仅根据本页的开关推断。

## 导入规则

### 当前模板事实

当前 `BusinesstypeExcelVO` 会导出/导入代码、名称、分组、物料与库区/库存状态范围、正向及撤销事务类型、在途策略、自动化与执行开关、流水号、打印模板、生失效时间、备注、审计列和是否可用。范围字段依赖相应字典转换；代码、名称、物料类型、物料状态、是否使用在途库为模板必填项。

服务导入会复用代码、名称、物料类型、物料状态、在途与可用状态等校验，但当前可见实现未见对库区、库位、流水号、打印模板及所有事务类型的完整服务端存在性校验。因此，导入成功不应直接视为所有引用已闭合。

### 建议的培训/人工维护模板

建议把导入拆成“基础策略建档”和“高级策略配置”两阶段，降低一次导入造成大范围单据行为变化的风险。

| 模板 | 建议包含 | 不建议人工导入 | 导入前校验 |
| --- | --- | --- | --- |
| 基础策略建档 | `code`、`name`、`grouping`、物料类型/状态、入/出库范围、入/出库事务类型、`useOnTheWay`、`onTheWayLocation`、`available`、备注 | 创建/更新人、创建/更新时间、乐观锁、扩展属性 | 代码唯一、字典值合法、库区可用且类型匹配、在途库位可用。 |
| 高级策略配置 | 自动流转、现场执行开关、流水号、打印模板、撤销事务类型 | 业务明细与扩展规则中的特定业务代码分支，除非已完成端到端验证 | 目标业务类型已存在；流水号/模板/事务类型存在且可用；由业务负责人复核影响。 |

> 当前模板仍包含审计字段，且更新者列的 Excel 标题也写为“创建者”。在模板治理完成前，应以导出的当前模板为系统事实、以上建议为目标模板，不可混用。

## 业务关系与示例占位

```mermaid
flowchart LR
  A[业务单据选择业务类型 code] --> B[读取物料、库存、库区与在途策略]
  B --> C[创建申请 / 任务 / 记录]
  C --> D[生成正向或撤销库存事务]
  C --> E[按策略生成流水号和打印输出]
  E -.当前入口.-> F[WMS 标签模板]
  F -.目标归属.-> G[INFRA 平台能力]
```
