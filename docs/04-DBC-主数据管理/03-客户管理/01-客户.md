# 客户

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：销售/交付协同人员、主数据维护人员、仓库发运人员及需要查询客户信息的业务人员。

## 业务目的与适用范围

客户主数据用于统一识别产品或服务的交付对象。销售订单、发货计划、销售出库、客户物料和交付月台等业务应引用同一客户资料，以确保交付对象、联系信息和业务协同口径一致。

本页维护客户主体；客户物料解决客户侧物料匹配，客户月台解决现场交接地点，两者均应在建立客户后按需要继续维护。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 客户代码、名称和简称 | 用于系统识别和日常协同。 |
| 联系信息与地址 | 支持订单、发运和异常沟通。 |
| 客户类型、结算或适用属性 | 区分客户业务口径；具体字段以页面为准。 |
| 是否可用及生效范围 | 防止已停止合作或未完成建档的客户被继续使用。 |

## 一项客户如何进入交付业务

```mermaid
flowchart LR
    A["确认客户主体与编码"] --> B["新增客户主数据"]
    B --> C["补充联系与业务属性"]
    C --> D["维护客户物料/客户月台"]
    D --> E["订单、发运与销售出库引用"]
    E --> F["按需停用或变更"]
```

客户资料建立后，仍需根据交付场景维护客户物料、收货地点或月台等关系。不要在销售出库单据中用临时文本替代已建立的客户主数据。

## 关键字段业务角色

下表只列影响客户主体识别与可用性的关键项；客户侧选料/包装见[客户物料](03-客户物料.md)。写法约定见[页面数据字典规范](../../02-业务模型/04-页面数据字典规范.md)。下游「仅可选可用客户」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)；停用过滤时点本页仍标 `❓`。

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- |
| 客户代码 | 主体业务识别号 | 新增必填、查重；更新限制 ❓（`GAP-046`） | 勿重复建档；已被订单/发运引用后慎改 |
| 名称 / 简称 | 正式与日常识别 | 新增均必填；查重 | 简称应销售/仓库可共用 |
| 类型 / 币种等 | 业务属性分类 | 字典或页面属性 | 不确定时先确认销售口径 |
| 是否可用 / 有效期 | 生命周期 | 停用后选择器过滤 ❓ | 停用前查在途订单、客户物料/月台 |

## 关键维护与变更

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 客户身份 | 代码、名称和简称是否属于同一真实主体。 | 代码应稳定，避免重复建档。见上表 P6。 |
| 联系与地址 | 是否满足订单、交付和异常沟通需要。 | 变更前评估在途订单和发运。 |
| 客户属性 | 是否影响销售、交付、结算或协同。 | 不确定时先确认业务口径。 |
| 启停/删除 | 是否仍被订单、发货或客户物料引用。 | 优先停用，避免直接删除已使用主体（`GAP-046`）。 |

### 字段说明总表

### 本页范围说明

| 操作 | 需要核对什么 | 当前限制或建议 |
| --- | --- | --- |
| 新增客户 | 代码、名称、简称和可用状态；必要的地址、联系人和业务属性。 | 代码、名称、简称在页面中均为必填，单项最长 50 个字符；已有相同客户时应维护原资料。 |
| 修改客户 | 名称、地址、联系人、币种、税率、类型或有效期是否影响在途交付。 | 修改前先确认订单、发运和客户物料是否会引用旧口径。代码或简称是否允许修改需用测试环境验证（`GAP-046`）。 |
| 停用或删除 | 是否仍存在在途订单、发运、客户物料或客户月台。 | 优先停用；删除保护和停用后的选择器过滤待测试确认。 |
| 批量导入 | 代码、名称、简称、可用状态和本次导入模式。 | 代码、简称、可用状态不可为空；名称、简称重复会形成错误。其它导入列以当前模板为准；创建与导入最低口径差异见 `GAP-046`。 |
建议列表优先显示客户代码、名称、简称、类型、可用状态和生效期；地址、联系人、币种、税率与审计信息可放在详情。常用筛选为客户代码、名称、类型和状态。

## 查询、详情与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 某客户有哪些交付资料 | 客户、客户物料、客户月台。 |
| 某订单/发货对应谁 | 销售订单、发货计划、销售出库记录。 |
| 为何无法选择客户 | 客户状态、业务类型、客户物料或交付地点。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 客户代码重复 | 先查既有资料，确认是否应维护原客户。 |
| 客户无法出现在交付页面 | 核对可用状态、业务配置和客户物料/地点资料。 |
| 需要变更名称或地址 | 先评估在途订单和发运，避免交接信息不一致。 |
