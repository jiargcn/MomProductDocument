## 当前事实卡（BATCH-01 首轮取证）

> 取证范围：`dev` 基线的 `basic_customerdock` DDL、Customerdock DO/VO/Service、导入类和 Web 页面。历史表中 `platformCode`、`platformName`、`defaultLocation` 等不是当前真实技术字段名。

### 真实字段、选择器与校验

| 主题 | 当前事实 |
| --- | --- |
| 真实对象 | `basic_customerdock`，主键 `id`；核心字段为 `code`、`customerCode`、`defaultLocationCode`、`warehouseCode`。 |
| DDL 约束 | `code`、`customer_code` 为非空；`default_location_code` 在 DDL 可空，但创建/更新 VO 标为必填。所有业务列长度为 `varchar(64)`，时间为 `timestamp(6)`，备注为 `text`。 |
| 客户选择器 | Web 从客户分页接口选择客户，筛选 `available=true`，回填 `customerCode`。 |
| 仓库/库位选择器 | 仓库来自仓库分页选择器；默认库位选择器筛选可用库位并要求先选择仓库，再以 `warehouseCode` 过滤。 |
| 后端校验 | 创建/更新校验月台代码唯一、客户代码非空、默认库位存在且可用；默认库位所属库区类型仅允许 `CUST` 或 `THIRD_PARTY`。 |
| 可用状态 | 创建服务在校验后强制写为 `true`；可用字段虽在 VO 中必填，创建表单规则与实际保存边界仍需测试。 |

### 编辑、导入与查询规则

- Web 编辑锁定月台代码和客户代码；后端更新仍会执行代码唯一与客户代码非空校验，但尚未确认是否把关联键回填为原值，接口绕过页面的变更边界待测试。
- 导入模板列为代码、客户代码、名称、描述、城市、地址、联系人姓名、联系人电话、默认库位、生效时间、失效时间、备注；仓库代码和可用状态在导入类中被注释，不属于当前模板。
- 导入服务会校验客户代码、月台代码及默认库位；默认库位的库区类型限制与普通创建一致。模式 2 会额外校验代码规则，模式的完整产品语义仍待导入组件验证。
- 当前列表展示代码、客户代码、名称、地址、联系人、仓库、默认库位、状态、周期和审计字段；客户代码与名称为主要查询入口。详情仍使用通用组件，建议分为“月台身份与客户、地址与联系人、仓库/默认库位、状态与周期、审计与销售单据跳转”。

### 当前文档校正与差距

- 当前业务键是月台代码 `code`，而非历史草稿的 `platformCode`；默认库位真实字段为 `defaultLocationCode`。
- DDL 仅有普通代码、客户代码索引，未见客户+月台或代码的唯一约束；客户存在性校验也仅验证代码非空。创建强制可用、导入忽略仓库字段、前后端关联键锁定边界未闭合，已登记为[产品差距总账](../../15-版本路线图/03-问题与待确认/产品差距总账.md)中的 GAP-047。

## 概述

客户月台记录客户收货地点的月台信息，用于发货计划中的物流匹配和到货预排。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| platformCode | 代码 | VARCHAR(50) | 必填 | 发货计划匹配（确定具体月台地址） | 月台在客户范围内的唯一编码，系统主键之一 |
| customerCode | 客户代码 | VARCHAR(50) | 必填 | 发货计划匹配（确定归属哪个客户） | 客户月台所属的客户编码，关联到客户主数据 |
| platformName | 名称 | VARCHAR(200) | 必填 | 物流调度显示（显示月台名称便于识别） | 月台的显示名称，如"1号月台-东门" |
| description | 描述 | VARCHAR(500) | 非必填 | 月台说明 | 月台的描述信息 |
| city | 城市 | VARCHAR(50) | 非必填 | 物流跟踪（城市定位） | 收货城市 |
| address | 地址 | VARCHAR(500) | 必填 | 发货单据打印（打印发货地址）、物流跟踪（司机导航地址） | 月台的详细收货地址 |
| contactPerson | 联系人姓名 | VARCHAR(100) | 非必填 | 物流跟踪联络（到货前电话确认） | 月台的收货联络人姓名 |
| contactPhone | 联系人电话 | VARCHAR(50) | 非必填 | 物流跟踪联络（电话通知） | 收货联络人的电话号码 |
| warehouseCode | 仓库 | VARCHAR(50) | 非必填 | （待补充） | 关联仓库 |
| defaultLocation | 默认库位 | VARCHAR(50) | 非必填 | （待补充） | 默认库位 |
| effectiveDate | 生效时间 | DATE | 非必填 | （待补充） | 生效时间 |
| invalidDate | 失效时间 | DATE | 非必填 | （待补充） | 失效时间 |
| remark | 备注 | VARCHAR(500) | 非必填 | | 月台的补充说明 |
