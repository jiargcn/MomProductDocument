# 客户月台

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：交付协同人员、发运人员、主数据维护人员。

## 业务目的与适用范围

客户月台用于维护客户现场的收货、卸货或交接地点，使发运和交付业务能明确“送给哪个客户、送到客户的哪个交接位置”。它是客户资料的补充，不替代本企业内部的仓库月台。

## 何时需要维护

客户新增交付地点、客户现场月台调整、发运频繁送错地点，或客户订单需要按地点区分交付时，应维护本资料。

## 月台如何进入交付流程

```mermaid
flowchart LR
    A["客户主数据"] --> B["客户月台"]
    B --> C["订单/发货计划选择交付地点"]
    C --> D["销售出库与运输交接"]
```

维护时应确认月台归属的客户、地址/地点描述和可用状态。客户有多个地点时，应让发运人员能清楚区分，避免只靠相似名称判断。

## 关键维护与变更

### 关键字段业务角色

「可用客户 / 先仓后位选默认库位」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)、[库位与仓储级联惯例](../../02-业务模型/13-库位与仓储级联惯例.md)。本页为**客户侧交付地点**，勿与企业内部月台混用。

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- |
| 月台代码 | 客户交付地点身份 | 必填；≤50 字符；服务查重、库约束 ❓（`GAP-047`） | 改码影响在途发运 |
| 归属客户 | 月台所属客户 | 仅可选**可用客户**；编辑锁定期望 | 客户存在性校验可能偏弱 |
| 仓库 / 默认库位 | 内部对接地点 | 先选仓库再按仓库+可用过滤库位 | 导入可能忽略仓库/可用字段（`GAP-047`） |
| 是否可用 | 可否被发运选用 | 创建常强制可用 | 有在途单时勿直接停用 |

| 维护点 | 业务判断 | 使用建议 |
| --- | --- | --- |
| 归属客户 | 月台是否属于当前客户。 | 先建立并核对客户主体。 |
| 地点标识 | 名称、地址或说明能否区分多个交付地点。 | 使用客户现场认可的交接称呼。 |
| 启停/替换 | 旧地点是否仍有在途发运。 | 先处理在途订单，再停用旧月台。 |

## 维护与查询说明

客户月台属于单文档对象：新增、编辑、导入、查询与字段语义在本页合并说明。

### 字段业务语义摘要

| 操作 | 需要核对什么 | 当前限制或建议 |
| --- | --- | --- |
| 新增月台 | 月台代码、归属客户、名称/地址、可用状态，以及需要时的仓库与默认库位。 | 月台代码页面必填且最长 50 个字符；客户选择器仅列出可用客户。 |
| 设置默认库位 | 所属仓库、可用库位与客户交接方案是否一致。 | 选择默认库位前需先选仓库；选择器按仓库和可用状态筛选库位。 |
| 修改或停用 | 是否有待发订单、发运计划或运输交接正在使用该地点。 | 先核对在途业务；代码和客户归属的修改限制待测试确认。 |
| 批量导入 | 月台代码、客户、默认库位、可用状态和地址资料。 | 导入会校验月台代码、客户代码与默认库位；错误回执和更新模式待补测试样例。 |

建议列表优先显示月台代码、名称、归属客户、城市/地址、默认库位、可用状态和有效期；联系人、描述和审计信息放入详情。常用筛选为客户、月台代码、名称和状态。

## 查询、详情与联查

| 查询目标 | 建议联查 |
| --- | --- |
| 某客户有哪些交付地点 | 客户、客户月台。 |
| 某次发运送到哪里 | 发货计划/销售出库、客户月台。 |
| 为什么地点无法选择 | 客户归属、月台状态和订单/发运页面条件。 |

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 月台归属错误 | 暂停用于新发运，核对客户、地址和在途业务后更正。 |
| 客户有多个相似地点 | 补充可识别名称和地址说明，避免用简称混淆。 |
| 已停用地点仍被使用 | 核对发运页面过滤规则和在途订单。 |
