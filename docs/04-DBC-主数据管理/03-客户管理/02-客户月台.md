# 客户月台

## 概述

客户月台记录客户收货地点的月台信息，用于发货计划中的物流匹配和到货预排。

## 字段说明

| 字段名 | 中文名 | 类型 | 约束 | 影响业务 | 备注 |
|--------|--------|------|------|----------|------|
| platformCode | 代码 | VARCHAR(50) | 必填 | 发货计划匹配（确定具体月台地址） | 月台在客户范围内的唯一编码，系统主键之一 |
| customerCode | 客户代码 | VARCHAR(50) | 必填 | 发货计划匹配（确定归属哪个客户） | 客户月台所属的客户编码，关联到客户主数据 |
| platformName | 名称 | VARCHAR(200) | 必填 | 物流调度显示（显示月台名称便于识别） | 月台的显示名称，如"1号月台-东门" |
| description | 描述 | VARCHAR(500) | 非必填 | 月台说明 | 月台的描述信息 |
| city | 城市 | VARCHAR(50) | 非必填 | 物流跟踪（城市定位） | 收货城市 |
| address | 地址 | VARCHAR(500) | 必填 | 发货单据打印（打印发货地址）、物流跟踪（司机导航地址） | 月台的详细收货地址 |
| contactPerson | 联系人姓名 | VARCHAR(100) | 非必填 | 物流跟踪联络（到货前电话确认） | 月台的收货联络人姓名 |
| contactPhone | 联系人电话 | VARCHAR(50) | 非必填 | 物流跟踪联络（电话通知） | 收货联络人的电话号码 |
| warehouseCode | 仓库 | VARCHAR(50) | 非必填 | （待补充） | 关联仓库 |
| defaultLocation | 默认库位 | VARCHAR(50) | 非必填 | （待补充） | 默认库位 |
| effectiveDate | 生效时间 | DATE | 非必填 | （待补充） | 生效时间 |
| invalidDate | 失效时间 | DATE | 非必填 | （待补充） | 失效时间 |
| remark | 备注 | VARCHAR(500) | 非必填 | | 月台的补充说明 |

## 关联关系

```
客户（1）←→（N）客户月台
客户月台 → 发货计划（到货地址）
```

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 联合主键 | customerCode + platformCode 唯一确定一个月台 |
| 来源表 | customerCode → 客户表 |
