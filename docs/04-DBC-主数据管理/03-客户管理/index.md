# 客户管理

> 阅读对象：测试、实施、运维（主）；销售协同与主数据维护人员（顺带）。

## 这一组业务解决什么问题

客户管理维护交付对象、交付地点（客户月台）和客户侧物料口径。读完本组文档，应能判断销售出库、交付或客户退货中客户、月台或客户物料选不到、地点不匹配，应该从客户资料的哪一层入手排查。

## 如何使用本组文档

| 你的目的 | 建议阅读 |
| --- | --- |
| 想理解客户如何与交付、发运建立对应关系 | 本页「这一组业务解决什么问题」与「关键业务对象与关系」。 |
| 要新建或维护客户、客户月台、客户物料 | 按下方「建议学习与操作顺序」逐项进入对应叶页。 |
| 交付业务中客户/月台/物料选不到 | 本页「常见问题与相关分组」。 |

## 建议学习与操作顺序

| 顺序 | 页面/业务对象 | 先解决什么 | 与下一步怎样衔接 |
| --- | --- | --- | --- |
| 1 | 客户 | 建立客户基础识别和可用状态。 | 是客户月台和客户物料的前提。 |
| 2 | 客户月台 | 维护客户侧交付或收货地点信息。 | 支持发运、交付和客户退货的地点判断。 |
| 3 | 客户物料 | 建立客户侧物料口径与系统物料的对应。 | 支持交付匹配和客户侧追溯。 |

## 常见问题与相关分组

客户、月台或客户物料无法在交付业务中选择时，先确认资料状态、客户与地点/物料的关联，以及对应销售业务的来源条件；物料自身问题回到[物料管理](../01-物料管理/index.md)，本组不重复维护物料本身的属性。
