# 供应商

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：主数据维护人员、采购协同人员、仓库收货人员及需要查询供应来源的业务人员。

## 业务目的与适用范围

供应商主数据用于统一识别企业的供货方。采购业务选择供应来源、仓库核对到货方、维护供应商物料关系时，均应引用同一份供应商资料，而不是在每张单据上重复录入名称、联系人和结算类信息。

本页讲清供应商资料为何维护、何时维护、修改后会影响什么。字段填写、批量导入、列表筛选和测试操作见[供应商维护与查询参考](03-供应商维护与查询参考.md)。供应商物料负责“某供应商供应何种物料”的匹配规则，另见本分组的供应商物料页面。

## 使用前准备

维护一项供应商前，至少应由采购或主数据责任人确认以下信息：

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 供应商代码与简称 | 代码是系统识别供应商的稳定标识；简称是日常业务快速辨识信息。 |
| 正式名称与联系信息 | 便于采购、到货异常和日常协同找到正确对象。 |
| 币种、类型、税率与生效区间 | 用于区分供应商属性和适用期间；具体是否在某业务中生效，仍以对应业务规则为准。 |
| 是否可用 | 避免将已停止合作或未完成准入的供应商继续用于新业务。 |
| 供应物料范围 | 若需要限制可采购物料，应在供应商建立后继续维护供应商物料关系。 |

## 一项供应商如何进入业务

```mermaid
flowchart LR
    A["确认合作主体与编码口径"] --> B["新增供应商主数据"]
    B --> C["补充联系人、属性与可用状态"]
    C --> D["按需维护供应商物料"]
    D --> E["采购与到货业务选择供应商"]
    E --> F["发生变更时评估引用后再维护或停用"]
```

供应商创建完成不代表其可以立即用于所有采购场景。物料匹配、采购订单状态、到货规则和权限由对应业务页面控制；供应商页面只负责提供可靠的主体资料和可用状态。

## 关键维护与变更

| 维护点 | 当前可确认的业务规则 | 使用建议 |
| --- | --- | --- |
| 代码 | 新增时必须填写，服务层会检查是否与已有供应商代码重复。 | 代码应稳定；不要为规避重复而添加临时后缀。 |
| 简称 | 新增时必须填写。 | 使用采购、仓库和财务能够共同识别的简称。 |
| 名称与联系信息 | 用于日常识别和联系。 | 变更前确认是否影响正在进行的采购或到货协同。 |
| 类型、币种、税率与有效期 | 用于表达供应商属性及其适用时间。 | 不确定口径时先向采购或主数据责任人确认。 |
| 代码与简称修改 | 当前更新逻辑会保留原代码和原简称。 | 若确需变更身份标识，应走主数据变更方案，不应绕过页面或接口。 |
| 可用/停用 | 可用性是供应商生命周期的重要控制信息。 | 停用前先确认未完成采购、到货和供应商物料关系；实际选择器过滤时点需测试验证。 |
| 删除 | 当前实现未能证明会拦截全部下游引用。 | 已被业务使用的供应商不建议删除，优先评估停用或归档。 |

## 查询、详情与联查

日常查询建议先用供应商代码或名称定位，再用类型、币种、生效区间等条件缩小范围。详情页应优先让读者判断“是谁、是否可用、如何联系、供应什么、在哪些业务中被引用”。

| 详情分组 | 应回答的问题 | 建议联查 |
| --- | --- | --- |
| 基本身份与状态 | 供应商是谁，当前是否可用。 | 供应商物料。 |
| 联系与结算属性 | 应联系谁，采用什么币种、税率或期限信息。 | 采购业务中的供应来源。 |
| 生命周期与审计 | 何时生效、何时失效、何时被维护。 | 变更记录（后续补充）。 |
| 业务引用 | 该供应商供应什么物料、是否已用于采购或到货。 | 供应商物料、采购订单、采购收货。 |

## 批量维护

供应商导入适合初始建档或按统一口径批量维护。导入前必须确认代码、简称和联系人邮件是否完整，并确认本次仅新增、仅更新还是允许两者同时处理；不要用导入绕过供应商准入和变更评估。

模板列、必填项、错误回执和处理方式见[供应商维护与查询参考](03-供应商维护与查询参考.md)。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 新增提示代码重复 | 先查询既有供应商，确认是否应维护原资料；不要建立同一主体的重复代码。 |
| 业务中找不到供应商 | 先核对供应商是否可用、是否已维护供应商物料，以及所在业务页面的选择条件。 |
| 需要修改代码或简称 | 不要直接改写；先确认引用范围并按主数据变更流程处理。 |
| 导入出现错误行 | 按错误回执逐行修正代码、简称、邮件或重复数据，再重新导入。 |
| 已停止合作 | 先检查未完成业务和引用，再执行停用或归档；不要直接删除。 |

## 当前限制与待确认事项

- 供应商停用后在采购、收货等选择器中的实际过滤时点尚需测试环境验证；
- 删除是否会拦截采购、收货、退货、供应商物料等全部引用，尚未形成端到端测试结论；
- 供应商详情的实际页签、跳转过滤条件和变更日志入口需要用测试环境截图确认；
- 前端按钮可见性不能单独证明后端接口权限，动作权限需在 System/RBAC 轮次验证。
