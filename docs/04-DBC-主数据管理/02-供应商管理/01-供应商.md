## 当前事实卡（BATCH-01 首轮取证）

> 取证范围：`dev` 基线的 `basic_supplier` DDL、`SupplierDO/VO/Service/Controller` 与 Web 供应商页面。下方历史字段表使用了若干非真实技术字段名，未被本节确认的内容不得作为接口、导入或测试依据。

### 真实对象与字段口径

当前持久化对象为 `basic_supplier`，主键为 `id`。真实业务字段如下；`creator`、`createTime`、`updater`、`updateTime`、`deleted`、`tenantId` 等属于系统审计或隔离字段，不应作为人工维护或导入模板字段。

| 真实字段 | DDL 类型/长度 | 创建接口必填 | 当前说明 |
| --- | --- | --- | --- |
| `code` | `varchar(64)`，非空 | 是 | 服务层创建时查询重码；Web 限制 50 字符并校验代码格式。 |
| `name` | `varchar(64)` | 否 | Web 可查询、列表显示和编辑；当前创建 VO 未标注必填。 |
| `shortName` | `varchar(64)`，非空 | 是 | 服务层要求非空；创建时未见数据库层唯一约束。 |
| `address`、`country`、`city`、`phone`、`fax`、`postId`、`contacts`、`bank` | 多数 `varchar(64)` | 否 | Web 对多数文本字段限制 50 字符，并对电话、传真、邮编应用前端格式校验。 |
| `email` | `varchar(128)`，默认 `admin@win.com` | 创建接口否；导入是 | Web 校验邮箱格式及最多 128 字符；导入服务要求非空。 |
| `currency`、`type` | `varchar(64)` | 否 | Web 使用币种、供应商类型字典；实际字典值需通过字典数据/测试环境确认。 |
| `taxRate` | `numeric(18,6)` | 否 | Web 数值控件限制为 `0–1`、6 位小数；该前端范围未见同等后端校验。 |
| `available` | `boolean`，默认 `true`、非空 | 是 | 创建 VO 必填；Web 默认只查询可用数据，并有启用/停用按钮。 |
| `activeTime`、`expireTime` | `timestamp(6)` | 否 | Web 在提交时校验失效时间大于生效时间；后端服务未见同等显式区间校验。 |
| `remark` | `text` | 否 | 当前列表和表单均默认隐藏。 |

### 新增、编辑与删除边界

- 创建服务校验：代码存在性/重复、简称非空、可用状态非空；其中代码唯一性由服务查询保障，DDL 仅有普通 `code` 索引，未见唯一约束。
- Web 新增提供新增、导入、导出；默认列表筛选 `available=true`，代码和名称为查询字段，代码、名称、简称、地址、国家、城市、电话、币种、税率、类型、生效/失效时间等为当前默认列表字段，创建/更新审计字段也默认展示。
- Web 编辑锁定 `code` 与 `shortName`；后端更新同样会将二者回填为原值，因此当前正常接口不允许改码或改简称。其它字段及可用状态可由更新/启停接口修改，引用影响仍待测试。
- 删除服务当前仅校验供应商存在后删除，未见采购单、收货单、退货单、供应商物料或 SCP 关联的引用拦截；实际物理/逻辑删除行为需结合 Mapper 与测试环境确认。
- 详情页当前使用通用 `Detail` 组件，尚未实现供应商专属分组和关联跳转。后续应至少规划“基本身份与状态、联系信息、结算属性、生命周期与审计、供应商物料/采购单据跳转”五组。

### 当前文档校正与关联

- 历史字段表中的 `supplierCode`、`supplierName`、`zipCode`、`contactPerson`、`contactEmail`、`bankName`、`supplierType`、`effectiveDate`、`invalidDate` 均不是当前后端/DDL 技术字段名，应逐步替换为本节真实字段。

- 代码唯一性缺少 DDL 强制约束、导入/创建必填不一致、前后端权限保护不一致，已登记为[产品差距总账](../../15-版本路线图/03-问题与待确认/产品差距总账.md)中的 GAP-044。

## 概述

供应商主数据记录与企业有采购业务关系的供应方信息，是采购订单、来料检验、付款结算的基础。
