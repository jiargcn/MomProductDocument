## 字段业务语义

本页模式归类：**P1 / P6 / P10** 为主。
**本页无 P2 业务主数据弹窗选择器**（类型/币种为字典）、**无 P3 / P4 级联选位、无 P7 / P8 / P9 / P11**（默认供应商优先级在供应商物料关系页说明，且 ❓）。
供货匹配选择器见[供应商物料-维护与查询参考](04-供应商物料-维护与查询参考.md)。写法约定见[页面数据字典规范](../../02-业务模型/04-页面数据字典规范.md)。

### 字段业务语义总表

| 字段（中文业务名） | 行为模式 | 业务作用 | 谁在何时维护/选择 | 业务约束摘要 | 影响哪些功能 | 变更或错选风险 |
| --- | --- | --- | --- | --- | --- | --- |
| 供应商代码 | P6 | 唯一识别 | 新增录入 | 必填、查重；更新锁定 | 采购、收货、供应商物料 | 重复/改码导致双主体 |
| 简称 | P6 | 快速辨识 | 新增必填 | 更新锁定 | 列表、单据展示 | 简称含糊 → 选错供方 |
| 名称 | P6 | 正式名称 | 新增/修改 | 按页面要求 | 协同、打印 | 与实物主体不符 |
| 类型 | P1 | 分类 | 字典 | 字典取值 | 筛选 | 分类口径不一致 |
| 币种 | P1 | 结算币种属性 | 字典 | 字典取值 | 采购结算理解 | 与合同币种不符 |
| 税率 | P1 | 税务属性 | 录入 | 0～1、≤6 位小数 | 结算沟通 | 错填不自动改历史单 |
| 是否可用 | P1 | 生命周期 | 启停 | 新增必填；选择器过滤 ❓ | 采购/收货选供应商 | 停用后在途单 |
| 生效/失效时间 | P10 | 适用窗 | 按需 | 过期过滤 ❓ | 准入期间控制 | 窗口错 |
| 联系人邮件 | P14 / 导入约束 | 联系 | 导入必填 | 页面与导入必填可能不同 | 协同 | 导入失败 |
| 审计字段 | P14 | 系统 | 系统 | 勿改 | 追溯 | — |

### 本页模式声明

- **无 P2 受限主数据选择器**：本页不选物料/月台；关系页才有。
- **无 P11**：是否存在“默认供应商”不在本主体页维护。

## 新增与录入限制

建议按“确认主体 → 设定代码与简称 → 补充联系和属性 → 设置可用状态 → 维护供应商物料”的顺序建档。

| 信息组 | 用户需要做什么 | 当前限制或核对方式 |
| --- | --- | --- |
| 身份识别 | 填写代码、名称和简称。 | 代码、简称为新增必填；代码不可与已有供应商重复。 |
| 联系信息 | 按需填写地址、国家/城市、电话、联系人和联系人邮件。 | 导入时联系人邮件为必填；新增页面与导入的必填口径不同，应分别核对。 |
| 业务属性 | 选择币种、类型，按需填写税率、银行、生效/失效时间。 | 币种、类型使用系统字典；税率页面限制为 0 至 1、最多 6 位小数。 |
| 生命周期 | 设置是否可用和备注。 | 新增时可用状态为必填；后续启停对下游选择的影响需测试确认。 |

## 修改、启停与删除

| 操作 | 当前行为 | 操作前应检查什么 |
| --- | --- | --- |
| 修改名称、联系方式和属性 | 更新服务会保存业务属性变更。 | 是否影响正在进行的采购或收货协同。 |
| 修改代码或简称 | 当前更新逻辑保留已有代码和简称。 | 如需改身份标识，走主数据变更方案。 |
| 启用/停用 | 页面存在可用状态管理入口；实际影响范围待测试确认。 | 未完成采购/到货、供应商物料及业务选择器。 |
| 删除 | 现有实现未能证明所有引用均被拦截。 | 是否已有采购、收货、退货或供应商物料引用；通常优先停用。 |

## 导入参考

### 模板应包含什么

| 分类 | 建议列 |
| --- | --- |
| 身份与联系 | 代码、名称、简称、地址、国家、城市、电话、传真、邮编、联系人、联系人邮件。 |
| 业务属性 | 银行、币种、税率、类型、生效时间、失效时间。 |
| 不应让用户填写 | 可用状态、备注、创建/更新时间、创建/更新人等系统或管理信息。 |

### 导入校验与处理

| 校验项 | 当前可确认的规则 | 使用建议 |
| --- | --- | --- |
| 空值 | 代码、简称、联系人邮件为空时会形成错误行。 | 先在模板中补齐三项再导入。 |
| 重复代码 | 同一导入文件中代码重复会报错；已有代码也需按导入模式处理。 | 导入前先做代码去重和存量核对。 |
| 新增/更新范围 | 导入模式控制不存在记录是否插入、已存在记录是否更新。 | 页面上模式名称与数值含义需以测试环境模板确认后再形成培训口径。 |
| 错误回执 | 系统返回错误行及原因。 | 逐行修正后重新导入，避免用重复导入覆盖不确定内容。 |

## 列表、详情与联查

| 推荐顺序 | 字段/条件 | 用途 |
| --- | --- | --- |
| 1 | 代码、名称、简称 | 快速确认目标供应商。 |
| 2 | 类型、币种、生效/失效时间 | 判断是否符合当前业务场景。 |
| 3 | 联系人、联系人邮件、电话 | 协同和异常处理时定位联系人。 |
| 4 | 创建/更新时间、维护人 | 辅助追溯资料维护。 |

详情页建议按“基本身份与状态、联系与结算属性、生命周期与审计、供应商物料与业务引用”分组；实际页面分组与跳转过滤条件待测试环境确认。

## 操作前快速核对

| 你准备做的操作 | 最少先核对什么 |
| --- | --- |
| 新增 | 主体是否已存在；代码、简称、可用状态是否明确。 |
| 修改 | 是否会影响未完成采购、收货或物料匹配。 |
| 停用/删除 | 是否仍存在下游引用；是否应改为停用。 |
| 导入 | 模板必填、同批代码重复、导入模式和错误回执。 |

## 仍待业务确认

- 供应商页面默认列表列、详情页签和字段显示顺序；
- 停用后的选择器过滤规则和删除引用保护；
- 导入模式的产品名称、覆盖范围及错误行回填流程；
- 供应商与 SCP 供应商账号、结算业务的实际关联边界。
