## 当前事实卡（BATCH-01 首轮取证）

> 取证范围：`dev` 基线的 `basic_supplieritem` DDL、Supplieritem VO/Service、Web 页面和导入类。下方历史字段表中的 `materialCode`、`platformCode`、`purchaseUnit` 等不是当前真实技术字段名。

### 真实模型、选择器与业务键

| 主题 | 当前事实 |
| --- | --- |
| 持久化对象 | `basic_supplieritem`；主键 `id`，核心关联字段为 `supplier_code` 与 `item_code`。 |
| 真实业务字段 | `supplierCode`、`itemCode`、`supplierItemCode`、`supplierUom`、`convertRate`、`packUnit`、`packQty`、`settlementType`、`available`、`activeTime`、`expireTime`、`allowOverShipment`、`dockCode`；`supplierName`、`itemName`、`itemUom` 是关联展示字段。 |
| 供应商选择器 | Web 从供应商分页接口构造选择器，筛选 `available=true`，以供应商 `code` 回填。 |
| 物料选择器 | Web 从物料分页接口构造选择器，筛选 `available=true`，以物料 `code` 回填。 |
| 月台选择器 | Web 预取可用月台列表，按代码保存、按名称显示；月台可选且可清空。 |
| 创建校验 | 服务校验供应商代码、物料代码、可用状态非空，并按 `supplierCode + itemCode` 查询重复。供应商/物料的实际存在性校验目前仅检查代码是否为空，未见对主数据查询存在性的强制验证。 |

### 新增、编辑、导入与接口挂接

- 创建时服务会清空输入的供应商名称、物料名称，再持久化；二者应视为关联展示数据，不应列入导入/维护的业务键。
- Web 编辑和后端更新均锁定 `supplierCode`、`itemCode`；更新服务再将二者回写为原值，当前正常接口不允许变更供应商—物料关系键。
- DDL 只有主键及普通索引，未见 `supplier_code + item_code` 复合唯一约束；该组合唯一性只由创建服务保障。
- 导入模板包括供应商代码、物料代码、月台代码、供应商物料代码、供应商计量单位、转换率、可用状态、允许超发、生效/失效时间、备注；包装单位、包装数量、结算方式字段在导入类中被注释，不应出现在当前模板。
- 导入会校验供应商/物料代码、允许超发、可用状态；按 `supplierCode + itemCode` 查找现有数据，模式非 3 时插入不存在记录、模式非 2 时更新已存在记录。模式的业务语义和跳过行反馈仍待测试环境核验。
- 创建、更新、删除会写入接口信息，类型为 `DBC-SCP`、名称“推送基础供应商物料”，记录目标 URL `/dbc/supplieritem/create`、`/update`、`/delete`。这证明存在待发送接口记录，不等同于 SCP 已成功接收或完成幂等处理。

### 当前文档校正与差距

- 历史表中的 `supplierName`、`materialName`、`stockUnit`、`purchaseUnit`、`allowOverIssue`、`conversionRate`、`platformCode` 等应分别校正或废弃，不能继续作为当前字段字典。

## 概述

供应商物料记录某供应商对特定物料的报价信息，是采购价格执行、供应商比价的依据。
