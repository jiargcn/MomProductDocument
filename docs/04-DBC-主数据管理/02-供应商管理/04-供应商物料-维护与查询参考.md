# 供应商物料-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[供应商物料](02-供应商物料.md)使用；本页用于维护供货匹配、批量导入、查询和问题定位。

## 快速定位

| 你要做什么 | 建议先看 |
| --- | --- |
| 为某供应商新增可供物料 | “新增供货关系”和“提交前核对”。 |
| 变更供应方物料号、计量或交付口径 | “修改、停用与替换”。 |
| 批量初始化或更新对应关系 | “导入或批量处理”。 |
| 排查采购或收货为何无法带入对应信息 | “列表、详情与联查”和“常见问题”。 |

## 新增供货关系

新增前应先确认供应商、物料和月台均已按当前现场口径建档并可用。每条关系以“供应商 + 物料”识别；同一组合重复提交会被拒绝。

| 需要准备或选择什么 | 为什么 | 当前核对方式 |
| --- | --- | --- |
| 供应商 | 确定供货主体。 | 从可用供应商中选择，不以手工名称替代。 |
| 物料 | 确定可供货对象。 | 从可用物料中选择，并确认采购、收货使用的是同一物料口径。 |
| 月台 | 说明到货交接地点，并可能影响采购收货时的地点带入。 | 从可用月台中选择；具体带入与拦截效果需在测试环境验证。 |
| 供应商物料代码 | 保留供应方使用的料号，便于来料单据、标签或协同核对。 | 可不填；如填写，应使用供应方稳定的识别口径，长度不超过 50 个字符。 |
| 采购计量与转换率 | 说明供应方交付单位与系统使用单位之间的换算口径。 | 转换率在页面中必填，且不得为负数；保存前由业务确认其换算含义。 |
| 是否允许超发、是否可用 | 决定该关系的使用边界。 | 创建或导入时均应明确，不依赖默认猜测。 |
| 生效/失效时间、备注 | 记录临时供货、切换或补充说明。 | 非必填；有时效要求时必须成对核对业务口径。 |

## 修改、停用与替换

| 操作 | 何时使用 | 先检查什么 | 结果或风险 |
| --- | --- | --- | --- |
| 修改交付属性 | 供应方料号、计量、月台、超发口径或有效期发生变化。 | 是否已有未完成采购、在途收货或退货使用该关系。 | 下游带入和追溯口径可能变化，应保留变更原因。 |
| 停用关系 | 合作终止、物料不再由该供应商供应，且不应继续被新业务选择。 | 是否仍有未完成订单、计划或到货。 | 不能以停用替代历史追溯；历史单据应仍能说明当时供应来源。 |
| 更换供应商或物料 | 原组合发生实质变化。 | 是否需要保留旧关系用于历史查询。 | 当前正常编辑不允许直接更换供应商或物料；建立新关系，并评估旧关系的停用时点。 |

## 导入或批量处理

导入适合供应商物料初始建档或统一修订。导入前先统一供应商代码、物料代码、月台及单位换算口径；不要用导入绕过采购与收货已在执行的变更评估。

| 模板内容 | 用户填写/系统填充 | 当前校验或注意事项 |
| --- | --- | --- |
| 供应商代码、物料代码 | 用户填写 | 是关系识别依据；不可为空，重复组合按导入模式处理。 |
| 月台代码 | 用户填写 | 页面新增要求选择月台；导入是否同样强制及错误提示需在测试环境确认。 |
| 供应商物料代码、供应商计量单位、转换率 | 用户填写 | 转换率不可为空；供应商物料代码最长 50 个字符。 |
| 是否允许超发、是否可用 | 用户填写 | 两项均应明确；空值会产生导入错误。 |
| 生效时间、失效时间、备注 | 按需填写 | 用于约束关系适用期或补充说明；日期先按统一格式准备。 |
| 供应商名称、物料名称、库存计量单位 | 系统关联展示 | 不应作为关系识别键或维护列手工填写。 |
| 包装单位、包装数量、结算方式 | 当前不纳入导入模板 | 若业务确有批量维护需要，先登记为产品需求，不自行扩展模板。 |

当前导入提供新增、更新或两者组合的处理方式；页面对各模式的产品名称、跳过行提示和错误回执仍需用测试环境补充截图和样例。

## 列表、详情与联查

| 查询目标 | 建议默认看什么 | 常用筛选 | 详情/跳转 |
| --- | --- | --- | --- |
| 某供应商可供应哪些物料 | 供应商代码/名称、物料代码/名称、供应商物料代码、可用状态。 | 供应商、是否可用、生效期。 | 跳转或返回供应商资料、物料资料。 |
| 某物料有哪些可选供应来源 | 物料代码/名称、供应商、月台、采购单位、超发口径。 | 物料、是否可用、是否允许超发。 | 联查采购计划、采购收货或来源单据。 |
| 到货地点或默认带入不符合预期 | 供应商、物料、月台、有效期。 | 供应商 + 物料组合。 | 继续核对月台及采购收货明细的实际带入结果。 |
| 追溯一笔采购的供应来源 | 供应商、物料、供应方物料代码和有效期。 | 供应商 + 物料 + 时间范围。 | 联查采购订单、采购收货记录；具体跳转入口待界面验证。 |

当前详情采用通用展示，尚未形成专属分组和快速跳转。建议后续按“关系身份、交付与换算、有效期与状态、审计与接口记录”分组，并提供供应商、物料、采购收货的快速入口。

## 终端操作

本对象当前作为主数据维护，未发现独立 PDA 维护入口。现场人员应在采购或收货终端使用已维护的关系；扫码、离线或异常提示规则需要在对应终端业务中验证。

## 操作前快速核对

1. 供应商、物料、月台是否都是当前可用且符合现场口径的对象？
2. 是否已存在相同的供应商 + 物料关系，或是否应先停用旧关系？
3. 转换率、超发口径和有效期是否已由采购、仓库共同确认？
4. 若关系正在被订单或收货使用，是否已评估变更生效时间和历史追溯影响？

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 供应商、物料或月台无法选择 | 核对对应主数据是否可用，以及当前人员是否具有维护范围；不要改用手工录入的替代代码。 |
| 保存时提示关系重复 | 查询已有关系，确认应更新交付属性、停用旧关系，还是本次供应关系本就不应新增。 |
| 采购或收货未带入预期地点/单位 | 先以供应商 + 物料查询关系，再核对月台、有效期和来源单据；将实际差异保留给测试环境验证。 |
| 导入后部分行未生效 | 下载或查看错误回执，逐行核对代码、转换率、超发口径、可用状态和导入模式。 |
