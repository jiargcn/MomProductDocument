# 系统概述

> 阅读对象：了解产品全景时从此页起步；测试 / 实施 / 运维 / 新开发找「从哪读起」时亦可对照下表。
> 说明：业务边界、实际部署与集成对象以当前项目环境、菜单与配置核验为准；未确认处见文中「待核验 / 规划中」标记。

## 产品定位

本产品文档将 MOM 作为制造运营业务的协同与执行中枢进行说明：统一管理主数据、供应链协同、仓储执行、生产执行、质量、设备、异常、排程与数据采集等能力。

旧版规划书提出“核心业务、支撑业务、公共业务”三层划分，可作为本文档梳理范围的参考；模块是否已在当前环境启用，须以菜单与配置另行核验。

一句话：**统一主数据与配置，让到货、库存、生产、质量、设备与异常在同一套业务对象上协同，而不是各系统各说各话。** 未在当前环境落地的能力（例如 **PS 排程为规划中**）不可按已上线对外承诺，见[模块边界](04-模块边界与系统上下文.md)。

## 核心能力

- 统一主数据与工厂业务建模
- 供应、仓储、生产、质量、设备与异常的协同闭环
- 单据、库存、批次/包装与业务记录追溯
- 报表、看板、标签打印、消息、接口、日志等公共能力

能力是否在**当前项目环境**全部启用，以菜单与配置核验为准，不以本页清单代替验收。
