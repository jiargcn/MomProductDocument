# 模块边界与系统上下文

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：了解各模块职责；测试 / 实施定位「权威落点」后进入业务主文档。
> 职责以现有模块文档第二轮结论为准；调用协议细节见 API 集成场景。

本页用一张表说清：**各模块解决什么问题、与谁协作、文档从哪进**。读完职责列可去模块首页开篇；测试 / 实施用「权威落点」判断跨模块争议该找谁。

## 如何使用本页

| 你的目的 | 怎么用 |
| --- | --- |
| 了解模块职责 | 扫「业务职责」列 → 打开 2～3 个模块首页开篇 |
| 测试 / 实施 | 先看权威落点与协作对象 → 进对应**主文档**做场景与配置（本页无配置差异细节，不宜只凭本页开验证单） |
| 新开发 | 本表 + 目标模块首页 + [核心业务流程](../02-业务模型/11-核心业务流程.md) |

## 1. 模块职责地图

| 模块 | 业务职责 | 主要协作对象 | 文档 |
| --- | --- | --- | --- |
| DBC | 主数据、工厂建模、策略和业务基础配置 | 全部业务模块 | [DBC](../04-DBC-主数据管理/index.md) |
| WMS | 仓储作业、库存、库内、盘点、仓储终端 | SCP、QMS、MES、EAM | [WMS](../05-WMS-库房管理/index.md) |
| MES | 工艺、计划执行、报工、追溯、线边终端 | WMS、QMS、EAM、ANDON、数采 | [MES](../06-MES-生产管理/index.md) |
| QMS | 检验、质量判定与评审 | WMS、MES、SCP | [QMS](../07-QMS-质量管理/index.md) |
| EAM | 维修保养执行、备件/工装；台账多在 DBC | MES、ANDON、WMS | [EAM](../08-EAM-设备管理/index.md) |
| ANDON | 异常呼叫、响应到岗 | MES、EAM、QMS | [ANDON](../09-ANDON-异常管理/index.md) |
| SCP | 采购协同、预测/计划、发货、跟踪、结算 | DBC、WMS、外部 | [SCP](../10-SCP-供应链平台/index.md) |
| PS | **规划中**：排程意图入口，本仓无可用实现，不可按已上线承诺 | MES/DBC（边界意向） | [PS](../11-PS-排程管理/index.md) |
| 数采 | 采集配置、日志、MQTT；Node-RED 薄弱 | MES、WMS、边缘 | [数采](../13-数采管理/index.md) |
| 系统与基础设施 | 组织权限、公共能力、运行与集成 | 全部模块 | [租户与认证](../12-系统管理/01-租户与认证/index.md)、[基础设施](../03-基础设施/index.md) |

## 2. 权威落点（摘要）

跨模块争议时，先按事实类型找权威模块，再进该模块主文档：

| 事实 | 权威 |
| --- | --- |
| 主数据身份 | DBC（SCP 并存对象同步未一律证实） |
| 库存事务/余额 | WMS |
| 生产执行/报工 | MES |
| 检验结论 | QMS |
| 维修工单 | EAM |
| 安灯到岗 | ANDON |

端到端见[核心业务流程](../02-业务模型/11-核心业务流程.md)；接口方向见[跨系统集成场景](../14-API参考/03-跨系统集成场景.md)。
