# 技术栈

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 说明：只写本仓库与配置中**已能核对**的选型；环境主机、账号与密钥不进入本文。部署拓扑见[部署架构](../03-基础设施/10-部署架构.md)。
> 本页为技术选型核对；产品定位与模块职责见[系统概述](05-系统概述.md)。

## 后端

| 组件 | 选型（已证实） | 备注 |
| --- | --- | --- |
| 开发语言 | Java 17 | 模块 POM 统一 `java.version` |
| 应用框架 | Spring Boot 3 体系 | 依赖形态见 `*-spring-boot-3-starter`、XXL-Job Spring Boot 3 工程 |
| 持久化 | MyBatis-Plus + Druid | 各业务模块普遍引入 |
| 数据库 | PostgreSQL | 配置 `db-type` / JDBC 驱动为 PostgreSQL |
| 缓存 | Redis（Redisson 客户端线索） | 配置中存在 redis / redisson；具体集群形态待运维确认 |
| 定时调度 | XXL-Job | 独立 `xxljob` 工程与管理端；业务页见[定时任务](../03-基础设施/08-定时任务与后台作业.md) |
| 报表引擎 | JimuReport（积木报表） | `win-module-report` 引入；业务说明见[报表](../03-基础设施/02-报表、看板与数据输出.md) |
| MQTT（现场） | mica-mqtt 等客户端依赖 | MES 等模块存在；数采通道见[数采](../13-数采管理/index.md) |

业务按模块拆分（本仓可见）：System、Infra、DBC、WMS、MES、QMS、EAM、ANDON、AGV、Report、Finance、SCP 等；IoT 网关（`iotgw`）为仓外依赖。
