# 多端与终端架构

> 状态：占位页已建立。端侧清单来自当前提供的源码目录；页面和权限范围待菜单与源码核验。

## 1. 设计目的

同一业务可能由管理端、协同端和现场终端共同完成。本页规定各端负责的操作类型，避免将 PDA 或线边端误认为独立业务域；读完可判断某个操作该去哪个端找、该去哪个模块的“终端操作”页看细节。

## 2. 当前已知端侧范围

| 端侧 | 参考职责 | 文档归属 |
| --- | --- | --- |
| WEB 主端 | 配置、查询、审批、管理与业务操作 | 对应业务模块页面。 |
| SCP 前端 | 供应链协同操作 | SCP 模块。 |
| PDA | 仓储现场扫描与执行 | WMS 的“终端操作”。 |
| 线边端 | 生产现场采集、投料、报工等 | MES 的“终端操作”。 |
| EAM PDA | 设备现场点检、保养、维修等 | EAM 的“终端操作”。 |

## 待确认事项
- 每个端侧可访问的菜单、角色和接口。
- 同一单据跨端操作时的状态、并发与权限限制。
- 离线、扫码、打印、设备接入及异常处理机制。

## 4. 编写规则

终端操作页面描述具体业务动作；本页仅描述端侧定位、数据一致性与权限边界。
