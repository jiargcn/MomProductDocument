# 维护订单

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> **资料状态：薄弱。** 本页只划清「PS 排程输入」与「MES 计划订单」边界，不抄 MES 字段。

## 业务目的与适用范围

本分组在产品规划中，是排程引擎的**输入侧订单/需求维护**：计划员录入或导入待排生产需求（工厂、序列、数量、交期、BOM 版本等），确认版本后再触发排程。

读完本页，应能：区分「要建可执行工单」应去 MES，还是仍在寻找未证实的 PS 维护订单；禁止把 MES 字段表贴进本页充实现状。

## 如何使用本页

| 你的目的 | 建议怎么做 |
| --- | --- |
| 了解 PS 输入侧规划意图 | 本页「规划中应覆盖的能力」 |
| 现场建工单、下发、开工 | 去 [MES 计划管理](../06-MES-生产管理/03-计划管理/index.md) |
| 判断有没有独立 PS 维护订单 | 对照「当前可确认状态」做环境核验 |
| 看排程执行入口（规划） | [执行生产排程](03-执行生产排程.md) |
