# 客户检验

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；客退质量、仓储退货协同等现场角色（顺带）；操作见[客户检验-维护与查询参考](客户检验-维护与查询参考.md)。

## 业务目的与适用范围

本分组菜单名称为**客户退回检验**（申请/任务/记录），用于客户退货入库后的质量确认。读完本页，应能立刻判断三件事：

1. **建单从哪触发**：主触发在 WMS 客户退货记录（规则「创建后建检验申请」开启时）；也可在本菜单手工建。
2. **为何没有申请**：规则是否关闭、是否重复回调号、是否缺少客户退货检验类型方案。
3. **为什么不是出货检验**：检验类型枚举虽有发货检验等码，但 QMS **无**独立出货 OQC ATR；出货质量协同联查销售出库/SCP，勿在本页培训出货放行。

退货仓储主链见 WMS 销售出库/客户退货相关页；方案见[检验配置](../01-检验配置/index.md)；不合格出口见[质量评审](../05-质量评审/index.md)。库存后续**事务以 WMS 为准**。

## 一笔客户退回检验如何完成

```mermaid
flowchart LR
    A["WMS 客户退货记录\n（规则开关开启）"] --> B["客户退回检验申请"]
    B --> C["任务执行"]
    C --> D["检验记录与结论"]
    D --> E["可选：评审 / 通知单"]
    D --> F["库存后续在 WMS"]
```

也可在 QMS 菜单手工建申请。通用 ATR 状态与判定口径同来料（申请九态、任务四态；接收/拒绝；使用决策四类）。

!!! example "写实示例：给定 → 期望"

    **路径 A · 退货自动建单**

    - **给定：** WMS 客户退货记录创建成功；规则「创建后建检验申请」开启；物料有客户退货检验类型方案。
    - **期望：** 生成客户退回检验申请；回调号形如「退货申请号:明细ID」防重；可处理为任务并录入结论。

    **路径 B · 规则关闭**

    - **给定：** 同上退货成功，但规则关闭。
    - **期望：** 不自动建单；手工在本菜单建申请仍可用（有权限与方案时）。

    **路径 C · 非出货 OQC**

    - **给定：** 现场要做「出货放行检验」，打开本分组菜单。
    - **期望：** 本页不适用；转到销售出库/SCP 协同；培训材料不得称本页为 OQC。

    **路径 D · 拒收后处置**

    - **给定：** 记录评估码拒绝或使用决策不合格；组织要求进评审/索赔。
    - **期望：** 可联查质量评审或 Q 通知单；库存隔离/报废到 **WMS** 核对——使用决策 ≠ 库存已变。

## 与销售 / SCP / WMS / 配置的边界

| 协同方 | 本页负责 | 不在本页展开 |
| --- | --- | --- |
| 检验配置 | 消费客户退货检验类型方案 | 方案主数据维护细节 |
| WMS 客户退货 | 消费建单触发；质量结论意图 | 退货收货库存事务、库位 |
| WMS 销售出库 | — | 出货拣配与发运（非本页 OQC） |
| SCP | 客诉/发货协同线索 | 采购预测与结算 |
| 质量评审 / Q1 | 客退不合格出口 | 索赔金额规则细节 |

## 关键判断

| 判断点 | 应先确认什么 | 影响 |
| --- | --- | --- |
| 退货后无检验单 | 规则开关、是否重复回调号、方案 | 避免重复建单或漏建 |
| 是否做出货检验 | 有无独立菜单/类型使用证据 | 无则不培训为 OQC |
| 结论后库存怎么变 | WMS 退货后续与隔离/报废 | QMS 不直接改余额 |
| 旁路建单找不到 | 寄售退货/发货调整回调号 | 勿只在本菜单死等 |

### 建议验证点

- 规则开启时，客户退货记录可生成客户退回检验申请且同回调号不重复。
- 规则关闭时不自动建单；手工申请仍可用。
- 任务可完成并形成记录结论；类型与方案为客户退货检验。
- 培训/验收材料不将本页称为出货 OQC。
- 未发布时不以使用决策假定库存已变；库存后续在 WMS 核对。
- 不合格样例可联查质量评审/通知单（按组织流程）。

## 限制与待确认

- `GAP-016`：客退检验现场状态机、旁路建单（寄售退货/发货调整）与回写规则待逐页核验。
- `FSEM-006`：方案/承接人选择器精确状态过滤与 P13 投影矩阵待测。
- 枚举含发货检验等类型，缺菜单 ≠ 已交付出货检验页；培训不得写成 OQC。
- 客户退货建单时部分 `reference*` 字段置空的业务意图待产品确认。
