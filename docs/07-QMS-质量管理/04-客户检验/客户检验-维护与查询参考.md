# 客户检验-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[客户检验](index.md)。面向测试、实施、运维的日常处理、查询与异常定位；对象为**客户退回检验** ATR。主线、出货边界与建议验证点以主文档为准。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 弄清触发与类型边界（客退 ≠ 出货 OQC） | 「字段说明」 |
| 退货后查是否建检验 | WMS 客户退货记录 + 规则开关；或本菜单申请 |
| 执行检验 | 客户退回检验任务/记录 |
| 查是否出货检验 | 本页不适用；转销售出库/质量配置 |
| 开一轮最小验收 | 「验收时建议核对」 |
| 想先理解客退进质量主线 | 返回[客户检验](index.md) |

## 字段说明

下表服务排障与维护，不替代主文档主线。

| 字段 | 它决定什么 | 约束摘要 | 配错/漏配时常见现象 |
| --- | --- | --- | --- |
| 退货记录触发开关 | 是否自动建申请 | 「创建后建检验申请」 | 关则漏建；开则须防重 |
| 客户/物料/数量 | 检验对象 | 多由退货带入 | 参考订单可能置空 |
| 类型=客户退货检验 | 与方案匹配 | 非发货检验菜单 | 当 OQC 培训 |
| 评估码/使用决策/ATR | 判定与门禁 | **库存事务归 WMS** | 决策≠库存已变 |

## 触发与联查

| 场景 | 操作提醒 |
| --- | --- |
| 自动建单 | 客户退货记录创建后，规则「退货记录创建后创建检验申请」开启；回调号形如「退货申请号:明细ID」防重。 |
| 手工建单 | 选客户退货检验类型方案；补包装数量。 |
| 联查仓储 | 退货记录号、包装号、批次；库存后续仍查 WMS。 |
| 旁路 | 寄售退货/发货调整若建单，用回调号联查，勿只在本菜单死等。 |
| 误当作 OQC | 停止；转销售出库/SCP，勿在本菜单验收出货放行。 |

## 申请 / 任务 / 记录

口径同来料：状态推进、步骤录入、评估码与使用决策、发布。重点核对物料、数量、包装与退货一致。发布后关注是否进评审；库存隔离/报废到 WMS 核对。

## 验收时建议核对

与主文档「建议验证点」同口径，便于现场勾选：

- [ ] 规则开启时，客户退货记录可生成客户退回检验申请且不重复。
- [ ] 规则关闭时不自动建单，手工申请仍可用。
- [ ] 任务可完成并形成记录结论；类型为客户退货检验。
- [ ] 培训材料不将本页称为出货 OQC。
- [ ] 使用决策已选但记录未发布时，不以库存已变为前提；库存事务归 WMS。
- [ ] 不合格可联查质量评审/通知单（按组织流程）。
