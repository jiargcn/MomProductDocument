# 质量评审-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[质量评审](index.md)。面向测试、实施、运维的日常处理、查询与异常定位；不合格闭环主线、与 WMS/通知单边界及建议验证点以主文档为准。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 弄清结论如何回写使用决策、与库存边界 | 「字段说明」 |
| 对检验不合格做 MRB | 「检验评审 ATR」 |
| 建客诉/索赔通知 | 「质量通知 Q1/Q2/Q3」 |
| 查仓储待评审库存 | WMS 质量评审（非本菜单） |
| 开一轮最小验收 | 「验收时建议核对」 |
| 想先理解不合格如何闭环 | 返回[质量评审](index.md) |

## 字段说明

下表服务排障与维护，不替代主文档主线。

| 字段 | 它决定什么 | 约束摘要 | 配错/漏配时常见现象 |
| --- | --- | --- | --- |
| 来源检验记录 | 挂接不合格事实 | 须可回溯 | 无来源难闭环 |
| 评审结论 | 让步/报废/返修 | 可回写使用决策 | 让步≠已放行 |
| 库存定位明细 | 处置对象 | 批次/包装/库位等 | 错处置 |
| Q1/Q2/Q3 | 通知/索赔 | 不替代评审结论 | 只建 Q 单无闭环 |
| 评审 ATR 状态 | 动作门禁 | 申请→任务→记录 | 非预期状态强发 |

## 检验评审 ATR

| 菜单 | 用途 |
| --- | --- |
| 检验评审申请 | 发起评审，关联检验记录/物料/包装/工单产线等。 |
| 检验评审任务 | 承接评审，填写结论与说明。 |
| 检验评审记录 | 查询已完成评审。 |

### 应核对

| 信息 | 提醒 |
| --- | --- |
| 检验记录号、检验类型 | 必须能回溯原检验。 |
| 明细库存定位 | 批次、包装、库位、仓库、库存状态。 |
| 评审结论 | 让步 / 报废 / 返修。 |
| 流程节点 / 评审人 / 时间 | 审批痕迹。 |

### 常用动作

申请提交与审批（以页面为准）；任务完成；完成后确认检验记录使用决策是否更新；再到 WMS 做库存动作。**勿在 QMS 手工改库存顶替处置。**

## 质量通知 Q1/Q2/Q3

| 单据 | 典型关注点 |
| --- | --- |
| Q1 | 客户、物料、数量、索赔原因/金额、责任人、优先级、状态。 |
| Q2 | 供应商、物料、关联 Q1/采购收货号、索赔与费用汇总、状态。 |
| Q3 | 问题描述、缺陷位置/类型、原因、金额、责任人；亦可有主子表形态。 |

动作以各通知单页面的新增、处理、关闭为准；勿与检验评审状态机混用。与评审 ATR 并存；是否自动互建见 `GAP-016`，培训勿编造必过自动链路。

## 联查

| 场景 | 建议 |
| --- | --- |
| 评审后库存未变 | 查 WMS 是否已做隔离/放行/报废；QMS 只给结论。 |
| 找不到评审单 | 用检验记录号反查；确认是否误走了 WMS 待评审物料。 |
| 索赔对不上 | Q1↔Q2 号、采购收货号、数量金额。 |
| 只见 Q 单无结论 | 按组织流程补检验评审；勿把通知单当 MRB 闭环。 |

## 验收时建议核对

与主文档「建议验证点」同口径，便于现场勾选：

- [ ] 从来料或生产不合格记录可建立检验评审申请并完成任务。
- [ ] 结论为让步/报废/返修之一；记录可查询。
- [ ] 任务完成后检验记录使用决策与映射预期一致（合格/报废/隔离）。
- [ ] 让步/报废样例在 WMS 侧核对库存动作；不以 QMS 评审完成代替余额结果。
- [ ] Q1/Q2/Q3 可独立建单查询；与评审关系按组织说明，不强行编造自动链路。
- [ ] 培训区分 QMS 检验评审与 WMS 质量评审菜单。
