# 质量评审-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[质量评审](index.md)。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 理解结论如何回写使用决策 | “字段业务语义” |
| 对检验不合格做 MRB | “检验评审 ATR” |
| 建客诉/索赔通知 | “质量通知 Q1/Q2/Q3” |
| 查仓储待评审库存 | WMS 质量评审（非本菜单） |

## 字段业务语义

本页模式归类：**P1 / P2 / P5 / P8 / P9 / P12**。

| 字段（中文业务名） | 行为模式 | 业务作用 | 业务约束摘要 | 影响哪些功能 | 变更或错选风险 |
| --- | --- | --- | --- | --- | --- |
| 来源检验记录 | P2 / P5 | 挂接不合格 | 须可回溯 | 评审合法性 | 无来源 |
| 评审结论 | P1 / P9 | 让步/报废/返修 | 可回写使用决策 | 下游处置 | 让步≠已放行 |
| 库存定位明细 | P2 / P8 | 处置对象 | 批次/包装/库位等 | WMS 处置 | 错处置 |
| Q1/Q2/Q3 | P1 / P12 | 通知/索赔 | 不替代评审结论 | 客诉闭环 | 只建 Q 单 |

## 检验评审 ATR

| 菜单 | 用途 |
| --- | --- |
| 检验评审申请 | 发起评审，关联检验记录/物料/包装/工单产线等。 |
| 检验评审任务 | 承接评审，填写结论与说明。 |
| 检验评审记录 | 查询已完成评审。 |

### 应核对

| 信息 | 提醒 |
| --- | --- |
| 检验记录号、检验类型 | 必须能回溯原检验。 |
| 明细库存定位 | 批次、包装、库位、仓库、库存状态。 |
| 评审结论 | 让步 / 报废 / 返修。 |
| 流程节点 / 评审人 / 时间 | 审批痕迹。 |

### 常用动作

申请提交与审批（以页面为准）；任务完成；完成后确认检验记录使用决策是否更新；再到 WMS 做库存动作。

## 质量通知 Q1/Q2/Q3

| 单据 | 典型关注点 |
| --- | --- |
| Q1 | 客户、物料、数量、索赔原因/金额、责任人、优先级、状态。 |
| Q2 | 供应商、物料、关联 Q1/采购收货号、索赔与费用汇总、状态。 |
| Q3 | 问题描述、缺陷位置/类型、原因、金额、责任人；亦可有主子表形态。 |

动作以各通知单页面的新增、处理、关闭为准；勿与检验评审状态机混用。

## 联查

| 场景 | 建议 |
| --- | --- |
| 评审后库存未变 | 查 WMS 是否已做隔离/放行/报废；QMS 只给结论。 |
| 找不到评审单 | 用检验记录号反查；确认是否走了 WMS 待评审物料。 |
| 索赔对不上 | Q1↔Q2 号、采购收货号、数量金额。 |

## 最小验收清单

- [ ] 从来料或生产不合格记录可建立检验评审申请并完成任务。
- [ ] 结论为让步/报废/返修之一；记录可查询。
- [ ] 任务完成后检验记录使用决策与映射预期一致（合格/报废/隔离）。
- [ ] Q1/Q2/Q3 可独立建单查询；与评审关系按组织说明，不强行编造自动链路。
- [ ] 培训区分 QMS 检验评审与 WMS 质量评审菜单。
