# QMS 质量管理

> 基线状态：已完成 `dev` 后端首轮取证。当前确认了检验配置、申请/任务/记录与质量评审的对象分层；检验结论如何联动库存、生产和供应链仍待逐项核验。

## 已核验的模块职责

QMS 负责检验规则与检验业务事实的维护。后端存在抽样方案、AQL、样本字码、检验方法、检验特性、检验阶段、检验方案/模板、物料检验配置和动态规则；同时存在检验申请、检验任务、检验记录及其主表、明细、包装、特性、定量/定性结果对象，以及申请/任务/记录评审对象。

| 能力域 | 当前已核验对象 | 文档边界 |
| --- | --- | --- |
| 检验配置 | 抽样、AQL、样本字码、检验方法、检验特性、检验阶段、方案、模板、物料检验配置、动态规则。 | 定义“应如何检验”；主数据引用与版本规则待核验。 |
| 检验执行 | 检验申请、检验任务、检验记录及主表/明细/包装/特性/定量/定性结果。 | 定义“实际检验了什么及其结果”；状态流待核验。 |
| 质量评审 | 申请、任务、记录三个层次的评审主表和明细。 | 评审的审批主体、结论和后续动作待核验。 |
| 质量协同 | 来料、生产、客户检验的分组入口均存在。 | 与 SCP、WMS、MES、ANDON 的触发和回写关系待核验。 |

## 关键边界

1. QMS 负责检验依据、检验执行及质量结论；库存状态变更、隔离、放行或退货的实际事务写入须在 WMS 与相关业务链路中核验。
2. QMS 的申请—任务—记录结构应引用[申请、任务与记录模型](../02-业务模型/01-申请任务记录模型.md)，具体质量对象不重复定义通用单据层次。
3. 当前后端对“检验记录”拆分较细，而菜单分组较粗；后续页面覆盖应按主表、明细、包装、特性、定量/定性结果补足查询与详情分组。

## 模块概述

本模块用于将质量要求配置为可执行的检验方案，并沉淀来料、生产及客户场景的检验申请、任务、记录和评审事实。

## 核心流程

> 待核验：应分别确认“来源业务触发检验申请 → 创建/分配检验任务 → 填写检验记录 → 形成质量结论/评审 → 回写库存或业务单据”的真实状态与接口链路。

## 接口规范
