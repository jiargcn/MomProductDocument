# QMS 质量管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施、运维（主）；质量工程师、体系工程师等现场角色（顺带）。

## 模块职责

QMS 把质量要求配成可执行的检验方案，并沉淀来料、生产过程、客户退回等场景的检验申请—任务—记录，以及不合格后的评审与质量通知。读完本页，应能：按「配置 → 三条触发链 → 评审闭环」选对分组；并分清库存处置归 WMS、报工细节归 MES。

库存隔离/放行/退货/报废等**事务写入以 WMS 为准**。通用 ATR 见[申请、任务与记录模型](../02-业务模型/01-申请任务记录模型.md)。

## 如何使用本模块

| 你的目的 | 建议怎么做 |
| --- | --- |
| 理解 QMS 总览与模块边界 | 本页：模块职责 → 核心流程 → 与其它模块边界 |
| 落地抽样/方案/动态规则 | 先 [检验配置](01-检验配置/index.md) |
| 跟一笔到货/过程/客退检验 | 按学习顺序进对应分组 |
| 排障：未建检、结论回写、不合格去向 | 先查边界表归属，再进分组常见问题/维护参考 |

## 建议学习顺序

不确定从哪读起时：先懂方案配置，再按到货/生产/客退三条链分别理解，最后看不合格如何闭环。

1. [检验配置](01-检验配置/index.md) — 先懂方案与抽样。
2. [来料检验](02-来料检验/index.md) — 到货确认触发主链。
3. [生产检验](03-生产检验/index.md) — 首件/末件/巡检/其他。
4. [客户检验](04-客户检验/index.md) — 客户退回检验（非出货 OQC）。
5. [质量评审](05-质量评审/index.md) — 让步/报废/返修与通知单。

操作步骤读各组「维护与查询参考」。

## 核心流程（总览）

```mermaid
flowchart TB
    CFG["检验配置：方案 / 抽样 / 方法"] --> IQC["来料检验"]
    CFG --> IPQC["生产检验"]
    CFG --> CR["客户退回检验"]
    W1["WMS 到货确认待检"] --> IQC
    W2["WMS 制品收货 / 手工"] --> IPQC
    W3["WMS 客户退货"] --> CR
    IQC --> REC["检验记录结论"]
    IPQC --> REC
    CR --> REC
    REC --> REV["检验评审 / 通知单"]
    REC --> WMS["WMS 库存处置"]
```

## 与其它模块边界
