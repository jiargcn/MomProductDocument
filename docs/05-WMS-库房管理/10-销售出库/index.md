# 销售出库

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：销售协同人员、仓库备货/发运人员、客户交付人员及需要追溯出库结果的业务人员。

## 业务目的与适用范围

销售出库将销售订单、发货计划或客户交付需求转化为仓库备货、发运和实际出库结果。它连接成品库存、客户交付、发货协同与库存扣减，确保“要发什么、从哪里发、实际发了多少、何时发给谁”能够完整追溯。

本页说明销售出库的业务主线。库存预留、批次/序列号分配、发运状态、接口回写和结算影响需以当前配置和测试环境继续验证。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 销售订单、发货计划或交付需求 | 明确本次出库的客户、物料、数量和时间要求。 |
| 客户、收货地点和交接信息 | 确保成品发往正确对象和地点。 |
| 可用库存、库位、批次/包装 | 决定可备货范围及追溯要求。 |
| 运输、承运或发运信息 | 支持实际装运和交付协同。 |
| 质量与放行状态 | 避免未放行、隔离或冻结库存被用于交付。 |

## 一笔销售出库如何完成

```mermaid
flowchart LR
    A["销售订单 / 发货计划"] --> B["备货申请或发货申请"]
    B --> C["备货/发运任务"]
    C --> D["拣货、扫码、复核与装运"]
    D --> E["发货/出库记录"]
    E --> F["库存事务与余额减少"]
    E --> G["客户交付、协同或结算后续"]
```

申请明确交付意图，任务组织仓库执行，记录保存实际发运结果。若发现库存不足、批次不匹配、客户信息错误或少发/多发，应保留差异并回到相应异常处理，而不能用正常发货记录掩盖问题。

【示例数据占位：客户订单 100 件，备货 100 件、实发 98 件、缺货 2 件。展示订单、任务、发运记录、库存结果和客户交付处理。】

## 关键判断、角色与动作

| 判断点 | 应先确认什么 | 对业务的影响 |
| --- | --- | --- |
| 是否可以备货 | 来源单据、交付时间、客户、物料和可用库存是否一致。 | 决定是否建立备货或发货工作。 |
| 从哪里拣货 | 库位、库存状态、批次/包装和质量放行情况。 | 决定实际交付对象与可追溯性。 |
| 是否可以发运 | 拣货结果、数量差异、客户/地点和运输交接信息。 | 决定是否形成实际出库记录。 |
| 出现交付差异如何处理 | 少发、多发、错料、错批次或运输异常原因。 | 决定是否补发、回退或建立异常协同。 |

销售或交付人员提供需求，仓库人员负责备货、复核和交接，运输/客户协同人员处理发运信息。实际审批、任务分配和动作权限需以当前配置确认。

## 库存与相关业务影响

销售出库记录应形成库存变动并减少相应库存结果。是否存在库存预留、批次先进先出、序列号强制扫描或安全库存预警，不能只依据历史草稿判断，应在测试环境中按物料和任务配置验证。

| 关联业务 | 应关注什么 |
| --- | --- |
| 销售/发货计划 | 来源订单、客户、交付数量和交期。 |
| 库存管理 | 拣货前后可用量、库位、批次/包装和事务记录。 |
| 客户主数据 | 客户、交付地点、客户物料或交接信息。 |
| 运输协同 | 承运、装运、发货单和交付结果。 |
| 结算或外部系统 | 发运结果是否需要回写、开票或结算。 |

## 查询、详情与联查

| 想解决的问题 | 推荐定位方式 | 建议联查 |
| --- | --- | --- |
| 哪些订单待备货/待发运 | 订单号、客户、发货计划、任务状态或交期。 | 销售来源、备货申请。 |
| 某客户实际发了什么 | 发货记录号、客户、订单或物料。 | 发运信息、库存事务。 |
| 为什么无法发货 | 任务状态、库存状态、批次/包装或质量放行信息。 | 库存余额、质量处理、来源订单。 |
| 出库是否已生效 | 发货记录和库存事务。 | 库存余额、客户/外部协同结果。 |

详情页建议按“销售来源与客户、物料与数量、备货与拣选、发运与交接、库存影响、异常与后续、系统信息”分组。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 库存有数但无法备货 | 核对库存状态、冻结、质量放行、库位和任务分配条件。 |
| 扫描批次/包装不匹配 | 停止操作，核对来源要求和可用库存，不要强行发货。 |
| 实发数量与订单不一致 | 记录少发/多发原因，并按交付异常流程处理。 |
| 发货后余额未变化 | 先查发货记录、库存事务和余额，再确认是否有异步或接口处理。 |

## 当前限制与待确认事项

- 备货、发货申请、任务和记录的真实状态及自动处理策略需测试确认；
- 库存预留、FIFO、批次/序列号规则和缺货处理需按物料配置验证；
- 客户交付、运输协同、外部回写和结算的完整闭环待跨模块测试；
- Web/PDA 的拣货、复核、装运和撤销/冲抵操作待补截图和权限验证。
