# 采购上架

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓库收货人员、上架执行人员、库存主管及需要查询库位结果的业务人员。

## 业务目的与适用范围

采购上架把已收货、需要进入正式存储位置的物料，从待处理状态转为可按库位管理和查询的库存结果。它连接采购收货、质量处理、库位选择与库存余额，避免“已经收货但不知道放在哪里、能否使用”的断点。

本页说明上架的业务主线与查询方式。上架策略、库位推荐、容量、扫描校验、状态值和终端动作应在后续采购上架-维护与查询参考和测试环境中逐项确认。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 来源收货及处理结果 | 确认哪些物料已经可以进入上架处理。 |
| 物料、数量、批次和包装 | 保证实际搬运对象与收货结果一致。 |
| 目标仓库、库区、库位及库存状态 | 决定物料最终在哪里、以何种状态存放。 |
| 库位和作业规则 | 确认是否需要扫描、是否允许修改推荐库位或数量。 |
| 质量或隔离结论 | 避免将待检、不合格或隔离物料误放入可用库位。 |

!!! example "📷 截图占位"
    采购上架申请或任务页面，标出来源收货、物料、目标库位、库存状态、批次/包装和扫描要求。

## 一笔采购上架如何完成

```mermaid
flowchart LR
    A["采购收货结果"] --> B["确认可上架范围"]
    B --> C["上架申请或任务"]
    C --> D["现场扫描并确认目标库位"]
    D --> E["上架记录"]
    E --> F["库存地点/状态结果更新"]
    F --> G["库存余额与后续可用查询"]
```

上架的核心不是简单输入一个库位，而是将来源收货中的物料、批次/包装、数量和库存状态与实际存放地点建立一致关系。现场发现库位不匹配、数量差异或质量限制时，应转入异常处理而不是覆盖来源信息。

!!! example "📝 示例数据占位"
    收货 100 件物料，分两次上架至两个库位；展示来源收货、上架任务、实上数量和余额地点变化。

## 关键判断、角色与动作

| 判断点 | 应先确认什么 | 对业务的影响 |
| --- | --- | --- |
| 是否可创建上架 | 来源收货是否完成，质量或库存状态是否允许继续。 | 决定是否进入上架任务。 |
| 上到哪里 | 推荐/指定库位是否与物料、状态和现场条件匹配。 | 决定后续拣选、盘点和可用范围。 |
| 是否可改数量或库位 | 当前任务配置、扫描要求和异常处理规则。 | 防止现场操作偏离来源结果。 |
| 是否已完成 | 是否已形成上架记录并在库存查询中显示正确地点。 | 决定是否关闭任务或继续处理差异。 |

仓库人员通常负责承接、扫描和上架；库存主管或业务处理人员负责处理库位、数量和状态异常。实际分配与权限需以测试环境验证。

### 关键字段业务角色

完整选择器与门禁见[维护与查询参考](采购上架-维护与查询参考.md)。上架衔接[采购收货](../03-采购收货/index.md)记录：主线差异是**地点/状态落地**，不是重新定义到货范围。

| 字段/配置点 | 在系统中的作用 | 关键行为要点（取值/范围/联动/门禁） | 维护或操作时要警惕什么 |
| --- | --- | --- | --- |
| 来源收货记录 | 确定待上架对象 | 从已收货、仍待上架的结果发起（❓ 精确过滤待确认） | 未收货或已上架完则选不到 |
| 目标仓库 / 库区 / 库位 | 正式存放地点 | 级联过滤；常要求扫描校验；任务可限制改位 | 扫错位导致后续拣选找不到料 |
| 库存状态 | 上架后可用范围 | 待检/隔离不得误入可用位（❓ 硬拦截范围待确认） | 状态错导致误发或不可用 |
| 待上/实上数量 | 控制上架量 | 可拆分上架 ❓；不得超过待上范围 | 数量与收货不一致需保留原因 |
| 批次 / 包装 / 托盘 | 与收货同一粒度落地 | 构成余额业务键，见[库存管理精度与唯一粒度](../../02-业务模型/08-库存管理精度与唯一粒度.md) | 漏采造成余额分裂错误 |
| 申请/任务/记录状态 | 门禁 | 任务已证实有执行、批量执行、取消接收 | 取消接收与库存预期清理 ❓ |

## 库存与相关业务影响

采购上架会改变物料的存放地点，并可能影响库存状态和后续业务的可选范围。上架完成后，采购收货、库存余额、拣选/发料、盘点等页面应能在一致的库位与追溯维度下查询到该物料。

| 关联业务 | 应关注什么 |
| --- | --- |
| 采购收货 | 来源、实收数量、批次/包装和质量结果。 |
| 库位管理 | 目标库位是否适用、是否需要容量或属性校验。 |
| 库存管理 | 上架前后余额的地点、状态和可用范围变化。 |
| 质量处理 | 待检、合格、隔离等状态是否限制上架去向。 |
| 后续出入库 | 拣选、发料、盘点能否按上架结果找到物料。 |

## 查询、详情与联查

| 想解决的问题 | 推荐定位方式 | 建议联查 |
| --- | --- | --- |
| 哪些物料尚未上架 | 来源收货、上架任务状态、供应商或物料。 | 收货记录、任务明细。 |
| 某物料实际放在哪里 | 上架记录号、物料、批次/包装。 | 库存余额、库位资料。 |
| 为何不能上架 | 任务号、目标库位、库存状态或扫描提示。 | 来源收货、质量结果、库位规则。 |
| 上架是否已生效 | 上架记录和库存余额。 | 最近库存事务、后续业务选择结果。 |

详情页建议按“来源收货、物料与数量、目标地点与状态、现场扫描/差异、库存影响、系统信息”分组。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 扫描库位不匹配 | 核对任务推荐库位、物料属性、库存状态和现场库位标签；不要跳过扫描限制。 |
| 任务数量与实物不一致 | 保留差异原因，回查来源收货和批次/包装，不要直接改成正常完成。 |
| 上架完成但余额地点未更新 | 先查上架记录，再查库存事务和库存余额。 |
| 待检物料被要求上架 | 先确认质量状态允许的库位或隔离处理方式。 |

## 当前限制与待确认事项

- 库位推荐、容量、混放、批次一致性等规则需由当前配置和测试环境验证；
- 任务是否允许拆分上架、部分完成、修改库位或数量需补实际场景；
- 上架记录的状态、撤销和反向库存影响尚需验证；
- Web/PDA 的扫描流程、异常提示和权限边界需补截图与测试账号验证。
