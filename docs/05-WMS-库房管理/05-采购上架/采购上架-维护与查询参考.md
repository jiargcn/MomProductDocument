# 采购上架-维护与查询参考

> 本页是[采购上架](index.md)的操作参考；上架任务已具备执行、批量执行和取消接收等动作，但实际状态与按钮前置条件仍需测试环境复核。

## 各阶段应维护什么

| 阶段 | 应完成的业务信息 | 不应绕过的限制 |
| --- | --- | --- |
| 上架申请 | 来源收货、待上架物料/数量、批次/包装、库存状态和目标地点。 | 质量或库存状态不允许时，不得直接创建可用库位上架结果。 |
| 上架任务 | 执行人、推荐/目标库位、待上架数量和扫描要求。 | 库位或数量不一致时必须保留异常原因。 |
| 上架记录 | 实上数量、实际库位、批次/包装、执行结果和时间。 | 记录完成后应回查库存地点/状态，不能只以任务完成代替结果确认。 |

## 导入与批量处理

当前申请、任务和记录均存在导入/导出对象；上架任务还可执行批量处理。具体哪些对象可通过页面导入、模板列、重复策略和部分完成规则待实测。

批量执行前应确保每条任务的来源、物料、数量、批次/包装、目标库位和库存状态均可成立；不应把系统编号、历史状态、审计信息作为业务导入字段。

| 位置 | 建议优先显示 | 快速跳转/操作 |

| 记录列表 | 记录号、物料/数量、实际库位、库存状态、执行时间。 | 库存事务、库存余额、库位资料。 |
