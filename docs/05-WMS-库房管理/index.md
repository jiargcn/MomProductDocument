# WMS 库房管理

> 基线状态：已完成采购收货—预计入—库存事务—库存余额的首轮源码取证。其余业务链、字段表和状态规则待按字段真实性校正任务继续核验。

## 已核验的模块职责

WMS 负责仓储业务执行和库存侧事实数据。当前后端存在入库、上架、发料、生产收/退料、销售出库、客户退货、调拨、计划外出入库、盘点、库存调整、包装/器具、预计入/出、库存余额、库存事务、余额变更历史、库存唯一标识与移动日志等对象；大量业务采用申请、任务、记录主明细结构。

| 能力域 | 当前已核验对象或链路 | 文档边界 |
| --- | --- | --- |
| 单据执行 | 多类申请、任务、记录及主表/明细/撤销记录控制器。 | 复用[申请、任务与记录模型](../02-业务模型/01-申请任务记录模型.md)，各业务只描述差异。 |
| 库存模型 | 预计入、预计出、库存事务、库存余额、余额差异、余额历史、库存 SN。 | [库存数据挂接模型](../02-业务模型/02-库存数据挂接模型.md)是统一入口。 |
| 首轮已证实链路 | 采购收货任务创建预计入；采购收货记录创建库存事务；库存事务服务更新库存余额。 | 该链路可作为纵向试点事实，详见采购收货和库存管理页面。 |
| 仓储作业 | 收货、上架、发料、生产收退、发货、调拨、盘点、调整、包装/器具。 | 各场景的状态、权限、库存影响和接口回写仍需逐链核验。 |
| 终端与输出 | PDA 承载现场扫码与执行；当前也存在标签/模板、导入导出及日志接口。 | 业务端负责触发与记录；标签/打印和导入治理遵循平台能力规范。 |

## 当前库存关键口径

库存余额的业务唯一粒度已由产品确认：批次、托盘、包装、物料、库存状态、库位。DDL 是否提供等价复合唯一约束仍是 GAP-005，不能因业务键已确认而误写为数据库约束已实现。

## 关键边界

1. WMS 是库存事务与余额的权威业务入口；SCP、MES、QMS、EAM 的业务动作是否调用或映射 WMS 库存模型，必须按实际服务链核验。
2. 标签、条码、打印在 WMS 中存在业务入口，不改变其长期平台能力归属。
3. 后续任何叶页字段表均须执行[字段技术名真实性校正任务](../15-版本路线图/字段技术名真实性校正任务清单.md)，不得由中文标签推导技术字段名。

## 模块概述

本模块用于将仓储业务意图落实为可执行任务与业务记录，并通过库存预计、事务和余额沉淀可追溯的库存事实。

## 核心流程

采购收货的已证实示例：业务来源 → 收货申请 → 收货任务（创建预计入）→ 收货记录（创建库存事务）→ 库存余额更新。其他业务链应按同一结构逐一验证，不能直接套用。
