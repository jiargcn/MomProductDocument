# 终端操作

> 状态：页面结构已建立，具体终端功能待按 WMS 业务链逐项取证。  
> 适用范围：PDA 与线边端中归属于 WMS 的操作入口。  
> 业务规则来源：以对应 WMS Web 页面、后端服务和数据库结构为准。

## BATCH-01 标准占位

> 状态：首轮占位增强。本页是 WMS 终端操作汇总入口，不替代各业务页中的终端执行规则。

终端操作不单独定义业务单据、任务、记录或库存过账规则。PDA/线边端只作为对应 WMS 业务页的执行入口补充；本页负责汇总终端入口、扫码对象、设备能力、离线/异常处理和与 Web 的差异。

### 已确认的终端边界

- PDA 源码还包含退货、发货、补料、报废、容器、标签/托盘等目录；本轮先列出已完成库存链路取证的高优先级入口，其他目录后续按对应 WMS 业务页补充。
- `menu.csv` 同时记录了 PDA 顶层菜单、WMS Web 菜单及一些历史停用/隐藏项。终端培训清单应以“测试环境实际菜单 + 已分配角色”为准，源码菜单只用作取证线索。
- 离线暂存、网络重试、重复扫码幂等、相机/扫码枪权限和异常提示，当前尚未获得可证实实现；这些内容保留为终端专项测试项，不做假设性说明。

### 后续页面补充格式

每个终端入口在对应 WMS 页面补充四项：**终端路由/菜单、任务状态前置条件、扫码与人工录入字段、提交后记录/事务/余额影响**。需要截图的场景另按图示与示例任务清单采集，避免用源码推断操作界面。

## 定位

本页不重复定义采购收货、上架、发料、盘点等业务规则，而是补充终端操作与 Web 操作之间的差异：扫码与标签识别、设备输入、简化录入、离线处理、按钮权限和异常反馈。
