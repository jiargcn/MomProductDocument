# 生产收料-维护与查询参考

> 本页是[生产收料](index.md)的操作参考。当前 WMS 菜单明确提供生产收料任务和生产收料记录；“生产收料申请”未在同一菜单链路中确认，应以 MES/生产来源或后续任务生成机制为准。

## 页面与业务对象

| 菜单入口 | 当前对象链 | 使用说明 |
| --- | --- | --- |
| 生产收料任务 | 生产来源 → 任务 → 记录 | 仓库/现场承接产出收料的执行入口。 |
| 生产收料记录 | 实际记录 → 库存事务/余额 | 查询实收数量、状态、地点与追溯结果。 |
| 产品收料相关对象 | 另一套请求、任务、记录对象。 | 与当前“生产收料”菜单的映射及适用边界待确认，不能混写。 |

## 执行与导入

| 阶段 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 来源与任务 | 生产订单、工序/工位、产出物料、数量、批次/包装、收料地点和状态。 | 来源、产出与质量状态不一致时暂停收料。 |
| 现场执行 | 实收数量、实际批次/包装、库位、库存状态、异常原因。 | 待检或隔离产出不得按合格库存处理。 |
| 收料记录 | 实收结果、质量分流、执行时间与人员。 | 以记录及库存事务核对是否真正入账。 |
| 批量导入 | 当前需先确认页面是否开放模板和导入。 | 不导入来源系统编号、历史状态、审计信息。 |

## 列表、详情与快速跳转

| 位置 | 建议优先显示 | 快速跳转 |
| --- | --- | --- |
| 任务列表 | 任务号、生产来源、工序/工位、物料/数量、收料地点、库存状态、状态。 | MES/工单、任务明细、现场终端。 |
| 记录列表 | 记录号、生产来源、实收数量、批次/包装、库位、库存状态、执行时间。 | 库存事务、库存余额、质量结果。 |
| 详情 | 生产来源、物料与数量、批次/包装、质量与状态、执行、库存影响、系统信息。 | 任务、MES/工单、库存、质量。 |

## 与库存预期、事务和余额的核对

生产收料的预期路径应为“MES/生产来源 → 收料任务 → 预计入库 → 收料记录 → 库存事务 → 库存余额”。当前已确认菜单中的任务和记录入口，以及独立的预计入对象；但任务由何时创建、是否必然生成预计入、质量分流后如何影响库存状态，仍需 MES 与 WMS 联合验证。实际入库是否完成，应以收料记录、库存事务和余额三者联查为准。
