## 字段业务语义

本页模式归类：**P1 / P2 / P3 / P4 / P5 / P7 / P8 / P9 / P12**。
相对出入库主链：没有“外部供应商/客户来源单”为主，而是 **P3 来源余额 + 目标地点/状态**；类型选错比字段填错更致命。盘点差禁止用本页消账。

写法约定见[页面数据字典规范](../../02-业务模型/04-页面数据字典规范.md)；粒度通例见[库存管理精度与唯一粒度](../../02-业务模型/08-库存管理精度与唯一粒度.md)。
**物料拆解等旁路无完整专表**（非标准 ATR，规则 ❓）。

| 字段（中文业务名） | 行为模式 | 业务作用 | 谁在何时维护/选择 | 业务约束摘要 | 影响哪些功能 | 变更或错选风险 |
| --- | --- | --- | --- | --- | --- | --- |
| 作业/业务类型 | P1 / P12 | 分流转移/状态/计划外/报废 | 建单时选对入口 | 菜单独立但部分共用对象族 | 审批、预计、事务路径 | 类型错 → 整单路径错 |
| 来源库位 / 来源余额 | P3 / P2 | 变动起点 | 申请/任务选择或扫描 | 粒度匹配且可用量足够 | 扣减/转移 | 余额不足或状态不符 |
| 目标库位 | P2 / P4 | 地点变化终点 | 转移/调拨/计划外入库 | 仓—区—位级联 | 余额新地点 | 跨仓残留 |
| 来源/目标库存状态 | P1 | 状态变化终点 | 状态转移必填 | 不得绕过质量放行/盘点 | 可用范围 | 用状态转移掩盖质量问题 |

| 批次 / 包装 / 托盘 | P8 / P3 | 变动粒度 | 随余额带入或扫描 | 余额业务键 | 追溯、盘点 | 漏采扣错行 |
| 申请/任务/记录状态 | P9 | 门禁 | 系统迁移 | 预计入创建分支不一致见 `GAP-042` | 执行、关闭 | 只看一种预期导致误判 |
| 创建人/时间等 | P14 | 审计 | 系统写入 | 勿当可维护项 | 追溯 | — |

### 分类型门禁与预期（P9 / 预期差异）

| 作业类型 | 预期（已证实方向） | 完成结果 | 特别风险 |
| --- | --- | --- | --- |
| 库内转移 / 调拨 / 状态转移 | 任务路径可建**预计出**；**预计入**存在分支不一致（`GAP-042`） | 记录完成 → 事务并清理相关预计 | 排查时不要只盯预计入是否出现 |
| 计划外出库 / 报废 | 申请侧可建**预计出** | 出库事务 | 无原因/无审批不得当“改数工具” |
| 计划外入库 | 申请侧可建**预计入** | 入库事务 | 同上 |
| 物料拆解等 | ❓ | ❓ | 不套用转移主链 |

### 联动 / 回填（摘要）

| 模式 | 本页要点 |
| --- | --- |
| P4 | 变更来源仓库 → 清空来源库位/余额；变更目标仓库 → 清空目标库位。 |
| P5 | 选中来源余额后回填并锁定追溯维度；目标在禁止改位时锁定。 |
| P7 / P8 | 数量受来源可用量约束；扫描不匹配必须停止。 |

## 页面与业务对象

| 菜单入口 | 当前对象链 | 使用说明 |
| --- | --- | --- |
| 调拨出库/入库 | 申请 → 任务 → 记录 | 组织间或仓库间转移；执行上属于库存转移业务类型。 |
| 库内转移 | 申请 → 任务 → 记录 | 改变地点的主要入口。 |
| 状态转移 | 申请 → 任务 → 记录 | 合格/隔离/报废/线边等状态转换入口。 |
| 计划外出库 | 申请 → 任务 → 记录 | 计划外减少库存。 |
| 计划外入库 | 申请 → 任务 → 记录 | 计划外增加库存。 |
| 报废出库 | 申请 → 任务 → 记录 | 受控报废处置。 |
| 物料拆解等 | 非标准三层或不完整三层 | 另按专项规则处理，不套用转移主链。 |

## 地点变化：调拨与库内转移

| 阶段 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 申请 | 作业类型、来源/目标地点、物料数量、批次/包装、原因。 | 来源与目标不清时不得发起。 |
| 任务 | 执行人、待移数量、扫描与库位要求。 | 余额不足或扫描不匹配时停止。 |
| 记录 | 实际移出/移入结果。 | 以记录和库存事务确认是否生效。 |

任务生成时可形成预计出；记录完成后形成库存事务并清理相关预计库存。预计入是否始终创建，存在路径差异，排查时不要只看一种预期。

## 状态转移

| 阶段 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 申请 | 来源状态、目标状态、物料与数量、业务原因。 | 不得用状态转移绕过质量放行或盘点调整。 |
| 任务/记录 | 实际状态变更结果、库位与追溯维度。 | 与质量隔离/放行结论冲突时暂停。 |

状态转移与库内转移共用对象族，但菜单和业务类型不同；查询时先确认入口类型。

## 计划外出入库

| 方向 | 应维护/确认 | 库存预期 |
| --- | --- | --- |
| 计划外出库 | 合法业务原因、审批、来源库存和数量。 | 申请侧可形成预计出。 |
| 计划外入库 | 合法业务原因、目标库位/状态和数量。 | 申请侧可形成预计入。 |

计划外业务不是“随便改数量”的入口。缺少原因、审批或凭证时不得执行。

## 报废出库

| 阶段 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 申请 | 报废原因、数量、批次/包装、责任或质量依据。 | 原因不清不得报废。 |
| 任务/记录 | 实际处置数量与结果。 | 完成后应能从记录追溯到出库事务。 |

## 库存影响与追溯

| 发生时点 | 系统业务结果 | 如何追溯 |
| --- | --- | --- |
| 生成任务 | 相关类型形成预计出/预计入。 | 从任务号查预计库存。 |
| 完成执行 | 形成业务记录和库存事务，清理相关预计。 | 从记录查库存事务。 |
| 事务完成 | 更新库存余额。 | 按物料、库位、批次、包装、库存状态查余额。 |

如果“做完了但库存没变”，依次检查：记录是否生成、事务是否生成、查询条件是否覆盖完整粒度、预计库存是否仍挂起。

## 仍待业务确认

- 库存转移预计入在人工/其它分支间的一致性；
- 各子业务撤销、反向事务和审批主体；
- 物料拆解、冻结解冻、形态转换的完整操作标准；
- 详情分组、按钮前置条件和 RBAC 边界。
