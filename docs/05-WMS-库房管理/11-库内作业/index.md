# 库内作业

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓库主管、库内作业人员、库存管理人员及需要追溯内部库存变化的业务人员。

## 业务目的与适用范围

库内作业处理库存仍在企业内部、但地点、状态、归属或数量需要发生变化的场景，例如调拨、转库、计划外出入库、报废和其它库存调整。它的目标是让每一次内部变化都有来源、执行过程和结果记录，避免直接改余额造成无法追溯的账实差异。

本页说明库内作业的共同业务逻辑。各子业务的字段、审批、状态、扫码和库存动作需在后续按具体页面补充，不能把某一类作业规则直接套用于全部库内操作。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 作业类型与业务原因 | 区分调拨、转库、报废、计划外出入库或其它调整。 |
| 源库存与目标去向 | 明确物料当前在哪里、要转到哪里或为何需要减少。 |
| 物料、数量、批次、包装和库存状态 | 确保变化作用于正确的库存追溯单元。 |
| 责任人、审批或授权 | 防止无来源、无权限的库存调整。 |
| 现场搬运或处置方式 | 决定是否需要扫描、复核、交接或报废凭证。 |

## 库内作业通用流程

```mermaid
flowchart LR
    A["库存调整需求"] --> B["按作业类型创建申请"]
    B --> C["审核或处理"]
    C --> D["库内执行任务"]
    D --> E["现场搬运、扫码或处置"]
    E --> F["业务记录"]
    F --> G["库存事务与余额结果"]
```

申请说明“为什么要变”，任务组织“谁去做”，记录保存“实际怎么变”。即使只是同仓库内转库，也应保留来源、目标和实际数量，确保库存查询能够解释数量或地点为何变化。

【示例数据占位：将一批物料从原料库位调拨到线边库位，以及一笔报废处置；展示申请、任务、记录和库存变化。】

## 常见作业类型与关键判断

| 作业类型 | 业务目标 | 关键判断 |
| --- | --- | --- |
| 调拨/转库 | 将库存从一个地点转到另一个地点。 | 来源与目标库位、数量、批次/包装、库存状态是否正确。 |
| 计划外出入库 | 处理不由常规采购、生产或销售链路直接触发的库存变化。 | 是否具备合法业务原因、审批和来源凭证。 |
| 报废 | 对无法继续使用的库存形成受控处置。 | 报废原因、数量、质量/责任要求和后续财务处理。 |
| 状态调整或其它内部处理 | 改变库存可用范围、归属或处理状态。 | 是否应由质量、盘点或其它专门业务处理，而非直接调整。 |

不同作业类型的审批、任务、记录和库存动作可能不同。遇到不确定场景，应先确认作业类型和对应业务页面，不应以“库内作业”作为万能的直接改库存入口。

## 库存与相关业务影响

库内作业会改变库存的地点、状态、数量或归属，但应始终保留可回查的库存事务。执行后，库存余额查询必须能反映正确的源/目标结果；与盘点、质量、生产、设备或自动化相关的变化还需转入相应模块继续处理。

| 关联业务 | 应关注什么 |
| --- | --- |
| 库存管理 | 源/目标库存、库存事务、余额和最后变化记录。 |
| 库位与工厂建模 | 来源/目标仓库、库区、库位和现场地点是否正确。 |
| 盘点管理 | 差异是否应通过盘点调整而非普通库内作业处理。 |
| 质量管理 | 状态变化是否由检验、隔离或放行结论触发。 |
| 终端操作 | 扫码搬运、容器移动或现场异常处理。 |

## 查询、详情与联查

| 想解决的问题 | 推荐定位方式 | 建议联查 |
| --- | --- | --- |
| 哪些库内作业待执行 | 作业类型、任务状态、来源/目标库位或执行人。 | 申请、任务明细。 |
| 某批库存为何换了地点/状态 | 物料、批次/包装、库存事务或业务记录。 | 来源申请、目标地点。 |
| 报废或调整是否已生效 | 业务记录、数量、原因和处理时间。 | 库存事务、库存余额、后续处置。 |
| 能否撤回或更正 | 当前状态、是否已经产生后续业务。 | 原记录、相关任务、库存结果。 |

详情页建议按“作业原因与类型、来源与目标、物料与数量、现场执行、库存影响、审批/异常、系统信息”分组。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 找不到可移动库存 | 核对物料、库位、库存状态、冻结和批次/包装。 |
| 来源与目标库存不一致 | 停止执行，核对作业类型、地点和扫描结果。 |
| 想直接调整数量 | 先判断是否应走盘点调整、报废或其它有原因的业务流程。 |
| 执行完成但库存没有变化 | 查询业务记录、库存事务和余额，并确认任务是否真正完成。 |

## 当前限制与待确认事项

- 容器移动、AGV/终端执行和扫码规则需结合现场页面补充；
- 库内作业与盘点调整、质量状态调整的职责边界需跨模块确认。
