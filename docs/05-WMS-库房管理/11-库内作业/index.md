# 库内作业

## 概述

库内作业是 WMS 库房管理模块中处理库存内部流转和调整的核心子系统，涵盖调拨、计划外、报废、转移等业务场景。库内作业通过申请-任务-记录的三阶段流程，确保所有库存变动可追溯、可管控。

库内作业的核心价值：

- **流程规范**：申请→审批→执行→记录的全链路闭环，防止无单据的随意库存变动
- **跨区协同**：支持仓库间调拨、库区流转等跨区域库存转移
- **质量管控**：不合格品隔离、报废处理等质量相关操作符合质量管理体系要求
- **数据修正**：人工修正系统库存数据，确保账实一致

### 核心实体说明

| 实体 | 中文名 | 说明 |
|------|--------|------|
| JobRequest | 作业申请 | 库内作业的发起单据，包含调拨申请、计划外申请、报废申请等 |
| JobTask | 作业任务 | 申请审批通过后生成的执行任务，指导库房人员执行具体作业 |
| JobRecord | 作业记录 | 作业执行完成后的正式记录，作为库存变动的业务凭证 |
| JobType | 作业类型 | 区分不同性质的库内作业（调拨出库/调拨入库/计划外/报废/转移等） |
| InventoryBalance | 库存余额 | 物料在特定仓库+库位+批次下的实时库存量 |
| InventoryTransaction | 库存事务 | 所有库存变动的流水记录，与作业记录关联 |
| Lot | 批次 | 物料的批次信息，支持批次追溯 |

### 关联关系

```
作业申请 1──n 作业任务（一个申请可生成多个任务，如分批调拨）
作业任务 1──1 作业记录（一个任务对应一条完成记录）
作业类型 1──n 作业申请（作业类型决定申请的业务性质）
库存余额 1──n 库存事务（每次库存变动生成一条事务）
物料 1──n 批次
物料 1──n 作业申请/作业任务/作业记录
```

### 作业类型分类

| 作业类型 | 代码 | 说明 |
|----------|------|------|
| 调拨出库 | TRANSFER_OUT | 从仓库/库区调出物料 |
| 调拨入库 | TRANSFER_IN | 向仓库/库区调入物料 |
| 计划外出库 | UNPLAN_OUT | 无单据的紧急出库 |
| 计划外入库 | UNPLAN_IN | 无单据的紧急入库 |
| 报废出库 | SCRAP_OUT | 质量问题物料的销毁出库 |
| 库存初始化 | INIT | 新物料首次入库或系统上线初始化 |
| 库内转移 | LOCATION_TRANSFER | 同仓库内库位调整 |
| 库存修改 | INV_MODIFY | 人工修正系统库存数据 |
| 不合格转隔离 | UQ_TO_QUARANTINE | 不合格品转入隔离区 |
| 合格转隔离 | Q_TO_QUARANTINE | 合格品转入隔离区（待检等） |
| 隔离转报废 | QUARANTINE_TO_SCRAP | 隔离区物料报废 |
| 合格转报废 | Q_TO_SCRAP | 合格品直接报废 |
| 报废转隔离 | SCRAP_TO_QUARANTINE | 报废品重新转入隔离区 |

## 字段约束说明

| 约束类型 | 说明 |
|----------|------|
| 字典项 | requestType（调拨出库/调拨入库/计划外出库/计划外入库/报废出库/库存初始化/库内转移/库存修改/质量状态转换）、status（待审批/审批中/已审批/已拒绝/已取消/待执行/执行中/已完成）、transferType（不合格转隔离/合格转隔离/隔离转报废/合格转报废/报废转隔离）、fromStatus/toStatus（正常/隔离/报废） |
| 联动影响 | requestQty > 源库区可用库存 → 申请被拒绝；taskQty < requestQty → 支持分批执行；modifyQty与currentQty差异超过10% → 需上报管理层审批 |
| 计算公式 | afterQty = beforeQty + qty（正数入库/负数出库）；targetQty = currentQty + modifyQty |
| 追溯机制 | 申请→任务→记录→事务的完整链路；质量状态转换直接关联检验报告单据 |
| 审批流程 | 标准作业：仓库主管审批；库存修改：仓库主管+财务双重审批；计划外作业：支持快速审批通道 |
