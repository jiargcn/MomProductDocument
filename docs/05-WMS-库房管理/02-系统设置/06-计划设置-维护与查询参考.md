# 计划设置-维护与查询参考

> 本页是[计划设置](04-计划设置.md)的操作参考。任何自动化设置变更均应先在测试环境验证，再在受控窗口内切换。

## 新增与编辑约束

| 信息组 | 应维护的信息 | 维护限制 |
| --- | --- | --- |
| 适用标识 | 代码、申请模式。 | 应明确服务的业务场景；代码重复与业务类型关联待验证。 |
| 自动处理 | 自动提交、自动通过、自动生成任务、直接生成记录。 | 可能改变审批与现场执行路径；必须提供回退方案。 |
| 生效控制 | 可用状态、生效/失效时间、备注。 | 切换前检查在途申请、任务和记录。 |

## 列表、详情与快速跳转

| 位置 | 建议优先显示 | 快速跳转/操作 |
| --- | --- | --- |
| 列表 | 代码、申请模式、四项自动处理、可用状态、有效期。 | 按可用状态与申请模式筛选。 |
| 详情 | 适用范围、自动处理策略、生效控制、变更与验证记录。 | 进入采购收货、退货或上架的申请/任务/记录样例验证。 |

## 最小验收清单

- [ ] 关闭自动处理时，申请、审核、任务和记录均按预期保留人工步骤。
- [ ] 启用自动处理时，自动生成对象、状态和执行人符合业务批准口径。
- [ ] 自动处理失败时，不会产生无法追溯的半完成业务。
- [ ] 切换配置后，在途业务和新建业务的表现符合预期。
