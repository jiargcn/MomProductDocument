# 系统日历

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。

## 这项配置解决什么问题

系统日历为指定模块维护可用的开始与结束时段，以及生效、失效和可用状态。它用于表达“某模块在什么期间按这套日历口径运行”，不等同于员工排班、账期或现场作业班次。

## 维护与查询说明

| 维护事项 | 当前可确认做法 | 操作提示 |
| --- | --- | --- |
| 新增 | 模块和可用状态为必填；可维护开始/结束时间、生效/失效时间和备注。 | 先确认该日历服务的模块与业务目的。 |
| 编辑 | 可调整时段和有效期。 | 已在使用的日历变更前，应评估正在处理的业务。 |
| 停用 | 通过可用状态停止后续使用。 | 不以停用代替历史时段的清理。 |
| 导入 | 模板包含模块、起止时间、可用状态、有效期和备注。 | 模板未标明必填校验，导入前应先做小样试导。 |
| 查询 | 列表优先显示模块、起止时间、可用状态和有效期。 | 常用筛选为模块、可用状态与有效期。 |

建议详情分为“适用模块、时间范围、生效控制、变更记录”；实际详情分组和与入库业务的跳转入口待补充截图验证。

## 当前边界与待确认事项

- 开始/结束时间是否允许交叉、重叠或跨期，以及保存时的校验规则尚未确认。
- 系统日历被哪些 WMS 业务实际读取、停用后是否即时生效尚未确认。
