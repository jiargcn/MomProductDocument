# 账期日历

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。

## 这项配置解决什么问题

账期日历维护年、月、期间起止和转换生效时间，为需要按会计或运营期间归集的 WMS 业务提供期间口径。它不等同于系统日历：系统日历关注模块时段，账期日历关注业务结果属于哪个期间。

## 维护与查询说明

| 维护事项 | 当前可确认做法 | 操作提示 |
| --- | --- | --- |
| 新增 | 年、月、期间开始/结束、转换生效时间和可用状态为必填；可维护描述、有效期和备注。 | 先确认期间边界与转换时点，避免跨期归属错误。 |
| 编辑 | 可维护说明、时间和有效期。 | 已产生业务结果的期间不应直接改写，应先确认关账、追溯和更正方案。 |
| 停用 | 用可用状态控制新业务的使用。 | 停用前确认尚未处理的在途业务。 |
| 导入 | 模板包含年月、期间时间、转换时间、可用状态、有效期和备注。 | 年月、时间与可用状态在模板中标记必填。 |
| 查询 | 列表优先显示年、月、期间起止、转换生效时间和可用状态。 | 常用筛选为年月、可用状态和有效期。 |

建议详情分为“期间识别、期间边界、转换控制、生效控制、变更记录”。

## 当前边界与待确认事项

- 年月唯一性、期间重叠、转换时间与期间边界的关系尚未确认。
- 收货、退货、上架等入库结果在何时取得账期、已关期间如何限制处理，需以测试环境验证。
