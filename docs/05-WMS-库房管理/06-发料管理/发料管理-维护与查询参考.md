# 发料管理-维护与查询参考

> 本页是[发料管理](index.md)的操作参考。发料、补料和生产退料属于相关但不同的业务入口；本阶段以普通发料为主线，补料与生产退料作为可联查分支说明。

## 页面与业务对象

| 菜单入口 | 当前对象链 | 使用说明 |
| --- | --- | --- |
| 发料申请 | 申请 → 任务 → 记录 | 面向有明确生产需求的正常发料。 |
| 发料任务 | 任务 → 现场执行 → 记录 | 仓库拣选、扫码和交接的主要入口。 |
| 发料记录 | 实际记录 → 库存事务/余额 | 查询实际发料及库存结果。 |
| 补料 | 申请 → 任务 → 记录 | 对现场缺口形成受控补充；规则待确认。 |
| 生产退料 | 申请 → 任务 → 记录 | 将生产现场物料退回明确去向；不与正常发料混用。 |

## 新增、执行与导入

| 阶段 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 发料申请 | 生产来源、领用地点、物料/数量、需求时间、批次/包装要求。 | 来源、用途和库存状态不清楚时不得发起。 |
| 发料任务 | 拣选库位、执行人、待发数量、现场交接要求。 | 扫描物料、包装或批次不匹配时必须走异常处理。 |
| 发料记录 | 实发数量、实际批次/包装、领用地点、差异原因。 | 应由记录形成库存结果；不得用余额编辑替代发料。 |
| 批量导入 | 申请层可见导入对象；模板、模式和前端入口待实测。 | 不导入系统编号、历史状态或审计信息。 |

## 与库存预期、事务和余额的核对

发料的预期路径应为“待执行发料任务 → 预计出库 → 发料记录 → 库存事务 → 库存余额”。当前已确认发料具有申请、任务和记录对象，但预计出库的生成/释放时点、少发补料与生产退料的处理方式尚未实测。因此，培训和问题排查应先从发料记录、库存事务和余额确认实际结果，不以预计出库代替实际扣减。
