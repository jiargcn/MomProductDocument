# 盘点管理-维护与查询参考

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 用途：配合[盘点管理业务说明](index.md)使用。盘点执行与盘点调整分属不同对象链；本页按阶段给出操作与查询要点。

## 快速定位

| 你要做什么 | 先看哪里 |
| --- | --- |
| 建盘点范围 | “计划与申请” |
| 现场实盘 | “任务与实盘记录” |
| 看账实差 | “差异对比” |
| 确认后改账 | “盘点调整” |
| 查库存是否已变 | “库存影响与追溯” |
| 使用 PDA | “终端执行参考” |

## 页面与业务对象

| 菜单入口 | 当前对象链 | 使用说明 |
| --- | --- | --- |
| 盘点计划 | 计划 | 约定范围、时间和组织方式。 |
| 盘点申请 | 申请 → 任务 | 落实可执行盘点工作。 |
| 盘点任务 | 任务 → 实盘记录 | 现场执行主入口。 |
| 盘点记录 | 实盘结果 | 查询实际盘点数据。 |
| 盘点对比 | 账面 vs 实盘 | 发现并定位差异。 |
| 盘点调整申请/记录 | 申请 → 记录 | 将已确认差异转为库存事务；无独立调整任务菜单。 |
| 差异库存相关入口 | 旁路对象 | 与现行调整主链关系待确认，不作为标准必经步骤。 |

## 计划与申请

| 应维护/确认 | 关键限制 |
| --- | --- |
| 仓库/库区/库位、物料、库存状态、盘点方式和责任人。 | 范围不清不得下发任务。 |
| 计划时间与是否需要冻结并发业务。 | 盘点期间收发会干扰差异解释。 |

申请/计划处理后应能形成可执行任务，并从库存余额带出账面基准。

## 任务与实盘记录

| 现场要确认什么 | 为什么重要 |
| --- | --- |
| 任务号、盘点范围和账面数量 | 防止盘错区域或拿错基准。 |
| 库位、库存状态、批次、包装/容器 | 保证与账面同一追溯口径比较。 |
| 实盘数量与异常原因 | 形成可靠实盘记录。 |

盘点任务和记录用于记实物，**不直接改库存余额**。数量异常应先复核，再进入差异对比。

### 常用动作

| 动作 | 业务结果 |
| --- | --- |
| 承接/执行任务 | 开始现场盘点。 |
| 形成盘点记录 | 保存实盘结果。 |
| 关闭任务 | 结束本次现场执行；具体状态待实测。 |

## 差异对比

| 你要查什么 | 推荐查看内容 |
| --- | --- |
| 是否有差异 | 同一库位/物料/批次/包装下的账面与实盘数量。 |
| 差异可能原因 | 漏扫、错库位、未完成收发、包装混淆、损耗。 |
| 下一步 | 复盘、解释差异，或进入盘点调整。 |

不要在对比阶段直接改余额。

## 盘点调整

| 阶段 | 应维护/确认 | 关键限制 |
| --- | --- | --- |
| 调整申请 | 差异来源、确认数量、原因、审批信息。 | 未确认原因不得调整。 |
| 调整记录 | 实际调整结果。 | 以库存事务和余额核对是否生效。 |

当前可证实实现：调整按正负数量生成入/出库存事务。不要按“先入差异库再出库”的历史说法操作或培训。

## 列表、详情与快速跳转

| 位置 | 建议优先显示 | 快速跳转 |
| --- | --- | --- |
| 计划/申请 | 单号、范围、方式、责任人、状态、时间。 | 任务。 |
| 任务列表 | 任务号、范围、执行人、状态。 | 申请、PDA、账面余额。 |
| 记录列表 | 记录号、库位、物料、实盘数量、批次/包装。 | 对比、账面余额。 |
| 对比 | 账面、实盘、差异数量、差异维度。 | 调整申请、相关收发。 |
| 调整 | 调整单号、差异来源、数量、状态、时间。 | 库存事务、库存余额。 |

详情建议分组：范围与计划、任务与执行、实盘明细、差异对比、调整与审批、库存影响、系统信息。

## 库存影响与追溯

| 发生时点 | 系统业务结果 | 如何追溯 |
| --- | --- | --- |
| 盘点任务/记录 | 形成实盘事实，不直接改余额。 | 从任务/记录查实盘。 |
| 差异对比 | 标出账实差。 | 从对比结果查差异维度。 |
| 盘点调整完成 | 生成库存事务并更新余额。 | 从调整记录查事务和余额。 |

若“已盘点但库存不变”，先确认是否只完成了实盘、尚未做调整；若已调整，再查事务和余额。
