# 盘点管理

## 概述

盘点管理是 WMS 库房管理中确保库存账实一致的核心模块。通过定期或不定期的库存盘点，发现并处理库存差异，保证账面库存与实际库存的准确性。盘点管理覆盖从盘点计划制定、盘点任务下发、扫码盘点执行、差异对比分析到盘点调整申请的全流程闭环管理。

盘点管理涉及两类核心业务实体：
- **盘点单据**：盘点计划、盘点申请、盘点任务、盘点记录、盘点对比、盘点调整申请、盘点调整记录
- **差异库存**：差异库存余额、差异库存出库申请、差异库存出库记录

### 实际对象与链路边界

当前实现存在两段相互关联、但不应混写为一个对象的链路：

| 链路 | 已确认对象表 | 已确认关系和作用 |
| --- | --- | --- |
| 盘点执行链路 | `plan_count_*`、`request_count_*`、`job_count_*`、`record_count_*` | 盘点计划提供范围；盘点申请生成盘点任务；任务执行产生盘点记录；对比服务从盘点记录中按初盘、复盘、抽盘阶段读取数据。 |
| 盘点调整链路 | `request_countadjust_*`、`record_countadjust_*`、库存事务/库存余额 | 盘点申请可根据差异盘点记录生成调整申请；调整申请处理后生成调整记录和库存事务，由事务服务更新库存余额。 |

- `CountRequestMainService` 以业务类型 `Count` 生成任务。明盘可按申请明细及拆分规则生成任务；盲盘可按盘点范围/库位生成任务。
- 盘点计划的 `isFreeze=TRUE` 且任务包含明盘明细时，服务会将相应余额标记为冻结，并写入 Redis 冻结标记；冻结的实际可见性、解除条件和并发边界仍需测试环境验证。
- `CountJobMainService` 执行时校验物料、盘点范围和库位。明盘还校验当前用户是否为任务承接人；完成后生成盘点记录。Web 任务页的“执行”主按钮处于注释状态，PDA `count/job/*` 承担现场盘点执行入口。
- `CountCompareServiceImpl` 以 `plan_number` 聚合记录：初盘阶段为基准，复盘取最近一次 `RECOUNT`，抽盘取 `AUDIT_COUNT`；对比键由物料、库位、库存状态、批次和包装号组成。
- 调整记录按 `adjustQty` 正负选择盘点调整业务类型配置的入库或出库事务类型，调用库存事务服务；调整前会以包装号、物料、批次、库存状态、库位核对余额数量。该链路是盘点差异影响库存余额的当前可证实入口。

### 流程说明

1. **盘点计划制定**：根据仓库/物料类型/ABC分类制定盘点策略，设定盘点周期和范围
2. **盘点申请创建**：由盘点计划触发或手动创建盘点申请单
3. **盘点申请审批**：审批通过后，系统自动生成盘点任务
4. **盘点任务下发**：盘点任务按库位/物料分配给指定仓管员
5. **扫码盘点执行**：仓管员使用 PDA/扫码枪扫描库位和物料，输入实际数量
6. **盘点对比分析**：系统自动对比账面数与实盘数，生成差异报表
7. **盘点调整申请**：对差异进行审批，生成盘点调整申请
8. **调整过账**：审批通过后，差异量过账到差异库存或正式库存
9. **差异库存管理**：盘点产生的差异量单独建账，可发起出库申请或报废处理

## 字段约束说明

| 约束类型 | 说明 |

| 业务规则 | 盘点任务未完成前不允许发起调整；差异库存余额可用数量必须大于等于0；差异库存出库数量不能超过可用数量 |

## 关联关系

```
盘点计划 ←→ 盘点申请（1对多）：一个盘点计划可触发多个盘点申请
盘点申请 ←→ 盘点任务（1对多）：一个盘点申请审批后生成多个盘点任务
盘点任务 ←→ 盘点记录（1对多）：一个盘点任务包含多条盘点记录
盘点任务 ←→ 盘点对比（1对1）：一个盘点任务生成一份对比报表
盘点对比 ←→ 盘点调整申请（1对多）：一份对比可发起多个调整申请
盘点调整申请 ←→ 盘点调整记录（1对多）：一个调整申请过账后生成多条调整记录
盘点对比 ←→ 差异库存余额（1对多）：盘点差异进入差异库存独立管理
差异库存余额 ←→ 差异库存出库申请（1对多）：差异库存可多次出库
差异库存出库申请 ←→ 差异库存出库记录（1对多）：一个出库申请可分多次出库
```
