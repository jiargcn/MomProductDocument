# 盘点管理

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：仓库主管、盘点人员、库存管理人员、财务/审计协同人员及需要处理库存差异的业务人员。

## 业务目的与适用范围

盘点管理用于比较系统库存与现场实物，发现差异后按受控流程确认和调整，使库存余额能够持续接近真实现场。它覆盖盘点计划、申请、任务、现场记录、差异对比和盘点调整等环节；重点不是“把数字改成一样”，而是保留差异来源、盘点过程和最终调整依据。

本页讲清盘点的业务主线。盘点范围、冻结规则、复盘机制、差异责任、审批和实际调整动作需在后续盘点管理-维护与查询参考与测试环境中确认。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 盘点范围与时间 | 明确盘哪些仓库、库区、库位、物料或库存状态。 |
| 盘点方式与责任人 | 决定全盘、循环盘点、抽盘等组织方式及任务分配。 |
| 账面库存基准 | 用于与实际扫描/计数结果进行比较。 |
| 批次、包装、容器和库存状态 | 防止仅按物料汇总而遗漏追溯维度。 |
| 作业冻结或并发规则 | 避免盘点期间继续收发造成无法解释的差异。 |

## 一次盘点如何完成

```mermaid
flowchart LR
    A["制定盘点计划或发起申请"] --> B["生成盘点任务"]
    B --> C["现场扫码/计数并形成盘点记录"]
    C --> D["账实差异对比"]
    D --> E["确认原因与盘点调整申请"]
    E --> F["盘点调整记录"]
    F --> G["库存事务与余额结果"]
```

盘点任务组织现场执行，盘点记录保存实盘结果，差异对比说明账实差在哪里，调整申请和记录负责将已经确认的差异转为受控库存结果。发现差异后，不应跳过对比和原因确认直接修改余额。

【示例数据占位：某库位账面 100 件、实盘 96 件。展示盘点任务、实盘记录、差异对比、调整申请、调整记录和库存结果。】

## 关键判断、角色与动作

| 判断点 | 应先确认什么 | 对业务的影响 |
| --- | --- | --- |
| 盘什么 | 仓库/库位、物料范围、库存状态、批次/包装和盘点方式。 | 决定任务范围和账面基准。 |
| 如何记实物 | 扫码、逐件/逐箱计数、容器信息和现场异常。 | 决定实盘记录的可靠性。 |
| 是否存在差异 | 账面数量、实盘数量、地点和追溯维度是否按同一口径比较。 | 决定是否进入差异分析。 |
| 是否可调整 | 差异原因、复核结果、审批和是否存在并发业务。 | 决定是否形成正式调整记录。 |

仓库主管通常组织范围与复核，盘点人员执行现场计数，库存/财务或管理人员确认差异和调整。实际职责、审批和按钮权限需以测试环境配置为准。

## 差异与库存影响

盘点差异可能来自漏扫、错库位、批次/包装混淆、收发未完成、现场损耗或系统记录异常。差异处理应先定位原因，再选择复盘、补录、暂停相关作业或建立调整；最终调整必须可从库存余额反查到盘点记录和调整记录。

| 关联业务 | 应关注什么 |
| --- | --- |
| 库存管理 | 盘点前账面余额、调整后的事务和余额结果。 |
| 库内作业 | 调拨、报废、计划外出入库是否造成未完成差异。 |
| 收货/发料/出库 | 盘点时间窗内是否存在尚未完成或未同步的业务。 |
| 质量管理 | 隔离、冻结或质量状态库存是否应单独盘点。 |
| 终端操作 | PDA 扫码、离线或重复扫描等现场风险。 |

## 查询、详情与联查

| 想解决的问题 | 推荐定位方式 | 建议联查 |
| --- | --- | --- |
| 哪些区域待盘点 | 计划号、任务状态、仓库/库位、盘点日期或执行人。 | 计划/申请、任务明细。 |
| 实盘结果是什么 | 盘点记录号、物料、库位、批次/包装。 | 账面余额、差异对比。 |
| 为什么产生差异 | 差异单、物料/库位、实盘与账面数量。 | 相关收发、库内作业、库存事务。 |
| 调整是否已生效 | 调整记录号、差异来源和处理时间。 | 库存事务、库存余额。 |

详情页建议按“盘点范围与计划、任务与执行、实盘明细、差异对比、调整与审批、库存影响、系统信息”分组。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| 同一物料盘出多条结果 | 核对库位、库存状态、批次、托盘和包装，不要只按物料合并。 |
| 盘点期间仍发生收发 | 记录时间窗和并发业务，按盘点规则决定冻结、重盘或差异解释。 |
| 扫码数量与账面差异很大 | 先复核扫描对象、库位和包装，再检查未完成业务或错放。 |
| 已确认差异但无法调整 | 核对审批、任务状态、相关库存冻结及是否应走其它业务。 |

## 当前限制与待确认事项

- 盘点计划、申请、任务、记录、对比、调整的实际状态与自动生成关系需测试验证；
- 全盘/循环盘点/抽盘的配置、冻结策略和任务分派规则需补场景；
- 差异库存、差异出库和盘点调整对财务/外部系统的影响需跨模块确认；
- PDA 扫码、重复扫描、离线处理和复盘操作需补真实截图与错误示例。
