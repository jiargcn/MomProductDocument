# 库存事务

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。具体查询操作见[库存管理-维护与查询参考](01-库存管理-维护与查询参考.md)。

## 这项查询解决什么问题

库存事务记录一次已经发生的库存变化。它是解释“为什么数量、地点或库存状态变了”的主要入口，而不是供日常人员绕开来源业务直接改库存的工具。

## 字段行为要点（P1 / P2 / P8 / P9）
本页关注：事务类型/业务类型、按记录号检索、变动落在何种粒度、由源业务完成动作产生。表内「行为模式」列（P 码）仅作辅证。事务是结果、不是去选余额，本页不单列拣选专表。完整总表见[库存管理-维护与查询参考](01-库存管理-维护与查询参考.md)「字段说明」。

| 要点 | 说明 |
| --- | --- |
| 主键追溯 | 优先用业务记录号 / 事务号，再带物料与时间窗 |
| 粒度字段 | 物料、库位、库存状态、批次、包装等应与余额口径一致 |
| 门禁 | 培训路径：源业务 → 记录 → 事务；人工创建是否开放见 `GAP-012` |
| 导出等动作 | 前端可有权限码，后端强制鉴权证据不足（`GAP-021`，并挂 `GAP-014`） |

## 与入库链的关系

采购收货记录已确认会形成库存事务；采购退货和采购上架完成后应通过事务验证其库存结果。上架通常更关注地点或状态变化，退货更关注可退对象与余额扣减，实际事务口径仍需分别核对。

## 查询与维护说明

| 场景 | 建议优先显示/筛选 | 联查路径 |
| --- | --- | --- |
| 解释一次变化 | 事务号、业务类型、事务类型、记录号、物料、数量、发生时间。 | 业务记录、来源任务、来源单据。 |
| 查地点或状态变化 | 物料、库位、库存状态、批次/包装、数量。 | 库存余额、库位资料。 |
| 发现异常 | 处理人、时间、来源记录和关联事务。 | 记录明细、接口/操作日志（待补）。 |

当前已确认存在创建库存事务和创建转运日志的服务入口，以及分页、高级查询与导出能力；正式培训应始终优先从来源业务形成事务。人工创建是否开放、可修改哪些信息和权限边界待实测。
