# 库存管理-维护与查询参考

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 用途：配合[库存管理](index.md)使用。本页用于定位预计入、库存事务和库存余额；不替代采购收货、发料、盘点等来源业务的操作说明。

## 快速定位

| 你要查什么 | 建议对象 | 首要定位条件 |
| --- | --- | --- |
| 哪些到货或任务尚未完成 | 预计入库 | 任务号、物料、库存状态、库位。 |
| 为什么库存数量发生变化 | 库存事务 | 事务号、业务记录号、物料、库存状态、包装或批次。 |
| 当前某物料还有多少 | 库存余额 | 物料、库位、库存状态、批次、托盘或包装。 |

## 预计入库：查询待完成预期

预计入库用于表达由任务带来的待完成入库预期，不等同于已入账库存。采购收货任务创建时会形成预计入；其它业务是否形成预计结果需按各自业务验证。

| 建议查看 | 用途 |
| --- | --- |
| 任务号、业务类型 | 找到预计来自哪项待执行工作。 |
| 物料、数量、单位 | 核对待完成的对象和数量。 |
| 库位、仓库、库存状态 | 判断预计会落入何处、以何种状态进入库存。 |
| 批次、托盘、包装 | 对需要追溯的物料核对库存维度。 |

## 库存事务：追溯一次变化

库存事务记录实际业务形成的一次库存变动。查询时优先从业务记录号或事务号定位；若只知道物料，应同时带上地点、状态和批次/包装缩小范围。

| 建议查看 | 用途 |
| --- | --- |
| 事务号、事务类型、业务类型 | 判断这是哪类变化。 |
| 业务记录号 | 反查实际收货、发料、盘点或其它业务结果。 |
| 物料、数量、单位 | 核对变化对象和变化量。 |
| 库存状态、库位、批次/包装 | 判断变化发生在什么库存维度。 |
| 发生时间、处理人 | 辅助追溯处理时点和责任线索。 |

当前实现存在批量创建库存事务的服务入口，但正常业务人员是否可直接人工创建、可修改哪些内容，应以页面权限和测试环境为准；培训中应优先通过来源业务形成事务。

## 库存余额：查看当前结果

余额查询应把物料、库位、库存状态、批次、托盘和包装作为一个整体理解。即使物料相同，只要其中一项不同，也可能是不同的库存结果和不同的可用范围。

| 推荐顺序 | 查询字段 | 为什么先看它 |
| --- | --- | --- |
| 1 | 物料、库位、库存状态 | 确认要找的物料在哪个地点、是否处于可用状态。 |
| 2 | 批次、托盘、包装 | 排除同物料不同追溯单元混在一起的情况。 |
| 3 | 数量、单位 | 判断当前结果是否满足需求。 |
| 4 | 仓库、库区、库位组、货主 | 在多仓、多归属场景中进一步定位。 |
| 5 | 最近事务号和时间 | 继续追溯最近一次变化。 |

## 详情与联查建议

| 详情分组 | 业务内容 | 快速跳转 |
| --- | --- | --- |
| 库存识别与数量 | 物料、数量、单位、货主。 | 物料基本信息。 |
| 地点与状态 | 仓库、库区、库位、库存状态、冻结信息。 | 仓库/库位资料。 |
| 批次与包装 | 批次、托盘、包装及有效期信息。 | 批次或包装追溯（后续补充）。 |
| 来源与最近变化 | 最近事务、业务记录、来源任务。 | 库存事务、对应业务记录。 |
| 系统信息 | 创建、更新时间和操作线索。 | 操作或接口日志（后续补充）。 |

## 查询前快速核对

| 场景 | 最少先核对什么 |
| --- | --- |
| 查“是否已入库” | 业务记录是否已生成，是否已有库存事务，再查余额。 |
| 查“为什么数量不同” | 物料、单位、库位、库存状态、批次/包装是否一致。 |
| 查“为什么不能使用” | 库存状态、冻结、地点和所在业务的选择条件。 |
| 查“任务是否还在等待” | 任务状态与预计入是否仍存在。 |

## 仍待业务确认

- 三类库存页面的真实详情 Tab、默认列表列和快速跳转过滤条件；
- 人工维护预计入、库存事务或库存余额的页面入口、权限和审计要求（`GAP-012`、`GAP-019`～`GAP-021`）；
- 余额唯一粒度与详情反查事务过滤条件的技术改进（`GAP-005`、`GAP-022`）；
- 冻结、质量状态、上架和盘点差异对可用库存的具体影响。
