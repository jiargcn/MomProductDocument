# 销售价格单

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。具体操作见[销售价格单-维护与查询参考](07-销售价格单-维护与查询参考.md)。

## 这项资料解决什么问题

销售价格单把特定客户、物料和币种下的价格及有效期集中维护，避免业务人员在订单或结算环节临时判断价格。它依赖客户物料关系：新增前必须已存在对应的客户—物料关联。

## 维护时先做的判断

| 判断 | 业务含义 |
| --- | --- |
| 适用客户与物料是否明确 | 防止价格被错误客户或物料使用；客户与物料须已建立客户物料关系。 |
| 币种、价格和可用状态是否完整 | 价格不可为负；缺少可用状态无法完成保存。 |
| 是否已存在生效配置 | 同一客户—物料（及币种、期间）并存或切换规则仍待验证（`WMS-SP-001`）。 |

## 关键字段业务角色

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- | --- |
| 客户代码 | P2 / P6 | 价格适用客户 | 必填；须为已存在客户；物料选择依赖客户 | 改客户等于改适用范围 |
| 物料代码 | P2 / P6 | 价格适用物料 | 必填；须与客户已有客户物料关系 | Web 编辑锁定；勿绕过页面改物料 |
| 币种 | P1 | 计价币种 | 必填；字典选择 | 与结算口径不一致会误导对账 |
| 价格 | P7 | 销售参考价 | 必填；不可为负 | 勿把临时询价当长期口径 |
| 是否可用 | P1 / P9 | 生命周期 | 必填；业务取价常要求可用 | 停用前评估订单/发货/结算是否仍读取 |
| 生效/失效时间 | P10 | 适用期间 | 可维护；期间校验与重叠拦截 ❓ | 不要假设系统会自动让旧价失效 |

## 当前可证实行为（相对接口旁路）

| 场景 | Web 日常操作 | 接口/服务旁路差异 | 培训口径 |
| --- | --- | --- | --- |
| 新增 | 校验客户存在、客户—物料已关联、币种/价格/可用 | 未见按「客户+物料+币种」的显式唯一拦截 | 先建客户物料，再维护价格；重复行风险见 `WMS-SP-001` |
| 编辑 | 客户代码、物料代码在表单中禁用 | 更新时服务会保留原物料代码；客户代码等仍可能被接口改写 | 日常按「适用范围不可改」执行；确需变更应新建并受控切换 |
| 导入 | 模板含客户、物料、币种、价格、可用与有效期 | 已有记录匹配键当前按物料代码定位，与「客户+物料」业务取价口径不一致 | 导入前小样试导；勿假设覆盖目标一定是「该客户下该物料」 |
| 取价线索 | 业务侧可按客户+物料查可用价格 | 有效期是否参与过滤未闭合 | 取价优先级与订单/发货/结算读取见 `WMS-SP-002` |
| 删除 | 可删 | 未见下游引用拦截 | 已使用价格优先停用，不直接删除（`WMS-SP-003`） |

## 使用与风险

- 已在用价格不宜直接覆盖历史口径；应采用新建并受控切换的方式处理调整。
- 有效期可录入，但系统尚未证实会校验时间先后、拦截重叠期间或按期间自动挑选价格。
- 导入匹配键与日常业务取价维度不一致时，批量覆盖可能改错行；必须以试导结果和错误回执为准。

## 待确认事项

- `WMS-SP-001`：客户+物料（+币种/期间）唯一性与导入匹配键是否应统一。
- `WMS-SP-002`：订单、发货、结算等场景的读取优先级与过滤条件。
- `WMS-SP-003`：已使用后的编辑、停用和删除保护。
