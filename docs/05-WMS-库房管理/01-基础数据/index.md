## 当前菜单与归属事实（BATCH-01 首轮取证）

> 取证依据：`reference/menu.csv`（`dev` 基线）和 Web 源码目录 `ui/src/views/wms/basicDataManage`。本节说明当前可见入口和归属边界，不替代每个实体页的字段、导入与权限规则。

### 当前 WMS 基础数据菜单

| 当前菜单分组 | 已确认页面 | 当前文档处理原则 |
| --- | --- | --- |
| 客户管理 | 销售价格单、客户发货预测。 | 属于 WMS 使用入口；客户、物料等被引用主数据仍应在 DBC 页面说明维护规则。 |

| 单据设置管理 | 事务类型、计划设置。 | 是申请、任务、记录与库存事务之间的重要配置入口；字段、状态和引用关系后续按实体单独取证。 |
| 标签管理 | 标签规则设置、标签信息、采购件标签、制造件标签、器具标签、库位标签、叫料标签、标签模板。 | 当前菜单位于 WMS，但长期目标归属为 Infra 平台能力；本页只说明 WMS 使用入口，平台能力的通用规则应在基础设施文档维护。 |

### 与 DBC / Infra 的边界

- 菜单数据同时显示：DBC 下有“业务类型”“单据设置管理”“系统全局规则”等配置入口；WMS 下也存在自己的单据设置、标签及日历入口。当前不能仅按菜单名称推定唯一维护责任。
- 标签、条码、模板和打印按已确认的产品方向应逐步收敛为 Infra 平台能力；在完成迁移前，WMS 页面仍是实际业务使用入口。后续文档必须标明“当前入口”与“目标归属”，避免把目标结构误写为现状。
- 采购、上架、发料、收货、盘点等业务页引用本分组配置时，应链接到具体配置实体页；本分组入口不重复维护字段字典。

### 后续实体取证顺序

1. 事务类型：入/出库存动作、业务类型、库存事务和余额更新的挂接。
2. 计划设置与日历：申请/任务生成、业务日期、账期控制和关账影响。
3. 标签与条码：模板、规则、打印任务、业务单据与终端扫描的关联；同步补 Infra 平台归属说明。
4. 科目与 ERP 映射：当前保存字段、导入规则、第三方接口与失败追溯。
