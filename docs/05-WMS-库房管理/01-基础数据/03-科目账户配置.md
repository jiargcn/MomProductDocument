# 科目账户配置

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。具体操作见[科目账户配置-维护与查询参考](08-科目账户配置-维护与查询参考.md)。

## 这项资料解决什么问题

科目账户配置把业务使用场景与会计账户、成本中心、项目及产品类规则对应起来，为库存变化和结算结果提供可解释的归属。

## 维护时最重要的判断

| 判断 | 业务含义 | 建议 |
| --- | --- | --- |
| 使用场景是否清楚 | 决定何时使用该账户。 | 以业务类型和结算规则为依据，不只按账户名称判断。 |
| 成本中心、项目是否有效 | 决定归集能否正确落位。 | 先从 ERP 成本中心/项目资料确认可用值。 |
| 是否由产品类带出 | 决定维护责任和修改边界。 | 勾选“来自产品类”时必须同时给出产品类类型；不要重复维护已有产品类规则。 |

## 关键字段业务角色

| 字段/配置点 | 行为模式 | 在系统中的作用 | 关键行为要点 | 维护时要警惕什么 |
| --- | --- | --- | --- | --- |
| 代码 | P6 | 科目配置识别号 | 新增必填；创建时按代码查重 | Web 更新保留原代码；勿绕过页面改码 |
| 成本中心类型 | P1 / P2 | 成本中心分类口径 | 导入必填且须属于已维护的成本中心类型 | 类型错误会导致导入失败或选错范围 |
| 会计账户 / 名称 | P12 | 入账解释 | 导入必填；页面亦可维护 | 名称应与财务口径一致 |
| 是否来自产品类 / 产品类类型 | P12 | 是否引用产品类账户 | 来自产品类时产品类类型必填；非来自则清空类型 | 与产品类页职责划分不清时先核对财务规则 |
| 项目是否必填 | P1 | 业务录单时项目约束 | 导入必填 | 错配会导致现场无法过账或漏项目 |
| 成本中心默认/限制范围 | P2 / P11 | 可选成本中心集合 | 导入可校验可用成本中心 | 填错会缩小或放错可选范围 |
| 是否可用 | P1 / P9 | 生命周期 | 新增必填；导入成功行常强制可用 | 停用影响业务选账户 |

## 当前可证实行为（相对接口旁路）

| 场景 | 当前可证实行为 | 培训口径 |
| --- | --- | --- |
| 新增 | 按代码查重；若标记来自产品类则产品类类型不能为空 | 代码应稳定；产品类带出规则一次配清 |
| 编辑 | 服务回填并保留原代码、原成本中心类型 | 日常按“代码与类型不可改”执行 |
| 导入 | 校验代码、类型、会计账户/名称、产品类来源、项目必填及成本中心范围；任一预检失败则本批不写入 | 必须先看错误回执再重导；不要假设部分成功 |
| 导入匹配 | 写入阶段按代码定位既有行（与预检中“代码+类型”表述不完全一致） | 同一代码多类型并存时以专项结论为准（`WMS-SA-001`） |
| 删除 | 未见下游业务引用拦截 | 在用配置优先停用 |

## 待确认事项

- `WMS-SA-001`：代码唯一性与导入策略（代码维 vs 代码+类型维）。
- `WMS-SA-002`：ERP 成本中心、项目来源与维护责任。
- `WMS-SA-003`：各业务场景选择规则。
