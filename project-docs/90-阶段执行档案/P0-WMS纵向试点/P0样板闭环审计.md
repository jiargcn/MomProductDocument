## 2. 可复用的已验证方法

| 方法 | P0 证据 | 后续使用规则 |
| --- | --- | --- |
| 字段真实性校正 | 物料页与库存三对象已按 DDL → DO → DTO/VO → 前端配置核对。 | 先写证据路径，再写技术字段；发现冲突进入差距总账。 |
| 动作与服务流转 | 采购收货、预计入删除、事务更新余额均已区分前端入口、后端服务与数据影响。 | 不把“保留 API”或“前端隐藏按钮”写成业务能力。 |
| 详情与跳转 | 采购收货的上下游跳转、余额页的事务 Tab 都已核对过滤条件。 | 必须写实际传参，并与业务唯一粒度比对。 |
| 导入规则 | 物料模板读取 VO 与保存 VO 的差异已记录。 | 同时核对模板、导入 VO、保存 VO、服务模式和错误回执。 |
| 权限取证 | 库存页已区分前端 `hasPermi` 与后端 `@PreAuthorize`。 | 页面可见性不能替代接口授权结论。 |
| 图示与示例 | 采购收货和库存页已配置流程图、库存粒度示例、截图占位。 | 复杂业务至少保留一个图示/示例任务，不凭空补业务数据。 |

## 4. 审计结论

P0 已达到“**可复制的文档样板**”标准：后续页面可以复用章节结构、证据层次、图示/示例任务和差距记录方式；不能复用任何具体业务字段、状态、导入、终端、权限或接口规则。

P0 尚未达到“**测试环境行为完全验证**”标准。未完成项已进入本页、[逐页取证与回填任务清单](../../01-工作总控/逐页取证与回填任务清单.md)和[问题与待确认总账](../../03-问题与待确认/问题与待确认总账.md)，不阻断后续逐页取证与已启动的四批占位范围。
