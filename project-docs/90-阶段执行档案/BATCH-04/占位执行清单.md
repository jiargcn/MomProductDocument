# BATCH-04 占位执行清单

> 状态：已启动。
> 范围：SCP + PS + API。
> 原则：只建立供应链协同、排程管理和 API 参考的结构与取证任务；不复制未经取证的订单字段、排程算法参数、接口路径、权限码或第三方回写规则。

## 1. 执行边界

| 项目 | 本批处理方式 |
| --- | --- |
| SCP | 供应商协同、采购订单、要货预测、要货计划、发货协同、采购跟踪、发票结算。 |
| PS | 排程基础数据、维护订单、执行生产排程、结果对比、结果查询、生产计划查询。 |
| API | 统一定义接口文档结构、字段真实性、权限、幂等、错误码、日志和第三方挂接口径。 |
| 业务模块引用 | SCP/PS 页面只写业务规则和挂接对象；接口细节在 API 参考或专项接口页维护。 |
| 第三方系统 | ERP、SAP、供应商门户、报表平台、排程引擎等只写待核验挂接矩阵，不猜测真实协议。 |

## 3. 每页首轮占位最低输出

| 章节 | 首轮最低要求 |
| --- | --- |
| 页面定位 | 明确页面属于供应链协同、排程输入、排程执行、结果查询或接口参考。 |
| 字段真实性 | 标记 DDL、DO、DTO、VO、前端配置、接口 DTO 待校正。 |
| 状态动作 | 保留订单确认、计划发布、发货、收货、排程执行、结果发布等动作占位。 |
| 接口挂接 | 标记 ERP/SAP/SCP/WMS/MES/报表/排程引擎的同步方向和待核验点。 |
| 权限与数据范围 | 区分采购方、供应商、计划员、管理员、接口账号和数据范围。 |
| 图示与示例 | 保留协同流程图、排程输入输出图、接口调用示例和结果样例。 |
| 差距记录 | 对字段失真、状态不明、接口不明、实现归属不明写入差距总账。 |
