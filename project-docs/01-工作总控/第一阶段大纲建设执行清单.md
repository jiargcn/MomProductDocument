# 第一阶段大纲建设执行清单

> 当前范围：仅 DBC 与 WMS。
> 前置完成：[P0 双样板复审与定稿说明](P0双样板复审与定稿说明.md)和[正式业务文档标准模板](../02-标准与方法/实体说明页标准模板.md)已定稿。
> 本清单的目标：一次性建立能被分派、能被培训、能继续填充的完整文档骨架；不在本轮做全量字段、代码或接口回填。

## 1. 本轮交付边界

| 要做 | 本轮不做 |
| --- | --- |
| DBC/WMS 模块首页改写为业务入门地图。 | System、Infrastructure、MES、QMS、EAM、ANDON、SCP、PS、API 的页面改写。 |
| 每个业务分组建立“分组目的、学习顺序、对象关系、页面清单、异常与素材任务”。 | 未经验证的状态机、权限规则、字段技术名和接口结论。 |
| 每个叶页一次性列全适用章节，并用可读站位说明将来写什么。 | 无差别填满字段表、导入模板、详情字段和源码证据。 |
| 为复杂页面登记流程图、状态图、截图和示例数据需求。 | 绘制尚未被当前资料或测试环境支持的固定图示。 |
| 建立主文档/详细参考的拆分建议。 | 为每个低频简单页面强制新增详细参考页。 |

## 6. 每个分组的交付检查

- [ ] 分组首页采用“分组目的 → 学习顺序 → 对象关系 → 页面清单 → 常见问题 → 素材任务”结构。
- [ ] 每个叶页一次性列出了适用的完整章节；没有空标题、空表或技术化站位。
- [ ] 复杂页面明确需要流程图、状态图、截图或示例数据中的哪些，而非机械全选。
- [ ] 页面明确应采用“一页完成”还是“主文档 + 详细参考”，并说明原因。
- [ ] 未确认规则写成可执行的补充任务，不写成当前事实。
- [ ] 每次改动后检查导航、相对链接和页面渲染；本波次结束后统一运行全站构建。

## 7. 问题不停工规则

缺截图、缺真实状态、缺权限结论或发现字段冲突时，保留业务化站位并登记内部问题，继续建设不依赖该结论的页面大纲。只有同一概念存在互斥定义、且会改变整个 DBC/WMS 信息架构时，才暂停并请求产品负责人决定。
