## 3. 必须回改项

### GATE-001：隔离历史技术正文

**问题：** DBC/WMS 的大量叶页虽已新增业务大纲，但原有 DDL、DO、DTO、VO、Controller、Service、事实卡、BATCH 等技术正文仍在正式 `docs/` 页面中。读者继续向下阅读仍会进入研发底稿式内容，违背“培训材料 + 业务字典”的目标。

**影响：** 不得以“已添加大纲”替代“正文已业务化”；第一阶段不得准入第二阶段。

**处理要求：**

1. 逐页提取仍有价值的业务结论，改写进“业务目的、典型流程、关键维护、查询联查、异常、限制”对应章节；
2. 将字段技术名、源码路径、表名、服务调用、证据冲突和差距编号迁入 `project-docs/` 的对应证据页或问题记录；
3. 删除或移出正式页中的技术事实卡和 BATCH 取证段落；
4. 每批迁移后运行技术标记扫描、`git diff --check` 和 `mkdocs build`；
5. P0 样板作为验收基准，不允许回退到技术底稿写法。

### GATE-002：完成产品负责人准入复核

在 GATE-001 完成后，请产品负责人用以下问题复核：

1. 新人能否在三分钟内说出 DBC/WMS 分别解决什么问题？
2. 培训人员能否用 P0 流程图和示例讲清一笔业务？
3. 使用者能否找到维护、查询、异常和上下游联查入口？
4. 页面是否仍有必须依赖研发背景才能看懂的正文？
