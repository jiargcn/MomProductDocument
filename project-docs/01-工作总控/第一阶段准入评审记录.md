本轮不以“文件存在”代替审阅。除人工复核两个模块首页和四个 P0 页面外，还按正式模板对页面标题、可分派站位、正文边界与本地链接执行了范围检查。结果如下：

| 审阅对象 | 覆盖与检查方式 | 结果 |
| --- | --- | --- |
| 模块首页 | 人工复核 DBC、WMS 各 1 页的模块定位、角色、学习顺序、协作边界和素材任务。 | 2/2 页面具备入门地图结构。 |
| DBC 分组 | 检查 8 个分组首页是否同时具备分组目的、学习顺序、对象关系、页面说明、常见问题和素材任务。 | 8/8 通过。 |
| WMS 分组 | 检查 12 个非 P0 分组首页是否具备目的、流程、角色动作、库存/终端、查询、限制和素材任务；采购收货按 P0 页面人工复核。 | 12/12 通过；采购收货另见 P0 结论。 |
| DBC 叶页 | 检查 36 个非 P0 叶页是否具备业务目的、维护时机、准备、典型使用、变更、查询、问题、限制和素材任务。 | 36/36 具备章节结构，但除供应商样板外均需补回真实业务内容。 |
| P0 主文档与维护与查询参考 | 人工复核物料主文档/维护与查询参考、采购收货主文档/维护与查询参考的阅读入口、互补边界、流程/关系图、查询与异常说明。 | 4/4 通过；未发现正式页直接链接内部证据底稿。 |
| 正文与链接边界 | 扫描 DBC 47 页、WMS 15 页的技术标记和内部底稿路径；检查本地相对链接。 | 技术标记 0；正式页直接内部证据路径 0；本地链接断链 0。该结果只能证明隔离，不足以证明正文可用于培训。 |

上述检查证明文档结构与表达边界；它不能替代“正式页确有业务内容”的人工审阅，也不能替代测试环境中的真实状态、权限、截图、示例数据和培训效果验证。

## 3. 准入闸门处理结果

### GATE-001：隔离历史技术正文（回改中）

**原问题：** DBC/WMS 的大量叶页虽已新增业务大纲，但原有 DDL、DO、DTO、VO、Controller、Service、事实卡、BATCH 等技术正文仍在正式 `docs/` 页面中。读者继续向下阅读仍会进入研发底稿式内容，违背“培训材料 + 业务字典”的目标。

**纠偏结论：** 技术字段名、源码路径、表名和证据冲突不应作为正式正文出现；但“隔离”不等于删除整页原有内容。每页必须先把既有内容分为可确认业务结论、内部证据、错误或未证实信息，再分别迁移。

**已完成的纠偏样板：**

1. P0 物料、采购收货保持“主文档 + 维护与查询参考 + 字段证据”质量基准；
2. 供应商已恢复业务主文档、维护与查询参考与独立字段证据页；
3. 库存管理已恢复业务主文档、维护与查询参考，并复用库存对象与库存粒度字段证据页；
4. 技术标记扫描结果仍为 DBC 47/47 页、WMS 15/15 页 0 个命中，但这只作为正文边界检查；
5. 余下页面必须先完成“三分法迁移”，不得把技术事实卡和 BATCH 取证段落带回正式页，也不得只留下占位。

**剩余回改要求：**

1. 从清理前版本逐页提取现有内容，先形成“可改写业务结论 / 应下沉证据 / 错误或未证实”迁移清单；
2. 正式页至少保留当前可确认的业务目的、典型使用、关键判断、查询/联查和常见问题；仅对确实缺少资料的内容使用可分派站位；
3. 每页均覆盖维护与查询信息；满足统一拆分判定的页面采用“主文档 + 维护与查询参考”，其余页面在主文档中合并相关章节；
4. 技术字段、源码路径、完整字段表、权限注解和差距编号迁入按业务对象拆分的 `project-docs/04-资料与证据/字段证据/`；
5. 每批回改后执行人工内容抽查、技术标记扫描、`git diff --check` 和 `mkdocs build`。

### GATE-002：完成产品负责人准入复核

在 GATE-001 完成后，请产品负责人用以下问题复核：

1. 新人能否在三分钟内说出 DBC/WMS 分别解决什么问题？

3. 使用者能否找到维护、查询、异常和上下游联查入口？
4. 页面是否仍有必须依赖研发背景才能看懂的正文？
