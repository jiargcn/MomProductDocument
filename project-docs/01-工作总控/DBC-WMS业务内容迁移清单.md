## 4. 完成判定

每页从“占位大纲”转为“可用第一阶段页面”，至少同时满足：

- 有可由当前资料支撑的业务目的与典型使用，不只有 `【待补充】`；
- 已有业务流程、查询、异常或变更影响时，保留并改写为业务语言；
- 技术字段、源码路径和取证冲突不留在正式正文；
- 每个业务均覆盖维护与查询信息；符合拆分判定的页面有主文档、同名维护与查询参考和按需建立的内部字段证据三层分工；
- 缺失的状态、权限、截图和示例以可分派站位或问题任务登记；
- 通过本地链接检查、`git diff --check` 和 `mkdocs build`。

完成全部 A、B、C 批次后，才可重新执行第一阶段 GATE-001/GATE-002 准入评审。
