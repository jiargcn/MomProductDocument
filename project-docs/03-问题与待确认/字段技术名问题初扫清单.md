# 字段技术名问题初扫清单

> 状态：FTA-001 初扫结果。  
> 扫描范围：`docs/**/*.md`。  
> 扫描时间：2026-07-15。  
> 使用方式：本页用于安排校正优先级，不直接替代字段映射表。

## 1. 初扫结论

现有文档中的字段表和英文技术名分布较广，不能按“少数页面修补”处理。后续必须先建立字段证据链，再回写业务页面。

本次初扫识别到的主要风险包括：

1. 多个旧页面存在根据中文名推导出的英文名，未能证明与后端 DO、DTO 或数据库列一致。
2. 部分页面把前端字段键、接口字段、数据库列和显示字段混写在同一张表中。
3. 一些模块入口页包含历史 ER 图或历史字段草稿，其中字段名很可能不是当前实现。
4. WMS、MES、QMS、EAM、SCP 的旧草稿页字段量较大，是后续误用风险最高的区域。
5. DBC 主数据页面数量多，虽然结构较清晰，但每张表仍需逐项用 DDL 和后端源码确认。

## 5. 校正规则

1. 不直接批量替换字段名。
2. 不用中文字段名反推英文技术名。
3. 不把前端字段键直接写成数据库列。
4. 不把显示字段、计算字段、选择器带回字段写成可导入字段。
5. 不把历史设计字段当作当前实现。
6. 无法确认时保留中文业务含义，技术字段写“待核验”。
