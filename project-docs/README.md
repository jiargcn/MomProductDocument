# MOM 产品文档建设工作台

> 范围：本目录是产品文档建设的内部工作区，不参与 MkDocs 产品站点发布。

这里保存用于建设、核验和持续维护正式产品文档的材料，包括任务总控、编写标准、问题记录、资料底表和阶段执行档案。它们帮助团队协作与追溯，但其中的待确认项、源码线索和历史结论均不能直接作为培训或业务事实。

## 目录职责

- `01-工作总控`：决定当前做什么、完成到哪里；每次阶段切换或恢复工作时优先更新。
- `02-标准与方法`：可复用的页面结构、取证规则和批量处理闸门。
- `03-问题与待确认`：产品差距、证据冲突和资料缺失；发现问题即登记，不阻塞其它可执行工作。
- `04-资料与证据`：输入模板与字段级核验底表。
- `90-阶段执行档案`：已完成阶段的过程证据，默认只读，仅在复盘或重开该阶段时更新。
- `99-结构治理`：目录边界、归档原则和迁移记录。

## 与产品站点的边界

- 面向用户、培训、实施和业务查询的稳定结论写入 `docs/`，并在 `mkdocs.yml` 中导航。
- 本目录只保存“如何形成结论”的过程材料，不进入产品站点导航。
- 已确认且需要公开说明的限制、变更和版本能力，应由产品负责人评审后转写到 `docs/15-版本路线图/`；不要把问题总账直接当作对外版本说明。
