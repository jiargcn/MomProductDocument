## 后续填充原则

1. 一张图只回答一个问题：流程、对象关系、状态或数据影响，不混合。
2. 状态图必须以测试环境验证为前提；无法验证时保留图示占位和需确认内容。
3. 截图必须脱敏，并在图旁说明读者应关注的按钮、字段或异常提示。
4. 示例数据应至少覆盖正常与一个异常分支，避免只展示理想流程。
