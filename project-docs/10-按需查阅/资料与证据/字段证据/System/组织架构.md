# System：组织架构字段证据

> 对应正式页面：[业务说明](../../../../docs/12-系统管理/02-组织架构/index.md)、[维护与查询参考](../../../../docs/12-系统管理/02-组织架构/组织架构-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。内部证据页。

## 1. 部门（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 表/对象 | `system_dept` / `DeptDO` | 租户内部门树。 |
| 树字段 | `parentId`（根 `0`）、`name`、`sort`、`status` | 标准树。 |
| 其它字段 | `leaderUserId`、`phone`、`email`、`deptNumber` | 负责人与联络信息。 |
| 菜单 | `system/dept`（`menu.csv` 部门管理） | System 入口。 |
| 租户 | 继承 `TenantBaseDO` | 部门数据按租户隔离。 |

## 2. 与用户、权限的关系

| 关系 | 证据 | 结论 |
| --- | --- | --- |
| 用户部门 | 用户对象含部门归属（配合数据权限） | 角色五档范围依赖用户部门。 |
| 数据范围 | `getDeptDataPermission` + `DeptDataPermissionRule` | 见数据权限证据页。 |
| 岗位 | `system_post` / `system_user_post` | 独立于部门树；见岗位证据页。 |

## 3. 与工厂/仓库边界

| 项 | 结论 |
| --- | --- |
| System 部门树是否包含工厂/仓库层级 | **否**（未证实为同一模型）。 |
| 工厂/仓库维护位置 | DBC/WMS 业务主数据。 |
| 旧规划“企业—工厂—部门” | 仅作参考；不可写成当前 System 已实现三级。 |
