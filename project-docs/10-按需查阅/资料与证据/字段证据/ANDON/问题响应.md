# 字段证据：ANDON / 问题响应

> 对应正式页面：[业务说明](../../../../docs/09-ANDON-异常管理/02-问题响应/index.md)、[维护与查询参考](../../../../docs/09-ANDON-异常管理/02-问题响应/问题响应-维护与查询参考.md)。
> 基线：测试环境 / `dev` / 2026-07-15。

## 1. 菜单与对象

| 菜单 | 对象 |
| --- | --- |
| 问题响应链配置 | `andon_problem_resp_chain_config` |
| 消息配置表 | `andon_message_config`（及发送 `andon_message_send`） |
| （后端有、菜单未单列） | 整改 `andon_problem_rect`、整改跟踪、问题文档、OEE 确认 |

## 2. 响应链配置字段

| 字段 | 含义 |
| --- | --- |
| problemClassification | 问题分类（生产/设备/模具/物流/质量/工艺/环境等） |
| problemPost / problemPostCode | 响应岗位 |
| stanDuration | 标准时长 |
| managerPost / managerPostCode | 上级岗位（升级对象） |
| intervalTime | 通知间隔 |
| msgId | 消息配置编号 |
| serialNumber | 岗位顺序（从 1） |

## 3. 消息配置

`msgId` / `msgType` / `msgDoc`。发送表另有投递记录字段。通道实现归属平台消息能力；ANDON 只配业务消息内容键。

## 4. 整改

`rectUser`、计划起止、`rectDoc`、`rectStatus`；跟踪表另有跟踪过程。与问题主记录的强绑定界面待环境确认。

## 5. 边界

- 响应链解决「谁在标准时长内到岗、超时通知谁」；**不**创建 EAM 维修工单（除非现场另有集成，当前服务层未证实）。
- 问题过程记录（到岗签到）是响应执行痕迹，见故障记录证据。

## 6. 待确认

- 整改/OEE 确认菜单在测试环境是否挂出。
- 与系统消息中心模板的映射。
