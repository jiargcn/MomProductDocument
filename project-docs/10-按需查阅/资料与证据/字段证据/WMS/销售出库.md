# WMS：销售出库字段证据

> 对应正式页面：[业务说明](../../../../docs/05-WMS-库房管理/10-销售出库/index.md)、[维护与查询参考](../../../../docs/05-WMS-库房管理/10-销售出库/销售出库-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。
> 内部证据页；不得从正式业务文档直接链接本页。

## 1. 对象关系

| 业务对象 | 数据对象 | 关键关联 | 证据结论 |
| --- | --- | --- | --- |
| 发货申请 | `request_deliver_main` / `request_deliver_detail` | 申请单号、主表 ID。 | 保存客户、来源单据、物料和待发货明细。 |
| 发货任务 | `job_deliver_main` / `job_deliver_detail` | 任务单号、来源申请单号。 | 保存现场可执行的发货任务与库存定位信息。 |
| 发货记录 | `record_deliver_main` / `record_deliver_detail` | 记录单号、来源申请、来源任务。 | 保存实际执行结果，是库存事务和后续处理来源。 |
| 发货撤销记录 | `record_deliver_cancel_main` / `record_deliver_cancel_detail` | 原发货记录。 | 保存撤销结果；独立菜单入口。 |
| 发货计划 | `plan_deliver_main` / `plan_deliver_detail` | 计划单号。 | 可作为申请来源；独立菜单。 |
| 预计出库 | `expectout`（库存对象） | 任务号等关联。 | 申请处理生成任务时创建；任务执行完成后清理。 |

## 2. 主表业务字段

| 中文名称 | 申请/任务/记录字段 | 说明 |
| --- | --- | --- |
| 业务类型、状态 | `business_type`、`status` | 三类对象均有；记录状态与申请/任务状态可能不是同一口径。 |
| 客户与交付地点 | `customer_code`、`customer_dock_code` | 客户代码与客户月台；导入时存在客户—月台一致性校验线索。 |
| 来源与计划 | `deliver_plan_number`、`so_number`、`customer_deliver_number`、`customer_asn_number` | 发货计划、销售订单、客户交货单等来源信息。 |
| 备货来源 | `stockup_record_number` | 可关联备货记录；备货链本身为独立对象族。 |
| 运输信息 | `carrier_code`、`transfer_mode`、`vehicle_plate_number` | 承运、运输方式、车牌；选择器来源待继续核验。 |
| 发货类型/数据来源 | `deliver_type`、`data_source` | 区分业务子类型与数据来源；成品直接发货等入口可能使用类型参数。 |
| 库存状态范围 | `in_inventory_statuses`、`out_inventory_statuses` | 限制可用来源/目标库存状态。 |
| 在途库位 | `use_on_the_way_location` | 是否使用在途库位；与寄售/在途子域边界待专项核验。 |
| 记录后续信息 | `out_transaction_type`、`interface_type`、`erp_voucher_number` | 用于库存和外部接口追溯。 |

## 3. 明细与库存定位字段

| 分类 | 关键字段 | 说明 |
| --- | --- | --- |
| 物料与数量 | 基类明细中的物料、单位、数量字段；申请明细另有 `unexecuted_qty` | 未执行数量用于跟踪剩余待发。 |
| 销售订单行 | `so_number`、`so_line` | 任务明细可保存订单号/行；申请/记录明细保留行号。 |
| 库存定位（申请） | `batch_id`、`package_number`、`pallet_number`、`inventory_status`、`location_code` 等 | 来自双向申请明细基类。 |
| 库存定位（任务/记录） | `from_*` / `to_*` 库位、状态、批次、包装、货主等 | 发货执行按来源侧核验余额并形成出库结果。 |
| 在途库位 | 任务明细 `on_the_way_location_code` | 在途场景专用；未证实前不写入正式主流程。 |

## 4. 页面、导入与终端证据

| 事项 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| Web 菜单 | `deliver-request-main`、`deliver-job-main`、`deliver-record-main`、`deliverRecordCancelMain`、`deliver-plan-main`；同组还有备货、客户退货、销售结算出库等。 | 各入口默认筛选、按钮可见条件。 |
| 申请处理 | `DeliverRequestMainService` 在处理后 `generateJob`，并创建预计出。 | 自动提交/同意/处理策略与直接生成记录分支。 |
| 任务执行 | `DeliverJobMainService` 执行后按库存管理精度、物料、包装/父包装、批次、来源库位和库存状态核验余额，生成记录与出库事务并清理预计出。 | 少发/多发默认策略、部分执行与取消恢复。 |
| 记录撤销 | `DeliverRecordMainService.cancelDeliverRecordByNumber` 可生成冲销事务与撤销记录；另有接口信息创建路径。 | 冲销接口仅在规则开启且接口类型匹配时异步生成（见 GAP-068）。 |
| 导入 | 申请服务存在 `importDeliverRequestList`、`importTemplate` 及客户月台/库位解析。 | 模板字段、更新模式和前端入口实测。 |
| PDA | `frontend/pda/src/pages/deliver` 含任务与记录页；菜单另有成品发货任务、成品直接发货。 | 承接时机、扫码校验和与 Web 状态一致性。 |

## 5. 相邻对象（不并入主链结论）

| 对象族 | 菜单/代码线索 | 处理原则 |
| --- | --- | --- |
| 备货 | `stockup-request/job/record` | 独立 ATR；可通过 `stockup_record_number` 关联发货。 |
| 客户退货 | `customerreturn-*` | 独立 ATR + 撤销记录；方向相反。 |
| 销售结算出库 | `sale-shipment-request/record` | 独立入口；不写成标准发货必经后续。 |
| 发货调整/寄售退货 | `deliveradjust-*`、`consignmentreturn-*` | 独立对象；保留问题登记。 |
