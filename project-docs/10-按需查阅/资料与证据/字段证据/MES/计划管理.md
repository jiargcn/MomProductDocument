# 字段证据：MES / 计划管理

> 对应正式页面：[业务说明](../../../../docs/06-MES-生产管理/03-计划管理/index.md)、[维护与查询参考](../../../../docs/06-MES-生产管理/03-计划管理/计划管理-维护与查询参考.md)。  
> 基线：测试环境目标 / `dev` / 2026-07-15。内部证据页。

## 1. 菜单与入口

| 证据 | 结论 |
| --- | --- |
| `reference/menu.csv`：生产订单管理 `mes/productionOrderMain/index` | 生产订单主入口。 |
| 生产工单管理 `mes/workorder/index` | 工单主入口。 |
| 工单完工状态 `mes/workOrderComplete/index`；工单状态流转日志 `mes/workorderStatusLog/index` | 查询辅助。 |
| 另有 `workOrderManagement`、`workOrderExecuteAuto`、`orderDay`、`workScheduling` 等 | 旁路/历史入口，本刀不展开细则。 |
| WMS `productionplan`「生产计划」 | **不是** MES 计划管理。 |

## 2. 持久化对象

| 业务对象 | 表 / DO | 说明 |
| --- | --- | --- |
| ERP 订单 | `mes_erp_order` / `ProductionOrderDO`（包名 productionOrder） | 包名易误解；表为 ERP 订单。 |
| 生产订单 | `mes_production_order` / `ProductionOrderMainDO` | 菜单生产订单管理。 |
| 生产工单（计划） | `mes_work_order` / `WorkOrderDO` | 含 `processRouteCode`、`taskType`、`parallelGroupId`、`workOrderSort` 等。 |
| 运行工单 | `mes_workorder_info_run` / `WorkorderInfoRunDO` | 下发生成；含任务模式与执行快照线索。 |
| 路线运行明细 | `BasicProcessrouteInfoDetailRunDO` 等 | 下发/开工复制模板到运行。 |

### 2.1 计划工单关键字段（WorkOrderDO）

`productOrderNo`、`workOrderNo`、`workOrderNoMaster`、`workOrderType`、`factoryCode`、`workroomCode`、`lineCode`、`shiftCode`、`materialCode`、`workOrderQuatity`、`materialUom`、`planStartTime`/`planEndTime`、`realStartTime`/`realEndTime`、`status`（字典 `mes_do_status`）、`processRouteCode`/`Name`、`bomCode`/`bomVersion`、`taskType`、`parallelGroupId`、`toStore`/`toInstrument`、`batchCode`/`customBatchCode`、`teamCode`、`workOrderSort`。

### 2.2 生产订单关键字段（ProductionOrderMainDO）

`orderNumber`、`mainOrder`、`erpOrder`、物料与数量族、`baseStartDate`/`baseEndDate`、`releaseDate`、实际起止、`systemStatus`、`status`、`orderType`、`productionWorkshop`、`lockedFlag`、`factory`、`available` 等。

## 3. 接口能力

| 映射 | 已见动作 |
| --- | --- |
| `/mes/production-order` | CRUD、page、senior、import/export；**订单关闭/取消/锁定/验证转换/执行转换** |
| `/mes/production-order-main` | CRUD、page、senior、import/export |
| `/mes/work-order` | CRUD、saveSplitOrderList、update-plan-sort、page/line、import/export；**publish / publish-batch / start / pending / restore / discontinue / close / finish / revoke**；并行组 create/sort/delete/get/deliver/start/pending/restore/discontinue/candidates/execution-layout/resolve-by-trace 等 |
| `/mes/work-order/bulk` | BULK 报工、人工尾箱、待装箱（执行侧，正式计划页不展开） |

下发实现：`WorkOrderDeliverImpl4.workOrderDeliverSingle/Batch` → 写运行工单并处理路线运行明细。  
开工等状态：`WorkOrderServiceImpl4`；复制快照：`ProcessrouteRunSnapshotService.copyTemplateToRun`。

## 4. 状态与枚举

| 项 | 证据 | 正式文档口径 |
| --- | --- | --- |
| 工单状态 | `WorkorderStatusEnum`：0 待下发…9 已退料 | 用中文名表，不在正文写 code |
| 字典 | `DictTypeConstants.MES_DO_STATUS = mes_do_status` | 工单执行状态 |
| 状态日志操作 | `WorkorderStatusLogOperationTypeEnum`：START/DELIVER/REVOKE/DISCONTINUE/CLOSE/FINISH/PENDING/RESTORE/…及自动完工类 | 维护参考「联查」 |
| ERP 转换 | `ErpOrderConvertStatusEnum`：未转换/已转换 | 业务名 |
| ERP 锁定 | `ErpOrderLockStatusEnum`：未锁定/锁定 | 业务名 |
| 任务类型 | `OrderTaskTypeEnum`：SINGLE/BATCH/BULK | 单件/批量/普通批量 |
| 任务模式（追溯） | `OrderTaskModeEnum`：SINGLE/BULK/BATCH | 与 task_type 对应，细节归执行 |

## 5. 明确否定的旧稿

| 旧稿 | 处理 |
| --- | --- |
| 虚构 `ERP_ORDER`/`PRODUCTION_ORDER`/`WORK_ORDER` 英文字段 ER | 废弃 |
| 独立 `MATERIAL_FEED`/`JOB_REPORT` 当作计划管理核心实体 | 投料/报工归执行与 WMS，本页不建伪 ER |
| 臆造 DRAFT/RELEASE 等英文工单状态 | 以 `WorkorderStatusEnum` 中文名为准 |
| 把 WMS 生产计划写成 MES 计划 | 边界已在正式页标明 |
