# 字段证据：SCP / 发货协同

> 对应正式页面：[业务说明](../../../../docs/10-SCP-供应链平台/05-发货协同/index.md)、[维护与查询参考](../../../../docs/10-SCP-供应链平台/05-发货协同/发货协同-维护与查询参考.md)。
> 基线：测试环境 / `dev` / 2026-07-15。

## 1. 菜单与前端

- `supplierdeliverRequestMain`、`supplierdeliverRecordMain`、`supplierPackage`。
- 控制器：`SupplierdeliverRequest*Controller`、`SupplierdeliverRecord*Controller`、`SupplierdeliverInspectionDetailController`。
- 首页：`takeSupplierdeliverCount` / `notTakeSupplierdeliverCount` 等。

## 2. 关键对象

| 业务对象 | 表 | 关键字段 |
| --- | --- | --- |
| 发货申请头 | `scp_request_supplierdeliver_main` | ppNumber、poNumber、poType、到仓月台/时间窗、承运相关 |
| 发货申请行 | `scp_request_supplierdeliver_detail` | poLine、itemCode、qty、包装、批次 |
| 发货记录头 | `scp_record_supplierdeliver_main` | asnNumber、requestNumber、status、purchasereceiptRequestFlag/Number、planArriveTime、vehiclePlateNumber |
| 发货记录行 | `scp_record_supplierdeliver_detail` | 明细数量与物料 |
| 检验明细 | `scp_supplierdeliver_inspection_detail` | 发货检验线索 |

## 3. 同步

- `SupplierdeliverRequestMainServiceImpl` / `SupplierdeliverRecord*ServiceImpl` 经 `ScpWmsInterfaceSyncService` 推送：
  - `RECORD_SUPPLIERDELIVER_MAIN` / `DETAIL`
  - `REQUEST_SUPPLIERDELIVER_*`
  - `TRANSACTION_BATCH` 等
- `interfaceType` 常见值：`SCP-WMS`。
- WMS 侧存在 `CreatePurchasereceiptRequestBySupplierdeliverRecordJob`（由发货记录建收货申请）线索。

## 4. 边界

- 库存收货执行在 WMS；SCP 负责 ASN 协同与推送。
- 勿与 WMS「成品发货」申请/任务/记录混淆。

## 5. 待确认

- 与 QMS 来料检验自动建单关系；Job 启用条件。
