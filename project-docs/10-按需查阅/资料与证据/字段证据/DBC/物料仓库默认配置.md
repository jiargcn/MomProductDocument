# DBC：物料仓库默认配置字段证据

> 对应正式页面：[物料库区配置管理](../../../../docs/04-DBC-主数据管理/01-物料管理/05-物料库区配置管理.md)、[维护与查询参考](../../../../docs/04-DBC-主数据管理/01-物料管理/09-物料库区配置管理-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。

| 事项 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| 页面实际对象 | 菜单名为“物料库区配置”，当前前后端主对象为物料—仓库默认配置；维护物料、仓库、管理精度、入库包装、可用状态和有效期。 | 菜单名是否将改为“物料仓库默认配置”，以及真正库区/库位策略页面的归属。 |
| 选择器 | 页面从可用物料、可用仓库中选择；包装选择器按物料过滤可用物料包装。 | 前端包装字段与后端字段的绑定是否能正常保存。 |
| 服务校验 | 新增/更新校验物料、仓库、管理精度、包装规格和可用状态；普通维护未见组合重复校验。 | 物料查询到空时的完整异常表现；是否需要增加物料+仓库唯一性。 |
| 编辑与删除 | 页面通常锁定物料和仓库；服务更新仍接收完整关系字段，删除未见下游引用检查。 | 旁路修改、删除对库存和作业的实际影响。 |
| 导入 | 模板包含物料、仓库（表头写“区域代码”）、管理精度、入库包装、可用状态、有效期、备注；按物料+仓库判断既有记录。 | 仅更新模式下不存在记录的回执；部分成功时跨系统同步行为。 |
| 外部挂接 | 普通新增、更新、删除会登记供应链平台同步信息。 | 实际发送、失败重试、第三方反馈，以及导入是否同样同步。 |

## 待核验项

| 编号 | 问题 | 影响 |
| --- | --- | --- |
| DBC-IW-001 | 页面名称、导入表头使用“库区/区域”字样，但实际选择和维护的是仓库。 | 培训、模板和业务口径可能误导为库区或库位策略。 |
| DBC-IW-002 | 前端使用的包装字段及选择对象，与后端保存/校验的包装字段及对象不一致。 | 普通 Web 新增或编辑可能保存失败，或产生错误包装口径。 |
| DBC-IW-003 | 物料+仓库组合未见普通维护的重复保护；页面锁定关系字段而服务更新仍可整体写入。 | 并发或旁路调用可能产生重复或错误变更。 |
| DBC-IW-004 | 删除未见库存、收货、上架等引用保护；实际库区/库位推荐能力也未确认。 | 变更与删除风险、以及产品能力边界不清。 |
