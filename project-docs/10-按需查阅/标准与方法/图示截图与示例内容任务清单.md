# 图示截图与示例内容任务清单

> 状态：规划中，作为页面字典和培训材料的增强任务。  
> 基线：测试环境 / `dev` 分支 / 2026-07-15。  
> 目标：避免文档只有文字和字段表；在复杂业务页中预留图示、截图、示例数据和示例流转位置，后续由自动生成或人工补齐。

## 1. 建设原则

图示、截图和示例内容不是装饰，而是为了降低理解成本。只有在以下场景才应加入：

1. 一个页面涉及三个以上对象或上下游单据。
2. 一个动作会产生任务、记录、预计量、事务、余额、接口推送等多个结果。
3. 规则需要通过示例数量、状态变化或异常分支才能讲清楚。
4. 用户需要知道页面上“看哪里、点哪里、查哪里”。
5. 字段表无法表达来源、流向、分组或优先级。

## 2. 内容类型

| 类型 | 用途 | 适合页面 | 占位写法 |
| --- | --- | --- | --- |
| 业务流程图 | 说明业务从触发到完成的主线和异常分支。 | 采购收货、发料、盘点、检验、维修、异常响应 | `【图示占位：说明起点、终点、关键分支和后续资料。】` |
| 数据流/挂接图 | 说明单据、任务、记录、预计量、事务、余额和外部接口之间的关系。 | WMS 库存链路、第三方接口、SCP 收发货、QMS 检验闭环 | `【图示占位：说明对象关系、读者要理解的结果和证据来源。】` |
| 状态流转图 | 说明状态、动作、权限和前置条件。 | 申请/任务/记录、审批、检验、维修工单、ANDON 问题处理 | `【图示占位：说明待验证状态、动作、回退分支和测试资料。】` |
| 详情页分组示意 | 说明详情页如何分组、哪些字段进入哪个区块。 | 主数据、单据、记录、查询页 | `【图示占位：说明分组目的、关注字段和所需页面截图。】` |
| 列表截图 | 说明默认列、查询条件、按钮和常用操作入口。 | 高频业务页、查询页、配置页 | `【截图占位：说明页面区域、需标出内容和脱敏要求。】` |
| 表单截图 | 说明新增/编辑表单、必填项、选择器和禁用字段。 | 主数据、新增申请、系统配置 | `【截图占位：说明表单区域、读者要理解的规则和脱敏要求。】` |
| 示例数据表 | 用一组具体数值解释字段和规则。 | 库存余额、事务、预计量、导入模板、结算 | `【示例数据占位：说明场景、关键数值、预期结果和资料来源。】` |
| 示例业务流转 | 用一条完整业务链讲清楚输入、操作、结果和查询入口。 | WMS/MES/QMS/SCP/EAM 主流程 | `【示例数据占位：说明起点、操作、结果、查询入口和异常分支。】` |
| 异常场景卡片 | 说明常见错误、冲销、撤销、短缺、超收、重复提交等。 | 任务执行、库存、接口、导入 | `【示例数据占位：说明异常触发、禁止绕过的操作、处理结果和所需截图。】` |

## 3. 优先级

| 批次 | 范围 | 必须补充的内容 | 原因 |
| --- | --- | --- | --- |
| P0 | 物料—采购收货—预计入—事务—余额 | 示例业务流转、库存挂接图、列表/表单截图占位、示例数据 | 当前是样板链路，决定后续写法。 |
| P0 | 申请—任务—记录通用模型 | 模型图、状态流转图、端到端示例 | 所有 WMS/MES/QMS/EAM 单据都复用。 |
| P0 | 库存数据挂接模型 | 数据流图、余额粒度示意、事务示例 | 库存逻辑复杂，纯文字容易误解。 |
| P1 | DBC 主数据 | 详情分组示意、表单截图、导入示例 | 主数据页面数量多，适合标准化。 |
| P1 | System/RBAC | 权限关系图、授权示例、数据范围示例 | 权限模型抽象，必须靠示例解释。 |
| P1 | QMS/EAM/ANDON | 状态流转图、异常场景卡片、端到端示例 | 状态与角色协作复杂。 |
| P2 | 低频查询和报表页 | 列表截图、查询示例、导出示例 | 主要用于查询培训。 |

## 4. 页面占位模板

每个需要补图或示例的页面，在对应章节插入以下结构之一：

```markdown
### 图示与示例占位

| 类型 | 放置位置 | 目的 | 资料来源 | 状态 |
| --- | --- | --- | --- | --- |
| 业务流程图 | 核心流程之后 | 说明主流程和异常分支 | 业务规则、源码、测试环境 | 待生成 |
| 列表截图 | 列表字段说明之前 | 说明默认列、查询条件和按钮 | 测试环境 | 待截图 |
| 示例数据 | 字段说明之后 | 用具体数据解释字段和规则 | 测试环境或构造样例 | 待补充 |
```

## 5. P0 样板链路占位要求

| 页面 | 需要增加的占位内容 | 后续填充方式 |

| 物料基本信息 | 新增表单截图、导入模板示例、详情页分组示意、物料被引用关系图 | 前端页面截图 + DDL/VO 证据 + 人工补业务说明 |
| 采购收货 | 端到端业务流转、申请/任务/记录状态流转、库存挂接图、示例数据 | 根据源码生成图示，测试环境补截图和示例单据 |
| 库存管理 | 预计入→事务→余额数据流图、余额粒度示意、事务示例、查询截图 | 根据 P0 字段证据底表生成图示，测试环境补截图 |
| 申请任务记录模型 | 通用模型图、状态动作矩阵、示例单据链 | 从 WMS 试点抽象沉淀 |
| 库存数据挂接模型 | 库存对象关系图、过账时序图、异常冲销示例 | 从采购收货和库存服务取证 |

## 6. 任务拆分

| 编号 | 任务 | 输入 | 产出 | 完成标准 |
| --- | --- | --- | --- | --- |
| VIZ-001 | 识别需要图示和示例的页面 | 页面覆盖总账、模块入口页、复杂业务页 | 图示/截图/示例占位清单 | 每个 P0/P1 页面标明是否需要图、截图、示例。 |
| VIZ-002 | 建立页面占位段落 | 实体说明页模板、业务页结构 | 页面内“图示与示例占位”段落 | 不破坏当前正文，位置合理。 |
| VIZ-003 | 生成模型类图示 | 业务模型、P0 字段证据底表 | Mermaid 或图片源文件 | 图能解释对象关系，不只是重复字段表。 |
| VIZ-004 | 准备测试环境截图清单 | 菜单路径、页面路由、测试账号 | 待截图清单 | 明确每张截图要截什么、遮挡什么、放哪里。 |
| VIZ-005 | 准备示例数据 | 测试环境或构造样例 | 示例单据号、物料、数量、状态变化表 | 示例数据能贯穿业务流转。 |
| VIZ-006 | 回写页面并校验 | 图示、截图、示例数据 | 完整页面 | 图、表、文字互相引用，构建通过。 |
| VIZ-007 | 建立维护规则 | 新增页面审查清单 | 防回归要求 | 复杂页面不能只写字段表，必须判断是否需要图示或示例。 |

## 7. 验收标准

1. P0 样板链路每个页面都有图示/截图/示例占位。
2. 每个占位都说明目的、资料来源和填充方式。
3. 图示不替代文字说明，截图不替代字段证据。
4. 示例数据必须能贯穿“输入—操作—结果—查询—异常”。
5. 后续自动生成图示或人工补截图时，不需要重新设计文档结构。
