# System：数据权限与决策权限字段证据

> 对应正式页面：[业务说明](../../../../docs/12-系统管理/03-用户与权限/02-数据权限与决策权限.md)、[维护与查询参考](../../../../docs/12-系统管理/03-用户与权限/数据权限与决策权限-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。内部证据页。

## 1. 角色数据范围（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 五档枚举 | `DataScopeEnum`：`ALL(1)` / `DEPT_CUSTOM(2)` / `DEPT_ONLY(3)` / `DEPT_AND_CHILD(4)` / `SELF(5)` | 仅部门/本人，无工厂/仓库/货主档。 |
| 角色字段 | `RoleDO.dataScope`、`dataScopeDeptIds` | 自定义范围挂部门 ID 列表。 |
| 配置入口 | 前端 `RoleDataPermissionForm`；后端 `PermissionController.assign-role-data-scope` | 角色侧配置。 |
| 默认值 | 新建角色默认 `ALL` | 未显式收窄前范围最宽。 |
| 聚合 | `PermissionServiceImpl.getDeptDataPermission` | 多角色并集；自定义并入本部门；无角色按 SELF。 |

## 2. 运行时过滤（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 规则引擎 | `DeptDataPermissionRule` + `DeptDataPermissionRuleCustomizer` | 仅对已 `addDeptColumn` / `addUserColumn` 的表追加条件。 |
| System 挂接 | System `DataPermissionConfiguration`：用户 `dept_id`、部门 `id`、用户 `id` | 用户/部门查询可受约束。 |
| WMS 挂接 | WMS `DataPermissionConfiguration` 中 `addDeptColumn` / `addUserColumn` **均为注释** | 角色部门范围未接到 WMS 业务表。 |
| 其它模块 | DBC/MES/QMS/EAM/ANDON/AGV 存在 Customizer Bean | 是否真正登记业务表需逐模块核对，本刀未建全清单。 |

## 3. 与岗位库位的边界（已证实）

| 机制 | 证据 | 与角色 `data_scope` 关系 |
| --- | --- | --- |
| 岗位库位 | `system_post_location` + WMS `@DataScopeLocation` / `DataScopeLocationAspect` | **独立机制**，按岗位关联库位过滤，不是角色五档。 |

## 4. 决策权限（边界结论）

- 未见独立“决策权限”主数据表或菜单。
- 高风险动作仍由 RBAC 权限标识 + 数据范围 + 业务状态 +（若有）业务侧审批共同决定。
- SCP 侧 `DefaultController` 对 `/admin-api/bpm/**` 返回模块已禁用提示；不能把通用审批引擎写成当前已启用 System 能力。

## 5. 本刀不覆盖

- 各业务模块已挂接部门规则的完整表清单
- 字典展示文案与枚举值的环境核对
- 岗位主数据与任务领取细则（见岗位证据页）
