# System：岗位、任务分配与审批主体字段证据

> 对应正式页面：[业务说明](../../../../docs/12-系统管理/03-用户与权限/03-岗位、任务分配与审批主体.md)、[维护与查询参考](../../../../docs/12-系统管理/03-用户与权限/岗位、任务分配与审批主体-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。内部证据页。

## 1. 岗位主数据（已证实）

| 业务对象 | 数据对象 | 说明 |
| --- | --- | --- |
| 岗位 | `system_post` | 岗位主数据；菜单 `system/post`。 |
| 用户↔岗位 | `system_user_post` | 多对多；与 `system_user_role` 独立。 |
| 岗位↔库位 | `system_post_location` | `PostLocationDO` / `PostLocationApi`。 |
| 岗位↔物料 | `system_post_item` | `PostItemDO` / `PostItemApi`。 |

未见岗位与角色的直接关联表。

## 2. WMS 消费（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 库位数据范围 | `@DataScopeLocation` + `DataScopeLocationAspect`，经 `postLocationApi.listByPostIds` | 按用户岗位库位过滤，**不是**角色 `data_scope`。 |
| 任务领取人 | 多类 Job 主表/VO 含 `acceptUserId` / `acceptUserName` | 业务任务承接字段；非 System 通用派工引擎。 |

## 3. 审批 / 工作流边界

| 项 | 证据 | 结论 |
| --- | --- | --- |
| BPM 禁用提示 | SCP `DefaultController` 对 `/admin-api/bpm/**` 返回模块已禁用 | 不能写成当前已启用全站审批中心。 |
| 业务侧引用 | 部分模块存在 `WorkFlowApi` 引用 | 是否在目标环境可用需单独取证；本刀不展开为 System 通用能力。 |

## 4. 与旧规划差异

| 旧规划表述 | 当前证据结论 |
| --- | --- |
| 任务优先分配给岗位 | 仅证实岗位库位/物料条件 + 业务领取人字段，未见通用“按岗位自动派单”引擎。 |
| 通用审批主体规则 | 未证实独立审批主体配置中心已启用。 |

## 5. 本刀不覆盖

- 各业务对 `PostItem` 的完整消费清单
- 转派/委托/超时升级细则
- 环境级 BPM 启停矩阵

## 问题登记

- 岗位消费与派发细则随业务页续作；不阻塞组织等其它 System 切片。
- 与 `GAP-070`（部门数据权限挂接）互补：库位隔离看岗位，部门隔离看角色范围。
