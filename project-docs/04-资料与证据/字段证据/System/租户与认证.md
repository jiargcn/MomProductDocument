# System：租户与认证字段证据

> 对应正式页面：[业务说明](../../../../docs/12-系统管理/01-租户与认证/index.md)、[维护与查询参考](../../../../docs/12-系统管理/01-租户与认证/租户与认证-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。内部证据页。

## 1. 租户（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 租户 | `system_tenant` / `TenantDO` | 名称、联系人、状态、域名、`packageId`、`expireTime`、`accountCount`。 |
| 套餐 | `TenantPackageDO.menuIds` | 套餐=菜单集合上限。 |
| 初始化 | `TenantServiceImpl` 创建角色并按套餐赋菜单 | 新租户可初始化管理角色菜单。 |
| 套餐变更 | `updateTenantRoleMenu` | 租户管理员直接赋套餐菜单；其它角色与套餐求交。 |
| 系统套餐 | `PACKAGE_ID_SYSTEM = 0` | 系统租户特殊标识。 |
| 菜单 | `system/tenant`、`system/tenantPackage` | `menu.csv` 租户列表/套餐。 |

## 2. 认证（已证实）

| 接口/能力 | 位置 | 结论 |
| --- | --- | --- |
| 密码登录 | `AuthController` `/login` | 主路径。 |
| 无验证码登录 | `/loginNoCode` | 存在；环境是否开放待确认。 |
| 退出 / 刷新 | `/logout`、`/refresh-token` | 已证实。 |
| 权限信息 | `/get-permission-info` | 返回用户、启用角色、启用菜单（供前端）。 |
| 短信/社交 | `/sms-login`、`/social-login` 等 | 接口存在；启用依赖配置。 |

## 3. 边界

- 部门/工厂隔离 ≠ 租户隔离；租户是更高层多租户边界。
- MES `tenantConfig` 等业务侧入口不并入本页主链。
- 密码策略、锁定、审计查询细节待运维专项。

## 问题登记

- SSO/短信/社交在目标环境启停清单待测。
- 会话超时与安全参数随 `GAP-015` 运维能力续作。
