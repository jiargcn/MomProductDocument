# WMS：采购收货字段证据

> 对应正式页面：[业务说明](../../../../docs/05-WMS-库房管理/03-采购收货/index.md)、[详细参考](../../../../docs/05-WMS-库房管理/03-采购收货/01-采购收货详细参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。

## 1. 对象关系

| 业务对象 | 数据对象 | 关键关联 | 证据结论 |
| --- | --- | --- | --- |
| 收货申请 | `request_purchasereceipt_main` / `request_purchasereceipt_detail` | 申请单号、主表 ID。 | 保存来源、供应商、订单和待收货明细。 |
| 收货任务 | `job_purchasereceipt_main` / `job_purchasereceipt_detail` | 任务单号、来源申请单号。 | 保存现场可执行的收货任务与库存定位信息。 |
| 收货记录 | `record_purchasereceipt_main` / `record_purchasereceipt_detail` | 记录单号、来源申请、来源任务。 | 保存实际执行结果，是库存事务和后续处理来源。 |

## 2. 主表业务字段

| 中文名称 | 申请/任务/记录字段 | 说明 |
| --- | --- | --- |
| 业务类型、状态 | `business_type`、`status` | 三类对象均有；记录状态与申请/任务状态可能不是同一口径。 |
| 供应商 | `supplier_code`、`supplier_name` | 供应商代码在三类主表中为非空；名称来源待继续核验。 |
| 来源单据 | `asn_number`、`pp_number`、`po_number` | 发货单、要货计划、采购订单等来源信息。 |
| 订单与来源类型 | `po_type`、`erp_order_type`、`source_type`、`fix_rate` | 订单、ERP 和固定/非固定转换率口径待专项核验。 |
| 运输信息 | `carrier_code`、`transfer_mode`、`vehicle_plate_number` | 选择器和字典来源待继续核验。 |
| 记录后续信息 | `in_transaction_type`、`interface_type`、`erp_interface_pushed`、`erp_voucher_number` | 用于库存和外部接口追溯。 |

## 3. 明细与库存定位字段

| 分类 | 关键字段 | 说明 |
| --- | --- | --- |
| 物料与数量 | `item_code`、`item_name`、`uom`、`qty`、`supplier_qty`、`supplier_uom`、`convert_rate` | 库存计量与采购计量需区分；转换规则待专项核验。 |
| 采购订单行 | `po_number`、`po_line`、`po_line_type` | 主表与明细可能都保存订单号，文档应区分层级。 |
| 库存定位 | `batch_id`、`package_number`、`pallet_number`、`inventory_status`、`location_code` | 是收货到库存过程的关键追溯维度。 |
| 组织与仓储 | `location_group_code`、`area_code`、`warehouse_code`、`owner_code` | 用于查询、校验和业务归属。 |
| 记录后续字段 | `receivable_qty`、`shortage_qty`、`putaway_request_number`、`inspect_result` | 支持短缺、上架和检验等后续处理。 |

## 4. 页面、导入与终端证据

| 事项 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| 申请新增 | 供应商、单据号、部门和自动策略存在接口必填或默认配置。 | 哪些由用户录入、哪些由系统生成。 |
| 来源选择 | 采购订单选择器筛选已发布状态且符合当前 ERP 订单类型。 | 订单类型口径及测试环境表现。 |
| 导入 | 有固定/非固定转换率两类模板；支持更新、追加、覆盖和错误文件。 | 数量、采购数量、单位和转换率的生成/校验。 |
| 页面动作 | 申请有提交/同意/驳回/处理等入口；任务有承接/放弃/拒收/撤销等入口；记录可撤销。 | 各状态前置条件、权限和下游闭环。 |
| PDA | 可按任务号或 ASN 定位；进入待处理详情自动承接；支持包装和库位扫描；拒收需填写原因。 | 数量、库位、批次、包装修改的默认配置。 |
