# WMS 采购上架：字段与行为证据

> 内部证据页；适用基线：测试环境 / `dev` 分支 / 2026-07-15。不得从正式业务文档直接链接本页。

## 已定位证据

- 采购上架包含申请、任务、记录及各自明细控制器与页面。
- 上架任务提供单条执行、批量执行和取消接收动作；任务与记录均支持查询与导出。
- 申请、任务、记录分别具备导入/导出对象，但实际页面入口待验证。

## 已确认事实

| 业务能力 | 当前事实 |
| --- | --- |
| 业务链 | 采购上架使用申请、任务、记录三层对象。 |
| 现场动作 | 任务层明确存在执行、批量执行、取消接收。 |
| 追溯关系 | 记录控制器同时引用申请、任务和记录对象，支持建立业务追溯链。 |

## 待确认问题

- WMS-PUT-001：任务接收、执行、批量执行、取消接收各自的状态前置条件。
- WMS-PUT-002：部分上架、拆分上架、改库位/数量、撤销与库存反向处理。
- WMS-PUT-003：库位推荐、容量、混放、批次一致性和质量状态校验。
- WMS-PUT-004：PDA 扫描入口与 Web 任务动作的职责边界。
