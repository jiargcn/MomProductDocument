# WMS 入库链事实层总览

> 内部工作底稿，不对产品站点发布。
> 范围：基础数据、系统设置、采购收货、采购退货、采购上架。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。

## 1. 用途与使用边界

本页是 WMS 入库链的内部证据入口，用于减少后续页面反复扫描菜单、前端和后端资料的成本。它只记录范围、对象关系、证据状态和待验证问题；每个业务对象的字段、动作和导入结论写入独立事实页。

正式 `docs/` 页面只转写已确认的业务结论，不直接链接本目录，也不以源码术语作为培训正文。

## 2. 本轮页面范围

| 分组 | 菜单/业务对象 | 当前文档形态 | 本轮目标 |
| --- | --- | --- | --- |
| 基础数据 | 销售价格单、产品类、科目账户、ERP 成本中心、ERP 项目信息。 | 仅分组页。 | 建立业务边界、维护/查询说明和对象事实页。 |
| 系统设置 | 系统日历、账期日历、事务类型、计划设置。 | 仅分组页。 | 建立受控配置说明、变更风险和对象事实页。 |
| 标签管理（系统设置关联） | 标签规则、标签信息、标签类型、采购/制造/器具/库位/叫料标签、标签模板。 | 仅分组页。 | 记录当前 WMS 入口与未来平台能力归属的边界；按实际对象建立页面。 |
| 采购收货 | 申请、任务、记录。 | 主文档 + 维护与查询参考 + 事实页。 | 复审并补足入库链共用事实、状态与后续上架关系。 |
| 采购退货 | 申请、任务、记录、撤销记录。 | 仅业务主文档。 | 建立维护与查询参考及对象事实页，核实库存冲减和撤销边界。 |
| 采购上架 | 申请、任务、记录。 | 仅业务主文档。 | 建立维护与查询参考及对象事实页，核实来源收货、库位与库存结果。 |

## 3. 入库链通用对象关系

```mermaid
flowchart LR
    A["受控基础资料与系统设置"] --> B["采购收货申请/任务/记录"]
    B --> C["待上架或可处理的收货结果"]
    C --> D["采购上架申请/任务/记录"]
    D --> E["库存地点、状态与余额结果"]
    B --> F["采购退货申请/任务/记录"]
    F --> G["库存冲减与供应商协同"]
```

这张图是本轮取证的工作假设：收货、上架和退货共享来源、物料、数量、批次/包装、地点、库存状态等追溯维度。各环节的创建时点、状态前置条件、库存事务和外部回写须分别由对象事实页和测试环境确认。

## 4. 事实层建设顺序

| 顺序 | 先沉淀什么 | 主要证据 | 后续可复用到 |
| --- | --- | --- | --- |
| 1 | 菜单与页面清单、对象边界。 | `reference/menu.csv`、前端路由。 | 导航、页面清单、终端入口说明。 |
| 2 | 新增/编辑、列表/详情、导入与选择器。 | 前端页面配置、请求对象、导入模型。 | 维护与查询参考。 |
| 3 | 申请—任务—记录的创建、处理、撤销与状态条件。 | 后端服务、动作接口、测试环境。 | 业务主文档、培训流程和状态图。 |
| 4 | 库存、质量、上架/退货、外部协同影响。 | 业务服务、库存模型、接口调用。 | 入库链关系图、异常处理和联查。 |
| 5 | 冲突与缺口。 | 前后端/DDL比对、测试结果。 | 问题登记与后续回填任务。 |
