## 已确认事实

| 主题 | 当前事实 |
| --- | --- |
| 发料链 | 普通发料具备申请、任务、记录三个对象。 |
| 生产收料链 | 当前菜单明确暴露任务、记录两个入口；任务存在执行请求对象。 |
| MES 关联 | 菜单同时存在 MES 叫料补料入口；生产收料的来源创建机制尚未确定。 |
| 并存对象 | 代码还存在另一套“产品收料”请求、任务、记录对象，不能直接等同于当前生产收料菜单。 |

## 待确认问题

- WMS-PROD-001：发料申请、任务接收/执行、记录生成及直接发料的状态前置条件。
- WMS-PROD-002：发料是否在任务创建时形成预计出，以及少发、超发、补料和退料对预计出/余额的处理。
- WMS-PROD-003：生产收料任务由 MES/工单在何时、以何种状态和字段生成；收料记录形成预计入、库存事务和余额的时点。
- WMS-PROD-004：生产收料与产品收料两套对象的菜单映射、适用范围、数据隔离及迁移关系。
- WMS-PROD-005：质量判定、隔离、报废、生产退料对库存状态与可用范围的具体影响。
- WMS-PROD-006：PDA/线边端的扫码、离线与异常处理职责边界。
