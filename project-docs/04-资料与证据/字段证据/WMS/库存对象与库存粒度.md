# WMS：库存对象与库存粒度字段证据

> 对应正式页面：[库存数据挂接模型](../../../../docs/02-业务模型/02-库存数据挂接模型.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。

## 1. 库存对象与挂接关系

| 业务对象 | 数据对象 | 与单据的挂接 | 当前结论 |
| --- | --- | --- | --- |
| 预计入库 | `transaction_expectin` | 按任务号、业务类型与库存维度挂接。 | 收货任务创建时形成待完成的入库预期。 |
| 预计出库 | `transaction_expectout` | 按任务号、业务类型与库存维度挂接。 | 已定位独立页面与后端对象；来源业务和生成时点待逐项确认。 |
| 库存事务 | `transaction_transaction` | 按记录号、业务类型、事务类型和库存维度挂接。 | 实际收货记录会形成库存事务。 |
| 库存余额 | `transaction_balance` | 按库存业务粒度汇总，保存最后事务号。 | 事务处理会更新库存余额。 |

| 物料与数量 | `item_code`、`qty` | 【待确认】 | `item_code`、`item_name`、`qty` | `item_code`、`item_name`、`qty` |
| 批次/包装/状态 | `batch_id`、`pallet_number`、`package_number`、`inventory_status` | 【待确认】 | 同左，另有小包装号 | 同左，另有小包装号 |
| 库位与归属 | `location_code`、`warehouse_code`、`owner_code` | 【待确认】 | 库位组、库区、仓库、货主 | 库位组、库区、仓库、货主 |
| 时间与冻结 | — | 【待确认】 | `put_in_time`、`expire_date`、`frozen` | `put_in_time`、`expire_date`、`frozen` |

## 3. 已确认的库存余额业务粒度

| 字段 | 数据库列 | 说明 |
| --- | --- | --- |
| 批次 ID | `batch_id` | 业务粒度字段。 |
| 托包装号 | `pallet_number` | 业务粒度字段。 |
| 箱包装号 | `package_number` | 业务粒度字段。 |
| 物料代码 | `item_code` | 业务粒度字段。 |
| 库存状态 | `inventory_status` | 业务粒度字段。 |
| 库位代码 | `location_code` | 业务粒度字段。 |

当前 DDL 未提供与该业务粒度完全一致的复合唯一约束。现有余额查询键也未包含托包装号，因此需继续作为技术差距跟踪。

## 4. 页面层与导入证据

| 对象 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| 预计入 | 接口层和页面均要求任务、业务、物料、库存状态、数量、库位等信息。 | 单位、仓库、货主的前后端规则差异。 |
| 库存事务 | 列表重点展示事务、业务、记录、物料、数量、状态、库位和包装。 | 是否允许人工新增/编辑。 |
| 库存余额 | 列表重点展示物料、数量、单位、状态、库位、仓库、批次和包装。 | 导入实际校验与初始化/迁移适用边界。 |

## 5. 待核验项

| 编号 | 问题 | 影响 |
| --- | --- | --- |
| P0-FIELD-004 | 余额业务粒度含托包装号，但当前查询键不含该字段。 | 余额匹配与唯一性判断。 |
| P0-FIELD-005 | 预计入未保存部分余额展示字段。 | 列表字段可能来自关联查询，不能直接假定来源。 |
| WMS-INV-001 | 已定位预计出库页面与对象，但尚未确认各类出库任务的生成与释放时点。 | 不能据此计算可用库存或承诺量。 |
| WMS-INV-002 | 库存事务存在创建入口，但正式页面的人工创建权限、编辑边界和审计要求未确认。 | 培训中不得把事务维护作为常规修正手段。 |
