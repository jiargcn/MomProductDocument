## 2. 持久化对象（业务名 ↔ 技术）

| 业务对象 | 表 / DO | 关键业务字段（技术名） |
| --- | --- | --- |
| 工艺路线头 | `mes_basic_processroute_info` / `BasicProcessrouteInfoDO` | `procrouteCode/Name/Type`、`routeVersion`、`itemCode/Name/Type`、`bom`/`bomVersion`、`status`、`graphJson`、`configJson`、`isPrevalidated`、`validRuleCode`、`remark` |
| 工序节点明细 | `mes_basic_processroute_info_detail` / `BasicProcessrouteInfoDetailDO` | 工序编码名称、顺序、前后工序、节拍、`nodeData`；`minOkTransferToNext`、`batchTransferGateStrategy`、`isParallel`、`maxParallelCount`、`jobAssignMode` |
| 工序—工位 | `mes_process_to_workstation` / `ProcessToWorkstationDO` | `procrouteCode`、`processCode/Name`、`workstationCode/Name`、`workstationType`、`isMaster`、`siteId` 等 |
| 路线—产品 | `basic_route_to_product` / `RouteToProductDO` | `procrouteCode/Name/Type/Version`、`itemCode/Name/Type`、`status` |
| 运行快照 | `BasicProcessrouteInfoRun*` / 对应 Run DO | 执行侧路线头+明细+图形快照；与工单不匹配有专用错误码 |

## 4. 枚举与校验（已证实）

| 项 | 证据 | 正式文档口径 |
| --- | --- | --- |
| `jobAssignMode` | `GRAB` / `DISPATCH`；空=未配置（Save/Resp VO Schema） | 抢单 / 派单 |
| `batchTransferGateStrategy` | `WO_MIN_OK`（工单累计合格达门槛）、`BATCH_PER_TASK`（按任务立即转）；空默认 WO_MIN_OK | 业务描述，不写枚举类名 |
| 并行校验 | `BASIC_PROCESSROUTE_DETAIL_PARALLEL_INVALID` | 并行配置非法则保存失败 |
| 分派模式非法 | `BASIC_PROCESSROUTE_DETAIL_JOB_ASSIGN_MODE_INVALID` | 仅允许空/GRAB/DISPATCH |
| 运行快照与工单不匹配 | `BASIC_PROCESSROUTE_RUN_WORK_ORDER_MISMATCH` | 联查时提示可能绑错快照 |

## 5. 明确否定的旧稿内容

| 旧稿说法 | 处理 |
| --- | --- |
| 独立实体名如 `MesRoute` 及虚构 ER | 废弃；以本页 DO 为准 |
| 状态 DRAFT/ACTIVE 等臆造英文 | 未在 DO 枚举中证实；正式页写“以页面字典为准” |
| “质检点”独立三表作为工艺管理核心 | 本刀未证实；节点扩展可作线索，不写死 |
| 工艺路线仅 DBC 维护、MES 只读同步 | 与当前菜单与 Controller 矛盾 |
