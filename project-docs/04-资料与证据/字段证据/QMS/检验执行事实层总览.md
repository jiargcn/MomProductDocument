## 2. 配置对象 → 执行对象引用

```text
AQL / 样本字码 / 采样方案 / 采样过程
        ↓
检验方法 + 检验特性（定量/定性）+ 检验阶段
        ↓
检验模板（programme template）→ 检验方案（按物料+检验类型）
        ↓
物料检验配置 / 动态规则 / 计数器 / 选定集
        ↓
检验申请 → 检验任务 → 检验记录（主表/明细/包装/特性/定量·定性结果）
        ↓（不合格出口，可选）
检验评审 ATR；Q1/Q2/Q3 通知单
```

方案关键能力字段（证据名）：`exemption`（免检）、`changeInventoryStatus`、`autoCreateRequest`、`generatedMode`/`crontab`、`mesProcessCode`、`splitRule`、`aql`、`inspectionLevel`、`dynamicUpdateCode`。

## 4. 状态与判定（中文培训名；code 仅证据）

| 类别 | 枚举 | 业务名 |
| --- | --- | --- |
| 申请状态 | `RequestStatusEnum` | 新增 / 审批中 / 审批通过 / 审批驳回 / 关闭 / 处理中 / 部分完成 / 已完成 / 中止 |
| 任务状态 | `JobStatusEnum` | 待处理 / 进行中 / 完成 / 关闭 |
| 检验类型 | `InspectionTypeEnum` | 1 采购收货检验；2 发货检验；3 过程检验；4 巡检；5 客户退货检验；9 在库；10 客户发货；11 首件；12 末件；13 性能 |
| 评估码 | `EvaluationCodeEnum` | 接收 / 拒绝 |
| 使用决策 | `UseDecisionEnum` | 全部合格 / 全部不合格 / 报废 / 隔离 |
| 评审结论 | `ReviewResultEnum` | 让步 / 报废 / 返修 |
| 严格度 | `InspectionSeverityEnum` | 宽松 / 正常 / 严格 |
| 检验水平 | `InspectionLevelEnum` | S1–S4 / G1–G3 |

记录发布后可回写上游；评审任务完成可按库存状态映射更新记录 `useDecision`（OK→全部合格，SCRAP→报废，HOLD→隔离）。

## 5. 触发与回写矩阵

| 链路 | 已证实 | 未证实 / 注意 |
| --- | --- | --- |
| 来料 ← WMS 到货确认 | 库存状态为待检行创建检验申请，`inspectionType=1`；可先查方案免检 | — |
| 来料 ← WMS 采购收货记录 | **已停用**：收货记录 `createInspectRequest` 仅打日志跳过（检验前置到到货确认） | 历史 MQ 兼容入口仍在 |
| 来料结论 → WMS | 记录发布：有 ASN 且类型=1 时调用到货检验结果回写 | 旧「采购收货检验结果」回写已注释 |
| 生产 ← WMS 制品收货 | 可创建检验申请，`inspectionType=4`（枚举业务名「巡检」） | 与菜单「巡检检验」是否同码并存需环境核对 |
| 生产 ← MES 报工/工单 | MES 模块侧未见直接建 QMS 申请 | 登记 `GAP-071` |
| 客户 ← WMS 客户退货 | 记录创建后按规则开关可建申请，`inspectionType=5` | 寄售退货/发货调整亦有同类入口 |
| 评审 ↔ 库存 | 评审明细含库存定位字段；结论可改 `useDecision` | **实际隔离/放行/报废事务以 WMS 为准**，不在 QMS 杜撰 |
| Q1/Q2/Q3 | 独立菜单与 DO；Q2 含供应商索赔/采购收货号等字段 | 与检验评审的强制先后关系未闭合 |

## 6. 否定旧稿

| 旧稿做法 | 处理 |
| --- | --- |
| 虚构英文字段 ER / 未核验状态名 | 废弃；以本页枚举与 DO 为准 |
| 声称采购收货完成必建检验 | 纠正为「到货确认 + 待检状态」 |
| 把客户检验写成出货 OQC | 菜单证实为「客户退回检验」；发货检验类型枚举存在但无独立菜单 ATR |
| 把 EAM 巡检 / WMS 质量评审并入 QMS 五组 | 边界分开写 |
