> 对应正式页面：[业务说明](../../../../docs/07-QMS-质量管理/05-质量评审/index.md)、[维护与查询参考](../../../../docs/07-QMS-质量管理/05-质量评审/质量评审-维护与查询参考.md)。
> 基线：测试环境 / `dev` / 2026-07-15。共享事实见[检验执行事实层总览](检验执行事实层总览.md)。

## 1. 菜单

- 检验评审申请/任务/记录：`qms/inspectionReview/inspectionReviewRequestMain` 等。
- q1/q2/q3：`qms/inspectionQ1` / `Q2` / `Q3`。
- WMS `quality-review`（待评审物料、质量评审记录）— **非**本 DO 族。

## 2. 持久化

| 对象 | 表 |
| --- | --- |
| 评审申请/任务/记录主表 | `request_review_main` / `job_review_main` / `record_review_main` |
| 评审明细 | `*_review_detail`（包装、批次、qty、inventoryStatus、location…） |
| Q1/Q2/Q3 | `basic_inspection_q1/q2/q3`；另 `inspection_q3_main/detail` |

`JobReviewMainServiceImpl` 完成时可 `updateUseDecision(inspectionRecordNumber, inventoryStatus)`，映射 OK→全部合格、SCRAP→报废、HOLD→隔离。

`ReviewResultEnum`：让步 / 报废 / 返修。

## 3. WMS 引用通知单

如报废申请/任务服务引用 `InspectionQ2Api`（索赔协同线索）— 正式页只写边界，不展开财务。

## 4. 否定旧稿

虚构评审 ER、把 WMS 待评审物料写成 QMS 评审主链、杜撰自动库存事务 → 废弃。
