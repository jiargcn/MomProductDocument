# 字段证据：QMS / 客户检验

> 对应正式页面：[业务说明](../../../../docs/07-QMS-质量管理/04-客户检验/index.md)、[维护与查询参考](../../../../docs/07-QMS-质量管理/04-客户检验/客户检验-维护与查询参考.md)。
> 基线：测试环境 / `dev` / 2026-07-15。共享事实见[检验执行事实层总览](检验执行事实层总览.md)。

## 1. 菜单

客户退回检验申请/任务/记录：`inspectionRequestCustomerReturn` 等。**无**「出货检验」独立 ATR 菜单。

## 2. 类型

`InspectionTypeEnum.CUSTOMER_RETURN_INSPECTION` code=`5`。另有 `2` 发货检验、`10` 客户发货检验 — 无对应本分组菜单。

## 3. WMS 触发

`CustomerreturnRecordMainService.createInspectRequest`：规则 `CREATE_INSPECTION_REQUEST_AFTER_CUSTOMER_RETURN_RECORD_CREATED`；`inspectionType="5"`；回调 `requestNumber+":"+detailId`。同类逻辑见于寄售退货、发货调整。

## 4. 否定旧稿

把本页写成出货 OQC / 客户放行主链 → 否定。

## 5. 待确认

建单时清空部分 `reference*` 字段的产品意图；旁路退货与菜单过滤。
