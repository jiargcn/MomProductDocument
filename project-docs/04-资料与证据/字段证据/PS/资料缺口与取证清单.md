## 2. 已能写进正式文档的内容（X）

1. PS 在产品规划中的定位：排程层，介于需求/订单与 MES 可执行工单之间。  
2. 站点仍保留 6 个业务分组作为导航骨架。  
3. 边界：执行与工单生命周期看 [MES 计划管理](../../../docs/06-MES-生产管理/03-计划管理/index.md)；工艺与产线主数据看 DBC。  
4. 明确声明：菜单、后端模块、字段、算法、接口均未证实。

| PS-GAP-002 | 是否存在独立后端模块或外部引擎 | 搜索部署清单、网关路由、第三方 APS；确认是否仓外系统 | P0 |

| PS-GAP-006 | 执行参数与结果版本 | 场景参数、版本号、异步任务、失败补偿 | P1 |

## 4. 取证完成后的回填顺序（建议）

1. 菜单与路由证实 → 更新各分组「当前可确认状态」。  
2. 按真实对象建 `project-docs/04-资料与证据/字段证据/PS/<对象>.md`。  
3. 正式页补字段表 + 维护与查询参考（仅对已证实对象）。  
4. 再写与 MES/DBC/ERP 的挂接矩阵；更新 [API 参考](../../../docs/14-API参考/index.md)。  
5. **仍禁止**在取证前恢复旧稿公式与假 ER。

- 不编造维护与查询参考页。  
- 不把 MES 计划管理字段/状态抄入 PS。  
- 不声称已对接外部排程引擎。  
- 不 push 远程。
