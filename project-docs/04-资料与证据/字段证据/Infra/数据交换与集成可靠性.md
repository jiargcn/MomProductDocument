# Infra：数据交换与集成可靠性证据

> 对应正式页面：[数据交换与集成可靠性](../../../../docs/03-基础设施/09-数据交换与集成可靠性.md)。
> 关联业务模型：[第三方接口与单据挂接](../../../../docs/02-业务模型/03-第三方接口与单据挂接.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。内部证据页。

## 1. 接口调用信息（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 表/对象 | `basic_interface_info` / `InterfaceInfoDO` | 含接口类型/名称、状态、url、req/resp、errorMsg、businessNumber、businessId、tenantId。 |
| 服务 | Infra `InterfaceInfoController`；WMS 等多处 `InterfaceInfoApi` 写入 | 平台服务 + 业务触发。 |
| 菜单 | `接口调用信息管理` → `wms/agvManage/interfaceInfo` | 当前菜单挂在业务路径，能力属交换轨迹。 |

## 2. 异步可靠性（已证实）

| 项 | 证据 | 结论 |
| --- | --- | --- |
| 失败记录 | 菜单 `api-mq-error`；`ApiMqErrorController` | 可查异常。 |
| 重试 | `/retry` → `apiInterfaceInfoConsumer.retry` | 支持人工重试。 |
| 成功历史 | 菜单 `api-mq-his` | 可查成功记录。 |
| 清理/重试作业 | `apiMqHisCleanJob`、`apiMqErrorJob` 等 `@XxlJob` | 与定时任务页联动。 |

## 3. 未证实

- 统一死信控制台、全站幂等键规范、自动对账大盘。

## 问题登记

- 菜单归属（WMS 路径 vs Infra）长期收敛待产品确认。
- 逐接口幂等/超时矩阵随业务续作。
