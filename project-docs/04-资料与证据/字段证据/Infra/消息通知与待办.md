# Infra/System：消息、通知与待办证据

> 对应正式页面：[消息、通知与待办](../../../../docs/03-基础设施/06-消息、通知与待办.md)、[消息通知](../../../../docs/12-系统管理/05-消息通知/index.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。内部证据页。

## 1. System 渠道（已证实）

| 能力 | 证据 | 结论 |
| --- | --- | --- |
| 通知公告 | `NoticeController`；菜单 `system/notice` | 公告维护。 |
| 站内信消息 | `NotifyMessageController`（我的消息、已读、未读数）；`system_notify_message` | 个人消息。 |
| 站内信模板 | `NotifyTemplateController` | 模板驱动内容。 |
| 邮件模板/日志 | `MailTemplateController`、`MailLogDO` | 渠道能力存在。 |
| 短信模板/日志 | `SmsTemplateController`、`SmsLogDO` | 渠道能力存在。 |
| 钉钉 | 钉钉模板/消息 Controller 与菜单 | 独立渠道入口。 |

## 2. 配置入口分散

| 菜单线索 | 模块 | 备注 |
| --- | --- | --- |
| 消息设置管理 | System | `system/messageSet` |
| 消息配置 | Infra | `infra/messageconfig` |
| 消息配置表 | ANDON | `andon/messageconfig` |
| 质量通知 | QMS | 业务域通知 |

不能写成单一消息配置中心。
