# 字段证据：ANDON / 故障记录

> 对应正式页面：[业务说明](../../../../docs/09-ANDON-异常管理/01-故障记录/index.md)、[维护与查询参考](../../../../docs/09-ANDON-异常管理/01-故障记录/故障记录-维护与查询参考.md)。
> 基线：测试环境 / `dev` / 2026-07-15。

## 1. 菜单

ANDON 根（id=4，`/andon`）下当前四项：

| 菜单名 | path / component |
| --- | --- |
| 安东故障记录 | `processRecord` / `/andon/processRecord/index` |
| 问题响应链配置 | `ProblemRespChainConfig` |
| 消息配置表 | `MessageConfig` |
| 停机项目表 | `ShutdownItem` |

说明：菜单名「安东故障记录」挂的是 **过程记录** 前端路径；后端另有 `problem-record`（问题事实）CRUD。培训时区分「问题主记录」与「过程/到岗记录」，勿只按菜单字面合并。

## 2. 问题主记录 `andon_problem_record`

关键字段与注释口径：

| 字段 | 业务含义（来自 DO 注释） |
| --- | --- |
| problemUser / problemPost | 问题用户、问题岗位 |
| pstartTime / pendTime | 问题起止 |
| estartTime / eendTime | 设备起止 |
| equipmentCode / workMoldCode / itemCode / workOrderNo | 设备、模具、零件、工单 |
| totalDuration | 总时长（分钟） |
| problemStatus | 0 呼叫中、1 结束 |
| problemAllStatus | 0 处理中、1 审结完成 |
| projectClassification | 01 安全 … 05 质量 |
| problemClassification | 01 人员 … 09 质量异常等 |
| problemSource | 01 安东异常、02 设备巡检、03 质量检验 |
| problemType | 缺陷/问题类型码表（毛刺、划伤等） |
| problemDescription / handMeasure | 描述、处理措施 |
| importance / urgentLevel | 重要度、紧急度（在机整改等） |
| analysis* / repairOrderStatus / attachment* | 分析登记、维修单状态、附件 |

服务实现目前为标准 CRUD，**未见**复杂自动状态机；升级/通知细节见响应链与过程记录。

## 3. 过程记录 `andon_problem_process_record`

关联 `problemId`；含通知类型/时间/内容、到岗人/时间、到岗状态与时长等。对应菜单「安东故障记录」更贴近过程协同视图。

## 4. 停机项目 `andon_shutdown_item`

设备编码、班次、停机项目/类型、计划时长、呼叫岗位等。用于停机分类与呼叫岗位，**不是** EAM 设备停机单据。

## 5. 否定旧稿

- 旧概述虚构 `andonId/lineId/andonLevel/critical` 英文状态机与 REST `/api/andon/faults` → 废弃。
- 不得把 EAM 维修工单状态机抄到 ANDON 故障页。
