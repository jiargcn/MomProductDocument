# 字段证据：EAM / 基础数据

> 对应正式页面：[业务说明](../../../../docs/08-EAM-设备管理/01-基础数据/index.md)、[维护与查询参考](../../../../docs/08-EAM-设备管理/01-基础数据/基础数据-维护与查询参考.md)。
> 基线：测试环境 / `dev` / 2026-07-15。

## 1. 菜单（EAM → 基础数据）

| 菜单名 | path / component | 说明 |
| --- | --- | --- |
| EAM基础配置 | `eam-basic-config` / `eam/eamBasicConfig` | 键值配置 |
| 故障类型 / 故障原因 | `basicFaultType` / `basicFaultCause` | 报修分类与标准工时 |
| 保养/巡检/点检方案与项、选择集 | `basicMaintenanceOption` 等 | 计划引用的方案底座 |
| 开拉方案/项（若启用） | `basic-kaila-option` 等 | 开拉计划底座 |
| 厂区班组角色 | `class-type-role` | 派工/班组角色 |
| 主要部件、部件关联 | `equipmentMain` / `relation-main-part` | 设备部件结构 |
| 生产商/供应商管理（EAM 菜单） | `equipment-manufacturer` / `equipment-supplier` | EAM 侧厂商表；与 DBC 生产商并存，归属勿混 |
| 备件基础、库区/库位（EAM） | `item` / `location` 等 | 备件主数据与 EAM 库位；部分菜单 status/visible 偏隐藏 |
| 车间/产线（EAM 基础） | `basicEamWorkshop` / `basicEamProductionline` | EAM 侧组织引用；工厂主数据仍以 DBC 为准 |
| 文档类型、表扩展属性、故障率参数、生产时长 | 对应 path | 辅助配置 |

根菜单：`EAM`（id=5，`/eam`）。

## 2. 关键对象（表名仅证据层）

| 业务对象 | 表 | 关键字段（业务含义） |
| --- | --- | --- |
| EAM 基础配置 | `basic_eam_config` | 标题、编码、配置值、备注 |
| 故障类型 | `basic_fault_type` | 编码/名称、类型、可关联设备、标准工时、描述 |
| 故障原因 | `basic_fault_cause` | 与类型配套的原因主数据 |
| 生产商/供应商（EAM） | `basic_equipment_manufacturer` 等 | 编号、名称、联系、地址 |
| 保养/巡检/点检方案与项 | `basic_*_option` / item / select set | 被计划 `optionCode` 引用 |

## 3. 边界

- **设备/工装台账主数据不在本组**：EAM「设备台账管理」「工装台账管理」菜单 component 指向 **DBC** 页面（`dbc/equipmentAccounts`、`dbc/toolAccounts`），证据见 DBC 设备/工装台账。
- EAM 基础数据中的生产商/供应商与 DBC 生产商可能并存；培训写「EAM 报修/台账引用侧」与「DBC 主数据侧」时勿合并成一张表。
- 车间/产线/工位权威主数据在 DBC；EAM 基础中的车间/产线为模块内引用配置，改主数据以 DBC 为准。
