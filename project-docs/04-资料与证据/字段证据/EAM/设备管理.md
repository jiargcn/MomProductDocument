## 2. 台账边界（已证实）

- 台账主数据 API：`EquipmentAccountsApi`（DBC）。验收、移动等服务通过 DBC API 读/补丁设备。
- 正式培训口径：**DBC = 设备是什么、在哪、是否可用；EAM = 验收/移动/变更/停机/报修维修执行。**

## 3. 报修 / 维修状态（枚举中文名）

报修：待审核、已撤回、审核中、已通过、已驳回、已创建、已确认、暂存、已作废、已退回。

维修工单：待派工、已逾期、已退单、待接单、待执行、执行中、待维修验证、待报修验证、已完成、已作废、暂停。

报修可带来源工单类型：保养 / 巡检 / 点检 / 开拉（`ReportRepairWorkOrderSourceEnum`）。

维修工单关键动作（控制器）：创建、派发、转单、执行/暂存执行、验证相关接口等。

## 4. 维修主数据业务字段（摘要）

`job_equipment_repair_main`：单号、类型、关联报修号、来源、紧急度/等级、设备编码名称、是否停机、起止与影响时长、故障类型、故障描述、班组角色、车间/产线/工序/工位、派发/接单/执行/提交/退回/作废操作人与时间、自动派工/执行/验证策略等。

报修申请 `request_equipment_report_repair`：设备、紧急度、故障类型、描述、停机、图片、组织位置、审批人、自动审核策略、来源工单号/类型等。

## 5. 否定旧稿

- 旧 EAM 概述中虚构 ER 英文字段与「购置申请→台账」闭环不得当培训事实。
- 不得把 DBC 台账字段表复制进 EAM 设备页当 EAM 自有表。
