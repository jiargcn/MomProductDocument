# DBC：客户物料字段证据

> 对应正式页面：[业务说明](../../../../docs/04-DBC-主数据管理/03-客户管理/03-客户物料.md)、[维护与查询参考](../../../../docs/04-DBC-主数据管理/03-客户管理/04-客户物料-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。

## 1. 对象与证据来源

| 项目 | 当前证据 |
| --- | --- |
| 业务对象 | 客户与内部物料之间的交付识别和包装关系。 |
| 数据对象 | `basic_customeritem`、`CustomeritemDO`。 |
| 页面规则 | 客户物料新增/编辑、列表、导入和导出页面。 |
| 主要来源 | `base.sql`、`CustomeritemServiceImpl`、客户物料导入对象、客户物料前端配置。 |

## 2. 关键字段口径

| 业务含义 | 后端属性 | 数据库列 | 文档结论 |
| --- | --- | --- | --- |
| 关系识别 | `customerCode`、`itemCode` | `customer_code`、`item_code` | 创建校验按客户与物料组合检查重复；数据库唯一约束待核验。 |
| 客户侧识别 | `customerItemCode` | `customer_item_code` | 页面可填，最长 50 字符并有格式校验。 |
| 包装与单位 | `packageUnit`、`packageStdQty`、`customerUom` | `package_unit`、`package_std_qty`、`customer_uom` | 页面要求包装规格、标准包装数量；数量最小值为 1。 |
| 生命周期 | `available`、`activeTime`、`expireTime`、`remark` | 对应可用、生效、失效、备注列 | 新建和导入路径均设置为可用；实际停用动作待测。 |

## 3. 新增、编辑、导入与下游证据

| 事项 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| 新增 | 客户、物料、包装规格、标准包装数量和可用状态参与校验；创建逻辑会将可用状态设为 true。 | 客户、物料、包装规格的实际存在性和可用性校验边界。 |
| 编辑 | 前端提供编辑能力；具体哪些关系键可改、引用后的保护规则未完成验证。 | 已被订单、标签或发运引用后的限制。 |
| 导入 | 导入校验客户代码、物料代码、包装规格、包装数量及客户物料代码；导入记录默认可用。 | 导入模式、错误回执和可用状态列的实际表现。 |
| 列表 | 前端支持客户、物料、客户物料代码筛选，展示包装、单位、可用状态、有效期和审计字段。 | 默认排序和用户关注字段顺序。 |
| 下游使用 | API 支持按客户 + 物料、按客户物料代码查询关系，供下游获取客户侧物料口径。 | 订单、发货、标签是否强制校验和异常提示。 |

## 4. 差异与待核验项

| 编号 | 问题 | 影响 |
| --- | --- | --- |
| DBC-CI-001 | 导入重复校验调用时传入的第二参数与关系键口径不一致，疑似将客户物料代码用于客户维度判断。 | 同批/历史重复关系可能无法按预期识别。 |
| DBC-CI-002 | 新建与导入均默认设置可用状态，页面的可用状态展示与实际启停流程需测试。 | 上线时机和停用培训口径。 |
| DBC-CI-003 | 数据库层面的客户代码 + 物料代码复合唯一约束未确认。 | 并发建档与数据治理风险。 |
| DBC-CI-004 | 当前详情缺少客户物料专属分组和交付快捷跳转。 | 查询效率和培训可读性不足。 |
