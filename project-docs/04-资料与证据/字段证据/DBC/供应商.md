## 1. 对象与证据来源

| 项目 | 当前证据 |
| --- | --- |
| 业务对象 | 供应商主数据。 |
| 数据对象 | `basic_supplier`、`SupplierDO`。 |
| 页面规则 | 供应商新增/编辑、列表、导入和导出页面。 |

## 2. 关键字段口径

| 业务含义 | 后端属性 | 数据库列 | 文档结论 |
| --- | --- | --- | --- |
| 供应商代码 | `code` | `code` | 新增必填；服务层按代码检查重复。 |
| 名称、简称 | `name`、`shortName` | `name`、`short_name` | 简称新增必填；更新时代码和简称会保留原值。 |
| 联系信息 | `address`、`contacts`、`email` 等 | 对应地址、联系人、邮件列 | 导入时联系人邮件为必填。 |
| 业务属性 | `currency`、`taxRate`、`type` | 对应币种、税率、类型列 | 币种、类型由前端字典构造；税率页面限制为 0–1、6 位小数。 |
| 生命周期 | `available`、`activeTime`、`expireTime` | 对应可用、生效、失效列 | 新增时可用状态必填；实际停用影响需测试验证。 |

## 3. 新增、编辑与导入证据

| 事项 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| 新增 | 服务层校验代码、简称和可用状态；代码重复会失败。 | 创建对象与页面对名称、邮件等必填表达是否完全一致。 |
| 编辑 | 更新逻辑将代码和简称回填为原值，正常更新不能改码或改简称。 | 已被采购/收货等引用时的业务保护。 |
| 删除 | 当前删除服务仅确认对象存在后执行删除。 | 是否为逻辑删除，以及是否存在跨业务引用拦截。 |
| 导入 | 同批代码重复、代码/简称/联系人邮件缺失会形成错误；导入模式控制新增/更新范围。 | 模式数值对应的产品名称、页面提示和错误回填体验。 |
| 权限 | 前端按钮存在权限配置线索；部分后端授权注解被注释。 | 后端接口实际强制授权与数据范围。 |

## 4. 字段技术名校正提醒

旧文档中 `supplierCode`、`supplierName`、`zipCode`、`contactPerson`、`contactEmail`、`bankName`、`supplierType`、`effectiveDate`、`invalidDate` 不是当前对象的属性名。正式业务页面不写这些技术名；如需技术核对，以本页字段映射和源码为准。
