# DBC：供应商物料字段证据

> 对应正式页面：[业务说明](../../../../docs/04-DBC-主数据管理/02-供应商管理/02-供应商物料.md)、[维护与查询参考](../../../../docs/04-DBC-主数据管理/02-供应商管理/04-供应商物料-维护与查询参考.md)。
> 基线：测试环境 / `dev` 分支 / 2026-07-15。

## 1. 对象与证据来源

| 项目 | 当前证据 |
| --- | --- |
| 业务对象 | 供应商与物料之间的供货匹配关系。 |
| 数据对象 | `basic_supplieritem`、`SupplieritemDO`。 |
| 页面规则 | 供应商物料新增/编辑、列表、导入和导出页面。 |
| 主要来源 | `base.sql`、`SupplieritemServiceImpl`、供应商物料导入对象、供应商物料前端配置。 |

## 2. 关键字段口径

| 业务含义 | 后端属性 | 数据库列 | 文档结论 |
| --- | --- | --- | --- |
| 关系识别 | `supplierCode`、`itemCode` | `supplier_code`、`item_code` | 创建服务按二者组合检查重复；更新时两者被保留为原值。 |
| 供应方料号与计量 | `supplierItemCode`、`supplierUom`、`convertRate` | `supplier_item_code`、`supplier_uom`、`convert_rate` | 页面要求转换率；供应方料号页面限制最多 50 字符。 |
| 交接与使用范围 | `dockCode`、`allowOverShipment`、`available` | `dock_code`、`allow_over_shipment`、`available` | 页面要求月台、可用状态；允许超发和可用状态在导入时不可为空。 |
| 生命周期 | `activeTime`、`expireTime`、`remark` | `active_time`、`expire_time`、`remark` | 可填写生效/失效时间和备注；实际业务拦截效果待测试。 |
| 关联展示 | `supplierName`、`itemName`、`itemUom` | 对应展示列 | 创建与更新会清空输入的供应商名称、物料名称；不应作为关系维护或导入键。 |

## 3. 新增、编辑、导入与下游证据

| 事项 | 已确认内容 | 待核验项 |
| --- | --- | --- |
| 新增 | 页面从可用供应商、可用物料构造选择器；服务检查供应商代码、物料代码、可用状态非空，并按供应商 + 物料检查重复。 | 服务对供应商、物料仅见非空检查，未见主数据存在性强制查询。 |
| 编辑 | 服务回写原供应商代码、物料代码；正常更新不能直接更换关系键。 | 被采购、收货或退货引用后的业务保护。 |
| 导入 | 导入检查供应商代码、物料代码、转换率、允许超发和可用状态；按供应商 + 物料决定新增或更新。 | 页面要求月台，导入校验未见月台非空检查；导入模式的界面名称和跳过行反馈待测试。 |
| 列表 | 前端已有供应商、物料、月台、超发、可用状态等筛选；展示计量、转换率、有效期和审计信息。 | 生产使用者最常用字段顺序与默认排序。 |
| 下游使用 | WMS 采购明细会按供应商 + 物料查询供应商物料，并读取供应商计量单位、月台关联的默认地点等信息。 | 采购收货、退货和计划对该关系的强制程度与异常提示。 |
| 外部接口 | 新增、更新、删除会生成“推送基础供应商物料”的接口记录。 | 接收系统是否成功消费、重试和幂等结果。 |

## 4. 差异与待核验项

| 编号 | 问题 | 影响 |
| --- | --- | --- |
| DBC-SI-001 | 数据库未见供应商代码 + 物料代码的复合唯一约束，当前主要依赖创建服务检查。 | 并发建档与数据治理风险。 |
| DBC-SI-002 | 页面新增要求月台，导入校验未见对应的非空检查。 | 页面与导入模板的必填口径可能不一致。 |
| DBC-SI-003 | 页面选择器过滤可用供应商和物料，但后端新增校验仅见代码非空。 | 非页面渠道导入或接口调用的主数据有效性风险。 |
| DBC-SI-004 | 当前详情是通用展示，未形成供应商物料专属分组或快速跳转。 | 查询效率和培训可读性不足。 |
