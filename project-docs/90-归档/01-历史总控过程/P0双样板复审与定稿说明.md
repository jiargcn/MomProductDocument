> 复审日期：2026-07-16
> 样板范围：[物料基本信息](../../docs/04-DBC-主数据管理/01-物料管理/01-物料基本信息.md)及其[物料基本信息-维护与查询参考](../../docs/04-DBC-主数据管理/01-物料管理/02-物料基本信息-维护与查询参考.md)，[采购收货](../../docs/05-WMS-库房管理/03-采购收货/index.md)及其[采购收货-维护与查询参考](../../docs/05-WMS-库房管理/03-采购收货/01-采购收货-维护与查询参考.md)。
> 结论用途：作为第一阶段 DBC/WMS 大面积建立模块说明和业务大纲前的唯一写法依据。

## 1. 本轮结论

两个样板现在可以作为“主文档 + 维护与查询参考”成对写法的标准：主文档让读者理解业务目的、典型流程、关键判断、影响和异常；维护与查询参考让读者完成维护、执行、导入、查询和联查。此结构适用于满足拆分判定的业务；简单业务可在主文档中合并维护与查询内容。技术字段映射和源码证据仍保留在内部底稿，不进入正式阅读路径。

这不是第一阶段的最终准入结论。测试环境截图、真实状态动作和脱敏示例尚未补齐，产品负责人仍需在第一阶段末做最终确认；但文档结构、文风、站位写法和页面分层已经足以开始 DBC/WMS 的**大纲建设**，尚不开始无差别的字段/代码回填。

## 2. 复审发现与已处理项

| 复审点 | 发现 | 已处理方式 |
| --- | --- | --- |
| 主文档和维护与查询参考的边界 | 两页存在少量重复，读者不易判断该从哪一页开始。 | 两组页面均增加“如何使用本组文档/快速定位”，明确主文档讲业务主线、维护与查询参考讲具体任务。 |
| 主文档的信息密度 | 主文档中仍有部分模板列和操作细节，容易再次向页面字典倾斜。 | 物料主文档将批量导入改为业务层的使用提示，并链接维护与查询参考；采购收货主文档补充关键判断点，避免用动作清单代替业务理解。 |
| 维护与查询参考的可操作性 | 参考页虽有表格，但缺少读者在提交前要核对什么的收口。 | 物料参考页补充维护顺序和变更评估提示；采购收货参考页补充按操作划分的快速核对。 |
| 版本和阅读边界 | 两份维护与查询参考缺少统一基线说明。 | 统一补充适用基线和“配合主文档使用”的定位。 |
| 技术内容混层 | 正式页已基本避免技术标签，但后续批量扩写缺少更完整模板约束。 | 将原实体页模板升级为[正式业务文档标准模板](../00-需关注/实体说明页标准模板.md)，明确五层文档位置和四类页面模板。 |

## 3. 已锁定的写作决定

| 决定 | 后续执行要求 |
| --- | --- |
| 主文档先讲业务，再讲规则 | 前三屏必须能回答“是什么、谁来用、怎样完成一次典型业务”。 |
| 维护与查询参考只补操作任务 | 符合拆分判定的业务配套同名维护与查询参考；简单业务将同类内容合并到主文档。参考页不复述端到端故事。 |
| 站位是未来内容大纲 | 每个未填段落使用 `【待补充】`、`【图示占位】`、`【截图占位】`、`【示例数据占位】`，并说明后续需要什么。 |
| 图示只表达一个业务问题 | 流程、状态、对象关系、截图和示例分别服务不同理解目标；不把 ER 图或服务调用图放进培训正文。 |
| 内部证据不外溢 | 表名、字段技术名、源码路径、冲突和差距编号只在 `project-docs/` 维护。 |
| 大面积调整先做大纲 | 第一阶段先重写 DBC/WMS 模块首页、业务分组页和页面完整大纲；不在大纲阶段补全字段、接口或技术细节。 |

## 4. 仍需验证、但不阻塞大纲建设的事项

| 事项 | 影响 | 后续资料或动作 |
| --- | --- | --- |
| 物料停用/删除对下游选择器的实际影响 | 影响培训中的操作边界。 | 在测试环境以已引用和未引用物料分别验证。 |
| 物料导入模板及错误回执 | 影响维护与查询参考是否能直接用于批量维护。 | 提供模板、错误文件和一组脱敏样例。 |
| 收货申请、任务、记录的真实状态动作 | 影响状态图和“什么状态能做什么”。 | 在测试环境按正常、少收、拒收、撤销场景验证。 |
| 收货 PDA 的扫码、数量/库位修改限制 | 影响现场操作培训。 | 补充测试账号、任务配置和脱敏截图。 |
| 上架、检验、退货、外部回写闭环 | 影响收货后的完整追溯说明。 | 在跨模块测试中补充关联号、结果和异常路径。 |
