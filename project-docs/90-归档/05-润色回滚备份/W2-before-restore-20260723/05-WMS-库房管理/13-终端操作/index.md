## 业务目的与适用范围

WMS 终端操作说明 PDA（及同属现场执行的移动入口）如何承接 Web 中的任务执行、扫码确认与异常处理。终端不另建业务规则。本页只覆盖 **WMS 相关 PDA**；MES 线边客户端、EAM PDA 不在本页。

读完应能说清：终端与 Web 边界、按业务链如何找入口，以及验证时应对照哪张 Web 主文档。

## 使用前准备

| 需要确认什么 | 为什么重要 |
| --- | --- |
| 当前要做的 Web 业务类型 | 决定打开哪一个 PDA 入口，避免串单。 |
| 任务号/来源号是否已生成 | 终端通常执行已存在的任务，而不是重新定义业务。 |
| 扫描对象与库存精度 | 物料、包装、批次、库位是否按任务要求采集。 |
| 账号与终端权限 | 决定能否看到并执行对应任务。 |
| 网络/登录状态 | 避免把本地未提交结果误认为已入账。 |

## 终端如何承接一笔业务

```mermaid
flowchart LR
    A["Web 建立申请并生成任务"] --> B["PDA 定位任务"]
    B --> C["扫码/录入现场结果"]
    C --> D["形成业务记录"]
    D --> E["库存事务与余额"]
    D --> F["回到 Web 查询与联查"]
```

典型分工是：Web 负责发起、处理、查询、调整和复杂协同；PDA 负责现场定位任务、扫描确认和快速异常提示。完成后仍应在 Web 用记录、库存事务和余额核对结果。

    一笔采购收货任务在 PDA 扫描包装和库位后完成，再在 Web 查看收货记录与库存余额。

### 关键判断

| 判断点 | 应先确认什么 | 对业务的影响 |
| --- | --- | --- |
| 该不该用 PDA | 是否已有可执行任务，以及现场是否需要扫码。 | 决定用终端还是 Web。 |
| 打开哪个入口 | 业务属于收货、发料、发货、库内、盘点还是生产结果。 | 防止执行错对象链。 |
| 扫描不匹配怎么办 | 任务限制、物料/包装/库位要求。 | 决定停止、重扫或转异常，而不是强行完成。 |
| 怎样确认已入账 | Web 记录、库存事务和余额。 | 避免把终端提交成功当成最终库存结论。 |

## 终端与 Web 的职责边界

| 事项 | Web | PDA/现场终端 |
| --- | --- | --- |
| 建立申请、审批/处理 | 主入口 | 通常不替代 |
| 承接并执行任务 | 可执行 | 主现场入口 |
| 复杂查询、联查、调整、撤销 | 主入口 | 仅少量快捷入口，且需回归 Web 核对 |
| 业务规则与状态机 | 以各业务页为准 | 只表现现场允许的动作 |
| 库存结果解释 | 事务/余额查询 | 不单独定义另一套库存逻辑 |

“直接收货/直接发货/直接转移”等终端入口可能存在，但不少为隐藏或受限路径，不能写成默认培训流程。

## 按业务链理解终端入口

| 业务链 | 终端补充什么 | 规则回到哪里 |
| --- | --- | --- |
| 采购收货/退货/上架 | 任务定位、包装/库位扫描、拒收原因 | 采购收货、退货、上架页 |
| 发料/生产收料 | 拣货扫码、交接、现场收料 | 发料管理、生产收料页 |
| 销售出库/备货/客户退货 | 拣货发运扫码、异常提示 | 销售出库页 |
| 库内转移/调拨/计划外/报废 | 移库扫码、状态变更确认 | 库内作业页 |
| 盘点 | 实盘扫码/计数 | 盘点管理页 |
| 制品收货/上架/完工撤销 | 产出接收与上架扫码 | 生产管理页 |

详细对照表见维护与查询参考。

## 查询、详情与联查

终端侧通常先按任务号、单据号或扫描码定位。核对最终结果时建议：

1. 在 PDA 确认提交成功；
2. 回到对应 Web 记录页；
3. 联查库存事务与库存余额；
4. 如有后续上架、质检、协同，再查后续单。

不要只在终端列表里判断“已经完成”。

## 常见问题与处理

| 情况 | 建议处理 |
| --- | --- |
| PDA 找不到任务 | 核对任务状态、执行人、业务入口和账号权限；必要时回 Web 查任务。 |
| 扫描通过但业务失败 | 记录提示信息，回 Web 看任务配置、库存余额和状态限制。 |
| 扫错包装/库位 | 停止并重扫；不要手工改数绕过任务限制。 |
| 把线边客户端当 WMS PDA | 先确认菜单归属；线边 MES 功能回 MES 文档。 |
| 终端显示成功但库存没有 | 查 Web 记录、事务、余额，以及是否仍有预计库存或异步处理。 |
