# MOM 制造运营管理系统

## 产品简介

MOM（Manufacturing Operations Management）制造运营管理系统，是一套面向离散制造业的全流程数字化管理平台，涵盖从主数据、库房、生产、质量、设备、异常、供应链到排程的核心业务闭环。

## 目标受众与阅读路径

正式业务页按**测试人员、实施人员**的「业务行为说明书」来写：功能范围清楚、逻辑可讲、配置可落地、验证可开单。深读主路径为：业务模块首页 → 分组 → **主文档** → 需要时再进同组「维护与查询参考」。

其余角色给路径提示（非并列优化目标）：

| 角色 | 怎么读 |
|------|--------|
| 新加入开发 | 顺带受益：模块首页 → 主文档 → 侧栏「认识系统」下的[业务模型](02-业务模型/10-领域模型.md)（共享约定）；争议时再查证据 |
| 售前 / 销售 | **浅层入口（约 15 分钟）**：见下方；**勿进**维护与查询参考、字段长表、技术证据 |
| 研发对照实现 | 以正式页已确认结论为准；缺口与争议见工作台问题总账与 `project-docs` 证据层——**非正式页文风目标** |

运维、部署类细节见「平台与集成」中的基础设施与系统管理，不作为本站业务页的润色北极星。

### 售前浅层路径（约 15 分钟）

按顺序扫读即可对外介绍；读完应能说清「MOM 管什么、不管什么、各模块各管一段」：

1. [系统概述](01-总体框架/05-系统概述.md) — 解决什么问题、从哪读起  
2. [系统边界与产品全景](01-总体框架/01-系统边界与产品全景.md) — 覆盖范围与外部边界  
3. [模块边界与系统上下文](01-总体框架/04-模块边界与系统上下文.md) — 各模块职责一览  
4. 侧栏「业务应用」任选 2～3 个模块首页，**只读「解决什么问题」**（或等价开篇边界）

**到此可停。** 不要进入叶页细节、维护与查询参考、字段长表；技术架构 / 技术栈、多组织与多端页仅在被问到部署/终端时再扫，不是默认浅层必读。

### 测试 / 实施深读（与浅层分流）

从目标业务的**模块首页**进入 → 分组 → **主文档**（配置如何起作用、建议验证点）→ 需要操作细节时再进「维护与查询参考」。不要把售前浅层当成已覆盖配置与验证；也不要把维护参考链给只做介绍的听众。

## 文档结构

侧栏按四层组织；**维护与查询参考**不单独占菜单，从各业务主文档内链进入（深读用，非售前入口）。

| 分区 | 内容 |
|------|------|
| 认识系统 | 总体框架（概述/边界/架构）与业务模型（领域、流程、共享业务规则）——浅层介绍优先从这里起步 |
| 业务应用 | DBC / WMS / MES / QMS / EAM / ANDON / SCP；PS 排程为**规划中**入口 |
| 平台与集成 | 基础设施、系统管理、数采、API 参考 |
| 版本路线图 | 当前版本功能清单与未来迭代方向 |

阅读提示：设备**身份台账**看 DBC「设备台账」；报修/巡检/保养等**运维执行**看 EAM。报表模板与嵌入方式见平台「报表看板与数据输出」；菜单侧配置入口见「报表管理」。
