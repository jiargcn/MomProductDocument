# 消息通知

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。  
> 阅读对象：**测试、实施（主）**；系统管理员与日常发通知的业务管理员（顺带）。售前一句即可：站内信≠业务待办。

## 这一组解决什么问题

System 消息通知提供用户可直接使用的公告、站内信等入口。平台渠道分层、邮件/短信/钉钉边界与跨模块待办说明见基础设施页：[消息、通知与待办](../../03-基础设施/06-消息、通知与待办.md)。

## 功能范围

| 本分组覆盖 | 不在本分组 |
| --- | --- |
| 通知公告、站内信/我的消息、站内信模板、钉钉模板/记录（环境启用时） | 把消息中心当成唯一跨模块待办中心 |
| 与业务待办的边界说明 | 渠道全景与业务触发矩阵 → 基础设施消息页 |

## 配置依赖概览

| 依赖 | 影响 |
| --- | --- |
| 消息菜单权限 | 谁能发公告/看记录 |
| 模板与发送动作 | 收得到/收不到站内信 |
| 邮件短信等通道环境配置 | 菜单有 ≠ 通道通 |

## 当前用户入口
| 入口 | 作用 |
| --- | --- |
| 通知公告 | 维护与发布广播类公告。 |
| 消息记录 / 我的消息 | 查看站内信、未读、标记已读。 |
| 消息模板（站内信） | 配置站内信模板内容与变量。 |
| 钉钉消息模板/记录 | 钉钉渠道配置与发送记录（环境启用时）。 |
| 消息设置/消息配置类菜单 | 存在分散配置入口；改前先确认业务域。 |

## 与业务待办的边界

- 站内信已读 ≠ 业务任务完成。
- 仓储任务领取、质检通知、ANDON 响应等以对应业务列表为准。
- 不要把 System 消息中心培训成唯一待办中心。

## 常见问题

| 情况 | 建议处理 |
| --- | --- |
| 收不到站内信 | 查消息记录是否生成、用户是否正确、模板是否发送。 |
| 公告发了没人看 | 确认公告状态/范围；重要事项仍需业务待办闭环。 |
| 找不到邮件短信配置 | 回基础设施消息页看渠道分层，并确认环境是否启用。 |

## 当前限制

- 业务事件触发矩阵未建全；
- 统一跨模块待办归集未证实；
- 详细渠道治理以基础设施消息页为准。
