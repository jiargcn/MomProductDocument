# 开发平台

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。  
> 阅读对象：**测试、实施、开发配置管理员（主）**。业务用户一般不依赖本页完成现场作业。售前可跳过。

## 这一组解决什么问题

开发平台承接面向扩展与配置的支撑能力（代码生成、系统参数配置、报表设计等），**不是**仓储/生产业务模块。业务规则写在业务页；本页只说明“有哪些配置入口、变更要注意什么”。

## 功能范围

| 本分组覆盖 | 不在本分组 |
| --- | --- |
| 代码生成、配置管理、报表设计器、文件配置等入口与风险提示 | 业务状态机与现场作业规则 |
| 与字典/菜单/权限页的分工 | 低代码/流程设计器启用范围（待环境确认） |

## 配置依赖概览

| 依赖 | 影响 |
| --- | --- |
| 环境隔离（禁止生产直接代码生成覆盖） | 定制被覆盖风险 |
| 租户上下文 / 缓存刷新 | 配置改了不生效 |
| 报表模板被多业务引用 | 改模板影响面 |

## 当前入口
| 入口 | 作用 | 风险提示 |
| --- | --- | --- |
| 代码生成 | 按表生成前后端骨架，加速开发。 | 仅用于受控开发环境；勿在生产直接生成覆盖业务定制。 |
| 配置管理 | 维护基础设施级参数配置。 | 变更可能影响全局行为，需评审与回滚预案。 |
| 报表设计器 | 积木报表模板维护。 | 见报表页；模板被多业务引用时需评估影响面。 |
| 文件配置 | 存储后端与主配置。 | 见文件页。 |
| 字典 / 菜单 / 权限 | System 主数据与授权。 | 见系统管理对应页，不在本页展开字段。 |

## 编写边界（必须遵守）

| 内容 | 应写在哪里 |
| --- | --- |
| 平台提供什么配置能力 | 本页 / 系统管理 |
| 配置如何监控、审计、发布 | 基础设施（日志、定时任务、数据交换等） |
| 业务如何使用配置结果 | 对应业务模块页面 |

## 关键判断

| 判断点 | 应先确认什么 |
| --- | --- |
| 配置改了不生效 | 是否需刷新缓存/重新登录、是否改错租户。 |
| 代码生成覆盖定制 | 是否在错误环境执行；对比版本管理。 |
| 业务异常被归因到“开发平台” | 先分清是业务规则、权限还是参数配置。 |

## 当前限制与待确认事项

- 低代码表单/流程设计器若菜单存在，启用范围待环境确认（审批引擎边界见岗位/BPM 说明）；
- 配置发布、灰度与回滚的标准作业程序待运维补齐；
- 开发平台变更的强制审计字段矩阵待抽样。
