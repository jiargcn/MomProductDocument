# 策略设置

> 适用基线：测试环境 / `dev` 分支 / 2026-07-15。
> 阅读对象：测试、实施（主）；业务配置人员（顺带）。

## 这一组解决什么问题 / 功能范围

策略设置维护影响单据、任务与规则执行方式的受控配置（班次/班组、业务类型、单据设置、单据开关、规则）。它们不是普通主数据：一次变更可能影响多类业务的创建、编号、可用范围或处理方式。

**范围外：** 各业务申请/任务/记录的现场作业细则在对应业务模块；共享通例见[单据类型、业务类型与单据配置](../../02-业务模型/05-单据类型、业务类型与单据配置.md)。

## 如何使用本组文档（测试 / 实施）

| 你的目的 | 建议阅读 |
| --- | --- |
| 讲清「改配置 → 单据/任务行为变化」并开验证 | **主文档**（尤其业务类型、单据设置；W1 定标） |
| 字段、导入、启停操作细节 | 同对象**维护与查询参考** |
| 评估生产环境变更风险 | 先读影响范围 → 测试环境验证 → 再变更 |

售前介绍请停在 [DBC 模块首页](../index.md)，**不要**直接改生产策略配置。

## 常见问题与相关分组
