# SCP 供应链平台

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。  
> 阅读对象：**测试、实施（主）**；采购、供应商门户用户（顺带）。售前只读「模块解决什么问题」，**勿**把 SCP 收货页写成仓库库存权威，也**勿**默认进入维护参考。

## 模块解决什么问题

SCP 是面向**供应商协同**的门户与协同单据域：采购订单确认、要货预测/计划协同、供应商发货（ASN）、到货/退货跟踪视图、发票与索赔/对账。独立前端 `scp-ui`、独立后端模块；通过 **SCP↔WMS 接口**推送发货记录、采购计划/订单、包装与条码等。

**库存事务、库内收货执行、余额以 WMS 为准。** SCP 侧亦有采购收货/退货/上架等对象（多供供应商侧查询或自 WMS 迁移），培训与验收时必须分清「协同可见」与「仓内权威」。

## 功能范围

| 在本模块 | 不在本模块（去哪看） |
| --- | --- |
| 订单发布与供应商确认 | ERP 主单权威与付款总账 |
| 要货预测 / 采购计划协同 | MES 生产计划/工单（与 SCP 直接联动**未证实**） |
| 发货申请→记录（ASN）及推送 WMS | WMS 收货任务执行与库存事务 → [采购收货](../05-WMS-库房管理/03-采购收货/index.md) |
| 采购跟踪视图（收货/退货/短缺等） | 把 SCP 页当作唯一库存真相 |
| 发票/索赔/对账日历（SCP 内对象） | 财务系统回写细节（待确认） |
| SCP 侧供应商/物料/仓库等副本与单据开关 | DBC 企业主数据治理权威 |

## 配置依赖概览

| 依赖层 | 先备 / 改什么 | 行为影响 |
| --- | --- | --- |
| SCP 基础数据 | 供应商、供应商物料、价格、包装、仓库/月台 | 建单/发货可选范围与计价 |
| 单据开关 / 策略 | 业务入口启用、计划/发货策略 | 菜单或动作是否可用 |
| 供应商用户绑定 | 用户↔供应商 | 门户「看得到/看不到」订单与计划 |
| 订单/计划状态 | 发布、确认/接受、关闭 | 能否发货、首页开放单统计 |
| autoPublish / autoAccept 等 | 计划侧开关 | 是否跳过人工发布/接受 |
| SCP↔WMS 接口 | 通道可用、映射与账期 | 推送失败则仓库无 ASN/计划 |
| DBC 主数据（并存） | 企业权威编码 | **勿默认**改 SCP 等于改 DBC（同步关系见文末未决） |

## 与其它模块边界

| 模块 | SCP 负责 | 不在 SCP 展开 |
| --- | --- | --- |
| DBC | 协同侧副本维护与引用 | 企业主数据治理权威 |
| WMS | ASN/计划/订单/包装条码等推送；跟踪查询 | 库内收货任务、库存事务与余额 |
| QMS | 发货检验明细等协同线索（若启用） | 来料检验 ATR 权威 |
| MES | — | 生产计划/工单；直接联动未证实 |
| ERP/财务 | 订单类型、已开票视图等线索 | 付款与总账规则 |
