# 采购订单

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。  
> 阅读对象：**测试、实施（主）**；采购员、供应商门户用户（顺带）。操作见[采购订单-维护与查询参考](采购订单-维护与查询参考.md)。售前停在[模块首页](../index.md)。

## 这一组解决什么问题

采购订单是 SCP 中连接企业与供应商的核心协同单据。采购方维护/发布订单后，供应商在门户查看并确认（或按计划策略自动接受）。订单与明细携带数量、已发/已收/已退等累计字段，作为发货与跟踪的依据；可向 WMS 登记推送。

订单类型含排程类与离散类；头状态见下文。旧稿「24 小时强制确认」等**未证实**，不以之为培训硬规则。

## 功能范围

| 本分组覆盖 | 不在本分组 |
| --- | --- |
| 订单创建/发布/确认/关闭与数量累计线索 | 库内收货执行（WMS）；ERP 主单权威 |
| 向 WMS 推送订单主/明细的协同侧 | ASN 物流细节 → [发货协同](../05-发货协同/index.md) |

```mermaid

    A["创建/导入订单"] --> B["发布 PUBLISHED"]
    B --> C["供应商确认或策略自动接受"]
```

## 主对象

| 对象 | 业务含义 | 使用者关心 |
| --- | --- | --- |
| 订单头 | 单号、供应商、类型、状态、下单/交期、税率、寄售、业务类型、数据来源等。 | 是否已发布、能否发货。 |
| 订单行 | 行号、物料、订购量、包装、单价/金额、已发/已收/已退/已上架量、超收比例、行状态等。 | 未交数量、行是否可发。 |

## 状态与关键动作

头状态：就绪、已发布、关闭、完成。

| 常见动作 | 业务结果 |
| --- | --- |
| 新增/修改/删除（受限） | 维护草稿或就绪数据。 |
| 发布 | 进入供应商可见/可确认范围。 |
| 确认/接受（供应商侧或自动） | 进入可发货协同。 |
| 关闭/完成 | 结束开放订单；首页「开放订单」统计据此聚合。 |

## 与计划 / 发货 / WMS / ERP 边界

| 协同方 | 本页负责 | 不在本页展开 |
| --- | --- | --- |
| 要货计划 | 可被计划引用 poNumber 等 | 计划发布与接受状态机 |
| 发货协同 | 提供可发订单行与数量上限线索 | ASN 物流与推送细节 |
| WMS | 可通过接口推送订单主/明细 | 库内收货执行 |
| ERP | 保留 erpOrderType、行税码等线索 | ERP 主单权威与回写协议 |
| DBC | 引用供应商/物料编码 | 主数据治理 |

## 关键判断

| 判断点 | 应先确认什么 | 影响 |
| --- | --- | --- |
| 供应商看不到订单 | 是否 PUBLISHED、供应商用户绑定 | 门户无单 |
| 无法发货 | 行剩余可发量、行状态、计划关联 | ASN 校验失败 |
| 数量对不上 | 行上 shipped/received/returned 累计 | 跟踪与对账偏差 |

### 关键字段业务角色

完整状态与选择器见[维护与查询参考](采购订单-维护与查询参考.md)。「可用供应商 / 可用物料」通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)；本页保留订单类型、发布状态与行累计等差异。

| 字段/配置点 | 在系统中的作用 | 关键行为要点 | 警惕什么 |
| --- | --- | --- | --- |
| 订单类型（排程/离散） | 协同路径分流 | SCHEDULE / DISCRETE | 类型错→后续入口不符 |
| 头状态 | 发布/关闭门禁 | READY→PUBLISHED→CLOSED/COMPLETED | 未发布供应商不可见 |
| 供应商 | 协同对象 | 可用供应商 | 绑错门户无单 |
| 行物料/订购量 | 可发上限来源 | 行累计 shipped/received 等 | 超发校验失败 |
| 超收比例 | 收货容差线索 | 行级 | 容差误解 |
| 寄售 / 业务类型 | 结算与策略线索 | 头字段 | 结算口径错 |
| 推送 WMS | 订单同步意图 | 接口类型已证实；同实例 ❓（`GAP-017`） | 以为 WMS 菜单即同单 |

### 选择器范围（骨架）

通例见[通用选择器过滤惯例](../../02-业务模型/12-通用选择器过滤惯例.md)。下表只写本页差异；发货侧「可发订单」精确状态集见 `FSEM-006`。

| 选择字段 | 选择对象 | 可选范围（当前可写） | 范围依赖 | 选不到时通常原因 |
| --- | --- | --- | --- | --- |
| 供应商 | SCP 供应商主数据 | 可用供应商；门户绑定另论 ❓（`GAP-017`） | — | 停用、未建、未绑定门户 |
| 行物料 | 物料主数据 | 可用；可采购用途 ❓（`FSEM-001`）；是否强制供应商物料关系 ❓（`GAP-045`） | 供应商、供应商物料 | 停用、无关系、用途关闭 |
| 订单类型 | 排程 / 离散 | SCHEDULE / DISCRETE | 建单入口 | 入口与类型不匹配 |
| 发货侧选订单/行 | 已发布订单行 | 头宜为已发布且有剩余可发；精确状态集 ❓（`FSEM-006`） | 头状态、行累计、供应商 | 未发布、已发完、已关闭/完成 |

## 限制与待确认

- 发布后变更是否需供应商再确认：依赖环境配置，待联调。
- 与 ERP 双向同步时机与失败补偿：待确认（`GAP-017`）。
- WMS 侧「采购订单」与 SCP 订单是否同一实例：**未证实**（`GAP-017`），文档按 SCP 侧对象描述。
- 下游发货可选订单精确状态集、P13 投影矩阵待测（`FSEM-006`）。
- 正文中的状态枚举名（就绪/已发布等）与行累计量线索（已发/已收等）来自已取证枚举与字段，**不是**由中文标签反译。
