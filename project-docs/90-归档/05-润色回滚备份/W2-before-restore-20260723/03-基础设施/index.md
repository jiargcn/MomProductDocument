# 平台能力与基础设施

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。  
> 阅读对象：**测试、实施、运维（主）**。售前只需知道「跨模块公共能力在此」；具体业务规则仍在各业务模块。  
> **说明：** 本组多为单文件叶页（非分组 `index`）；叶页深度润色属后续波次。本页只做**模块地图**。

## 模块解决什么问题

本模块描述跨业务复用的**平台能力与运行底座**：标签打印、报表、查询视图、导入导出、文件、消息、日志审计、定时任务、数据交换可靠性等——回答「能力如何提供、如何配置、如何治理」，而不是「某一笔收货/报工怎么做」。

## 功能范围

| 在本模块 | 不在本模块 |
| --- | --- |
| 公共能力的配置入口、分层口径、运维治理要点 | WMS/MES/QMS 等具体业务状态机与现场作业规则 |
| 与 System 权限/租户的衔接说明 | RBAC/组织树主维护 → [系统管理](../12-系统管理/01-租户与认证/index.md) 等 |
| 标签等「当前入口在业务模块、长期归属平台」的边界声明 | 把业务页写成平台 API/源码说明 |

## 配置依赖概览

| 依赖 | 影响 |
| --- | --- |
| 租户套餐 / 角色菜单 | 平台菜单是否可见 |
| 环境参数（存储、短信/邮件通道、调度中心等） | 能力是否真正可用（菜单有 ≠ 通道通） |
| 业务模块内嵌入口 | 如标签当前主入口可能在 WMS；改平台模板仍影响多业务 |
| 运维权限分离 | 高风险清理/重试/改参应与普通业务角色分离 |

## 编写边界

1. 本模块写「能力如何提供、如何配置、如何治理」。  
2. 业务模块写「业务在何处使用该能力、产生什么数据」。  
3. 权限 → [RBAC](../12-系统管理/03-用户与权限/01-RBAC权限模型.md)；认证 → [租户与认证](../12-系统管理/01-租户与认证/index.md)。

## 长期归属约定

标签、条码和打印属于平台能力。即使当前配置入口位于 WMS/DBC，也只表示业务入口现状；模板、编码、渲染、输出、设备适配与审计按平台能力治理。
