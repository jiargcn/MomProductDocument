# 模块边界与系统上下文

> 适用基线：测试环境目标 / `dev` 分支 / 2026-07-15。
> 阅读对象：售前扫职责（浅层）；测试 / 实施定位「权威落点」后进入业务主文档（深读）。
> 职责以现有模块文档结论为准；调用协议细节见 API 集成场景。

本页用一张表说清：**各模块解决什么问题、与谁协作、文档从哪进**。售前读完职责列即可去模块首页；测试 / 实施用「权威落点」判断跨模块争议该找谁。

## 如何使用本页

| 你的目的 | 怎么用 | 不要做 |
| --- | --- | --- |
| 售前 / 对外介绍 | 扫「解决什么问题」列 → 打开 2～3 个模块首页开篇 | 进入维护与查询参考、字段长表 |
| 测试 / 实施 | 先看权威落点与协作对象 → 进对应**主文档**做场景与配置 | 只凭本页开验证单（本页无配置差异细节） |
| 新开发 | 本表 + 目标模块首页 + [核心业务流程](../02-业务模型/11-核心业务流程.md) | 把技术架构页当业务边界的替代 |

## 模块职责地图

| 模块 | 解决什么问题（一句话） | 主要协作对象 | 文档 |
| --- | --- | --- | --- |
| DBC | 统一主数据、工厂建模与策略，让下游「选得对、认得住」 | 全部业务模块 | [DBC](../04-DBC-主数据管理/index.md) |
| WMS | 仓储作业、库存余额与库内/盘点执行 | SCP、QMS、MES、EAM | [WMS](../05-WMS-库房管理/index.md) |
| MES | 工艺与计划落地为报工、追溯与线边执行 | WMS、QMS、EAM、ANDON、数采 | [MES](../06-MES-生产管理/index.md) |
| QMS | 检验任务、质量判定与评审 | WMS、MES、SCP | [QMS](../07-QMS-质量管理/index.md) |
| EAM | 维修保养等运维执行；设备身份台账多在 DBC | MES、ANDON、WMS | [EAM](../08-EAM-设备管理/index.md) |
| ANDON | 异常呼叫、响应到岗 | MES、EAM、QMS | [ANDON](../09-ANDON-异常管理/index.md) |
| SCP | 采购协同、预测/计划、发货、跟踪、结算 | DBC、WMS、外部 | [SCP](../10-SCP-供应链平台/index.md) |
| PS | **规划中**：排程意图入口，本仓无可用实现，不可按已上线承诺 | MES / DBC（边界意向） | [PS](../11-PS-排程管理/index.md) |
| 数采 | 采集点配置、日志与边缘上报 | MES、WMS、边缘 | [数采](../13-数采管理/index.md) |
| 系统与基础设施 | 组织权限、公共能力、运行与集成 | 全部模块 | [租户与认证](../12-系统管理/01-租户与认证/index.md)、[基础设施](../03-基础设施/index.md) |

## 权威落点（摘要）

跨模块争议时，先按事实类型找权威模块，再进该模块主文档：

| 事实 | 权威 |
| --- | --- |
| 主数据身份 | DBC（SCP 并存对象同步未一律证实） |
| 库存事务 / 余额 | WMS |
| 生产执行 / 报工 | MES |
| 检验结论 | QMS |
| 维修工单 | EAM |
| 安灯到岗 | ANDON |

端到端流程见[核心业务流程](../02-业务模型/11-核心业务流程.md)；接口方向见[跨系统集成场景](../14-API参考/03-跨系统集成场景.md)。

## 限制与待确认

下列项**不打断**上表已确认职责；落地前需按环境核验：
