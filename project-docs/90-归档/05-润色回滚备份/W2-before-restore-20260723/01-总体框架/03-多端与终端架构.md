# 多端与终端架构

> 状态：端侧清单来自当前已知工程范围；各端菜单、角色与权限细节待菜单与实现核验。
> 阅读对象：实施与终端方案讨论；**不是**售前 15 分钟浅层主链（被问到 PDA / 线边端时再打开）。

本页规定各端负责的操作类型，避免把 PDA 或线边端误当成独立业务域——终端是**同一业务**的现场入口，权威业务规则仍在对应业务模块主文档。

## 设计目的

同一业务可能由管理端、协同端和现场终端共同完成。本页只定端侧定位与边界；具体动作、状态与配置以各业务主文档为准。

## 当前已知端侧范围

| 端侧 | 参考职责 | 文档归属 |
| --- | --- | --- |
| WEB 主端 | 配置、查询、审批、管理与业务操作 | 对应业务模块页面 |
| SCP 前端 | 供应链协同操作 | [SCP](../10-SCP-供应链平台/index.md) |
| PDA | 仓储现场扫描与执行 | WMS「终端操作」相关分组 |
| 线边端 | 生产现场采集、投料、报工等 | MES「终端操作」相关分组 |
| EAM PDA | 设备现场点检、保养、维修等 | EAM「终端操作」相关分组 |

## 编写与阅读规则

- 终端操作页描述具体业务动作；本页仅描述端侧定位、数据一致性与权限边界。
- 售前介绍默认停在[系统概述](05-系统概述.md) / [模块边界](04-模块边界与系统上下文.md)；不要从本页链进维护参考。
