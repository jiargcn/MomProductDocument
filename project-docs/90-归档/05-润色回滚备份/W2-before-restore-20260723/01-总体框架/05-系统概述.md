# 系统概述

> 状态：产品定位与阅读入口已整理；当前环境实际启用的模块、租户范围与外部集成对象，仍以测试环境 / `dev` 菜单与配置核验为准（见[系统边界](01-系统边界与产品全景.md)待确认项）。
> 阅读对象：售前浅层入门（主）；测试 / 实施 / 新开发找「从哪读起」时亦从此页分流。

本页用两屏说清：MOM **解决什么问题**、**边界在哪**、**从哪读起**。读完应能选对浅层或深读路径，而不是直接掉进字段表。

## MOM 解决什么问题

面向离散制造，把「主数据与工厂建模 → 供应协同 → 仓储执行 → 生产执行 → 质量判定 → 设备运维 / 异常响应」串成可协同、可追溯的运营闭环；报表、标签、消息、接口等作为公共能力支撑业务执行。

一句话：**统一主数据与配置，让到货、库存、生产、质量、设备与异常在同一套业务对象上协同，而不是各系统各说各话。**

## 核心能力（扫读）

- 统一主数据与工厂业务建模（DBC）
- 供应、仓储、生产、质量、设备与异常的协同闭环
- 单据、库存、批次/包装与业务记录追溯
- 报表、看板、标签打印、消息、接口、日志等公共能力

能力是否在**当前项目环境**全部启用，以菜单与配置核验为准，不以本页清单代替验收。

## 文档原则（诚实口径）

1. 旧规划书仅用于提炼稳定理念和待核验范围，**不作为**当前实现事实。
2. 当前实现以测试环境、`dev` 分支、菜单、源码与 DDL 等证据为准；正式页只写已确认业务结论。
3. 无法确认的内容使用「待核验 / 规划中 / `❓`」等标记，**不用推测补全**；缺口详情在工作台问题总账，不在本页展开成长清单。
